Chromium Digest: 2025-12-09

Overview

On 2025-12-09, the Chromium main branch saw 639 commits, with 474 deemed relevant to developers. This summary covers changes from commit (472c212)... to (99d36c9).... The day was marked by a large number of breaking changes across core content, UI, Autofill, and GPU APIs, requiring significant updates for many embedders. The Web Platform saw improvements to CSS Selectors and animations, while experimental features like Vertical Tabs and WebNN continued to evolve.

Breaking Changes

  • ⚠️ BREAKING The pure virtual method content::ContentBrowserClient::RegisterWebUIInterfaceBrokers has been removed. Embedders must now implement two new methods: RegisterTrustedWebUIInterfaceBrokers and RegisterUntrustedWebUIInterfaceBrokers to separate interfaces for chrome:// and chrome-untrusted:// schemes. (2b5bd6d)
  • ⚠️ BREAKING The child_id field of the core public struct content::GlobalRenderFrameHostId has been changed from a primitive int to the strongly-typed content::ChildProcessId. Any code directly accessing this field must be updated. (8a18733)
  • ⚠️ BREAKING The previous_render_frame_host_id field has been removed from the public content::NavigationRequestInfo struct. Embedders relying on this field for navigation tracking must update their code. (de7c74e)
  • ⚠️ BREAKING The autofill_agent.mojom interface has changed. The ApplyFieldsAction method now requires FillId and supports_refill parameters, breaking existing implementations. (58de6fa)
  • ⚠️ BREAKING The public method AutofillManager::NumDetectedForms() has been removed from the components/autofill API. (5af0dc3)
  • ⚠️ BREAKING The net::device_bound_sessions API signature has changed. Methods in SessionService and related classes now require a DbscRequest& wrapper instead of a raw URLRequest*. (124289e)
  • ⚠️ BREAKING On Linux, the ui::LinuxUi::CreatePrintDialog method signature has changed to return a std::unique_ptr instead of a raw pointer, and the corresponding ReleaseDialog method has been removed. (203ad26)
  • ⚠️ BREAKING Also on Linux, the static GtkUi::GetPlatform() method has been removed. Constructors for SelectFileDialogLinuxGtk and PrintDialogGtk have been changed, breaking embedders using these GTK-based UI components. (7ba746d)
  • ⚠️ BREAKING The DesktopMediaPickerController::Show() method signature was refactored. The includable_web_contents_filter argument was removed and must now be set in the Params struct. (3a1a659)
  • ⚠️ BREAKING The MemoryDumpManager::RegisterDumpProvider method signature has changed. It now accepts a MemoryDumpProvider::Name class instead of a const char*, requiring embedders to update their calls and potentially add provider names to histogram variants. (d368323)
  • ⚠️ BREAKING The gpu_rasterization field has been removed from the public gpu::Capabilities struct. (17f29af)
  • ⚠️ BREAKING The RemoteAppsLacrosBridge interface has been removed from the stable crosapi.mojom interface for ChromeOS integrations. (2929b15)
  • ⚠️ BREAKING The account_id parameter has been removed from the public IdentityManager::RefreshAccountInfoIfStale method. (13d5a96)
  • ⚠️ BREAKING The public static method ChromeML::Get() signature has changed, removing the library_name parameter. (d3b417e)
  • ⚠️ BREAKING Legacy IPC ParamTraits for gfx::BufferUsageAndFormat have been removed, breaking any code that serializes this struct over legacy IPC. (1f0b4ca)

Web Platform & Blink

  • The :lang() pseudo-class has been updated to support extended language ranges as per the CSS Selectors Level 4 specification. This aligns with web standards but may affect sites relying on the previous, more restrictive behavior. (acc7766)
  • The onanimationcancel event handler has been exposed to GlobalEventHandlers, making it available on elements and windows for web developers. (f649d65)
  • The mechanism for "light dismiss" (e.g., clicking outside a popover or <dialog>) has been changed to use the click event instead of pointerdown/pointerup, fixing several bugs and improving spec alignment. (8a0c5fc)
  • The autofill event and related APIs have been added to blink::Document in support of the WICG address-autofill proposal. (018378f) (471ea39)
  • For the experimental Route Matching API, the CSS function urlpattern() was renamed to url-pattern() to align with spec changes. (8dbacc6)

UI & UX

  • Development continues on the experimental Vertical Tabs feature, with the addition of a dedicated VerticalTabGroupHeaderView and fixes for toolbar margins and selection events. (1e3afb7) (09d9932) (3f30da6)
  • On Android, a crash in MediaSessionHelper was fixed by pooling BroadcastReceiver instances instead of creating one per session. (86559d2)
  • For Android WebView, the First Contentful Paint (FCP) metric is now plumbed through AwWebPerformanceMetricsObserver for consistent performance monitoring. (5999046)
  • On iOS, stale Safety Check notifications are now automatically removed from the Notification Center when the underlying issue is resolved. (8588e07)

Core Internals & Networking

  • To combat ad-based history manipulation, new flags (is_entry_created_by_ad_, is_ad_entry_creator_) have been added to NavigationEntry to track history entries created by ads. This is currently a data-only change. (543e658)
  • Initial navigations to WebUI pages will now skip running NavigationThrottles and CommitDeferringConditions to improve performance and reliability. (9d040bc)
  • The WebHistoryService was refactored to return structured results instead of a generic base::Value, improving type safety. (671a12e)
  • The ASSIGN_OR_RETURN macro in base was updated to better support implicit conversions for std::optional and pointers. (36e4f9d)

Performance, GPU, & Media

  • The system for detecting workstation lock status has been improved to use OS-level observers instead of polling, providing more timely updates for APIs like the Idle Detector. (319bb38)
  • The experimental InputScenarioPriorityBoost feature, which dynamically adjusted renderer thread priority, has been removed. (300e9a6)
  • The std::optional<gfx::HDRMetadata> type has been replaced with gfx::HDRMetadata across the media stack to de-optionalize the type and simplify its usage. (59ae1c8)
  • For the experimental WebNN API, the DirectML execution provider will now be avoided on software GPUs to prevent crashes. (29da2cb)

Security & Privacy

  • The legacy Safe Browsing Extended Reporting (SBER) preference is being deprecated. Reporting is now gated by the user's protection level (e.g., Enhanced Protection) rather than the old SBER toggle. (04061b4)

Accessibility

  • A new accessibility tree builder, PdfAccessibilityTreeBuilderStructure, was added to build trees directly from PDF structure trees for tagged PDFs, providing more accurate semantic structure. (d66dd88)
  • A new WidgetAXManager::Observer interface was added to allow components like WebView to listen for accessibility enablement and correctly configure their accessibility tree. (e522f75)

Extensions

  • The declarativeNetRequest API was updated to expose the supported keys of the RuleCondition dictionary, allowing extensions to perform feature detection. (ab42e9f)
  • The system.display extension API definition was converted from the legacy .idl format to the modern .webidl format as part of an ongoing modernization effort. (e8ea5f7)

Build System & Cleanup

  • The libc++ standard library dependency was rolled forward. This applies the [[nodiscard]] attribute to several standard components, which may produce new compiler warnings. (49c3f9b)
  • Numerous expired or unused feature flags were removed, including web-app-manifest-protocol-handlers, merchant-trust-signals, LastVSyncArgsKillswitch, and WebAppManifestIconUpdating. (5c7e4d6) (8baafb3) (6c2f4b5) (71e18aa)
