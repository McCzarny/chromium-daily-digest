Chromium Digest: 2026-04-13

Overview

On 2026-04-13, the Chromium main branch saw 613 commits, with 457 identified as relevant to developers. This day was marked by numerous breaking changes to core APIs, build configurations, and the removal of the Text Autosizer feature. The first major commit of the day was (129c0a6) and the last was (c2e9a49).

Core APIs & Services

  • ⚠️ BREAKING BREAKING: The ContentBrowserClient interface was updated. ShouldTreatURLSchemeAsFirstPartyWhenTopLevel(string_view) is replaced by ShouldTreatAsFirstPartyWhenTopLevel(const url::Origin&). Embedders overriding this method must update their implementation to use the url::Origin parameter. (9f52b02)
  • ⚠️ BREAKING BREAKING: The signature for the virtual method net::CookieStore::SetCanonicalCookieAsync has changed. A default argument was removed, and callers must now explicitly provide the cookie_access_result parameter. Embedders who subclass net::CookieStore will need to update their implementations. (4d84d8a)
  • ⚠️ BREAKING BREAKING: AccountSettingService has been refactored into a pure virtual interface. Consumers must update their code to depend on the AccountSettingService interface instead of the concrete AccountSettingServiceImpl class. (8a38787)
  • ⚠️ BREAKING BREAKING: Core base/metrics APIs have been modified. Histogram constructors and PersistentCreate methods now require a name_hash parameter, breaking low-level integrations with the metrics system. (2edb1dc)
  • ⚠️ BREAKING BREAKING: The public mojom interface PrintManagerHost::ShowScriptedPrintPreview() has been changed, removing the is_modifiable boolean parameter. Embedders implementing this interface must update their method signature. (d9e628f)
  • ⚠️ BREAKING BREAKING: The obsolete MitigationsEnabledFor3pcd function has been removed from CookieSettings, and the corresponding SetMitigationsEnabledFor3pcd method has been removed from the network::mojom::CookieManager Mojo interface. (1190624)
  • ⚠️ BREAKING BREAKING: The platform-independent utility function base::subtle::YieldProcessor() has been removed from base/synchronization/lock_subtle.h via a revert. (cd356a8)
  • The VideoFrame::WrapSharedImage method overload that accepted a coded_size has been removed. This value can now be inferred directly from the shared image. (b14f02a) (04cfedf)
  • The chrome://policy/logs page has been migrated from the legacy WebUI messaging system to a modern, type-safe Mojo interface. (6a07b82)
  • The AIManager has been hardened with more explicit checks and granular error messages related to enterprise policies and user settings for on-device AI models. (9381fd2)

Build System & Infrastructure

  • ⚠️ BREAKING BREAKING: The //chrome/browser/net directory has been modularized. Source files were moved from the monolithic //chrome/browser:browser target into new :net and :impl targets, which will break downstream build configurations that directly reference these files. (888790c)
  • Enterprise-related source files have been moved from //chrome/browser into a dedicated //chrome/browser/enterprise build module to improve code organization. (8a22588)
  • The use_afl GN argument and AFL_DRIVER_DONT_DEFER environment variable have been removed to streamline fuzzer build configurations. (b691d02)
  • The deprecated TestBrowserClosedWaiter test utility has been removed and its callers migrated to the BrowserDestroyedObserver API. (dd0cca1)

Web Platform & Blink

  • ⚠️ BREAKING BREAKING: The Text Autosizer feature has been completely removed. The text_autosizing_enabled preference in WebPreferences and related logic in AwSettings are deleted. Embedders relying on this for text inflation will need to implement their own scaling logic. (4797e6b)
  • ⚠️ BREAKING BREAKING: The LayoutReinsertOnInFlowStateChange feature flag and its associated code have been removed. This includes the removal of several virtual methods from Blink layout classes, such as LayoutBlockFlow::ChildBecameNonInline, which may break downstream forks that override them. (f5d8e01)
  • The behavior of the stable Web API navigation.navigate() has changed. Calls with a javascript: URL will now reject with a NotSupportedError instead of being silently ignored, which could break sites relying on the previous behavior. (868ac71)
  • To align with the HTML spec, <meta> tags used for responsive iframes are now only considered valid if they appear within the document's <head>. (129c0a6)
  • A security issue where attr() values in CSS shorthands could bypass taint checks has been fixed. (0921752)
  • The SVG parser for <polygon> and <polyline> points has been refactored to parse numbers individually, improving alignment with spec-defined error handling. (5885775)

Android

  • ⚠️ BREAKING BREAKING: A refactor of PersistentStoreCleaner was reverted, changing its public Java API from instance-based methods back to static methods. Consumers of the recently introduced factory and instance methods will be broken. (66e6c2a)
  • Navigation logic was updated to support the SWITCH_TO_TAB disposition, allowing navigation to an existing tab with a matching URL instead of always opening a new one. (ca13bcf)
  • Syncing of site search engines (template URLs) is now enabled on Android, controlled by the kSyncSearchEnginesAndroid feature flag. (6c0a722)
  • The BottomSheetController now exposes a getPeekHeightPx() method, allowing for better integration with browser controls. (ceadac5)
  • The TabModelSelector interface now includes a getProfile() method to provide easier access to the associated profile. (8f196ec)

Security & Privacy

  • Cross-Origin Resource Policy (CORP) checks are now being enforced for the Service Worker Static Routing API's cache source. This change is gated by the kServiceWorkerStaticRouterCORPCheck feature flag. (e2b9187)
  • Client-Side Detection (CSD) for malicious sites will now trigger a check upon initial detection of a credit card form, in addition to the existing trigger on user interaction. This is controlled by the EnableDetectionTrigger feature parameter. (5543142)
  • On macOS, opening the applescript: scheme as an external protocol is now disallowed to enhance security. (71c38a1)
  • A path confusion vulnerability in the Payment Method Manifest crawler was fixed by using the post-redirect URL to resolve relative application paths. (aa2bdb2)

Developer Tools (CDP)

  • The Target.TargetInfo object for worker targets will now include a parentFrameId property. This allows DevTools clients to correctly attribute a worker to its owning frame in the target tree. (2fce35d)

Autofill & Passwords

  • The Autofill popup menu has been updated to support nested flyout menus, enabling richer UI for complex data types. (3bbb3c5)
  • The Mojo IPC for password generation requests has been updated to use FieldRendererId instead of indices, fixing a potential browser crash caused by out-of-sync frame data. (61c155e)

iOS

  • Implementations for Siri Shortcuts have been added, allowing users to open history, settings, passwords, and new incognito tabs via voice commands. (d921def) (99a1d6d) (9a3fea0) (279eb95)
  • Fullscreen support has been improved for iOS 17/18 by adjusting the WebView frame to correctly account for system UI changes. (d93e050)

Graphics & Rendering

  • A "flush-only" sync mechanism has been introduced for TreesInViz. This allows a renderer to notify Viz of resource deletions upon becoming invisible without triggering a full, unnecessary draw cycle. (5751c03)
  • BufferQueue usage is now enabled on Windows with partial delegation, which involves allocating a buffer queue to each drawn render pass to improve performance. (de96932)
