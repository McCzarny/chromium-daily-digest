Summary for 2026-04-03 on branch 'main'

Chromium Digest: 2026-04-03

Overview

This summary covers 463 commits on the main branch, from commit (3622de3) to (1e376a2), with 340 identified as relevant to developers. The day was marked by an unusually high number of breaking API changes across UI, services, and media components, requiring significant code updates for embedders. Key changes include the removal of the WebUI tab strip, stricter Content Security Policy enforcement, and numerous public API removals in the sign-in, accessibility, and PWA subsystems.

Breaking Changes

  • ⚠️ BREAKING The kWebUITabStrip feature has been disabled by default on all platforms, including ChromeOS. This effectively removes the WebUI-based tab strip as a stable feature. (880ccd9)
  • ⚠️ BREAKING The IdentityManagerProvider class and its public header identity_manager_provider.h have been removed. Consumers must now depend on IdentityManagerFactory directly to get the IdentityManager. (4db6a2e)
  • ⚠️ BREAKING The public method is_secondary_widget() has been removed from the core ui/views/widget/widget.h interface, which was available for embedders to use. (c61eb90)
  • ⚠️ BREAKING The deprecated PWA identity update dialog flow has been removed. This includes the public function WebAppUiManager::ShowWebAppIdentityUpdateDialog and the web_app_callback_app_identity.h header. (e05db21)
  • ⚠️ BREAKING Content Security Policy (CSP) now treats <link rel=modulepreload> without an as attribute as a script (script-src-elem). This stricter enforcement may break sites relying on the previous, more lenient behavior. (ec9209f)
  • ⚠️ BREAKING The public API for views accessibility live regions has been changed. Several deprecated methods like SetLiveAtomic, SetLiveStatus, and SetLiveRelevant have been removed from view_accessibility.h in favor of a new unified approach. (2493686) (e3b1238) (c7c4e05) (7d4d666)
  • ⚠️ BREAKING A new NEW_SPLIT_VIEW value has been added to the WindowOpenDisposition enum. This can cause compile-time failures in embedder code that uses switch statements on this enum without a default case. (7f076b4)
  • ⚠️ BREAKING On Android, the public API for media color space handling has been refactored. The JniHdrMetadata class API has changed, and MediaCodecBridge creation methods now require new color space parameters. (6f0ebd2)
  • ⚠️ BREAKING On Android, the public ButtonData class for toolbar buttons has been refactored to use a builder pattern, removing the previous public constructor and changing how buttons are configured. (736bbb6)
  • ⚠️ BREAKING Multiple public APIs have been removed or altered, including: the GlicWindowController::Create() factory and GlicKeyedService::toggleUI() signature; numerous OAuth scope constants from gaia_constants.h; FormFieldData::set_host_form_signature(); and WorkingDirectoryIndependentServerNameFromUTF8 from named_mojo_ipc_util.h. (b02d14f) (9a12d39) (8b1ec69) (9780ea5) (1e376a2)

Security & Privacy

  • The File System Access API now blocks write access to */.git/hooks directories on Windows to mitigate a potential remote code execution vector. (fb916b2) (5ada3cd)
  • A vulnerability in Direct Sockets that allowed UDP multicast to bypass local network permissions has been fixed. (d3e49ac)
  • The FrameSerializer now strips SVG <script> elements in addition to HTML ones, preventing script execution in saved MHTML documents. (51ac3bc)
  • Several Use-After-Free vulnerabilities were fixed in the Fuchsia ContentDirectoryURLLoader, the GPU process during OOM, and in aura::Window notification loops. (7b454b0) (daeb9ba) (43bf0cb)
  • A potential cross-origin information leak via LCP Critical Path Predictor hints during redirects has been fixed by clearing the hint on redirection. (ef0bfae)

Web Platform & APIs

  • Support has been added for the OpenXR mesh extension (XR_EXT_hand_tracking_data_source), expanding the capabilities of the WebXR API. (7a7e2c5)
  • The Gamepad API on macOS now defaults to using the modern GameController framework for PlayStation and Xbox controllers. On Windows, the enable-windows-gaming-input-data-fetcher flag was removed, and enumeration was improved to prevent duplicate device entries. (711fe95) (2ccddd5) (4aa5edf) (2a73708)
  • For PWAs, shortcut URLs are now validated during manifest parsing and on launch to ensure they are in-scope of the web app. (c6e8cc1)
  • The UA stylesheet rule that set border-color: gray for <table> has been removed behind the TableDefaultBorderColorCurrentColor feature flag, allowing table borders to correctly default to currentColor as per the spec. (69afd54)
  • The Prompt API feature name has been updated to LanguageModel to align with emerging web standards. (cc6081b)

UI & UX

  • ⚠️ BREAKING A new NEW_SPLIT_VIEW WindowOpenDisposition was added to support opening links in a split view, typically via Ctrl+Alt+Click. This is a breaking change for embedders with switch statements on the enum. (7f076b4)
  • On Android, touch arbitration logic for the Tab Bottom Sheet was refactored to use a synchronous 'Gesture Zone', improving scrolling reliability and interaction. (b26bc8c)
  • On iOS, fullscreen mode will now exit on non-same-document navigations, aligning with legacy behavior and improving predictability. (695e711)
  • The New Tab Page 'Simplified UI' for Action Chips has been permanently removed, making the grid/flex-wrap view the only option. (e54421d)

Services & Components

  • In Autofill, the feature flags for CVC storage and filling (kAutofillEnableCvcStorageAndFilling) have been removed as the feature is now fully launched and enabled by default. (9249343)
  • Logic for the 'fallback to default DoH provider' feature has been moved into the network service and is now gated by the Enhanced Safe Browsing preference, preventing probes for ineligible users. (ac79bac) (baf926c) (83aea84)
  • The SplitHostCacheByNetworkAnonymizationKey feature flag was revived to allow experiments to selectively disable DNS partitioning while keeping other network state partitioned by the Network Anonymization Key. (8176f94)

Core & Internals

  • In Mojo, support was added for a direct return type for result<T, E> interfaces, enabling simpler, straight-line asynchronous code without requiring a callback for the result. (ff6c30c)
  • A large-scale refactoring updated many callers of LayoutObject::Style() to use the null-safe StyleRef() variant, improving code safety and reducing potential null pointer dereferences. (c76de26)
  • ⚠️ BREAKING The public API signature of PaintCanvasVideoRenderer::CopyVideoFrameTexturesToGLTextureViaIntermediateSI was changed to require a VideoFrameSharedImageCache* argument, breaking existing callers. (8c4ade7)
  • Numerous components, including //chrome/browser/speech, //chrome/browser/device_reauth, and //chrome/browser/collaboration, underwent significant refactoring to modularize build dependencies and clean up includes. (b1d2c70) (c51ae4c) (340a3ca)
