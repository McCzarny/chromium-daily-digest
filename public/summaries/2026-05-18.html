Summary for 2026-05-18 on branch 'main'

Chromium Digest: 2026-05-18

Overview

On a busy day with 568 total commits (392 relevant), the main branch saw a significant number of breaking API changes across core browser, sync, and media components. Major refactoring efforts continue, alongside security hardening and the ongoing 'GlowUp' UI refresh. The first and last relevant commits were (de2beb1) and (f48d543).

Breaking Changes

  • ⚠️Over 20 enterprise policies have been removed, including AppLaunchAutomation and LegacySameSiteCookieBehaviorEnabled. Administrative scripts and configurations relying on these policies will no longer function. (bf54c2e) (94ea7ec)
    PolicyEnterpriseChromeOS
  • ⚠️Widely-used browser lookup helper functions in chrome/browser/ui/browser_finder.h (e.g., FindTabbedBrowser, FindBrowserWithProfile) have been removed. Developers must migrate to the new ProfileBrowserCollection and GlobalBrowserCollection APIs. (1d867aa)
    BrowserUITabs
  • ⚠️The public WebContentsDelegate::OnDidBlockNavigation method signature was changed to include initiator_url and initiator_origin. Embedders implementing this delegate must update their code to match the new signature. (64a041d)
    NavigationBrowser
  • ⚠️The public SyncService interface has been modified, removing the deprecated IsSyncFeatureEnabled() and IsSyncFeatureActive() methods. The behavior of GetSelectedOsTypes() has also changed, which may impact embedder logic. (9efc428)
    Sync
  • ⚠️The core MojomParse trait in Rust Mojo bindings now requires a generic Context parameter, breaking all existing implementations. Code must be updated to use new conversion methods instead of standard From/Into traits. (bf955d0) (4fb6547)
    BaseNetwork
  • ⚠️The public Java OmniboxActionFactory on Android was refactored from an interface to a class with static factory methods. Downstream projects using the previous factory interface must update their code. (7018851)
    AndroidUI
  • ⚠️The ExtensionKeybindingRegistry::Delegate class has been removed and the ExtensionKeybindingRegistry constructor signature has changed, breaking integrations that manage extension keyboard shortcuts. (f08221c)
    ExtensionsUI
  • ⚠️The return type of WASAPIAudioOutputStream::RenderAudioFromSource on Windows was changed from bool to HRESULT, breaking binary compatibility for consumers of this public media API. (a43cf79)
    MediaWindows
  • ⚠️A dependency roll for HarfBuzz now requires a minimum FreeType version of 2.8.0. This is a breaking change for build systems that may be using an older version of FreeType. (8c34f88)
    BuildFonts
  • ⚠️The public BrowserFocusController::Delegate class has been removed as part of a refactor, breaking embedders of //chrome who may have been using or subclassing this delegate. (64c1527)
    UIBrowser
  • ⚠️The public API setRendererLibraryPrefetchMode has been removed from Android WebView's AwContentsStatics and the support library, breaking any embedder using it. (f4dcadf)
    AndroidPerformance
  • ⚠️The SharingService API was refactored, renaming the public method SendUnencryptedMessageToDevice to SendIosPushMessageToDevice and removing the underlying delegate mechanism. (cf82324)
    BrowseriOS

Security

  • A Time-of-Check Time-of-Use (TOCTOU) race condition in speech recognition was fixed, which could have allowed an extension to bypass audio capture permissions. (0086720)
    SecurityExtensionsMedia
  • The password manager's check for cross-origin iframes was improved to validate the entire ancestor chain, preventing potential password leakage in complex 'sandwich' (A-B-A) iframe scenarios. (b80505d)
    PasswordManagerSecurity
  • The Android WebShare API now validates that shared URLs use HTTP or HTTPS, and will terminate a renderer that attempts to pass an invalid scheme. (38749a0)
    SecurityAndroidBlink
  • Validation for ICC color profiles fetched from the X11 root window was hardened to prevent potential memory exhaustion and integer overflows. (826fa20)
    SecurityLinuxGPU
  • An ASLR bypass vulnerability was fixed by replacing raw pointer logging with opaque tokens in the audio service. (552949a)
    SecurityMedia

Web Platform & Blink

  • A new syntax for line-clamp has been implemented: max-lines: <integer> auto. This allows clamping by both line count and height, improving web compatibility. (7eec7df)
    BlinkCSS
  • Print requests originating from fenced frames are now rejected. A compromised renderer attempting to print from a fenced frame will be terminated. (07e3684)
    BlinkPrintingSecurity
  • The runtime flag SvgTransformOnNestedSvgElement has been removed, making the feature permanently enabled and allowing transform attributes on nested <svg> elements. (c4963ac)
    BlinkHTML

UI & UX

  • The "GlowUp" UI refresh continues with new rounded icons and themed colors across many surfaces, including Downloads, Extensions, Certificate Manager, and the Message Center. (08cd8e0) (50404b4) (1db29e1) (227aeee)
    UIThemes
  • The Omnibox context menu has been simplified by removing several items, including emoji, writing direction, and autofill options, as part of the 'GlowUp' refresh. (e9141f4)
    UI
  • A new "Lock Screen" trigger was implemented for the Smart Restart feature, which can automatically restart the browser for pending updates when the screen is locked. (188f353)
    UpdaterUI
  • A bug was fixed where holding the Esc key would not exit fullscreen mode if focus was on a native UI element like the Omnibox. (92368d9)
    UIBrowser

Android

  • Windowed Incognito mode is now enabled by default on large-screen Android devices. (e57ef26)
    AndroidUIPrivacy
  • The fontSettings.getFontList() extension API on desktop Android was updated to return a list of generic font families (e.g., serif, sans-serif) to provide compatibility, as Android lacks a system API to enumerate all installed fonts. (a9f439b)
    AndroidExtensionsFonts
  • A prompt to upgrade from approximate to precise location is being introduced for the Android Geolocation API. (b50ce6e)
    AndroidPrivacy
  • A rendering issue with zero-copy camera frames using Dawn/Graphite was fixed, which previously caused swapped U and V planes (pink/green colors). (e153883)
    AndroidMediaGPU

Sync & Accounts

  • A crash caused by corrupted local Nigori (encryption) data is now prevented by validating the data on startup and allowing for a safe reset. (0ab182f)
    SyncCrypto
  • The process for setting up a new custom passphrase was refactored to move key derivation into the SyncEngine, simplifying the API surface. (fc62658)
    SyncCrypto

Build & Infrastructure

  • A new performance bot, android-pixel10_webview-perf, was added to CI and Pinpoint configurations to run WebView performance tests. (3cb8f70)
    InfraTestingAndroid
  • The Rust toolchain was repackaged and several third-party Rust crates were updated. (748a3f2) (362b960)
    BuildInfra
