Chromium Weekly: 2026 Week 15

Overview

An exceptional week for API evolution, this summary covers 14 relevant changes from 2976 commits across 6 days. The week was dominated by a wave of breaking changes across nearly every core subsystem as part of a broad effort to modernize APIs, improve security, and refactor the codebase under projects like 'Bedrock'. Key events include a fundamental overhaul of the Android JNI layer, the removal of experimental features like Desktop Link Preview, and the raising of the minimum iOS deployment target to 18.0.

API Modernization & Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE The Android JNI layer was fundamentally refactored. The public JniUtil.java and JniInit.java classes were removed, with functionality consolidated into JniZero.java. This is a major breaking change for Android embedders interacting with JNI. (d75bfa0) (cafc078)
  • ⚠️ BREAKING BREAKING CHANGE The minimum required iOS deployment target has been raised to 18.0. Builds targeting older iOS versions will now fail. (f870b78)
  • ⚠️ BREAKING BREAKING CHANGE Core task management APIs in base::task::SequenceManager were updated, with SetDefaultTaskRunner being replaced by SetDefaultTaskQueue. Additionally, base::ObserverList now disallows re-entrant notifications by default on desktop platforms, which may cause DCHECK failures in existing code. (4b73a31) (83dc1d0)
  • ⚠️ BREAKING BREAKING CHANGE As part of the 'Bedrock' project, legacy browser management functions like chrome::GetTotalBrowserCount() and chrome::FindBrowserWithID have been removed in favor of a new BrowserCollection API. (ebcb57e) (fb39bae) (2a4a86d)
  • ⚠️ BREAKING BREAKING CHANGE Numerous core content, browser, and extensions interfaces received new pure virtual methods or signature changes, requiring embedders to update their implementations. Affected interfaces include content::RenderWidgetHostView, content::RenderFrameHostManager::Delegate, ContentBrowserClient, and extensions::RuntimeAPIDelegate. (1e42bd2) (b83f5c5) (7d3c950) (011e693)
  • ⚠️ BREAKING BREAKING CHANGE Core networking and storage APIs were updated for type safety and modernization. Methods on net::URLRequestJob and net::HttpTransaction now return the type-safe base::ByteSize instead of int64_t, and the StorageAccessHandle::Estimate() mojom method was similarly updated. (72bf5dc) (79bbb97) (05ad8e7)

Web Platform & APIs

  • Several experimental features were removed from the codebase, including Desktop Link Preview (kLinkPreview), ServiceWorkerAutoPreload, and CSSSelectorFragmentAnchor, cleaning up the platform and reducing maintenance overhead. (831a9eb) (2bb2c50) (d57f326)
  • ⚠️ BREAKING BREAKING CHANGE The behavior of several web APIs was changed to align with specifications. element.scrollIntoView() now resolves its promise only after smooth scrolling completes, the Highlight API now uses live iteration instead of a snapshot, and support for width and height as presentation attributes on nested <svg> elements has been removed. (cd669ef) (df5462c) (facfdaa)
  • The WebCrypto API was expanded to support the post-quantum key encapsulation mechanisms ML-KEM-768 and ML-KEM-1024, a significant step towards post-quantum cryptography readiness. (b582009)
  • The SharedWorkerExtendedLifetime feature is now enabled by default, allowing shared workers to live longer under certain conditions to improve reliability for complex web applications. (070a08b)

Security & Privacy

  • Numerous Use-After-Free vulnerabilities were fixed across the codebase in components like WebContentsImpl, ServiceWorkerCacheStorageMatcher, PasswordFormManager, and TransientWindowManager, improving browser stability and security. (899e77f) (d17fe11) (c542194) (ab2ec9c)
  • Security was hardened in several areas: the H.264 parser now validates slice data to prevent out-of-bounds access, renderer input for texture coordinates is now validated in the browser process, and a same-origin check is now enforced for DedicatedWorker main script fetches. (94ad12b) (acafd83) (7d3c950)

Code Health & Infrastructure

  • The TypeScript compiler was upgraded to v6.0.2. Developers working on WebUI may need to update their code to address new deprecations and stricter type checking. (83a142e)
  • A large-scale refactoring continued to migrate components from the legacy base::Singleton pattern to the modern base::NoDestructor, particularly across Keyed Service Factories. (6d036b4)
