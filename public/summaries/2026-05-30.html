Chromium Digest: 2026-05-30

Overview

This report summarizes key changes across 87 analyzed commits, focusing on significant architectural migrations, security hardening, and API modifications. The activity spans from commit (1638833) to (1639041).

Critical Behavioral & API Changes

  • ⚠️The alarms.create() method now enforces a strict 1024-character limit on alarm names, promoting a runtime error instead of a previous warning. This represents a significant behavioral change for extension developers relying on dynamic alarm keys. (c0032de)
  • HTTPS-First Mode (HFM) logic now prevents incorrect fallbacks to HTTP when encountering client-side blocks (e.g., Safe Browsing), and the balanced mode setting is now synced with Enhanced Safe Browsing states. (9c97035) (79b48f0)
    SSLPrivacy

Security & Memory Safety

  • Critical memory safety patches for Bluetooth subsystems, migrating to owning net::IOBufferWithSize to prevent Use-After-Free and Use-After-Unmap vulnerabilities in asynchronous socket writes. (8265efd) (e940e42)
    BluetoothSecurity
  • Refined browser-side verification for popup content extraction, ensuring that only active, visible widgets can extract content, mitigating potential background-process exploits. (366d386)
    SecurityBrowser

Infrastructure & Ash Modernization

  • Continued architectural decoupling of Ash from ash::ProfileHelper by migrating to BrowserContextHelper and components/user_manager. Test suites modernized to use event-based wait primitives like SearchResultsChangedWaiter instead of RunUntilIdle(). (bd9fe23) (05b5543) (53bdd62)
    AshChromeOSTesting
  • Added FFI support for base::Feature to allow Rust-based modules to interact directly with Finch field trials, furthering Chromium's Rust adoption strategy. (30ed7c9)
    BaseInfra

Feature Integration & UI/UX

  • Expanded Glic feature support with Enterprise DLP integration for drag-and-drop, wiring for Android UI components, and new accessibility labels. (1eb4f40) (c6d88d1)
    AIAndroidUI
  • Implemented new PWA height constraints and strike-based UX for Pix account linking, improving anti-spoofing protections and user communication. (43bc35a) (8b67bb7)
    PaymentsAutofillUI
