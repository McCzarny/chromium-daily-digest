Chromium Digest: 2025-11-19

Overview

This digest summarizes a day of development on the Chromium main branch, covering 662 commits from (29bb9f0) to (a96996f). Today's changes feature a significant number of breaking changes across the codebase, including the renaming of CSS `display: masonry`, the removal of the foundational `blink::Supplementable` class, and major API signature changes in core components.

Major Breaking Changes

  • ‚ö†Ô∏è BREAKING The experimental CSS feature display: masonry has been renamed to display: grid-lanes, along with related properties like inline-masonry. Web content, polyfills, or tests using the old masonry keyword will break. (76cc361) (7d1e648)
  • ‚ö†Ô∏è BREAKING The foundational blink::Supplementable base class and its header have been completely removed. This is a major refactoring that requires updates to any downstream code that used this mechanism to attach extra data to Blink objects. (4c9ba80) (ee79c91)
  • ‚ö†Ô∏è BREAKING The public Blink interface blink::MemoryPressureListener has been removed. Downstream projects must migrate to the base::MemoryPressureListener interface to handle memory pressure events. (c022244) (cf35b90)
  • ‚ö†Ô∏è BREAKING A roll of the libc++ dependency removed several standard library headers, including . Code that explicitly includes these now-removed files will fail to build. (afb865d) (1cdc8d8)
  • ‚ö†Ô∏è BREAKING The persistent cache component's API was refactored, removing the BackendParams class in favor of a new PendingBackend model. This alters the public API for creating and initializing cache backends. (da81816)
  • ‚ö†Ô∏è BREAKING Methods in base::flat_map and base::flat_tree now use templated signatures to support heterogeneous lookup. This change may break code relying on the previous, non-templated signatures. (3de1d9d)

Web Platform & APIs
