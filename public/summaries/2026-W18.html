Week 18 of 2026 (2026-04-26 to 2026-05-01)

Chromium Weekly: 2026 Week 18

Overview

This week in Chromium saw an exceptionally high volume of breaking API changes, impacting core components from the compositor and UI to Android sign-in and the base library. Major features like Web Feed and the WebUI Tab Strip were completely removed. Alongside widespread refactoring efforts, security was significantly hardened against renderer exploits and policy bypasses, while new user-facing features like Vertical Tabs by default and a 'Listen to this page' option were introduced. The week's activity covered 1949 relevant commits out of 2635 total across 5 days.

Major Breaking Changes & API Removals

  • ⚠️ BREAKING BREAKING CHANGE The Web Feed (Following) and WebUI Tab Strip features have been completely removed from the codebase. All associated components, APIs, and feature flags like kWebFeedSort have been deleted, and builds enabling enable_webui_tab_strip will now fail. (3c759c0) (17de746) (c6e0135) (7532c85)
  • ⚠️ BREAKING BREAKING CHANGE The public C++ compositor APIs were significantly refactored. Key virtual methods on LayerTreeHostImpl were moved to a new ClientLayerTreeHostImpl subclass, and the core cc::LayerTreeHostClient class was renamed to cc::LayerTreeHostDelegate. (144eb65) (afd22cd)
  • ⚠️ BREAKING BREAKING CHANGE Public Java interfaces for sign-in on Android (IdentityManager, SigninManager) were heavily modified. The deprecated ConsentLevel enum and related methods were removed, requiring embedders to update their sign-in and sign-out flows. (22ad081) (11da9dd) (1c80e19) (436359d)
  • ⚠️ BREAKING BREAKING CHANGE The widely used chrome::FindBrowserWithTab function is being removed, and fullscreen-related methods have been moved from BrowserWindow to a new BrowserWindowFullscreenController. Callers must migrate to new APIs like GlobalBrowserCollection. (48bd889) (d1db08e)
  • ⚠️ BREAKING BREAKING CHANGE Numerous core C++ and component APIs were broken. content::WebContentsDelegate gained a new pure virtual method IsDocumentPictureInPictureBlockedBySystem. The webapps::ManifestId was refactored from a GURL to a wrapper class. base::RandInt and base::DateIntervalFormat were removed from the base library. (bcd4839) (73319ea) (5f591f6) (1c49b4d)
  • ⚠️ BREAKING BREAKING CHANGE The public remoting component APIs were cleaned up with the removal of non-WebRTC transport protocols and legacy authenticators. The remoting::RemoteAudioInput class was also renamed to remoting::AudioInjector. (e548872) (32d22ac) (6b2679d)
  • ⚠️ BREAKING BREAKING CHANGE The public OmniboxPopupPresenterBase interface now requires implementations to return std::optional<base::TimeDelta> from ShouldDeferUntilVisualStateReady and implement the new pure virtual method ShouldDetachWebContentsOnHide. (c708812)

Security Hardening

  • Browser defenses against compromised renderers were strengthened. A check for invalid navigation headers was upgraded from a crash report to a full renderer process kill, and IPC validation was improved in EventRouter to prevent spoofed extension events. (e0a0f3e) (6271562)
  • Multiple policy bypasses were fixed. A Cross-Origin Resource Policy (CORP) bypass in the Service Worker Static Router was patched, and a CSP bypass in Manifest V2 extensions that allowed script execution was fixed. (b671011) (5046d3c) (99e2591)
  • Several Use-After-Free vulnerabilities were addressed in components like AXTreeSourceAndroid, the macOS FilePathWatcherFSEvents, and the extension UserScriptSet. (55bfe49) (cb6f7a4) (fe455e2)
  • Data leaks were plugged in several areas. Sensitive information is now stripped from COEP/DIP violation reports, a DevTools session detachment issue leaking cookies was fixed, and the Web Share API was prevented from sharing local file: URLs. (59bf0e2) (b1cb06e) (65fe959)
  • The RejectWeakCiphertext feature is now enabled by default to block weak symmetric ciphers in TLS, and initial support for ML-DSA signatures in TLS was added behind a feature flag. (5b951ab) (a2638b6)

Web Platform & Blink

  • The CSSGapDecoration feature for styling gaps in flexbox and grid layouts is now enabled by default. Additionally, support was added for calc() expressions mixing <angle> and <percentage> types in conic-gradient(). (4934ed0) (2f0622a)
  • As part of feature cleanup, the window.fence.notifyEvent API and related Fenced Frames code were removed. The experimental ::interest-hint pseudo-element was also renamed to ::interest-button to align with specifications. (3cca452) (e9430ad) (bb7289f)
  • A new kWebNNModelCompilation sandbox type was added for the WebNN API, which will be used to isolate ONNX Runtime graph compilation in a future utility process. (1adb86c)
  • ⚠️ BREAKING For extensions, support was added for custom context menu items in the browser's tab menu, enabled by the kExtensionTabContextMenu flag. The webstorePrivate permission was also removed. (ad49992) (de56b30)

UI & UX

  • The Vertical Tabs feature is now enabled by default on Windows, Mac, and Linux. (20e810a)
  • A new 'Listen to this page' item was added to the desktop context menu, controlled by the kImprovedReadAloud feature flag. (021e219)
  • The tab hover card UI was enhanced to display the memory usage of the hovered tab. (605b071)
  • The voice search UI was updated with a new 'Aurora' recording animation and an explicit stop button, gated by a feature flag. (efb148a) (6a4b098)

Mobile Platforms

  • On Android, the UnthrottleAsyncTouchMoves feature is now enabled by default (excluding WebView) to improve gesture responsiveness. In WebView, Google Mobile Services (GMS) calls are now deferred until after Chromium startup by default to improve performance. (d40e41a) (305be3b)
  • On iOS, app startup performance was improved by deferring the initialization of the Tab Grid UI until it is needed. The Tab Switcher Overflow Menu is also now enabled by default. (0f67083) (0113879) (d4b0cb7)
  • The Android AutocompleteController API was updated to accept a WebContents parameter, enabling more contextual suggestions. (2b80341) (c96df14) (4924e16)

Core Internals & Refactoring

  • ⚠️ BREAKING A significant effort continued to replace raw integer IDs with strongly-typed wrappers. This includes migrating callers to use content::ChildProcessId in ProcessMap and content::GlobalRenderFrameHostId in speech recognition APIs. (ab62ab0) (2686e85) (5ee42da)
  • ⚠️ BREAKING BREAKING CHANGE The JNI generation contract for Android has changed. The module_name must now be specified in the generate_jni GN build target instead of the @NativeMethods Java annotation, requiring updates to both source and build files. (629046f)
  • The core browser navigator logic was extracted from //chrome/browser/ui into a dedicated //chrome/browser/ui/navigator build target and directory to improve modularity. (a7ea861) (815c5cf)
  • Support for the legacy pre-mmap Safe Browsing file format has been removed. Instead of migrating old databases, Chromium will now return an error, prompting a full database update. (158b83a)
