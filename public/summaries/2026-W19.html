Week 19 of 2026 (2026-05-03 to 2026-05-08)

Chromium Weekly: 2026 Week 19

Overview

This week saw an exceptionally high volume of breaking API changes across the codebase, impacting the compositor, GPU, content, Android, and Autofill APIs. A total of 2690 relevant commits landed out of 3586 total across 6 days. Despite the churn, several new web platform features were shipped, including popover=hint and new CSS media pseudo-classes, while security was hardened around file pickers and user activation.

Major Breaking Changes

  • ⚠️BREAKING CHANGE: A major refactoring of the Compositor and GPU APIs occurred. Core interfaces like cc::LayerTreeHostImplClient and cc::LayerTreeHostSchedulingClient were renamed to use a ...Delegate suffix. Additionally, constructors for SharedImageBacking and its subclasses now require a SharedImageInfo struct, and the tex_coord_rect field was removed from CompositorRenderPassDrawQuad, breaking direct interaction with compositor frame data. A new pure virtual method, CouldDisplayBeforeUnloadDialog(), was added to content::RenderFrameHost. The return type of content::WebContents::ForSecurityDropFullscreen was also changed to prevent a use-after-free vulnerability. (01d357d) (8229aec) (058c2d6)

  • ⚠️BREAKING CHANGE: Several breaking changes were made to the Android and JNI layers. The public WebView API getAwContents() now blocks until initialization, potentially causing ANRs. A new pure virtual method SetDrawPaused(bool) was added to content::Compositor. The JNI layer also saw breaking changes, including the removal of the ("module_name") parameter in the @NativeMethods annotation. (97e5ca6) (6c313d1) (8567d0e)

Web Platform

  • Several new CSS and HTML features were enabled by default. This includes the popover=hint attribute for non-modal popovers, new media pseudo-classes like :playing and :paused, support for shape-outside basic shapes, and the page-margin-safety descriptor for printing. Additionally, standard text-decoration properties are now honored on SVG text. (a4e8e34) (cefbd1e) (9611ae0) (798c128) (ec86856)
    BlinkCSSHTML
  • ⚠️BREAKING CHANGE: Web platform APIs saw significant behavioral and breaking changes. The scrollIntoView() API's default alignment changed to "auto". The Long Tasks API now redacts attribution for cross-origin iframes for privacy. On Android, the web-serial-wired-devices-android feature was removed entirely, and Web Bluetooth now correctly blocks notifications for blocklisted characteristics. (5972c54) (f34f0ba) (1113e33) (8b64a2a)
    BlinkPrivacyUSBBluetooth
  • The focusgroup accessibility feature was enabled by default, graduating from its origin trial. This feature allows developers to define a navigation boundary for composite widgets, improving keyboard navigation and interoperability for custom components. (5fdbeaa)
    AccessibilityBlinkHTML

Security & Platform Specifics

  • Security was enhanced across the platform. The File System Access API is now more restrictive, blocking file pickers in cross-origin subframes by default. The browser now enforces that a page must have user activation to show a beforeunload dialog. Additionally, workers created from data: URLs now correctly receive a new, unique opaque origin. (3fd5e2c) (aa69447) (ee6bfd1)
    SecurityBlinkUI
  • ⚠️BREAKING CHANGE: Major features were removed from specific platforms. On ChromeOS, the ability to install Debian (.deb) packages from the Files app UI was removed, and the document_scan Crosapi was deleted. The cross-platform Web Feed (Following feed) feature was also completely removed. (fb90834) (46d30e8) (19de65f)
    ChromeOSLinuxUI
  • On Android, the native PDF viewer has been refactored to be loaded on demand as a dynamic feature module. This change improves initial startup performance and reduces the memory footprint of the browser. (15ba7f5) (bd7f626)
