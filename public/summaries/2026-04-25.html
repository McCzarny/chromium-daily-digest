Chromium Digest: 2026-04-25

Overview

This digest covers 55 commits, highlighting several major behavioral changes to the Popover, WebGL, Geolocation, and Web App Manifest APIs. Key updates include the removal of an iOS-specific Autofill API, security enhancements for Live Captions and the Geolocation service, and numerous bug fixes across rendering, UI, and media components.

Web Platform & APIs

  • ⚠️ BREAKING BREAKING: The Popover API implementation has been updated to align with recent specification changes. This introduces stricter re-entrancy protection, throwing a DOMExceptionCode::kInvalidStateError if showPopover() is called during an existing show/hide operation. Additionally, popover=auto elements opened within a popover=hint will now be 'downgraded' to hint behavior instead of throwing an error. Web content relying on the previous, more lenient behavior may break. (712f6f0)
  • ⚠️ BREAKING BREAKING: Web App Manifest validation is now stricter. The browser process will now enforce that start_url, id, scope, share_target.action, file_handlers.action, and protocol_handlers.url are all same-origin with the document. Manifests with cross-origin URLs in these fields will now be rejected, which may break web apps that previously relied on the more lenient validation. (0b080ce)
  • The WebTransport API now supports the congestionControl hint as a constructor option, allowing developers to suggest a preference for 'throughput' or 'low-latency' algorithms, aligning with the W3C specification. (7faefb2)
  • Corrected multiple bugs in HTML forms, including a fix for updating <selectedcontent> in listboxes (87144c6512872f082d6265584bec96e0287d5aeb), ensuring :valid/:invalid pseudo-classes update correctly for <select multiple size=1> (6819a106c96188395dda3c08050ab97185b6ac9d), and preventing a crash by disconnecting the SelectMutationObserver when a <select> element is removed from the document (36ab6e149a627aa217e8de7f882442a5f23220bd). (87144c6) (6819a10) (36ab6e1)
  • The HTML parser's fast path has been fixed to correctly handle nested <li> elements, aligning with a patch from WebKit. (c96815f)
  • Resolved threading violations and race conditions in the Android implementation of the WebShare API, improving stability during file transfers. (5678c0f)

Security & Privacy

  • ⚠️ BREAKING BREAKING: The Geolocation API's behavior has changed to mitigate a privacy risk. The 15-minute throttle on frequent requests for approximate locations will now persist even if all clients disconnect and reconnect. This prevents malicious sites from bypassing the throttle to reconstruct a precise location from multiple approximate fixes. Applications relying on the throttle resetting on disconnect will be affected. (b76dfb0)
  • Fixed a JSON injection and out-of-bounds read vulnerability in the Live Caption translation feature. The code now uses structured JSON generation (base::WriteJson) instead of StringPrintf to prevent malicious peers from injecting arbitrary JSON or causing a denial of service. (a1ab0fb)
  • The Chrome Elevation Service on Windows now uses an explicitly system-locked secure temporary directory (base::GetSecureTempDirectory()) for its recovery child process, providing defense-in-depth against potential insecure C:\Windows\Temp configurations. (3297b1b)

GPU & Graphics

  • ⚠️ BREAKING BREAKING: WebGL behavior has been changed for spec compliance. Attempting to relink a GL program while transform feedback is active will now fail and generate a GL_INVALID_OPERATION error. Previously, this operation may have succeeded, leading to state desynchronization. WebGL applications that relink programs during active transform feedback will need to be updated. (8617532)
  • A race condition in the Windows hardware video encoders (D3D12 and Media Foundation) has been fixed. Read access on a VideoImageRepresentation is now correctly maintained for the entire duration of an encode operation, preventing potential heap corruption from concurrent write access. (28ec09a)
  • Fixed a bug where transform feedback could be accidentally left active when switching to a newly created GL context, by ensuring the state restorer is properly initialized first. (f41b46e)
  • Support for backdrop-filter images has been added to the HTML-in-canvas drawing feature, allowing these effects to be correctly rendered when using drawImage() with an HTML element. (27dafe4)

Autofill & Omnibox

  • ⚠️ BREAKING BREAKING: The PlusAddressEmailOverride feature has been removed. This includes the removal of kPlusAddressEmailOverride from the AutofillSnackbarType enum on Android and, more significantly, the removal of the showPlusAddressEmailOverrideNotification method from the public AutofillClientIOSBridge protocol. iOS embedders using this method will need to remove it from their implementations. (1fd3309)
  • The abandoned In-Product Help feature for Plus Address creation suggestions (kIPHPlusAddressCreateSuggestionFeature) and all its associated code have been removed. (292966b)

iOS

  • Internal sign-in logic on iOS has been refactored, moving EPC (External Privacy Context) fetching from SigninAccountCapabilitiesSceneAgent to the centralized SystemIdentityManager. (f468cf4)

UI & Accessibility

  • An accessibility bug has been fixed where screen readers would not announce an updated tab group name. The accessible name for tabs within a group is now properly updated when the group's title changes. (fb35a0d)
  • The behavior of the Android framebust message now matches desktop: the message is now correctly dismissed on all cross-document main frame navigations. (589cfed)
  • Fixed an accessibility tree consistency failure related to <br> elements that have DOM children added via script. The logic now correctly marks the node as dirty to ensure the accessibility tree is updated. (84f83aa)

Developer Experience & Tooling

  • A new test helper, content::ReadyForInputObserver, has been added to help write more reliable browser tests. It allows tests to wait until a frame's input router is active before sending synthetic input, reducing flakiness caused by the kDropInputEventsWhilePaintHolding feature. (4398d46)
  • Several Extensions API tests (SystemStorageApiTest, SocketsTcpServerApiTest) have been migrated from the deprecated "app shell" environment to run as part of the standard chrome/browser_tests suite, improving test reliability and realism. (dc4393c) (0aeb02c)
  • The check-std-ranges-pipe-operator clang plugin option has been turned into a no-op, as part of a cleanup to eventually remove the option entirely. (b6dc473)
