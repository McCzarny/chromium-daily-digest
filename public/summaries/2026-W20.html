Week 20 of 2026 (2026-05-10 to 2026-05-15)

Chromium Weekly: 2026 Week 20

Overview

This week saw 2517 relevant commits out of 3344 total. The period was characterized by an exceptionally high volume of breaking API changes affecting core C++, Java, and Mojo interfaces across nearly all major subsystems. Significant developments include the removal of the <webview> tag on desktop, major refactorings in PartitionAlloc and the compositor, and numerous security hardening measures.

Breaking Changes & API Removals

  • ⚠️BREAKING CHANGE: The <webview> tag and chrome.webviewTag API have been removed from Chrome Apps on Windows, Mac, and Linux, restricting the feature to ChromeOS only. This breaks a stable API used by desktop Chrome Apps. (dfa28c4)
    ExtensionsBrowserChromeOS
  • ⚠️BREAKING CHANGE: Core PartitionAlloc headers, such as partition_root.h, have been moved into an internal/ subdirectory. This public API change breaks builds that directly include these headers from their old locations. (cd836a8) (7c766eb)
    BaseBuild
  • ⚠️BREAKING CHANGE: The monolithic //chrome/browser build target has been broken into many smaller source_set targets. Embedders must update their BUILD.gn files to depend on the new, more specific targets. (a235e83)
    BuildBrowser
  • ⚠️BREAKING CHANGE: Numerous core C++ APIs were broken. Highlights include the removal of ycbcr_info() from media::VideoFrame, a new color customization API for views::Textfield, new pure virtual methods on content::WebContentsObserver, and a signature change to blink::URLLoaderThrottle::BeforeWillRedirectRequest. (55b4406) (b44a41b) (4a7f0a4) (8c41713)
    MediaUIBlinkNetwork
  • ⚠️BREAKING CHANGE: Web-facing behavior was altered in several ways. The Web Crypto SubtleCrypto.verify() return type is now a spec-compliant Promise<boolean>, ARIA role parsing is now stricter, and WebUSB now blocks standard control transfers via the generic controlTransfer method. (14ff7bb) (09e48bc) (3169e98)
    BlinkCryptoAccessibilityUSB
  • ⚠️BREAKING CHANGE: The long-deprecated app_shell binary and its associated sources have been completely removed from the extensions system, breaking any remaining build dependencies. (75cd2c6) (e2a0758) (937f006)
    ExtensionsBuild

Security & Privacy

  • Memory safety was enhanced by enabling PartitionAlloc's MiracleObject (PartitionAllocWithAdvancedChecks) by default for the Browser UI process, helping to detect and prevent Use-After-Free vulnerabilities. (34c2ad4)
    SecurityBasePerformance
  • Sandbox protections were hardened on multiple platforms. On Windows, the network service now brokers connections to non-publicly-routable addresses to avoid OS firewall issues. On Linux, AppArmor profiles are now included in installer packages, fixing sandbox issues on newer distributions. (3129fd2) (d4c2b79)
    SecurityNetworkWindowsLinux
  • Multiple potential Site Isolation bypasses were fixed. MHTML subframes are now forced to commit with opaque origins, and the Web Serial API now blocks access for non-default StoragePartitions to prevent cross-origin data access. (e4b92a6) (16e3c60)
    SecurityBlinkStorage
  • User privacy was improved in several areas. Glic (Gemini in Chrome) screenshots now ensure sensitive fields in out-of-process iframes are redacted. Additionally, active sensor connections are now severed immediately when a user revokes the 'Sensors' permission for a frame. (f588883) (19452b0)
    PrivacySecurityAIBlink

Web Platform & Rendering

  • The CSS text-fit property has been enabled by default and is now considered stable. Additionally, the chain value for the overscroll-behavior property has shipped, allowing developers to explicitly control scroll chaining. (5294214) (424d2b9)
    BlinkCSS
  • The TreesInViz compositor architecture, which moves more layer tree management into the Viz process for better performance and isolation, is now enabled by default on most platforms. (b6929d0)
    CompositorGPU
  • A Service Worker interceptor has been added to the download manager, allowing service workers to handle downloads initiated from browser UI actions like 'Save As'. (a2fbe85)
    ServiceWorkerDownloadsBlink
  • The async clipboard read() and readText() APIs were updated to use non-blocking Mojo IPCs, preventing the browser UI from freezing when pasting content from applications that are slow to respond. (a274f36)
    BlinkPerformanceUI

Performance & Memory

  • The compositor's ResourcePool will now discard all resources when the viewport is resized, significantly reducing memory spikes and improving performance during window resizing operations. (d6a5cb9)
    PerformanceCompositor
  • Layout performance for pages with complex text has been improved by caching SkStrike objects per font, reducing expensive digest computations, especially for CJK characters. (51e12ce)
    BlinkPerformanceFonts
  • The migration from base::MemoryPressureListener to the more flexible base::MemoryConsumer API continues, allowing for more granular memory interventions and better resource management. (a5ea88b) (cd4a600) (3eec1c7)
    PerformanceBase

Platform Specific

  • On Android, the multi-week refactoring of the Custom Chrome Tabs (CCT) toolbar was completed. The OpenH264 software encoder is now enabled by default for WebRTC and WebCodecs, and startup performance was improved by deferring ContentView creation for frozen background tabs. (e6f2c6b) (a4f5a3b) (3cbee39)
    AndroidUIMediaPerformance
  • On ChromeOS, the large-scale effort to decouple Ash from the browser continues, with numerous components migrated to use lower-level Ash APIs. In a breaking change, several methods like CreatePrintJob and CancelPrintJob were removed from the crosapi::mojom::LocalPrinter interface. (4d70581) (0847328) (aca3120)
    ChromeOSAshPrinting
  • On iOS, entry points for Gemini and other AI features were unified into a generalized coordinator. For security, passkey support was integrated into the manual fill mediator, allowing passkeys to be displayed in the fallback UI. (41840d4) (e22b581)
    iOSUIAIPasswordManager
