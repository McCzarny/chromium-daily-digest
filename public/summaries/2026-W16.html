Chromium Weekly: 2026 Week 16

Overview

This week in Chromium saw an exceptionally high volume of breaking changes, driven by major refactoring projects in the Password Manager, networking stack, and core browser components. Key developments include the complete removal of the legacy Text Autosizer feature and synchronous OSCrypt backend, significant security hardening for web workers and media requests, and continued progress in modularizing the main //chrome/browser build target. Across 6 days, developers landed 2643 relevant commits out of 3514 total.

Major API & Feature Removals

  • ⚠️ BREAKING BREAKING CHANGE A massive refactoring of the Password Manager has begun, replacing the PasswordForm struct with StoredCredential. This breaks the public API of LoginDatabase and other components, requiring significant updates for embedders interacting with the password store. (fa3099d) (167b1b4) (a4434fb)
  • ⚠️ BREAKING BREAKING CHANGE The legacy synchronous OSCrypt backend (components/os_crypt/sync) has been completely removed. Embedders must migrate from the public OSCrypt::EncryptString and OSCrypt::DecryptString APIs to the modern os_crypt/async backend. (a4cab4a) (21b80f4)
  • ⚠️ BREAKING BREAKING CHANGE The legacy Text Autosizer feature has been completely removed from the Blink rendering engine. Embedders who relied on this for automatic text inflation on mobile will need to implement their own scaling logic. (4797e6b) (f8c89b4) (2c0162e)
  • ⚠️ BREAKING BREAKING CHANGE The chrome.usb, chrome.bluetooth, chrome.bluetoothLowEnergy, and chrome.bluetoothSocket APIs have been removed from Windows, Mac, and Linux builds. This breaks Chrome Apps on desktop platforms that rely on direct hardware access, which is now restricted to ChromeOS. (32b3c6d) (1e44322)
  • ⚠️ BREAKING BREAKING CHANGE Core networking Mojo interfaces were overhauled, replacing opaque [Native] types for HttpResponseHeaders, SSLInfo, and CertVerifyResult with fully structured Mojo types. This breaks direct integrations with the network service. (c565345) (d825538)
  • ⚠️ BREAKING BREAKING CHANGE Numerous C++ and Java interfaces saw breaking changes. Notable examples include adding new pure virtual methods to content::WebContentsObserver and content::TracingDelegate, and changing signatures in ContentBrowserClient and browser-finding utilities. (17d438b) (853791d) (9f52b02) (c7811f0)

Security & Privacy

  • ⚠️ BREAKING BREAKING CHANGE To align with web standards, data: URL Web Workers now receive unique, opaque origins by default. This security measure may break applications that previously relied on these workers inheriting their creator's origin. (8d1a1e1)
  • ⚠️ BREAKING BREAKING CHANGE Cross-origin media requests using HTTP Range headers are now disallowed for security reasons. This change primarily impacts HLS (HTTP Live Streaming) playback scenarios that fetch media segments from cross-origin servers. (d2e04a1)
  • ⚠️ BREAKING BREAKING CHANGE The permissions.addHostAccessRequest and permissions.removeHostAccessRequest extension APIs now require a user gesture. This behavioral change mitigates clickjacking attacks by preventing programmatic requests for host permissions. (1aca864)
  • ⚠️ BREAKING BREAKING CHANGE The ChromeOS auto-enrollment process will now trigger a mandatory device wipe if it detects the device is already owned. This is a significant behavioral change for enterprise device setup workflows, intended to harden security. (a602cec)
  • DevTools agent hosts for Service Workers and Shared Workers will now only bind their Mojo pipes when a DevTools session is actively attached, providing defense-in-depth against a compromised renderer. (4be40bb) (233b6b2)

Web Platform & Rendering

  • A new performance.speculations.preloads API was added, allowing web developers to programmatically access measurement data for resources loaded via the Speculation Rules API. (2f2079d)
  • CSS capabilities were enhanced: AccentColor and AccentColorText keywords are now enabled by default, and the shape-outside property was extended to support path() and shape() functions. (00200ea) (14981db)
  • Performance was improved in two key areas: the HTML parser was optimized to avoid O(n^2) behavior on pages with large inline scripts, and the compositor will now merge layers for sticky and fixed position elements to reduce overhead. (1e0d769) (dc6a5be)
  • ⚠️ BREAKING BREAKING CHANGE Several behaviors were changed to align with web standards, including making navigation.navigate() reject javascript: URLs, having navigator.credentials.get() throw an error when called without a credential type, and tightening the parsing of the CSS :lang() pseudo-class. (868ac71) (b17a514) (95aa378)

Build System & Code Health

  • The major "Bedrock" initiative to modularize the codebase continued at a rapid pace. Numerous components, including networking (//chrome/browser/net), enterprise, privacy sandbox, notifications, and the password manager, were moved out of the monolithic //chrome/browser target to improve build times and code organization. (888790c) (92b2a6c) (4ae8b1e) (9ddae42)

Android

  • Several features were removed, including the "Creator" page associated with Web Feeds and the "Contextual Tasks" native page. (5653d2b) (35007fa)
  • Syncing of site search engines is now enabled via the kSyncSearchEnginesAndroid flag, and Link Capturing for PWAs is enabled by default on desktop Android. (6c0a722) (30a7542)
  • The tab strip fade effect was refactored to use compositor layers instead of PNG assets, allowing for variable-length fades and improved performance. (eeef6ab)
