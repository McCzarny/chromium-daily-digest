Chromium Digest: 2026-04-04

Overview

This digest analyzes 38 Chromium commits, from commit (1dfe4d5) to (bc63f70). The analysis identifies one major breaking change in the media stack and highlights significant developments in the WebUI toolbar, compositor internals, and extensions platform.

Media

  • ⚠️ BREAKING Breaking Change: The HlsDataSourceProviderImpl::DataSourceFactory class has been moved and refactored into the more generic media::DataSource::Factory. This change breaks embedders who provide a custom factory to HlsDataSourceProviderImpl, as the constructor signature has changed from taking a std::unique_ptr<HlsDataSourceProviderImpl::DataSourceFactory> to std::unique_ptr<DataSource::Factory>. This relocation decouples the factory from HLS-specific code, enabling more flexible data source creation. (75bbf79)

UI & UX

  • The WebUI-based toolbar continues to evolve with the implementation of SetActionElementIdentifier and UpdateActionState for pinned actions, allowing them to be used as anchors for other UI elements. A visual divider has also been added to separate pinned and popped-out actions. (fcb45c1) (5b3d5f4) (3df9083)
  • A change to the user-agent stylesheet that removed border-color: gray for <table> elements was reverted due to widespread test failures, restoring the previous default table border appearance. (bad6a01)
  • The kTaskScopedSidePanel feature flag for Contextual Tasks was removed, simplifying the feature's behavior by defaulting to a single implementation for side panel scope and removing the alternative code path. (1a873b2)
  • On Android, mouse events are now correctly blocked by the scrim and the find-in-page search bar, preventing unintended clicks on content underneath. Additionally, snackbars will no longer be automatically dismissed when a bottom sheet closes, correcting a recent regression. (b11fc4f) (bc63f70)
  • Various UI components saw refactoring and bug fixes, including the composebox component, Glic (Google Lens in Chrome) metrics, and the Vertical Tabs UI. (7f61143) (e2cc342) (3edf72c) (69a7fe2)

Compositor & Rendering

  • A significant internal refactoring in the compositor (cc) changes how layer update rects are managed. Mutations to a cc::Layer's update rect are now allowed during a paint callback, with the state being snapshotted on the main thread before commit. This improves the internal consistency of the rendering pipeline by centralizing state capture in LayerTreeHost. (1bf9d9e)
  • The behavior of the readonly attribute on form controls has been cleaned up and documented to align with browser interoperability, ensuring it consistently bars constraint validation across all applicable input types. (587216c)

Extensions

  • Several enterprise-focused extension APIs (enterprise.login, enterprise.kioskInput, enterprise.hardwarePlatform, enterprise.deviceAttributes) have been converted from the legacy JSON schema format to the modern WebIDL format as part of an ongoing platform modernization effort. (596b815) (09c68dd) (345d69d) (16b705d)
  • A bug in the EventRouter was fixed where it would not correctly terminate processing after receiving a malformed message, improving the robustness of the event system. (ba05b12)

Internals & Bug Fixes

  • A fix was implemented to prevent re-entrant navigations during legacy back-forward cache activations, improving navigation stability. (2f22e14)
  • On Android, the browser's Navigate() function was corrected to avoid clobbering an ongoing navigation when inserting a pre-existing WebContents into the tab strip. (cf49e17)
  • A memory ordering bug in ipcz (an IPC transport layer) was fixed in NodeName::LoadAcquire to ensure correct synchronization between threads. (1dfe4d5)
  • Several minor bug fixes were landed, including corrections for dangling pointers in Ash UI tests, a use-after-free in ToastController, and various internal refactorings and test suite improvements. (a3f0962) (16a592d) (53245ff) (3c4f586)
