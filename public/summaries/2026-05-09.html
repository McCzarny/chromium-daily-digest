Chromium Digest: 2026-05-09

Overview

This digest covers 57 commits from May 9, 2026, from commit (e8f4b02) to (37bfc56). Four major breaking changes were identified in core dependencies, UI element tracking, extensions, and security handling. Other key updates include the introduction of a new MIME handler fallback API, AI and personal context API development, and numerous bug fixes and refactors across Android, Ash, and WebUI components.

Infrastructure & Build

  • ⚠️A dependency roll of libc++ removes Chromium's custom versions of standard C headers, including fenv.h, ctype.h, and inttypes.h. This is a core dependency update that may cause build failures for embedders who were implicitly or explicitly relying on definitions from the now-removed custom headers. (abbeaf7)
    BuildInfra
  • The shared public agents repository has been added as a new dependency in //agents/shared to contain shareable agentic configurations. Additionally, unused Python utilities for chrome_proxy were removed, and various OWNERS files and documentation were updated. (e754fe8) (6a4b6f7) (029c2f1) (0e193a9) (954bfad)
    InfraAIBuild

Security

  • ⚠️As a security hardening measure, renderers that incorrectly provide an Origin header on navigation requests will now be terminated. This replaces the previous, non-fatal DumpWithoutCrashing behavior and may cause previously stable renderers with this bug to crash unexpectedly. (87ecad0)
    SecurityNetwork
  • Several site isolation feature flags, including enable-site-isolation-for-password-sites and isolate-origins, have been marked as never-expiring to ensure their continued availability for debugging and user opt-in scenarios. (2ed859d)
    SecurityPrivacy

Extensions

  • ⚠️The Web Store component hosted app is no longer loaded on non-ChromeOS platforms (e.g., Windows, Mac, Linux). This removes a long-standing component that was an implementation detail for the old webstore origin. This will break any automation, tests, or embedder features that relied on its presence. (22c9e8a)
    ExtensionsChromeOS
  • A new API, chrome.mimeHandler.abortAndFallbackToNativeHandler(), has been introduced. This allows third-party MIME handler extensions to surrender a response back to the browser's native handler, preventing the extension from being locked into rendering an error for unsupported content subtypes. (d39ac48)
    ExtensionsPDF

UI & User Education

  • ⚠️The ui::TrackedElementWebUI API for anchoring UI to WebUI content has been refactored. Tracked elements are now considered invisible when their containing WebContents is hidden. This behavioral change to a core UI framework component will require consumers to adapt to the new visibility rules. (c00f0b9)
    UI
  • The Omnibox UI is being refactored with the implementation of the OmniboxPopupFullPresenter class. In a separate change, files uploaded from Google Drive via the Omnibox context menu are now added as context to the searchbox. (e8f4b02) (6437bfc)
    UIAI
  • On Windows, Stylus Handwriting support has been extended to function correctly when the text input element is within a cross-process iframe (OOPIF). (d09496a)
    UIWindowsInput

Android

  • Several UI bug fixes and improvements were made, including resolving a memory leak in ToolbarProgressBar, fixing Side Panel visibility restoration across window resizes, and refactoring the Tab Bottom Sheet's WebView resizing to prevent visual artifacts during animations. (7552f56) (52daeae) (eac15fc)
    AndroidUI
  • The PDF toolbar is now inflated using a ViewStub, deferring its creation until it is actually needed. This improves performance by avoiding unnecessary layout inflation on PDF load. (2f29a3e)
    AndroidPDFPerformance

AI & Personalization

  • A client-side API skeleton for the Context Memory Service (CMS) has been defined. This introduces the FetchContext method to the PersonalContextService interface, allowing clients to retrieve context from the service. (adc0707)
    AIPrivacy
  • A comprehensive test harness, including schemas and a runner script, has been added for the MAGI multi-agent protocol to prevent regressions in prompt instructions and state machine transitions. (375df35) (962cb01)
    AITesting
  • On iOS, a bug was fixed that caused a Gemini session to start automatically when the Page Action Menu was opened on a managed account, rather than waiting for user intent. (a9c08a4)
    iOSAI

Ash & ChromeOS

  • A series of tests for network UI components, including NetworkListMobileHeaderViewTest and WifiToggleNotificationControllerTest, were modernized to use targeted synchronization APIs instead of the less reliable RunUntilIdle() method. (d1d4fe9) (77e4fae) (34ac26f) (1c261d6)
    AshTestingNetwork
  • A dangling pointer in ash::Shell has been fixed, and other minor refactors were performed to improve code health. (34f46e9) (295ca52)
    Ash

Web Platform

  • A clock drift bug in WebrtcAudioModule that caused an audible cracking sound has been fixed. The module now adjusts the number of audio frames it pulls based on the actual time elapsed since the last poll, ensuring synchronization with the audio sink. (29e7f53)
    WebRTCMedia
  • WebGL behavior is now more spec-compliant: ETC/ETC1 compressed texture formats are disallowed in texStorage calls until the corresponding WEBGL_compressed_texture_etc or WEBGL_compressed_texture_etc1 extension is explicitly enabled by the application. (6e73bdc)
    WebGLGPU
  • A bug fix in Blink's rendering engine ensures that backgrounds paint correctly up to the border-box origin when the experimental border-shape CSS property is used, as the previous optimization incorrectly contracted the background area. (b1b03e1)
    BlinkCSS
  • The PDFium engine was refactored to prevent potential dangling pointers by deferring the deletion of page objects after they are removed from a page, allowing for safe reloading of text data before the objects are freed. (34fa380)
    PDF
