Chromium Weekly: 2026 Week 17

Overview

This week saw a torrent of breaking changes driven by a massive effort to modularize the core //chrome/browser target and modernize major APIs in the UI, graphics, and services layers. Across 2,474 relevant commits out of 3,326 total, several user-facing features like Web Feed and Plus Addresses were completely removed. Key developments include significant security hardening of IPC boundaries, the enabling of new rendering performance optimizations, and major refactoring of Android's multi-window and sign-in APIs.

API Modernization & Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE: The legacy BrowserList API was completely removed, along with several helper functions in browser_finder.h. Functionality was also stripped from the core BrowserWindow interface, including methods for DevTools, QR code generation, and "send tab to self", which are now handled by dedicated controllers or features. (499cfae) (ba141b8) (c9a8aa7) (d383d4e) (ab08725) (cf0a15d)
  • ⚠️ BREAKING BREAKING CHANGE: Several user-facing features were completely removed, resulting in significant API breakage. The Web Feed feature was dismantled across dozens of commits, removing its UI, Java bridges, and feature flags. The Autofill "Plus Address" feature was also removed, including its AutofillClient delegate. Other removals include the WebUITabStrip, the "Glass Toolbar" feature, and the experimental Page Summary button. (c292b11) (bafb125) (1a47ae7) (ff574ae) (d65a964) (4e27221) (0679b56) (f71a274)
  • ⚠️ BREAKING BREAKING CHANGE: A massive effort continued to break up the monolithic //chrome/browser target. Key components like browser lifetime, sessions, sign-in, and navigation predictors were moved into their own build targets. This requires downstream embedders to update their build dependencies to include the new, more granular public API targets. (94c8534) (b904862) (3f51165) (5ea02e3)
  • ⚠️ BREAKING BREAKING CHANGE: The public Java API surface for Sync and Sign-in on Android was significantly reduced. Numerous methods were removed from SyncService (e.g., isInitialSyncFeatureSetupComplete, isSyncFeatureActive) and SigninManager (revokeSyncConsent), breaking embedders using the Sync-as-a-feature API. (d78abe4) (7fcfbca) (6e99a22) (f42b9fc) (0639353)
  • ⚠️ BREAKING BREAKING CHANGE: Core graphics and compositor APIs saw major refactoring. cc::LayerTreeHostImpl::Create() was removed in favor of specific implementations, cc::OwnedLayerImplList non-const accessors were removed, and in Blink, LayoutBox margin accessors were replaced by MarginOutsets(). These changes enforce stricter API usage but require significant code updates for embedders. (9d32df7) (c5db8df) (6cb32f8) (b1585a0)

Security & Privacy

  • IPC and service boundaries were hardened. DevTools was hardened against untrusted clients loading file:// resources, Mojo IPC on Windows gained improved process validation to prevent pipe squatting, and the File System Access API now prevents compromised renderers from monitoring files outside their scope. (28c26ea) (8e5202f) (0e083d3) (12f2a15)
  • Numerous Use-After-Free (UAF) and memory safety vulnerabilities were addressed in components including SpdyStream, RenderFrameHostImpl, the Presentation API, and Bluetooth handling. A Time-of-Check to Time-of-Use (TOCTOU) vulnerability in clipboard handling was also fixed. (6810eeb) (f2c3442) (d3038a7) (959627a) (e59c8af)
  • A vulnerability allowing third-party SharedWorkers to bypass SameSite cookie restrictions when creating WebSocket connections has been fixed. In a related change, WebView's cookie selection logic for intercepted requests was corrected to properly respect Same-Site rules. (2b49183) (7b41029)

Web Platform & Rendering

  • A new MatchLocalFont Mojo API was added to the FontDataService to handle @font-face src: local() lookups in the browser process. This is part of a larger migration away from legacy font proxies, with an initial implementation on Windows using DirectWrite. (662a4cc) (01a0932)
  • Several rendering and caching optimizations were enabled by default. MergeFixedLayers and MergeStickyLayers are now on to improve rendering performance, and the BackForwardCachePauseMicrotasks feature was enabled to improve BFCache behavior by pausing microtask execution for cached pages. (d80eb10) (7f5cf31)
  • For Media Source Extensions (MSE), non-IDR I-frames with SEI recovery points in H.264 streams are now correctly promoted to keyframes, fixing seeking issues in open-GOP content for players like hls.js. (53b9408)
  • Content Security Policy for CSS Modules is now stricter, with <style type=module> elements now enforcing the script-src policy in addition to style-src. (fa5ca80)

Platform-Specific Updates

  • On Android, significant refactoring of multi-window logic now allows creating a new browser window from an existing WebContents, improving flexibility for embedders. The WebSerialWiredDevicesAndroid feature, allowing web pages to connect to wired serial devices, was also enabled by default. (edfa5f2) (6e38ae5) (8cb21e2)
  • ⚠️ BREAKING BREAKING CHANGE: The AuthenticationService API on iOS was updated to remove methods requiring a ConsentLevel parameter as part of a large-scale sign-in refactoring. Enterprise policy support was also enhanced with checks for passkey creation and logic to protect the Tab Grid from screenshots. (c2453a6) (e6b6aed) (dbab372) (1ebf897)
