Chromium Digest: 2026-05-05

Overview

A summary of 803 commits on the main branch, with 614 considered relevant. This digest focuses on developer-impacting changes, including an unusually high number of breaking API modifications across multiple components. The first relevant commit was (3f267f3) and the last was (38c1ba6).

Breaking Changes

  • ⚠️The public C++ interface cc::LayerTreeHostSchedulingClient has been renamed to cc::LayerTreeHostSchedulingDelegate. Embedders implementing this compositor scheduling interface must update their class definitions. (53e1246)
    CompositorBlink
  • ⚠️The signatures of several public API functions in the extensions subsystem, such as AllowCrossRendererResourceLoad, have been changed to use the strongly-typed content::ChildProcessId instead of a raw int. (d348d44)
    Extensions
  • ⚠️The extensions::mojom::EventRouter interface was refactored, moving service_worker_context from a struct to a direct parameter on IPC methods. Additionally, the C++ API extensions::MimeTypesHandler::GetHandler() now returns a const pointer. (907c3bf) (a086295)
    ExtensionsServiceWorker
  • ⚠️The virtual method SharedImageBackingFactory::CreateSharedImage now accepts a gpu::SharedImageInfo struct instead of individual parameters. Custom factory implementations must be updated. (c06e765)
    GPUCompositor
  • ⚠️The method signature for DirectRenderer::DrawFrame, a core public class in the Viz display component, has been changed to add a new const AggregatedFrame& parameter. Subclasses must be updated. (e14bcb1)
    CompositorGPU
  • ⚠️On macOS, synchronous methods in the public TextInputClientMac interface have been renamed (e.g., GetCharacterIndexAtPoint is now SyncGetCharacterIndexAtPoint) and new asynchronous versions have been introduced. Embedders must update their calls. (60aa06c)
    macOSUI
  • ⚠️Several SuggestionGenerator methods in Autofill now accept a non-const AutofillClient*. Separately, the public AutocompleteInput class was refactored to replace an enum with a boolean for keyword mode. (4bd9046) (1a40eab)
    AutofillUI
  • ⚠️The virtual method permissions::PermissionContextBase::NotifyPermissionSet has a new const content::PermissionResult* parameter. Subclasses must update their overrides. (5f2014a)
    BlinkPrivacy
  • ⚠️Focus-related methods (FocusInactivePopupForAccessibility, RotatePaneFocus, etc.) have been removed from the public BrowserWindow interface and moved to a new BrowserFocusController class. (16c69f3)
    UIBrowserAccessibility
  • ⚠️The public low-level type alias subtle::AtomicWord has been removed from base/atomicops.h, which will break any code that uses it directly. (a9b7f6a)
    Base
  • ⚠️On Android, the OMA DRM download feature has been completely removed. Separately, the behavior of Navigate() has been altered, as the browser parameter is now optional, which may cause some navigations to open a new window unexpectedly. (a491df8) (0b478c5)
    AndroidDownloadsNavigation
  • ⚠️On ChromeOS, the ability to install Debian (.deb) packages directly from the Files app UI has been removed and is now blocked with a warning dialog. (fb90834)

    iOS

    UI

Security and Privacy

  • The Web Share API is now hardened in the browser process, which will validate provided URLs and terminate a compromised renderer attempting to use non-web schemes like file://. (3f267f3)
    SecurityBlink
  • A potential NTLM credential leak has been mitigated in the getFileSystemInfo enterprise API by adding path validation to block network paths. (e04526f)
    EnterpriseSecurityNetwork
  • A fix in the Windows MachineIdProvider prevents a heap out-of-bounds read when a storage driver returns a malformed descriptor header. (236ef5f)
    SecurityWindows
  • Web Bluetooth will now correctly abort pending watchAdvertisements requests when a page loses visibility, preventing a potential privacy bypass. (e6b4397)
    BluetoothPrivacy
  • The maximum size of HID report descriptors is now limited to 65535 bytes to prevent out-of-memory or timeout failures during parsing. (e2d3268)
    SecurityUSB

Web Platform

  • The CSSSafePrintableInset feature is now enabled by default, shipping support for the @page descriptor page-margin-safety for controlling print margins. (798c128)
    BlinkCSSPrinting
  • The image.decode() API will now correctly resolve its promise even if the image decode fails due to memory constraints, aligning with the specification. (cb0c0d8)
    BlinkHTML
  • A bug in AudioBufferSourceNode has been fixed where the duration was incorrectly handled when looping with a playbackRate other than 1.0, ensuring playback stops correctly. (381fd85)
    BlinkMedia
  • Icon URL schemes in Web App Manifests are now restricted to http, https, data, or the document's scheme to mitigate Server-Side Request Forgery (SSRF) risks. (7cb6a9b)
    BlinkSecurity
  • Support for MIPS intrinsic optimizations in WebAudio has been removed. Builds on MIPS will now fall back to a scalar implementation, which may cause a performance regression for WebAudio on that architecture. (3b8474a)
    MediaPerformance

Platforms

UI and Features
