Chromium Weekly: 2025 Week 46

Overview

This week in Chromium development saw significant architectural refactoring, with 406 relevant commits analyzed out of 576 total across 4 days. The long-running effort to remove the `Supplementable` pattern from Blink made major progress, fundamentally changing how objects are extended. Other key themes include continued work on Project Bedrock to decouple core browser components, numerous API-breaking changes to improve security and modernize the codebase, and performance enhancements like lazy string building.

Breaking Changes & API Evolution

  • ⚠️ BREAKING BREAKING CHANGE: A major architectural refactor is underway to remove the Supplementable pattern from Blink, replacing its hash-table-based mechanism with more performant alternatives. Classes like Page, Screen, HTMLMediaElement, and HTMLVideoElement have been updated, which may break downstream projects that relied on this extension point. (a0c6dba) (cdd031f) (4acc768) (0d09159) (fa22d92) (9ae0d40)
  • ⚠️ BREAKING BREAKING CHANGE: As part of a security and modernization effort known as "spanification," numerous functions across the codebase were updated to use safer types like std::span and std::string_view instead of raw pointers. This changes many function signatures and requires call sites to be updated. (132358c) (cdd3e5c) (4238934) (d27794e)
  • ⚠️ BREAKING BREAKING CHANGE: Project Bedrock continues to decouple core components by migrating observers in services like session_service_base and idle_service away from the global BrowserList to a per-profile model. (0e5b5d1) (1f82c9e) (643ce22) (2c22a9a)
  • ⚠️ BREAKING BREAKING CHANGE: Several APIs were removed or altered. Notable changes include the removal of the setHitTestRegions API for HTML-in-canvas, the removal of the unused OnUpdateError from CrxInstaller, and a signature change for AutofillDriver::ExtractForm(). (3a9e6b2) (d279473) (5063b6a)
  • ⚠️ BREAKING BREAKING CHANGE: The UI and Mojom interfaces for ARC ADB Sideloading have been removed from ChromeOS Settings. (6f1bec5)

Performance

  • A new base::LazyStringBuilder class was introduced to improve performance by delaying string construction and avoiding intermediate allocations, particularly useful in scenarios with many concatenations. (6f3703c)
  • Perceived performance on session restore is improved by pre-loading favicons for tabs that the user is about to restore manually. (9b8ef9b)
  • Networking objects like net::IsolationInfo and NetworkIsolationKey are now cheaper to copy due to the use of ref-counted data members, improving performance in areas like subresource loading. (d9614f9)

Security & Privacy

  • V8 introduced a 'site familiarity' heuristic that may disable certain optimizations on unfamiliar websites. This trades some performance for increased security by reducing the potential attack surface on sites with low user engagement. (9beb380)
  • The Secure DNS (DNS-over-HTTPS) feature was enhanced. In 'automatic' mode, Chrome can now fall back to a well-known DoH provider if the user's system DNS does not support it, increasing the adoption of encrypted DNS. (76cab8c) (984ddf8)
  • Navigation gating is now enabled by default, showing a confirmation dialog before navigating to URLs on the Optimization Guide's blocklist of potentially sensitive sites. (b0bc944)

UI & UX

  • The legacy UI for video Picture-in-Picture has been removed, making the modern 2024 controls the default and only option, simplifying the codebase. (272e694)
  • The New Tab Page search box ('realbox') gained support for dragging and dropping files and images, and was updated with new rotating placeholder text. (4cfc988) (9d1cd82) (29a75ee)
  • The browser layout system was refactored to simplify support for vertical and WebUI tabstrips, and the vertical tabstrip state is now persisted across sessions. (c70b396) (e38dca2)
  • On Android, the extension installation dialog now displays the list of requested permissions to the user. (9ad9658)

Web Platform & Blink

  • ⚠️ BREAKING BREAKING CHANGE: To align with W3C specifications, the totalSamplesDuration statistic in RTCAudioPlayoutStats now correctly includes the duration of audio glitches, which may affect monitoring applications. (c30f598)
  • ⚠️ BREAKING BREAKING CHANGE: Experimental CSS gap decoration properties were renamed to align with specification changes, such as column-rule-outset becoming column-rule-inset. (45245de) (0fada3d)
  • The built-in configurations for the Sanitizer API were updated to match the latest WICG specification, ensuring interoperability and standard behavior. (a630adc) (50b5044)
