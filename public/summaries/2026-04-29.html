Chromium Digest: 2026-04-29

Overview

A busy day with 699 total commits, of which 518 were considered relevant after filtering. The day was marked by a very high number of breaking changes to core APIs in areas like the compositor, UI, base library, and networking. Major refactors also continued in the layout engine and browser-finding logic. The first commit was (9bc9e6b) and the last was (7eab0f4).

Breaking Changes

  • ⚠️ BREAKING In Blink's layout engine, several public virtual methods on LayoutBoxModelObject (e.g., BorderTop(), PaddingRight()) have been removed. Code must be updated to use the consolidated BorderOutsets() and PaddingOutsets() methods. (b2246d7)
  • ⚠️ BREAKING Key public virtual methods on the compositor's LayerTreeHostImpl (e.g., BeginCommit, CreatePendingTree) have been moved to a new ClientLayerTreeHostImpl subclass. Embedders who subclass or directly call these methods must update to the new class. (144eb65)
  • ⚠️ BREAKING The signature for SharedImageFactory::CreateSharedImage has changed. It now accepts a SharedImageInfo struct instead of a long list of parameters. (08ee0a7)
  • ⚠️ BREAKING Public methods related to fullscreen state (e.g., ShouldHideUIForFullscreen()) have been removed from the BrowserWindow and Browser interfaces. This functionality now resides in the new BrowserWindowFullscreenController. (d1db08e)
  • ⚠️ BREAKING The widely used function chrome::FindBrowserWithTab is being removed. Embedders must migrate to new APIs like GlobalBrowserCollection to find a Browser* from a WebContents*. (48bd889)
  • ⚠️ BREAKING The base::RandInt function has been removed from base/rand_util.h. Callers must migrate to base::RandIntInclusive. (5f591f6)
  • ⚠️ BREAKING The password_manager::PasswordStoreConsumer interface was changed, replacing the virtual method OnGetPasswordStoreResults with OnGetPasswordStoreResultsOrErrorFrom. (18f3b6c)
  • ⚠️ BREAKING The public method writable_hdr_metadata() has been removed from the media::VideoDecoderConfig class. (5794b9e)
  • ⚠️ BREAKING The MADV_FREE-based discardable memory feature, including the public function base::GetDiscardableMemoryBacking(), has been removed from base. (73c9221)
  • ⚠️ BREAKING Legacy non-WebRTC remoting components have been removed, including public headers and the CreateChannelAuthenticator() pure virtual method from remoting::protocol::Authenticator. (e548872)
  • ⚠️ BREAKING The public RTCLoggingDispatcher Mojo interface was modified. FinishDiagnosticLogging now requires a metadata parameter, and the Mojo file's directory was renamed, breaking include paths. (8e0ea31)
  • ⚠️ BREAKING On Android, the SideUiObserver interface method onPreSideUiSpecsChange now returns an android.transition.Transition instead of an android.animation.Animator. (3f4d377)
  • ⚠️ BREAKING The public AndroidAutofillClient class now inherits from content::WebContentsObserver, altering its class hierarchy and requiring subclasses to handle new lifecycle methods. (7ae919c)
  • ⚠️ BREAKING The behavior of RTCPeerConnection has changed with the enforcement of the Connection-Allowlist header. Construction can now be blocked by this header, which may require server-side configuration changes. (ec55185)
  • ⚠️ BREAKING The behavior of ProtocolHandlerRegistry for Off-The-Record (OTR) profiles has changed. The registry is no longer shared with the original profile, isolating handlers. (7495f64)

Security

  • The RejectWeakCiphertext feature, which blocks weak symmetric ciphers in TLS, is now enabled by default. (5b951ab)
  • A logic flaw in DevTools that could leak raw header access and HttpOnly cookies after a session was detached has been fixed. (b1cb06e)
  • The Omnibox SearchSuggestionParser now validates that navigation suggestions use only HTTP or HTTPS schemes, preventing execution of malicious javascript: URLs. (f1feef3)
  • A Use-After-Free vulnerability in AXTreeSourceAndroid was fixed by ensuring a raw pointer is not retained across tree mutations that could invalidate it. (55bfe49)
  • Input validation was added to the VAAPI video decoders and encoders for AV1, H264, and VP8 to reject malformed data and prevent potential crashes or vulnerabilities. (e616e27) (1647d70) (390e0ed)

Blink & Web Platform

  • Support was added for calc() expressions that mix <angle> and <percentage> types in conic-gradient() color stops, aligning with the CSS Images 4 specification. (2f0622a)
  • The CSS line-clamp feature had its continue and block-ellipsis keywords renamed to align with new specification resolutions. (10dbb90)
  • A new kWebNNModelCompilation sandbox type was added for the WebNN API. This will be used for an upcoming compiler utility process to isolate ONNX Runtime graph compilation. (1adb86c)
  • The streaming sanitizer was updated to better handle text node normalization, aligning its behavior more closely with the HTML parser. (5aa3eb2)

UI & UX

  • The API to play specific segments of a Lottie animation (playSegments) has been made public, allowing for more granular control over animations. (bc01b08)
  • A tab's memory usage is now displayed on the tab hover card UI. (605b071)
  • A new feature flag, kWebUIOmniboxFullPopupV2, was introduced to allow parallel development on a next-generation WebUI omnibox popup. (099d60d)
  • The TranslateLanguageListModel interface was updated to expose language codes via GetTargetLanguageCodeAt, preparing for a new searchable language selection UI. (210dc00)

Extensions

  • Support for adding custom context menu items to the browser's tab context menu has been implemented. Extensions with the contextMenus permission can specify "tab" as a context, enabled by the kExtensionTabContextMenu feature flag. (ad49992)
  • ⚠️ BREAKING Method signatures in the public extensions::ProcessMap API were changed to use the strongly-typed content::ChildProcessId instead of int. (2686e85)
  • A use-after-free vulnerability in UserScriptSet::UpdateUserScripts was fixed by ensuring memory is handled correctly during script updates. (fe455e2)

Mobile (Android & iOS)

  • On Android, the UnthrottleAsyncTouchMoves feature is now enabled by default (excluding WebView) to improve gesture responsiveness. (d40e41a)
  • On Android, the location bar was refactored, centralizing phone-specific button logic into the LocationBarCoordinator. (2d80323) (cf69f5d) (829b178)
  • On iOS, support for native map previews via universal links has been added. (0440587)
  • ⚠️ BREAKING On iOS, the public virtual method BuildExternalPrivacyContext was removed from the SystemIdentityManager class. (6246ffa)

Code Removals & Infrastructure

  • Support for the pre-mmap Safe Browsing file format has been removed. Instead of migrating, Chromium will now return an error, prompting a full database update. (158b83a)
  • The Single Web Feed feature, previously used for the Creator Page in the Feed, has been removed along with its associated code and metrics. (7532c85)
  • The build system's proto_library template was updated to publicly propagate dependencies, fixing build issues where transitive headers could not be found. (7107857)
  • The unused translation_unit tool has been removed from the codebase. (0f53c60)
