Chromium Digest: 2026-05-08

Overview

This daily summary covers 677 commits to the Chromium main branch on May 8, 2026, with 496 commits filtered as relevant to developers. The day was marked by an exceptionally high number of breaking API changes across core components like Content, UI, and Android JNI, requiring significant updates for embedders. Notable feature work includes enabling focusgroup by default and hardening security around file pickers. The first relevant commit was fa57518e6fa156ae52dc1ced184692a457706a4d and the last was d06297d18862dbe4e2d48633ec210c8befc5ade2.

Breaking Changes

  • ⚠️The public method SiteInstance::GetSiteURL() has been removed from the core content API. Embedders must update their code to use SiteInstance::GetSecurityPrincipal().GetDeprecatedSiteURL() instead. (01d357d)
    ContentSecurity
  • ⚠️A new pure virtual function, CouldDisplayBeforeUnloadDialog(), has been added to the public content::RenderFrameHost interface. Embedders who subclass RenderFrameHost must implement this new method. (8229aec)
    ContentSecurity

Security

  • The origin of workers created from data: URLs is now correctly set to a new, unique opaque origin per the HTML spec. This is a behavioral change that may break content that relied on the previous non-compliant behavior of inheriting the creator's origin. (ee6bfd1)
    BlinkSecurityServiceWorker
  • The File System Access API has been hardened by restricting cross-origin subframes from showing a file picker. A new ContentBrowserClient method, IsCrossOriginSubframeAllowedToShowFilePicker, allows embedders to grant explicit exemptions. (3fd5e2c)
    SecurityBlinkContent
  • A browser-side check now strictly enforces that a page must have user activation to show a beforeunload dialog. (aa69447)
    SecurityUI
  • On iOS, WebAuthn requests are now correctly restricted to secure (HTTPS) origins. (1dc7714)
    iOSSecurityWebRTC

Web Platform

  • The focusgroup accessibility feature has been enabled by default, graduating from its origin trial and improving keyboard navigation for custom components. (5fdbeaa)
    AccessibilityBlinkHTML

  • SVG text elements now honor standard text-decoration CSS properties, as the SvgEnableTextDecorationCssStyling feature has been enabled by default. (ec86856)
