Breaking Changes

  • ⚠️ BREAKING A major refactoring of core networking Mojo interfaces removes deprecated [Native] types for SSLInfo, CertVerifyResult, HttpResponseHeaders, and others. This will break embedders using these network service interfaces directly. (c565345)
  • ⚠️ BREAKING The public API chrome::FindTabbedBrowser() has been removed from browser_finder.h. Embedders must migrate to the new ProfileBrowserCollection class, which also returns a different type (BrowserWindowInterface*). (c7811f0)
  • ⚠️ BREAKING The public constructor for ui::gl::HDRMetadataHelperWin has been removed. Code must be updated to use the new static Create() factory method. (10dba43)
  • ⚠️ BREAKING The public API HelpBubble::AddOnCloseCallback() has been removed. It is replaced by two new methods: AddOnClosingCallback() and AddOnClosedCallback(). (e7b9096)
  • ⚠️ BREAKING The NetworkContext::SetNetworkConditions API signature has changed to require a new throttling_client_id parameter, affecting DevTools and network simulation tooling. (c9c9dc6)
  • ⚠️ BREAKING The WebRTC RTCDataChannel.send() method will now throw a TypeError if a message exceeds maxMessageSize. This aligns with the spec but changes previous behavior where it would fail silently. (dfbbc3f)
  • ⚠️ BREAKING The public C++ API content::WebContents::SetTabSwitchStartTime has a new signature to improve tab switch metrics. (424a1a6)
  • ⚠️ BREAKING In the VIZ component, TextureDrawQuad::SetNew and SetAll no longer have a default parameter. Callers must now explicitly provide the is_tex_coords_normalized value. (c2fc4f3)
  • ⚠️ BREAKING The ToolbarButtonProvider API has been refactored, removing GetAppMenuButton() and replacing it with GetAppMenuControl(). (0800caa)
  • ⚠️ BREAKING The public Java methods isNightModeEnabled() and isForceDarkWebContentEnabled() have been removed from TabWebContentsDelegateAndroid. A new WebContentsThemeClient should be used instead. (7156569)
  • ⚠️ BREAKING The SendTabToSelfModel::AddEntry method signature has changed, adding a new commit_confirmation callback. All implementations of this interface must be updated. (7ff606d)
  • ⚠️ BREAKING The return value contract for AutofillClient::GetProfile() has changed. It is now guaranteed to return a non-null Profile*, removing the need for null checks. (9b77d05)
  • ⚠️ BREAKING Public string constants for Perfetto data source names have been removed from the perfetto_service.mojom interface. They are now only available in the C++ header services/tracing/public/cpp/perfetto/perfetto_data_source_names.h. (61a9b97)
  • ⚠️ BREAKING The public Shutdown() method has been removed from the PolicyInvalidator class. Cleanup is now handled via RAII. (2248ccf)
  • ⚠️ BREAKING On iOS, the public protocol SuccessfulReauthTimeAccessor has been removed as part of a simplification of the ReauthenticationModule. (18b43a7)
  • ⚠️ BREAKING The command-line switch kWebViewStartupTasksPlusMultiProcess has been removed as the associated feature is now enabled by default. (69cb556)
  • ⚠️ BREAKING The public Java interface PaymentApp.InstrumentDetailsCallback has changed the signature of its onInstrumentDetailsError method to support more detailed error reporting. (3609d22)
  • ⚠️ BREAKING Several public metrics-gathering functions have been removed, including ash::app_list::RecordZeroStateSuggestionOpenTypeHistogram and BookmarkStats::RecordBookmarkAllTabs. (8692e47) (3255c80)

Security and Privacy

  • To provide defense-in-depth against a compromised renderer, DevTools agent hosts for Service Workers and Shared Workers will now only bind their Mojo pipes when a DevTools session is actively attached. (4be40bb) (233b6b2)
  • The chrome.scripting.executeScript API now correctly enforces host permissions on all subframes, not just the top-level one, closing a potential IPC validation bypass. (e05c69e)
  • A recursion depth limit has been added to bookmark Pickle deserialization to prevent stack overflow from a maliciously crafted clipboard or drag-and-drop payload. (0e3f531)
  • The browser process now verifies that a Service Worker's main script fetch request matches the registered script URL, preventing a policy downgrade attack from a compromised renderer. (0f7954e)
  • A potential path traversal vulnerability in feedback_util::ZipString was fixed by validating the filename before use. (30d636a)
  • On Android, the system now ensures that memory file descriptors (memfds) are size-sealed before being handled, preventing Time-of-Check to Time-of-Use (TOCTOU) issues. (e5c6c0b)
  • Support for pre-M79 raw DPAPI ciphertext has been removed from os_crypt, eliminating a legacy fallback path. (db40741)
  • A bug in the cross-fade() CSS function that could leak EXIF orientation information from cross-origin images has been fixed. (1d5d289)

Web Platform and Rendering

  • The RTCDataChannel.send() method will now throw a TypeError if a message exceeds maxMessageSize, aligning with web standards. (dfbbc3f)
  • The CSS overflow: clip property will no longer show scrollbars on the non-scrollable axis, aligning its behavior with overflow: hidden. (2f04f82)
  • For WebGL, pending GL errors are now correctly preserved across a canvas resize, fixing a conformance issue. (73d0650)
  • The Navigation API now aligns with the spec to scroll to the top of the document when navigation.scroll() targets a non-existent fragment. (d6d5d3b)
  • A crash in TextIterator was fixed for cases where EmitsOriginalText() was enabled, which could cause out-of-bounds access with truncated text. (2dd1234)
  • The deprecated blink::WebString::FromUTF8() method was removed but then quickly reverted due to build failures. The API remains available. (64bea51) (0890a5c)
  • The FontSrcLocalMatching feature flag, enabled since M77, has been removed, making improved local font matching a permanent feature. (304f0cb)

Graphics and Media

  • The public constructor for ui::gl::HDRMetadataHelperWin has been removed and replaced with a static Create() factory method to remove a D3D11 dependency. (10dba43)
  • Hardware DRM playback failures on multi-GPU Windows systems have been addressed by ensuring the virtual video window is positioned on the correct monitor to select the right GPU adapter. (2992ee7)
  • A deadlock in WebAudio that could occur when stopping an AudioDestination with output buffering bypassed has been resolved. (dd97aa7)
  • In the VIZ component, TextureDrawQuad constructors now require callers to explicitly provide the is_tex_coords_normalized value. (c2fc4f3)
  • For ARC video encoding, a valid REC709 color space is now used by default. (092cbcc)

UI and UX

  • The HelpBubble API has been refactored for stability, replacing AddOnCloseCallback() with the more specific AddOnClosingCallback() and AddOnClosedCallback(). (e7b9096)
  • The chrome::FindTabbedBrowser() utility function has been removed. It is replaced by a new API on the ProfileBrowserCollection class. (c7811f0)
  • On Android, the BottomSheet can now act as a browser control, allowing it to be offset by the top-controls, by integrating with the BottomControlsStacker. (a2b879a)
  • On iOS, significant work was done on the App Bar and New Tab Page to fix layout issues in landscape and fullscreen modes. (68b3fae) (b3ca998) (8d832b4)
  • The multi-step PWA installation dialog has been refactored to include a new introductory view. (40c6553) (6623918)
  • On Android, the TabWebContentsDelegateAndroid API for themeing has been refactored, removing isNightModeEnabled() in favor of a new WebContentsThemeClient. (7156569)

Services

  • The AutofillClient::GetProfile() method contract has changed; it is now guaranteed to return a non-null Profile*, removing the need for null checks in client code. (9b77d05)
  • The SendTabToSelfModel::AddEntry method signature has changed to include a new commit_confirmation callback, breaking custom implementations of the model. (7ff606d)
  • The AutofillAiDefaultAvailability feature has been enabled by default on desktop, changing the opt-in flow for AI-powered Autofill features. (1f566c4)
  • For password sharing, a check was added to ensure the origin of a form's URL matches the signon realm's origin, preventing potential spoofing. (ff624fe)
  • The kMigrateAccountPrefs feature flag has been removed, finalizing the migration of account-scoped preferences on mobile platforms to align with desktop. (ac84d7d)
  • A use-after-free crash in FormFiller::UndoAutofill has been fixed. (c7c2ce4)

Networking and Performance

  • A major refactoring of core networking Mojo interfaces removes deprecated [Native] types, requiring significant updates for embedders using these interfaces. (c565345)
  • The NetworkContext::SetNetworkConditions API now requires a throttling_client_id to support multiple concurrent DevTools sessions applying throttling rules. (c9c9dc6)
  • A new caching mechanism was added to GURL::SchemeIsHTTPOrHTTPS() to improve performance on the navigation critical path. (ae2b997)
  • On Android, the UI update for loading state notifications has been moved out of the critical path of navigation, reducing blocking time. (9d95efd)
  • The AnchorElementPreloader was updated to select the correct Network Anonymization Key (NAK) based on the initiating frame, improving preconnect accuracy. (e3fbf66)

Build and Infrastructure

  • The effort to modularize the codebase continues, with sources for segmentation_platform, page_load_metrics, and media_galleries being moved into their own build targets. (80b4488) (0ec6be5) (5b56cb8)
  • cc::RefCountedBuffer now inherits from base::RefCountedThreadSafe instead of base::RefCounted to ensure atomic ref-counting and prevent concurrency issues. (5625e13)
  • New FYI CI builders for Linux Arm64 have been added to the build infrastructure. (e13b855) (1aab138)
  • The InProcessBrowserTestMixin was extracted into its own header to break dependency chains and improve build times. (9e207fd)
