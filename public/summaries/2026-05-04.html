Summary for 2026-05-04 on branch 'main'

Chromium Digest: 2026-05-04

Overview

This daily summary covers 567 commits on the main branch from May 4, 2026, with 427 commits identified as relevant to developers. The changes span from commit (162a1ba) to (4802e28). A significant number of breaking changes were landed, impacting core compositor, UI, media, and autofill APIs.

Breaking Changes

  • ⚠️The public C++ function Atomic64::Acquire_Load has been removed from the core header base/atomicops.h. Embedders directly using this low-level atomic operation must migrate to a different implementation. (ef2ad96)
    Base
  • ⚠️The core compositor interface cc::LayerTreeHostImplClient has been renamed to cc::LayerTreeHostImplDelegate. Embedders implementing or referring to this class must update their code. (67cf5cf)
    CompositorBlink
  • ⚠️The public method SetVSyncDisplayID on several viz and gl interfaces (e.g., ui::gl::Presenter, viz::OutputSurface) now requires a force_update parameter, breaking existing calls and implementations. (a5ab2ae)
    GPUUImacOS
  • ⚠️The CanvasResourceProviderSharedImage interface has been refactored. The method GetSharedImageUsageFlags() is now a pure virtual, which will break custom canvas resource provider implementations. (f6ecb37) (2e24745)
    BlinkGPU
  • ⚠️The PasswordStoreConsumer interface has been updated to use StoredCredential. The public method OnGetPasswordStoreResultsOrErrorFrom now passes a vector of StoredCredential instead of PasswordForm, breaking custom consumers. (2b93d3b)
    PasswordManager
  • ⚠️The public method BrowserAutofillManager::DidShowSuggestions has changed its signature to accept a const FormGlobalId& instead of a const FormData&, requiring updates to any code calling it. (50dc42f)
    Autofill
  • ⚠️The public constructor for media::FFmpegVideoDecoder now takes a std::unique_ptr<MediaLog> instead of a raw MediaLog*. Embedders creating instances of this class directly must update their code. (16b2782)
    Media
  • ⚠️The static method views::BubbleDialogDelegate::CreateBubble has been renamed to CreateBubbleDeprecated. This is an API removal that will cause build failures for any code still calling the old method. (5ee0e6a)
    UI
  • ⚠️The StyleImage::IsCorsSameOrigin method signature has been changed, removing its String& parameter. Embedders subclassing or directly calling this Blink API will need to update their code. (644d893)
    BlinkCSS
  • ⚠️The public methods SetShouldShowLeadingSeparator and SetShouldShowTrailingSeparator have been removed from the MultiContentsView class. (04ff39d)
    UI

Blink & Web Platform

  • The CSS shape-outside property is now more powerful, with support for path(), shape(), rect(), and xywh() basic shapes enabled by default. (9611ae0) (ed8a8cb)
    BlinkCSS
  • A new JavaScript API, Element.matchContainer(), has been introduced behind the ElementMatchContainer flag, allowing developers to programmatically evaluate if an element satisfies the conditions of a CSS container query. (8308463)
    BlinkCSS
  • To align with web standards, <option> and <optgroup> elements inside a disabled <select> will now correctly match the :disabled CSS pseudo-class. (a76ffb0)
    BlinkCSS
  • The experimental accuracyMode attribute has been removed from the GeolocationPosition web API as part of a specification update. (e71de3b)
    BlinkPrivacy
  • The Gamepad API is gaining support for raw input events via a new Origin Trial, providing access to unmapped and unfiltered controller data. (4eac740)
    BlinkWebRTC
  • In WebAudio, memory allocation for PeriodicWave is now fallible. Instead of crashing on out-of-memory errors, it will now throw a NotSupportedError. (cd69090)
    BlinkMedia
  • The HTMLOrForeignElement IDL type has been renamed to HTMLOrSVGOrMathMLElement to align with WHATWG specifications. (0c96db4)
    BlinkHTML

Security

  • The browser process now strictly verifies the sessionId in all DevTools protocol messages from renderer-side sessions. A compromised renderer attempting to spoof or omit the sessionId will now be terminated. (de2829e)
    SecurityDevTools
  • A Use-After-Free vulnerability in the passthrough command decoder's DoBufferData implementation has been patched. (1353277)
    SecurityGPU
  • A Use-After-Free in FileSelectHelper was fixed by replacing manual refcounting with a WebContents-scoped manager. (2e5d150)
    SecurityStorage
  • On iOS, the on-disk ArchivableCredentialStore file, which could contain plaintext passwords, is now explicitly excluded from device backups. (977f350)
    SecurityiOSPasswordManager
  • Renderer-initiated navigations from inactive RenderFrameHosts are now consistently disallowed to enhance security. (96f0257)
    SecurityNavigation

Core, Performance & GPU

  • On macOS, the CALayerTreeCoordinator now allocates a new CAContext on resize to ensure atomic updates between the browser and GPU processes, fixing potential flickering. (acca2e2)
    CompositorGPUmacOS
  • The CastStreamingMediaVideoEncoder feature is now enabled by default, improving performance for Cast streaming. (ee0ca84)
    MediaGPU
  • Several instances of SplitString were replaced with the more performant SplitStringPiece across the codebase, reducing string copies. (ca753e1) (2b78ba8) (baa0337)
    PerformanceBase
  • A data race in TaskAttributionId was fixed by making its internal ID generator atomic, improving scheduler reliability. (ca652e2)
    BasePerformance

UI & UX

Services

    AutofillStorageBase

    AndroidCSS

    iOSSecurityPayments
