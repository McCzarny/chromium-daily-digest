Summary for 2026-05-01 on branch 'main'

Chromium Digest: 2026-05-01

Overview

A busy day with 603 commits, of which 431 were deemed relevant for this summary. The day saw a significant number of breaking API changes and refactorings, alongside security enhancements and new features. Activity spanned from the first commit 2b2726ca9d5289a827b6802d96192467d2aa6caa to the last 22921c2492c410545d874d82e16946f48c678ad5.

Breaking Changes

  • ⚠️ BREAKING The core compositor class cc::LayerTreeHostClient has been renamed to cc::LayerTreeHostDelegate. Embedders implementing this class for LayerTreeHost must update to the new name. (afd22cd)
  • ⚠️ BREAKING The enterprise policy ForceForegroundPriorityForOrigins is renamed to ForceForegroundPriorityForUrls. Its matching logic now uses full URLs instead of origins, requiring administrators to update policy configurations. (605fbfc)
  • ⚠️ BREAKING For non-branded builds, the default Google crash server URL has been removed. Embedders must now explicitly configure their own crash reporting endpoint. (e7f4186)
  • ⚠️ BREAKING The public API WebView.findAddress has been removed from Android WebView. This was obsolete on Android P+ but is a breaking change for embedders targeting older versions. (35557c4)
  • ⚠️ BREAKING The signature of the public virtual method PdfInkModuleClient::DrawText has been changed to include a new InkTextId parameter. Embedders implementing this interface must update their code. (4ae99a5)
  • ⚠️ BREAKING The signature for views::BubbleDialogDelegate::CreateBubble has changed. The default parameter for widget ownership was removed, and all callers must now explicitly provide it. (3301350)
  • ⚠️ BREAKING Method signatures in the public speech recognition API, including SpeechRecognitionManagerDelegate, have been changed to use content::GlobalRenderFrameHostId instead of separate render_process_id and render_frame_id integers. (5ee42da) (c15d856)
  • ⚠️ BREAKING The public method BrowserView::toolbar_height_side_panel() has been renamed to BrowserView::side_panel(). Code using this accessor for the browser's side panel must be updated. (49935f8)
  • ⚠️ BREAKING The public class remoting::RemoteAudioInput has been renamed to remoting::AudioInjector. Embedders using the remoting component's audio injection feature must update to the new class name. (6b2679d)
  • ⚠️ BREAKING The JNI generation contract has been changed. The module_name must now be specified in the generate_jni GN build target instead of the @NativeMethods Java annotation, requiring updates to both source and build files. (629046f)
  • ⚠️ BREAKING The public constructor HashValue(base::span<const uint8_t> hash) has been removed from net/base/hash_value.h. Code must be updated to use the HashValue(HashValueTag, base::span) constructor instead. (a91b8f4)
  • ⚠️ BREAKING The public methods set_should_show_outline and should_show_outline have been removed from the SidePanelEntry UI class. (b105956)

Security

  • A check for invalid navigation headers sent by a renderer process was upgraded from a DumpWithoutCrashing to a full renderer process kill, hardening the browser against misbehaving renderers. (e0a0f3e)
  • A use-after-free vulnerability in the macOS implementation of FilePathWatcherFSEvents was fixed by introducing a ref-counted context. (cb6f7a4)
  • A potential local privilege escalation vulnerability in Chrome Remote Desktop on Windows was fixed by preventing the service from following reparse points when writing configuration files. (f67fb61)
  • A sanitization bypass in FrameSerializer was fixed. Mixed-case scripting attributes (e.g., ONLOAD) in MHTML files could previously lead to script execution on re-opening. (19cb047)
  • Security for the Service Worker Static Routing API was strengthened to prevent a Cross-Origin Resource Policy (CORP) bypass. (5046d3c)
  • Validation for main thread event listener IPCs in EventRouter was strengthened to prevent a compromised renderer from spoofing extension events. (6271562)
  • Support for ML-DSA signatures in TLS has been added behind a feature flag. (a2638b6)

Web Platform & Blink

  • The CSSGapDecoration feature for styling gaps in flexbox and grid layouts has been marked as stable and is now enabled by default. (4934ed0)
  • The window.fence.notifyEvent API and related Fenced Frames code have been removed, as the feature is no longer launching. (3cca452)
  • A new tools permissions policy was introduced for the Web Model Context API (WebMCP), allowing cross-origin iframes to register and use tools when granted permission. (ea1d52c)
  • The chrome.histograms WebUI API has been updated to accept floating-point values, which are rounded to the nearest integer. This improves usability with APIs like performance.now(). (1622742)
  • Support for the spaces value in the ruby-overhang CSS property was added behind a feature flag. (85c5a07)

Android & WebView

  • The Web Feed feature has been removed, including its network/storage code and the kWebFeedContentOrder preference, as part of its ongoing deprecation. (c6e0135)
  • In WebView, Google Mobile Services (GMS) calls are now deferred until after Chromium startup by default to improve performance. This is controlled by the WebViewDeferStartupGmsCalls flag. (305be3b)
  • The Android contact picker will use the native system picker on Android 17+ when a feature flag is enabled, removing the need for the READ_CONTACTS permission. (30c0d18)
  • A prototype of a WebUI-based New Tab Page is being introduced, guarded by a GN argument and a runtime flag. (42937e4)
  • A crash in the Android BottomSheet was fixed, which occurred when changing device orientation while the keyboard was visible. (f2f79ca)

UI & UX

  • The voice search UI has been updated with a new "Aurora" recording animation and an explicit stop/submit button, gated by a feature flag. (efb148a) (6a4b098)
  • The Side Panel Bookmarks "edit dialog" was migrated from Polymer to Lit. (6786396)
  • A fix was implemented for the placement of window menus on Linux with X11, which were appearing in the wrong location due to a DIP vs. pixel coordinate mix-up. (3880f83)
  • Text annotations in the PDF viewer can now be deleted with the Backspace key in addition to the Delete key. (3790f18)
  • A race condition was fixed in tab dragging where detaching tabs could fail if the tab strip's selection model changed during the drag operation. (9000f56)

Extensions

  • A change that required a user gesture for the addHostAccessRequest API was reverted, restoring the ability for extensions to call it programmatically. (a76e95f)
  • The nacl_arch property in the runtime.getPlatformInfo() API has been deprecated, as Native Client (NaCl) was removed in M139. (6f14e2a)

Code Health & Refactoring

  • The browser_navigator files have been moved from chrome/browser/ui/ to a new chrome/browser/ui/navigator/ subdirectory, requiring updates to include paths. (815c5cf)
  • Code related to glic and lens has been modularized out of the main //chrome/browser target into their own source sets to improve dependency management. (4646bd8) (bc61a54)
  • A large number of tests in ash/ were migrated away from using the generic RunUntilIdle() to more targeted and reliable synchronization primitives, improving test stability. (78dd3d4) (60e3a08) (53b6e88)
  • A new SQL-based shared HTTP cache backend was implemented to map NetworkIsolationKey to a database ID, part of ongoing networking stack improvements. (7a02630)
