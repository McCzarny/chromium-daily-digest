Chromium Digest: 2026-04-24

Overview

On this day, 717 commits were merged to the main branch, with 541 being identified as relevant to developers. This summary covers a significant number of breaking API changes across the UI framework, WebRTC, prefetching, and core content APIs. Major refactoring efforts continue in the compositor, build system, and browser componentization. The first commit was c55d041505a613bca037523002843de2f7d646e4 and the last was 5d155b4bce588dba996810da80cf4c00632293f8.

Breaking Changes

  • ⚠️ BREAKING The core compositor class cc::LayerTreeHostImpl has been refactored. Its public static factory method Create() was removed, and the implementation was split into ClientLayerTreeHostImpl and VizLayerTreeHostImpl. Embedders who directly instantiate this class will need to update their code. (9d32df7)
  • ⚠️ BREAKING The content::public::browser::WebContents interface no longer has the GetCaptureHandleConfig() method. This functionality has been moved to Page, and the corresponding delegate method was changed from SetCaptureHandleConfig to OnCaptureHandleConfigUpdate. (359ad9a)
  • ⚠️ BREAKING The PrefetchContainer::Observer interface has been updated. The OnGotInitialEligibility and OnPrefetchCompletedOrFailed methods no longer pass eligibility and completion status as parameters. Observers must now retrieve this data directly from the PrefetchContainer object. (219dedd) (56b1776)
  • ⚠️ BREAKING The ui/views bubble API has two breaking changes: BubbleDialogDelegate::SetHighlightedButton has been removed in favor of SetHighlightedElement, and the factory method BubbleDialogDelegate::CreateBubble now requires an explicit Widget::InitParams::Ownership parameter. (b382a3d) (fa368f1)
  • ⚠️ BREAKING The public Java interface SidePanelContainerCoordinator.init() on Android now requires a SidePanelCoordinatorAndroid parameter to support bidirectional communication. (ad6a6ec)
  • ⚠️ BREAKING The PeerConnectionTrackerHostObserver interface has been updated. The OnPeerConnectionSessionIdSet method now requires a base::OnceClosure callback, which will break existing observer implementations. (52749a3)
  • ⚠️ BREAKING The PasswordManagerDriver mojom interface has removed the ShowPasswordSuggestions method. Embedders must migrate to using AskForValuesToFill on the AutofillDriver interface. (8635789)
  • ⚠️ BREAKING The blink::String::Number<T>() template method has been removed and replaced with explicit overloads for standard integer types. Code using this template with other types (e.g., short) will no longer compile. (52798a5)
  • ⚠️ BREAKING The AccountInfo::Builder has a significant behavioral change. When the kGaiaAccountIdEnforcement feature is enabled, account_id is now derived from gaia_id, and calls to SetAccountId() are ignored. (0029f3a)
  • ⚠️ BREAKING On iOS, a new virtual method WillShutDown() has been added to the FullscreenBrowserAgentObserver interface, requiring all implementers to add it to their class definition. (32fe61b)
  • ⚠️ BREAKING Several public headers have been changed, breaking includes and path lookups: DIR_INTERNAL_PLUGINS was removed from chrome_paths.h; shelf-related prefs were moved to ash_pref_names.h; leak_detection_delegate_interface.h was renamed; and the RTC_USE_H264 build flag was removed from Blink's public config. (46850ff) (c4dca43) (edd9f64) (e81b84e)
  • ⚠️ BREAKING The public C-API for the On-Device Model Service, services/on_device_model/ml/chrome_ml_api.h, has changed the signature for the GetTokenizerParams function. (db585a0)

Web Platform & Rendering

  • For Media Source Extensions (MSE), non-IDR I-frames with SEI recovery points in H.264 streams are now promoted to keyframes. This fixes seeking issues in open-GOP content for players like hls.js. (53b9408)
  • Content Security Policy for CSS Modules is now stricter: <style type=module> elements will now enforce the script-src policy in addition to style-src. (fa5ca80)
  • Numerous fixes were landed for CSS Grid Masonry, addressing crashes and incorrect layout behavior, particularly with subgrids. (715a527) (d8ff8bc) (cd6ba4d)
  • Performance of ContainerNode::ReplaceChildren() has been improved by using a single RemoveChildren() call, reducing style recalculation overhead. (1c6a39e)
  • On Windows, local font lookup is now routed through the FontDataService when the feature is enabled, part of a migration away from the legacy DWriteFontProxy interface. (01a0932) (6c6ce5d)
  • The InputVizard feature for improved input handling is now enabled by default on Android. (b1f0797)
  • A race condition in WebGL has been addressed by ensuring element array data is read exactly once. (013d0c8)

UI & Features

  • The WebUITabStrip feature, which was behind a disabled-by-default flag, has been completely removed from the codebase. (4e27221)
  • On Android, a new JNI method has been added to Tab to allow Java code to asynchronously query the memory footprint of a tab's renderer, intended for use in UI like hover cards. (fb82a78)
  • On iOS, a new runtime flag, delayLoadingResources, has been added to CWVEarlyInitFlags to allow deferring resource bundle initialization, improving startup performance for WebView clients. (d2cf47f)
  • The PDF viewer's Ink Signatures feature received a new private API, chrome.pdfViewerPrivate.getTextInfo(), to improve font handling performance. (262b57c) (dd6626c)
  • The Page Summary feature and its associated UI elements have been removed from Android, including public constants in FeatureConstants.java. (77ddc42) (59ca8a4)

Services & APIs

  • The SamplingHeapProfiler and its mojo interface now support a Stop() method, allowing clients like Electron to safely toggle the heap profiler at runtime without crashing the service. (f106c51)
  • A new API has been added to SessionSyncService to support storing tab screenshots, likely for use in features like tab resumption. (a611ade)
  • The OneTimeCodeFieldParser has been integrated into Autofill's form parsing heuristics to improve the detection of One-Time Password (OTP) fields. (9097ee6) (266913f)
  • The Send Tab To Self feature has been enhanced to use Autofill's form and field signatures for more reliable form filling on the receiving device. (ad9a1e9) (3dbb736)

Security & Stability

  • Multiple Use-After-Free (UAF) vulnerabilities have been addressed in RenderFrameHostImpl::ExitFullscreen(), WidgetBaseInputHandler, the Presentation API, and Bluetooth connection handling on macOS. (f2c3442) (d3038a7) (959627a) (cc8c915)
  • The tab freezing policy now uses base::LiveTicks instead of base::TimeTicks for its unfreeze timer. This prevents the timer from advancing while the system is suspended, avoiding a "thundering herd" of tabs being unfrozen simultaneously upon system resume. (162b410)
  • A dangling pointer in update_client::Component was fixed by switching from base::raw_ref to base::WeakPtr in callbacks, preventing crashes during component updates. (d0b245d)
  • Input validation was added to the accessibility system to prevent the renderer from sending reserved or browser-specific AXNodeID values, preventing potential collisions. (65fba3d)
  • On iOS, scheme validation was added for auto-generated search engines to prevent potential Universal Cross-Site Scripting (UXSS) vulnerabilities. (c1bb5bb)

Build & Infrastructure

  • Significant effort was made to modularize core browser components by moving sources out of the monolithic //chrome/browser:browser target, including //chrome/browser/signin and //chrome/browser/navigation_predictor. (3f51165) (5ea02e3)
  • The monolithic generate_bindings_all build action was split into 11 per-type actions, enabling parallel execution and reducing build times. (f7c2ded)
  • A new feature has been added to PartitionAlloc to reserve a "zero segment" at the beginning of the user-space address space. This is a prerequisite for the V8 Sandbox. (9924b55)
  • Major dependencies libc++ and perfetto have been rolled forward, incorporating numerous upstream changes. (612b748) (7326987)
