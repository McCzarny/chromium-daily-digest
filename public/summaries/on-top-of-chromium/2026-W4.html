Week 4 of 2026 (2026-01-18 to 2026-01-24)

Chromium Weekly: 2026 Week 4

Overview

This week in Chromium saw an exceptional number of breaking changes across core systems, with 938 relevant commits out of 1237 total. Developers should pay close attention to the complete removal of the legacy SigninManager and the mechanical renaming of base::Value::Dict and base::Value::List. Other significant changes include the removal of the "Fast Checkout" feature, numerous modifications to the content and extensions APIs, and the stabilization of several UI and security features.

Major Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE: The legacy SigninManager and its factory have been completely removed from chrome/browser/signin. This is a major API removal affecting all desktop sign-in and identity integrations, requiring migration to modern patterns like IdentityManager. (ef1e523) (a4da4de)
  • ⚠️ BREAKING BREAKING CHANGE: The widely used base::Value::Dict and base::Value::List have been renamed to base::DictValue and base::ListValue respectively. This mechanical change requires embedders to update all usages of these fundamental data types. (8e25bad) (42b022b) (d673b3a)
  • ⚠️ BREAKING BREAKING CHANGE: The public content::RenderProcessHost interface has two new pure virtual functions: IncrementPendingReuseRefCount and DecrementPendingReuseRefCount. Embedders with custom RenderProcessHost implementations must add these methods to avoid build failures. (7f786d9)
  • ⚠️ BREAKING BREAKING CHANGE: The entire "Fast Checkout" feature and the legacy TabLoader class for session restoration have been removed. Embedders relying on these for custom payment or session logic must migrate to newer mechanisms. (52e96c0) (7eca812)

APIs & Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The extensions::ManagementPolicy::Provider::UserMayInstall method is now asynchronous, requiring updates to custom providers. Other improvements include a fix for the webRequest.onAuthRequired event with service workers, correct firing of the tabGroups.onUpdate event, and a new command-line flag (--permit-dnr-rules-on-webviews) to apply Declarative Net Request rules to webviews. (b79a8c6) (6407790) (a2234fe) (bfcea3d)
  • The PWA installation flow has been changed so that the <install> HTML element will now always bypass the permission prompt, treating the request as implicitly granted by the user's click. (8d4441f)
  • The DevTools Protocol command Target.createTarget now accepts an optional focus parameter, allowing automation tools to create a new foreground tab without bringing the entire browser window to the front. (89de898)
  • ⚠️ BREAKING BREAKING CHANGE: Support for the "Preview" protocol has been completely removed from the Digital Identity API. Calls attempting to use this protocol will now fail. (a40eb38)

Security & Policy

  • The clipboard API has been hardened: ReadAvailableCustomAndStandardFormats() and ReadUnsanitizedCustomFormat() now require renderer paste permissions, preventing compromised renderers from reading custom clipboard formats without user activation. (67ed1f7)
  • ⚠️ BREAKING BREAKING CHANGE: A significant behavioral change was introduced for Incognito mode enterprise policies. The IncognitoModeUrlAllowlist policy can now override an IncognitoModeAvailability policy set to 'disabled', potentially enabling Incognito mode unexpectedly in managed environments. (01a54f3)
  • A new enterprise policy, RestrictPdfSaveToGoogleDriveAccountsToPattern, has been added, allowing administrators to restrict which Google accounts can be used for the 'Save to Drive' feature in the PDF viewer. (c86d1da)
  • WebAuthn Passkey support has been stabilized with the removal of the kWebAuthnPasskeyUpgrade feature flag. Support was also added for the CTAP2.2 hmac-secret-mc extension, improving PRF-on-Create for certain authenticators. (eb53ce0) (9ad0302)

Browser UI & Session Management

  • ⚠️ BREAKING BREAKING CHANGE: The session restore mechanism now preserves the collapsed state of tab groups. This required adding a new pure virtual method, GetGroupIdForSavedGroup, to the public sessions::LiveTabContext interface, which embedders must implement. (1ae9a24)
  • ⚠️ BREAKING BREAKING CHANGE: The core browser navigation functions chrome::CanGoBack and chrome::GoBack have been modified to incorporate "back-to-opener" logic, altering the fundamental behavior of the back button in some scenarios. (81bc0fc)
  • The Tab Search button is now enabled by default, altering the default appearance of the browser's top chrome. (eeb683f)
  • The experimental Vertical Tabs feature received numerous fixes for drag-and-drop rendering, group editor behavior, and now correctly shows placeholder titles for loading tabs. (db67e0e) (dd67882) (f682b0e) (9c249d5)

Core Internals & Testing

  • ⚠️ BREAKING BREAKING CHANGE: Numerous base library utilities have been removed or altered, including base::Contains, histogram helpers like AddScaled(), and base::HistogramFlattener. The signature for base::ToVector was also changed. (4e6249c) (9b7ede3) (7cf7b7d) (a2039f6)
  • ⚠️ BREAKING BREAKING CHANGE: Widespread file path relocations for public headers like child_process_id.h (to content/public/common/) and split_tab_id.h (to components/split_tabs/) require build configuration updates. (0f00d98) (fd58fdc)
  • A large internal rename of "Supervised User" to "Family Link" was performed across the codebase to align with current product branding. (97b35f3)
  • Trace events for base::WaitableEvent and base::ConditionVariable have been improved to capture the caller's source location, making performance analysis of thread synchronization more actionable. (35892d8)
