Chromium Weekly: 2026 Week 2

Overview

A week marked by an exceptionally high number of breaking changes, requiring significant attention from developers building on Chromium. Out of 2549 total commits across 5 days, 1847 were identified as relevant. Major changes impacted core UI interfaces for tab management, security policies on macOS, device monitoring capabilities, and several fundamental APIs in the base, networking, and graphics libraries.

Browser & UI

  • ⚠️ BREAKING BREAKING CHANGE: A series of breaking changes significantly refactored core tab management UI interfaces. Embedders with custom UI will need to update implementations for TabStripModel (return type of selection_model() changed), BrowserView (tabstrip() method removed), TabDragTarget (new GetAttachedContext() method), TabListInterface (new GetTabGroupTabIndices() method), and TabStripRegionView (new GetTabRendererData() method). (5242d38) (f55ce20) (4eee675) (cbf7c2c) (cfabb7e)
  • ⚠️ BREAKING BREAKING CHANGE: Public static methods GetGuestBrowserCount() and IsOffTheRecordBrowserInUse() have been removed from the BrowserList class. Embedders must migrate to equivalent functions in chrome/browser/ui/browser_finder.h. (83633e5)
  • ⚠️ BREAKING BREAKING CHANGE: The public method RenderWidgetHost::WillSendInputEventToRenderer in content/public/browser/render_widget_host.h has been renamed to SimulateUserInteraction. (064f6d1)

Security & Policy

  • ⚠️ BREAKING BREAKING CHANGE: On macOS, loading of third-party in-process plugins is now blocked by default for the CHILD_PLUGIN process type. This security hardening measure will break embedders relying on this mechanism for binaries not signed with the same Team ID. (c54ef9c)
  • ⚠️ BREAKING BREAKING CHANGE: Support for detecting MTP/PTP devices via the public StorageMonitor component has been completely removed on both Windows and macOS, breaking any functionality that relied on it for media device interaction. (aef221a) (8e0605c)
  • ⚠️ BREAKING BREAKING CHANGE: The bssl::TrustStore base class in BoringSSL now has a new pure virtual function, GetTrustedMTCIssuerOf. Custom subclasses must implement this method to avoid build failures. (504a326)
  • A new WebAppInstallForceList group policy has been implemented, allowing enterprise administrators to prevent users from installing web apps from the browser on Windows and macOS. (06504a5)

APIs & Core Libraries

  • ⚠️ BREAKING BREAKING CHANGE: The entire //ui/gfx/ipc/skia component, which provided legacy IPC serialization for Skia data types, has been removed. Embedders must find alternative serialization methods. (6ef8408)
  • ⚠️ BREAKING BREAKING CHANGE: The base::MemoryPressureListener API was significantly refactored. The static SetNotificationsSuppressed method was removed in favor of a new reference-counted base::MemoryPressureSuppressionToken system. (63f5b8d)
  • ⚠️ BREAKING BREAKING CHANGE: Several core utility functions were removed from public headers. base::ByteSwap is removed in favor of std::byteswap, and an overload of url::CanonOutputT::Append was removed in favor of the std::basic_string_view version. (5869e71) (9989ff4)
  • ⚠️ BREAKING BREAKING CHANGE: Public serializer functions for session and local storage have been made private, breaking any external code that relied on them to interact with the underlying LevelDB storage. (23e8879) (dee78ad)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The sidePanel.close() API's behavior has changed. When called with a tabId, it no longer falls back to closing the global side panel and will now return an error if no tab-specific panel is enabled, potentially breaking extensions that relied on the fallback. (cf0517b)
  • The runtime.onMessageExternal listener now supports returning a Promise for asynchronous responses, allowing for more modern async handling in extensions. (b093c11)
  • The UnpackedInstaller class was moved from chrome/browser/extensions to the more generic extensions/browser directory, making it more accessible to non-Chrome embedders who need to install unpacked extensions. (e87a1a6)

Networking

  • ⚠️ BREAKING BREAKING CHANGE: Deprecated public proxy APIs have been removed from the Cronet networking library's Java interface. Embedders must now use static factory methods for Proxy and ProxyOptions. (3670059)
  • ⚠️ BREAKING BREAKING CHANGE: The disable_idle_sockets_close_on_memory_pressure parameter has been removed from the public network.mojom.NetworkContextParams interface. (6cbcf51)
