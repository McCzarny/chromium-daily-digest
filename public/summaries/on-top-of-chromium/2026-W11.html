Week 11 of 2026 (2026-03-08 to 2026-03-14)

Chromium Weekly: 2026 Week 11

Overview

This week's activity, with 509 relevant commits out of 707 total, introduced a wave of API modernizations and security enhancements. A significant number of breaking changes affected core networking, UI, and service APIs, requiring embedder attention. Key highlights include enabling the network service sandbox by default on Windows, major refactoring of the clipboard and tab strip APIs, and the removal of several legacy features.

Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE Multiple core networking APIs have been modified. The ConnectionAllowlist mojo struct now uses enums, the ClientSocketPool::Preconnect callback returns a bool instead of a net::Error, HttpStream::GetTotal...Bytes() methods return net::ByteSize, and the SetTrackingProtectionEnabledFor3pcd method was removed from the CookieManager mojo interface. (9d5b5cf) (ad97a90) (5895f46) (9bf8133)
  • ⚠️ BREAKING BREAKING CHANGE The UI and clipboard APIs saw significant refactoring. The synchronous ui::Clipboard::IsFormatAvailable has been removed in favor of an asynchronous GetAllAvailableFormats. For context menus, blink::WebMenuSourceType was replaced by ui::mojom::MenuSourceType, and the virtual method OnWidgetClosing was removed from WebUIBubbleDialogView. (54777b8) (75a48e0) (d18477d)
  • ⚠️ BREAKING BREAKING CHANGE Core service APIs were updated. HistoryService methods now use a strongly-typed history::ClusterId, AutofillClient has new methods for AI failure notifications, and the content::ChildProcessHost::LaunchAsChildProcess method for launching plugin processes has been removed entirely. (9cb1107) (b1627f2) (12d359c)
  • ⚠️ BREAKING BREAKING CHANGE On Windows, the automatic fallback to the ANGLE D3D9 backend has been removed. If D3D11 initialization fails, Chromium will now use software rasterization, potentially impacting performance on older systems. (282348e)
  • ⚠️ BREAKING BREAKING CHANGE The navigator.storage.estimate() API will now always report a fixed quota in Incognito mode to prevent privacy leaks. This is a behavioral change to a stable API. (c48f71d)
  • ⚠️ BREAKING BREAKING CHANGE Extension messaging behavior has changed. Attempting to send a SharedArrayBuffer in a message, which previously failed silently, will now throw an error. (7c8c8fb)

Security

  • The network service is now sandboxed by default on Windows, significantly hardening the browser's security posture against network-based attacks. (a0ad012)
  • Security was tightened for web standards. The Authorization header is now correctly stripped on cross-origin fetch() redirects when keepalive is used, and a browser-side check now enforces the allow-modals sandbox attribute to prevent compromised renderers from showing dialogs. (5fb6646) (7e25766)
  • For the new Built-In AI APIs, browser-side permission policy checks have been added. If a compromised frame attempts to use these APIs without permission, the renderer process will be terminated. (a3d6e7c)

APIs and Browser Internals

  • The Direct Sockets API has been extended to support Source-Specific Multicast (SSM) for UDP sockets, allowing isolated web applications to join and leave multicast groups with source filtering. (0302377)
  • The ServiceWorkerAutoPreload feature is now enabled by default, a performance optimization that may alter resource loading behavior for web applications that use Service Workers. (d500868)
  • A new method, ContentBrowserClient::IsTopChromeWebUIURL, was introduced to provide a reliable way for embedders to identify processes hosting privileged Top Chrome WebUI. (0747310)
  • The MainIdleBypassScheduler feature has been enabled by default, improving the stability and predictability of the requestIdleCallback API for scheduling low-priority tasks. (b25cdb8)

Extensions

  • Browser stability has been improved by making it more resilient to messages from stale extension service workers. Instead of crashing, the browser will now safely drop IPC messages from outdated worker instances. (de506e6) (d12bcc8) (10acabd)
  • A crash in the chrome.cookies API was fixed by wrapping Mojo callbacks, ensuring they are always invoked even if the network service disconnects during an API call. (70dd2cc)
  • Modernization of extension APIs continues with the conversion of the userScripts API from JSON schema to WebIDL. (fab7b96)

UI and Testing

  • A new HitTestCallback was added to ui::views::FrameView, providing a unified, cross-platform hook for embedders to implement custom draggable regions in non-client areas. This replaces the now-removed macOS-specific NativeFrameViewMacClient. (7c1ec2a) (5a2ee1d)
  • A new testing utility, CrashFrameProcess, has been added to content/public/test/browser_test_utils.h to simplify writing browser tests that validate behavior when a renderer process crashes. (efe7d29)
  • As part of ongoing dependency hygiene efforts, numerous components within chrome/browser were modularized into their own build targets, including page_info, webapps, memory, and site_isolation. (f7272c2) (c26595f) (15e29f9) (9c845fc)
