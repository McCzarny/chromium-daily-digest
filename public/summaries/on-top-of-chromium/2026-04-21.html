Chromium Digest: 2026-04-21

Overview

This summary covers 684 commits to the Chromium main branch on April 21st, 2026, from which 496 were identified as relevant to developers building on Chromium. This digest focuses on a significant number of breaking API changes, particularly in Remoting and core browser interfaces, alongside security enhancements and build system modernizations. The first relevant commit was 054c47b1b245e62ccc13ccb3e6f9209eab288a35 and the last was 69a4fe10c99acd66f172d6770d5d5a25f35747d5.

Breaking Changes: API & Frameworks

  • ⚠️ BREAKING The public content::DirectSocketsDelegate interface has been simplified. Three virtual methods related to Private Network Access (RenderFrameHasDirectSocketsPNAContentSetting, SharedWorkerHasDirectSocketsPNAContentSetting, ServiceWorkerHasDirectSocketsPNAContentSetting) have been removed, requiring embedders who implement this delegate to update their code. (c86e354)
  • ⚠️ BREAKING The public BrowserWindow interface has been refactored. Several sharing-related methods have been removed and relocated to a new BrowserWindowFeatures class, breaking embedders that call these methods directly on BrowserWindow. (c20a9e0)
  • ⚠️ BREAKING The public TemplateURL class in components/search_engines now uses string_view instead of const std::string& for several method parameters to improve performance. This requires embedders to update their calls to these methods. (ee1830e)
  • ⚠️ BREAKING The WebAppCommandScheduler::LaunchApp public method has been renamed to WebAppCommandScheduler::LaunchAppFromCommandLine. Embedders launching web apps via command line parameters must update their code to use the new method name. (d189391)
  • ⚠️ BREAKING A revert has removed the recently added ContextMenuDelegate::BuildMenuAsync() method from the public components/renderer_context_menu interface. Embedders who adopted the async method must revert to the synchronous BuildMenu() implementation. (378dd7b)
  • ⚠️ BREAKING The constructor signature for CloudPolicyCoreStatusProvider and its subclasses has changed. Embedders using this public interface for custom policy UIs must update their code to pass a CloudPolicyManager* and Profile* instead of a CloudPolicyCore*. (db41c1d)
  • ⚠️ BREAKING In the WebAuthn implementation, the nested TouchToFillControllerWebAuthnDelegate::CredentialReceiver class has been extracted into a new, standalone public interface TouchToFillCredentialReceiver. Code referencing the old nested class path will break. (ca5d5ca)
  • ⚠️ BREAKING Several unused public methods, including HasPrefetchBeenConsideredToServe() and StartBlockUntilHead(), have been removed from the PrefetchContainer class, which is part of the public content API. (4ec75a6)
  • ⚠️ BREAKING The public method AimEligibilityService::IsCountry has been removed and GetCountryCode has been made private. Consumers of this service can no longer use it for country-based checks. (671fe30)
  • ⚠️ BREAKING The public static helper method ClipboardFormatType::FileContentZeroType() has been removed. Code should be updated to use FileContentAtIndexType(0) instead. This was part of a change to improve drag-and-drop interoperability on Windows. (c47189a)

Breaking Changes: Remoting

  • ⚠️ BREAKING Major components of the legacy (non-WebRTC) remoting protocol have been removed. This includes public classes like BufferedSocketWriter, MessageReader, and MessageDecoder from the messaging pipeline. (2b28768)
  • ⚠️ BREAKING The legacy Chromotocol video pipeline has been deleted. This removes the public VideoEncoder and VideoDecoder interfaces and their implementations. Embedders must use the modern WebRTC-based architecture for remoting solutions. (9cc0984)

Breaking Changes: Build System

  • ⚠️ BREAKING A new "unified modules" system has been implemented for builds with use_clang_modules=true. This fundamentally changes how libc++ and system headers are handled and will break custom build configurations that relied on the previous module structure or targets like libcxx_headers. (9416dff)

Security & Permissions

  • ⚠️ BREAKING Behavior Change: The permission system has been updated to allow ephemeral (one-time) and persistent grants to coexist for the same origin. A new ContentSettingsConstraints parameter in HostContentSettingsMap::SetWebsiteSetting controls this behavior, which may impact embedders' permission management logic. (93ced04)
  • A Universal Cross-Site Scripting (UXSS) vulnerability in the Reading Mode side panel has been patched. The fix ensures strings passed from extensions to the ttsEngine API are properly sanitized. (eb38552)
  • A security vulnerability in download resumption that could lead to incorrect hash calculations and bypass Safe Browsing checks has been fixed. (fc030cd)
  • To harden against forged location information, the X-Country and X-Geo-Level-1 headers are no longer trusted when the variations seed is fetched over an insecure HTTP connection. (536eb1d)
  • Enterprise Data Controls policies are now enforced for the "Search for..." context menu item, which will be hidden or show a warning based on policy. (9b14dce)

Browser UI & Features

  • A new reusable dialog delegate, TopChromeWebUIDialogDelegate, has been introduced and used in several places to reduce boilerplate and standardize the creation of top-level WebUI dialogs. (0e80a62) (f14a8c0) (1784e6c)
  • The "Send Tab to Self" feature has been improved to wait for a confirmation from the Sync server before showing the "Sent" UI to the user, with a timeout to prevent the UI from hanging. (c3cd37a) (e38dedb)
  • On macOS, a fix has been implemented to correctly handle double-click-to-zoom actions on custom draggable regions, such as the empty space in a vertical tab strip. (2be0ae4)
  • The AutocompleteController's observer list now explicitly allows re-entrancy, fixing a regression where UI updates could not trigger a new autocomplete query. (415cfa4)
  • An uninstall button has been added to the PWA frame toolbar, which appears on the first launch after installation but is hidden for preinstalled default apps. (244798a) (0482137)

Build System & Performance

  • To improve linking performance for large targets on Windows, the /prefetch-inputs flag is now enabled by default for lld-link during cold cache builds. (8e295b0)
  • To reduce latency before navigation, database updates for the AutocompleteActionPredictor are now deferred to an asynchronous, low-priority task. (5d63702)
  • The DirectML (DML) backend for the experimental WebNN API has been removed, as the ONNX Runtime backend is now preferred on Windows. (ccff00c)
  • The ongoing "Bedrock" project to improve modularity continues, with //chrome/browser/ssl and //chrome/browser/new_tab_page being moved into their own dedicated build targets. (124fb31) (22a94a8)
