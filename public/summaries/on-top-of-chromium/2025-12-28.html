Chromium Digest: 2025-12-28

Overview

This digest analyzes 5 commits, with 3 selected after filtering for relevance to browser-level projects on Windows and macOS. The key changes include a significant, potentially breaking modernization of core Mojo APIs, an internal memory safety fix for Windows, and a minor UI bug fix in the Omnibox. Core APIs

  • ⚠️ BREAKING Mojo's public binding headers (clone_traits.h, equals_traits.h, etc.) have been refactored to use C++20 concepts instead of SFINAE (std::void_t) for trait detection. This modernizes the codebase but alters the underlying template machinery of public interfaces. This is a breaking change for advanced embedders who may have been specializing or otherwise interacting with the previous SFINAE-based trait implementations. (694c89f)
  • An internal memory safety fix was applied to Windows-specific utility code (base/win/winrt_storage_util.cc). The change updates buffer handling for COM interface methods to correctly use UNSAFE_BUFFERS, resolving a UNSAFE_TODO. This does not affect any public APIs. (b8e26d0)

Browser UI

  • A bug in the Omnibox has been fixed that caused 'search in tab' chips to appear incorrectly when the user had typed input. The logic now correctly ensures that only one type of tab chip is shown at a time. (72cb7ea)
