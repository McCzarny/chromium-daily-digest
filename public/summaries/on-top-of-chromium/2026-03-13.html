Chromium Digest: 2026-03-13

Overview

A busy day on the main branch with 606 commits, of which 437 were analyzed for developer impact. The day saw a significant number of breaking changes, particularly in the UI clipboard and core service APIs, requiring attention from embedders. Changes span from commit (1c93014) to (9bf8133).

Breaking Changes: Core APIs & Services

  • ⚠️ BREAKING As part of the Third-Party Cookie Deprecation (3PCD) cleanup, the SetTrackingProtectionEnabledFor3pcd method and tracking_protection_enabled_for_3pcd parameter have been removed from the public network::mojom::CookieManager interface. Embedders configuring the network service via this mojom will have a build break. (9bf8133)
  • ⚠️ BREAKING The public content::ServiceWorkerContextObserverSynchronous interface signature has been modified. The OnStoppingSync and OnStoppedSync methods now include a blink::ServiceWorkerToken parameter to provide more specific worker instance information, requiring embedders to update their observer implementations. (aa8f4cd)
  • ⚠️ BREAKING The ResolveTokenRequest method in the federated_auth_request.mojom interface and the corresponding IdentityRegistryDelegate::OnResolve method have been refactored. Parameters are now grouped into a union, breaking existing implementations of the FedCM API. (1be409c)
  • ⚠️ BREAKING The 'User Notes' feature has been removed. This includes the removal of the AnnotationType.kUserNote enum value from the public third_party/blink/public/mojom/annotation/annotation.mojom interface, which will break any client code referencing this annotation type. (27f8c22)
  • ⚠️ BREAKING The ExtensionInstallPolicyService::CanInstallExtension method signature has changed. It now returns a std::optional<std::u16string> to provide a localized message explaining why an extension is blocked, instead of a simple boolean. Integrations that check for extension installation permissions must be updated. (f53de3f)

Breaking Changes: UI & Platform Clipboard

  • ⚠️ BREAKING The core clipboard API has been significantly refactored to be more asynchronous. The synchronous ui::Clipboard::IsFormatAvailable method has been removed entirely and replaced by the asynchronous GetAllAvailableFormats. This requires callers to adapt to a callback-based pattern. (54777b8)
  • ⚠️ BREAKING The ui::PlatformClipboard::IsSelectionOwner method signature has been reverted to its synchronous form (bool IsSelectionOwner(...)) after a brief period as an asynchronous method. Any code that adapted to the recent asynchronous version will break and must be reverted. (f80ba53) (8dc8619)
  • ⚠️ BREAKING In the Ozone platform interface, PlatformClipboard::OfferClipboardData has been made fully asynchronous by removing its synchronous OfferDataClosure callback. This breaks the API contract for clients implementing or using this part of the Ozone platform. (9dc355d)

APIs, Extensions, and Security

  • A major internal refactoring of the Tab Strip Service API was completed to make it platform-agnostic. While this is a large architectural change involving new adapters and interfaces, it is designed to be internal and should not break embedders using the established public interfaces. (c9b4d76)
  • The PDFiumEngineClient interface for the PDF viewer was modernized to use base::span for buffer handling in its SubmitForm and InstallBuffer methods, improving type safety for embedders. (0299cee)
  • On Windows, Google Credential Provider (GCPW) sign-in flows now support security keys. The GCPW process acts as a broker to access the security key device, improving 2-Step Verification support in enterprise environments. (b85ad72)
  • The CopySharedImageToGLTextureViaTextureCopy method has been moved from an internal implementation to the public gpu::gles2::GLES2Interface, making it available for embedders to use directly for GPU operations. (6482daf)

Browser UI & Features

  • The feature flag for loyalty card filling in Autofill (kAutofillEnableLoyaltyCardsFilling) has been removed, enabling the feature by default for all users. (9d45f2f)
  • On macOS, window restoration behavior has been improved to always restore windows to their correct 'spaces' (virtual desktops), even when the application is manually quit and restarted. (f9a6ce0)
  • The First-Run Experience (FRE) sign-in page now includes a 'Don't sign in' button on the native toolbar, providing a clearer path for users who wish to skip sign-in. (ee4d985)

Build & Infrastructure

  • Work continues on improving dependency hygiene by modularizing several browser components into their own build targets, including //chrome/browser/memory, //chrome/browser/unified_consent, and //chrome/browser/ui/frame. (15e29f9) (ae443f5) (91da409)
