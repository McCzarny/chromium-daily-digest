Chromium Digest: 2025-12-03

Overview

On December 3rd, 2025, the Chromium main branch saw 596 commits, with 428 deemed relevant to developers building on the platform. This day was marked by a significant number of breaking API changes across Media, GPU, WebAuthn, and core browser interfaces, requiring careful review and updates from embedders. The changes span from commit (6c7ffdd) to (d46e733).

APIs & Core Interfaces

  • ⚠️ BREAKING BREAKING: The media::VideoFrame API underwent a major refactoring. The public enum VideoFrame::STORAGE_GPU_MEMORY_BUFFER was renamed to VideoFrame::STORAGE_MAPPABLE_SHARED_IMAGE, the public method VideoFrame::MapGMBOrSharedImage() was renamed to VideoFrame::MapSharedImage(), and the related LegacyGpuMemoryBufferForVideo class was removed. Embedders interacting with video frames will need to update their code. (42fcda8) (db2f79e) (8ca93f6)
  • ⚠️ BREAKING BREAKING: The DawnCachingInterface has been refactored. The GpuChannelManager constructor signature was changed to accept a GpuPersistentCacheCollection, and an overload of DawnCachingInterfaceFactory::CreateInstance was removed. This affects embedders integrating with Dawn for GPU operations. (4cc3088)
  • ⚠️ BREAKING BREAKING: The public member AccountInfo::hosted_domain has been made private. Embedders must now use the new public method AccountInfo::GetHostedDomain() to access this information. (265e995)
  • ⚠️ BREAKING BREAKING: The signature of content::SpeechRecognitionDispatcherHost::Create has been changed to accept a RenderFrameHost* instead of an int render_frame_id. This change is part of a larger refactoring to improve embedder flexibility in overriding Mojo interfaces. (6207e68)
  • ⚠️ BREAKING BREAKING: The API for observing Isolated Web App runtime data in iwa_runtime_data_provider.h has been modified. The IwaRuntimeData::Observer pattern was replaced with a CallbackListSubscription, breaking existing consumers. (26a966b)
  • ⚠️ BREAKING BREAKING: The launched feature flag kZeroSuggestPrefetching was removed, along with its associated public helper function OmniboxFieldTrial::IsZeroSuggestPrefetchingEnabled(). Code that calls this function will no longer compile. (150a9ac)
  • The ContextualTasksService::Observer interface was updated with new methods OnTaskAssociatedToTab and OnTaskDisassociatedFromTab to notify observers when a tab's association with a task changes. (c0e582a)
  • A use-after-free bug in base::StackSampler was fixed by making its internal data structure thread-safe, improving stability during browser shutdown. (4f629fe)

Security & WebAuthn

  • ⚠️ BREAKING BREAKING: Public FIDO headers have been moved from device/fido/ to device/fido/public/ without providing forwarding headers. Downstream projects that include the old header paths will experience build failures. (0615b88)
  • ⚠️ BREAKING BREAKING: The public virtual method GetUnShadowedPasskeys() has been removed from the PasskeyModel class. Consumers should migrate to the GetPasskeys() method. (c1498b4)
  • ⚠️ BREAKING BREAKING: The public API for creating Safe Browsing interstitial pages has changed. The should_trigger_reporting parameter was removed from the CreateSafeBrowsingPage method and SafeBrowsingBlockingPage constructor. (2757ee1)
  • The PasskeyModel API was refactored to introduce unified GetPasskeys and GetPasskey functions for fetching credentials, simplifying client code. (a3f9ac7) (bb19ec2)
  • A new client-side phishing detection trigger, ImageEmbeddingMatch, was implemented to compare page image embeddings against a list of target embeddings. (6e095bf)
  • The WebHistoryService will now use the more specific webhistory OAuth scope instead of the broad sync scope for its requests, improving security posture. (83fee2d)

Policy

  • ⚠️ BREAKING BREAKING: The UserAgentReduction enterprise policy and its associated public helper functions in embedder_support (e.g., GetUserAgentReductionFromPrefs) have been removed. Embedders relying on this policy must update their code. (64f74a0)
  • ⚠️ BREAKING BREAKING: Policy preference name constants in the public header components/policy/core/common/policy_pref_names.h were converted from extern variables to inline constexpr. This may break builds that are not C++17 compatible or have specific linkage expectations. (bbd0a2b)
  • A new policy domain, POLICY_DOMAIN_EXTENSION_INSTALL, was introduced to support enterprise extension installation policies. (5387010)
  • The logic for applying the SecondaryGoogleAccountSigninAllowed policy has been refined to only check cookie settings for accounts.google.com, preventing cases where blocking cookies on google.com would unintentionally break secondary account sign-in. (952ce66)

Networking & Storage

  • ⚠️ BREAKING BREAKING: The signature of net::disk_cache::CreateCacheBackend has been changed to include a new cache_encryption_delegate parameter. Embedders who create their own disk cache backends will need to update their calls to provide this new parameter. (f7ba4f3)
  • An internal refactoring introduced a CacheFile abstraction to prepare for transparent disk cache encryption. This replaces direct base::File usage within the disk cache implementation. (cd57155)
  • A bug was fixed where HTTP 425 (Too Early) responses were incorrectly retried for all idempotent requests. The fix ensures retries only happen when TLS early data was actually sent. (1a07b4e)

Browser UI & Features

  • A new ExtensionsToolbarContainerViewModel was introduced to separate UI logic from data modeling for the extensions toolbar, improving code structure and preparing for cross-platform logic reuse. (441fb4c)
  • The text in the tab context menu now dynamically updates to reflect the vertical tabs state, for example, showing "Close Tabs Below" instead of "Close Tabs to the Right". (7576257)
  • A bug causing the Omnibox popup to appear in the wrong position after moving a browser window has been fixed by synchronizing its bounds before showing. (c067a28)
  • On macOS, a window focus issue that occurred when switching windows via notification clicks has been resolved by deferring the processing of windowDidResignKey events. (c9401b7)

Testing & Infrastructure

  • ⚠️ BREAKING BREAKING: Support for the --headless=old command-line switch has been completely removed. Scripts or applications still using this value will now see it treated as an invalid argument. (b18a8a9)
  • The --enable-unsafe-webgpu command-line flag was added to the list of flags that never expire, ensuring developers can continue using it for testing upcoming WebGPU features. (c87c61a)
  • A new test-only API, BrowserManagerService::AddBrowserForTesting(), was added to support the ongoing transition away from BrowserList in unit tests. (961e7f0)
