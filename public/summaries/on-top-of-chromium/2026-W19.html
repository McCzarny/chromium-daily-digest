Chromium Weekly: 2026 Week 19

Overview

This week's development cycle, spanning 3,643 total commits with 2,692 relevant changes, was dominated by a wave of breaking API changes aimed at modernizing core browser architecture, storage, and extension systems. Major refactoring efforts touched the Browser class, SiteInstance, and SQLite integration, while the extensions platform saw significant updates to MIME handling and the removal of the legacy Web Store app.

Core APIs & Browser Internals

  • ⚠️BREAKING CHANGE: The core Browser class underwent significant refactoring to slim down its interface. Shutdown and unload logic was moved to a new UnloadController, and focus management methods were moved to BrowserFocusController. Embedders with custom window closing logic or those who directly called focus methods on the BrowserWindow interface will need to update their code. (37c7b9f) (16c69f3) (beaae0f)
    BrowserUIAPI
  • ⚠️BREAKING CHANGE: The public method SiteInstance::GetSiteURL() has been removed. As part of a large-scale refactoring to centralize security information in SecurityPrincipal, embedders must now use SiteInstance::GetSecurityPrincipal().GetDeprecatedSiteURL() to access the site URL. (01d357d)
    BrowserSecurityAPI
  • ⚠️BREAKING CHANGE: Multiple legacy APIs were removed from the base library as part of ongoing cleanup. Notable removals include base::subtle::Atomic64::Acquire_Load and the base::trace_event::TraceScopedTrackableObject class template, requiring downstream projects to migrate to modern equivalents. (ef2ad96) (00b4862)
    BaseInfra

Extensions & Web Store

  • ⚠️BREAKING CHANGE: The component-hosted app for the Chrome Web Store is no longer loaded by default on Windows, macOS, and Linux. Navigations to the webstore URL are now treated as a regular website, removing the special privileges previously granted by the hosted app model. This change is controlled by the kWebstoreHostedApp feature flag. (22c9e8a)
    ExtensionsBrowserSecurity
  • ⚠️BREAKING CHANGE: The handling of MIME types by extensions saw significant changes. A new chrome.mimeHandler.abortAndFallbackToNativeHandler() API was introduced, allowing extensions to cede control to native handlers. Concurrently, extension frames handling MIME types are now treated as the top-level context for network and storage partitioning, which may alter security and storage behavior for some extensions. (d39ac48) (45b03bb)
    ExtensionsAPINetwork
  • ⚠️BREAKING CHANGE: Several core extension APIs were updated for improved type safety and correctness. The extensions::WebRequestInfo struct now uses a strongly-typed content::GlobalRenderFrameHostId instead of raw integers, and Extension::GetManifestData() now returns a const pointer, enforcing read-only access to the manifest. (0f97ab9) (9d8833a)
    ExtensionsAPI

Storage & Networking

  • ⚠️BREAKING CHANGE: The bundled SQLite library was upgraded to version 3.53.1. To prevent database corruption, a new runtime CHECK has been added that will intentionally crash the process if multiple unsynchronized connections are opened to the same sandboxed database file. Embedders must audit their database connection patterns to avoid this fatal error. (030b805) (48c4bd9) (87e5fd4)
    StorageBaseBuild
  • ⚠️BREAKING CHANGE: Navigation security policy has been tightened significantly. Renderers that incorrectly provide an Origin header on a navigation request will now be terminated. This stricter enforcement, enabled by default via the kKillOnUnexpectedOriginHeader feature, replaces the previous behavior of logging an error without crashing. (87ecad0)
    SecurityNetworkBrowser

UI & User Experience

  • ⚠️BREAKING CHANGE: The modern, FlexLayout-based infobar UI is now permanently enabled. The kInfobarRefresh feature flag and all associated legacy code have been removed, making this a mandatory visual and structural change for all infobars across the browser. (69e4d33) (7e49209) (92a80ce) (29c1235)
    UIBrowser
  • ⚠️BREAKING CHANGE: The user education framework for WebUI was significantly refactored. HelpBubbleHandler was consolidated into TrackedElementHandler, and several classes were removed. This will affect how help bubbles are anchored and how their visibility is tracked, especially when the parent WebContents is hidden. (c00f0b9)
    UIAPIBrowser
  • A new enterprise policy, IsolatedModeSettings, has been introduced. This policy allows administrators to replace the standard Incognito mode with a new "Isolated mode" profile type, offering a different set of privacy and security guarantees for enterprise environments. (b2955dd)
    EnterprisePolicySecurity

Build System & Testing

  • Major testing infrastructure cleanup resulted in the removal of the extensions_browsertests and app_shell_unittests build targets. Tests from these suites have been migrated into the main browser_tests binary, simplifying the build graph and test execution for embedders. (3df42d8) (28e3d31) (361726b)
    BuildTestingExtensions
  • ⚠️BREAKING CHANGE: A new build dependency on //third_party/llvm-libc:headers is now required by V8. Embedders of V8 or Chromium will need to add this build target to their configurations to avoid compilation failures. (365c552)
    BuildV8
