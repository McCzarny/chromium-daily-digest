Summary for 2025-11-19 on branch 'main'

Chromium Digest: 2025-11-19

Overview

A total of 662 commits were made to the 'main' branch. After filtering, 451 relevant commits were analyzed. The day's changes span from commit (29bb9f0) to (a96996f).

Breaking Changes

  • ⚠️ BREAKING The NavigationHandle::SetRequestHeader and RemoveRequestHeader methods now accept std::string_view instead of const std::string&. This change improves performance by avoiding string allocations but requires downstream projects to update their code to the new method signature. (7d6e063)
  • ⚠️ BREAKING The obsolete BackgroundModeAllowRestart feature flag and related code have been removed, simplifying the background mode logic for extensions. Projects relying on this flag for specific restart behavior will need to adapt. (14a0eb2)

API

  • As the Protected Audience API is being deprecated, the associated k-anonymity service has been disabled by default. Chrome will no longer contact the k-anonymity server for ad status or reporting. (7da13e3)
  • The DevTools Protocol (CDP) has been extended to support Direct Sockets Multicast capabilities. This includes new events for when a socket joins or leaves a multicast group. (34c5740)
  • A new error message has been added to getDisplayMedia for cases where invalid capture constraints are provided. (46501b5)
  • A new utility function, IsEligibleUser, has been introduced in components/wallet/core/browser to centralize eligibility checks for wallet-related features, now including a country-based allowlist. (5722174)
  • The C++ MetricsReporter for WebUI now includes a Mark(name, time) method, providing more flexibility for recording performance metrics from C++. (29bb9f0)

Extensions

  • The extensions.webRequest API can now parse a SecurityInfo object, giving extensions access to detailed security information about a web request, such as the TLS certificate and protocol version. (971ad9c)
  • A significant code refactoring moves extension permission-related files from //chrome/browser/permissions to //extensions/browser/permissions, as they no longer have dependencies on the //chrome module. (bc9fccb)

Security

  • A new "approval-only" mode has been introduced for the remote debugging server, guarded by the kDevToolsApprovalMode feature flag. When active, it requires user confirmation via a dialog for each new debugging connection and displays an infobar while connections are active, enhancing security. (7c12e9d) (4f8dd59) (0b9af34) (3d94716)
  • The Device Bound Session Credentials (DBSC) API has been updated to allow multiple session creation requests to proceed at the same time, which is required for complex sign-in flows. (3792496)
  • Code using older OpenSSL EVP_CIPHER APIs has been migrated to newer, size_t-based, bounds-checked versions for improved memory safety. (e07db00)
  • The PasswordManagerBlocklistPolicy implementation has been moved from //chrome/browser/policy to //components/password_manager/core/browser, consolidating the policy logic with the feature's core code. (08b0ed5)

Performance

  • A new feature, kHttpCacheInitializeBackendEarly, has been added to initialize the HTTP cache backend earlier in the startup process. This aims to reduce I/O latency on the critical path of the first network request. (35767c0)
  • The WinSboxNoDbghelpWarmup feature is now enabled by default, which should improve the performance of sandbox initialization on Windows. (c24330d)
  • The MouseCursorMonitor on Windows is now event-driven, improving the performance of cursor rendering in remote desktop scenarios. (cd4d28a)

UI/UX

  • The "Split View" feature, which allows two tabs to be viewed side-by-side in one window, is now enabled by default on Windows and Mac in preparation for a full launch. (e59bbd8)
  • On macOS, the Picture-in-Picture (PiP) window is now correctly excluded from screen captures initiated via getDisplayMedia. (cc36e60)
  • A new card has been added to the user Profile Menu to display errors related to passkey unlocking. (579599f)
  • Pressure sensitivity is now supported for PDF Ink signatures, allowing for more natural-feeling digital signatures. (61cbeee) (934ed53)

Base & Infrastructure

  • The base::flat_map and base::flat_tree containers have been refactored to improve support for heterogeneous lookups, allowing search and insertion with keys of different types when a transparent comparator is used. (3de1d9d)
  • A new Mojo dialer for the Updater service has been implemented for Windows. (e3fb01b)

Testing

  • A new tool, run_action.py, has been added to build/bench to allow developers to benchmark the build performance of specific files locally. This will help analyze the impact of different build configurations on compilation time. (6faf918)
