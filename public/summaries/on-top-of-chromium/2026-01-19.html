Summary for 2026-01-19 on branch 'main'

Chromium Digest: 2026-01-19

Overview

This summary covers 436 commits on the main branch for January 19, 2026, with 267 identified as relevant to developers working on Chromium-based projects. This day saw several significant breaking changes, including the removal of the legacy TabLoader class, the deletion of several base::HistogramBase helper functions, and a behavioral change to Incognito enterprise policies. The day's activity spans from commit (7b049db) to (a1a578a).

Breaking Changes

  • ⚠️ BREAKING The legacy TabLoader class and its helpers (TabLoaderDelegate, TabLoaderTester, SessionRestorePolicy) have been removed from chrome/browser/sessions. Embedders relying on TabLoader for custom session restore logic must migrate to the new mechanism managed by BackgroundTabLoadingPolicy. (7eca812)
  • ⚠️ BREAKING The public API helper functions AddScaled(), AddKilo(), and AddKiB() have been removed from base/metrics/histogram_base.h. Projects using these must now implement scaling logic manually before calling AddCount(). (9b7ede3)
  • ⚠️ BREAKING A significant behavioral change was introduced for Incognito mode enterprise policies. The IncognitoModeUrlAllowlist policy can now override an IncognitoModeAvailability policy set to 'disabled', forcing it to 'enabled'. This may cause Incognito mode to become available unexpectedly in managed environments. (01a54f3)
  • ⚠️ BREAKING The public header child_process_id.h has been moved from content/public/browser/ to content/public/common/. Code including this header directly must update its include path to avoid build failures. (0f00d98)
  • ⚠️ BREAKING The supervised_user::UrlFormatter class has been removed from components/supervised_user as part of a refactoring, breaking any code that used this public class. (18386c6)
  • ⚠️ BREAKING In the public VR service mojom interface, an enum value was renamed from SUBMIT_AS_MAILBOX_HOLDER to SUBMIT_AS_TEST. This signature change will break clients using this interface. (0a15604)

Browser & Session Management

  • A significant but reverted change began migrating the core BrowserListObserver to a new BrowserCollectionObserver. The change was reverted on the same day due to test failures. (25c87a7) (5013988)
  • The BackgroundTabLoadingPolicy was refactored to use the standard Performance Manager ownership system instead of a custom singleton, aligning it with modern practices. (fd30d10)
  • Usage of TabStripModel in contextual_tasks was replaced with the platform-agnostic TabListInterface to improve code sharing. (3ea620d)

UI & UX

  • The experimental Vertical Tabs feature received several improvements, including layout manager fixes and support for horizontal animations when adding or removing pinned tabs. (db67e0e) (17b5a4e) (8357c88)
  • Zoom controls are now centrally disabled for TopChrome WebUIs (e.g., Omnibox popup) to ensure they always render at 100% zoom. (d78cc4f)
  • On macOS, a bug causing incorrect WebContents focus events in PWAs when using the Find in Page control has been fixed. (99c4920)
  • The FluentOverlayScrollbar feature flag was removed, and its functionality is now merged into the main OverlayScrollbar flag. (3139933)

Security & Identity

  • The kWebAuthnPasskeyUpgrade feature flag was removed, as the feature is now stable and enabled by default. (eb53ce0)
  • WebAuthn now supports the CTAP2.2 hmac-secret-mc extension, improving PRF-on-Create support for certain authenticators. (9ad0302)
  • The WebAuthn immediate authentication dialog will now display the password source if there is a mismatch, improving user clarity. (7950611)
  • The kBiometricAuthenticationBeforeFilling preference was made read-only to prevent modification via developer tools. (040eb92)

Services & Components

  • The internal SQL library was enhanced with new BindTime() and BindTimeDelta() methods on sql::Statement, providing a standardized way to serialize time-based values. (dc42eac)
  • The WebHistoryService was updated to support a new deletion API, controlled by the kWebHistoryUseNewApi feature flag. (dbc294b)
  • New metrics were added to measure the queue size of delayed network calls during the sign-in process, helping diagnose network state discrepancies. (59cd1e9)
  • The password manager's sync bridge was updated to use absl::flat_hash_set for improved performance. (43a67e5)

Testing & Infrastructure

  • A race condition in ChromeDriver that could lead to a crash has been fixed by adding proper locking when a command is processed while its associated webview is being disposed. (583277c)
  • The build system was updated to ensure the target_platform is always passed to grit actions, fixing issues with remote builds. (f720f89)
  • The test fixture for MutableProfileOAuth2TokenServiceDelegate was parameterized to improve test coverage for access token fetching logic. (27bb20b)
