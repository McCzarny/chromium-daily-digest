Week 48 of 2025 (2025-11-23 to 2025-11-29)

Chromium Weekly: 2025 Week 48

Overview

A turbulent week in Chromium development saw 1933 relevant commits out of 2680 total, introducing a vast number of breaking changes across the codebase. The most significant development was the enabling of the Network Service Sandbox by default on Windows, a major security milestone. Multiple core APIs in networking, identity, and UI were modified, requiring significant attention from embedders. Instability was also a theme, with a major refactor of the AccountInfo identity struct being partially reverted after causing crashes.

Security

  • ⚠️ BREAKING BREAKING CHANGE The Network Service Sandbox is now enabled by default on Windows. This is a major behavioral change that enhances security by restricting the network process's access to the system, but it may break embedders who rely on the previous, more permissive environment. (c6bd7f0)
  • The remote debugging server has been hardened with a new "approval mode" controllable via the kDevToolsRemoteDebuggingEnabled preference and the kDevToolsRemoteDebuggingAllowed enterprise policy. This requires user approval for new connections, improving security for developers. (2e576ad) (2748580)
  • ⚠️ BREAKING BREAKING CHANGE The core WebAuthn public API AuthenticatorCommon::MakeCredential has been updated to include a new payment_options parameter, requiring embedders to update their calls. (5d55cd3)
  • An initial implementation of a proxied Unexportable Key Service was added, laying the groundwork for the network process to interact with hardware-backed keys (like the macOS Secure Enclave) via the browser process. (269bac8) (a6cfb36)

Core APIs & Internals

  • ⚠️ BREAKING BREAKING CHANGE The AccountInfo identity struct underwent a volatile refactoring to privatize the hosted_domain field. The changes, which required all consumers to use a new GetHostedDomain() accessor, caused crashes and were partially reverted late in the week. Embedders should monitor this area closely for instability. (da1dfa4) (d85bfc8) (3428bfe) (659bbd1)
  • ⚠️ BREAKING BREAKING CHANGE The callback for the fundamental networking utility network::SimpleURLLoader::DownloadToString now receives a std::optional<std::string> instead of a std::unique_ptr<std::string>, requiring updates to all client code. (a19f517)
  • ⚠️ BREAKING BREAKING CHANGE A new pure virtual method, GetOAuthConsumerFromId, was added to the SigninClient interface. Embedders with custom SigninClient implementations must add this method to avoid build failures. (5657390)
  • ⚠️ BREAKING BREAKING CHANGE Transitive includes for ui/gfx/geometry/rect.h and rect_f.h have been removed from many public headers like content/public/browser/web_contents.h. Code that relied on these headers to implicitly include gfx::Rect or gfx::RectF will fail to build without adding the includes directly. (5ac7f34)
  • ⚠️ BREAKING BREAKING CHANGE The base::MemoryPressureListener API will now dispatch a notification when memory pressure returns to MEMORY_PRESSURE_LEVEL_NONE. Custom listeners must be updated to handle this new notification to avoid unexpected behavior. (d139d7a)
  • ⚠️ BREAKING BREAKING CHANGE The entire components/fingerprinting_protection_filter component has been deleted from the codebase. Projects with a dependency on this component will no longer build. (b6d76c1)

Browser UI & Features

  • The new tabbed browser layout, part of the "Chrome Next" UI refresh and supporting the Toolbar Height Side Panel (THSB), is now enabled by default. (76dd43c) (838d53b)
  • ⚠️ BREAKING BREAKING CHANGE The experimental "Tab Scrolling" feature (scrollable-tabstrip) has been completely removed from the codebase. The tabstrip will now revert to the default behavior of shrinking tabs when the strip is full. (ef29bab)
  • Work continues on the "Tab Group Focusing" feature, with a new OnTabGroupFocusChanged method added to the TabStripModelObserver interface to notify listeners of group focus state changes. (de85415) (c7b09ea)

Extensions & Developer Tools

  • ⚠️ BREAKING BREAKING CHANGE The ability for extensions to delete passwords via chrome.browsingData.removePasswords has been removed. Calls to this function are now a no-op and will not affect the password store. (9fb257e)
  • ⚠️ BREAKING BREAKING CHANGE The Chrome DevTools Protocol (CDP) has moved the StyleSheetId type from the CSS domain to the DOM domain. Automation clients referencing CSS.StyleSheetId must be updated to use DOM.StyleSheetId. (23a0a9f)
  • The remote debugging server can now automatically find an available port if the default port 9222 is in use. The active server address is now displayed on chrome://inspect. (a4e85e3) (8a418d9)
  • The WebRequestPersistFilteredEvents feature is now enabled by default. This may alter the behavior of extensions using the chrome.webRequest API by changing how event listeners are handled across navigations. (49be6c3)

Enterprise

  • A new enterprise policy, SearchContentSharingSettings, was added to allow administrators to control content sharing with AI features, independent of other AI-related settings. (6e7bf35)
  • A new StaticStorageQuotaEnabled enterprise policy has been added, allowing administrators to override the default storage quota estimation behavior. (4ff2cdf)
  • ⚠️ BREAKING BREAKING CHANGE The value savedTabGroups has been removed as a valid option from the SyncTypesListDisabled enterprise policy. Configurations using this value to disable tab group syncing are no longer valid. (1746a94)
