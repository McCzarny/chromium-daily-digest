Chromium Digest: 2026-04-09

Overview

This daily summary for April 9, 2026 covers 528 relevant commits out of a total of 721 on the main branch. A significant number of breaking changes were landed, primarily focused on refactoring core browser APIs, removing legacy functionality, and cleaning up data structures. The first relevant commit was (f3cfa32) and the last was (1b28d58).

Breaking Changes

  • ⚠️ BREAKING The widely used public functions chrome::FindBrowserWithWindow and chrome::FindBrowserWithID have been removed. Embedders must update their code to use the new GlobalBrowserCollection singleton, for example: GlobalBrowserCollection::GetInstance()->FindBrowserWithWindow(). The return type for the window lookup has also changed from Browser* to BrowserWindowInterface*, requiring further refactoring. (2a4a86d) (fb39bae)
  • ⚠️ BREAKING The integer values of the public media::Channels enum in media/base/channel_layout.h have been changed to align with the WAVEFORMATEXTENSIBLE standard. Any code that relies on the specific numeric values of these channels for audio processing will break. (898ad6b)
  • ⚠️ BREAKING A new pure virtual method, NotifySwappedRWHVChildFrameFromRenderManager, has been added to the content::RenderFrameHostManager::Delegate interface. Custom implementations of this delegate must now provide an implementation for this method to avoid build failures. (b83f5c5)
  • ⚠️ BREAKING In the base threading library, Thread::Delegate::AddTaskObserver and the corresponding task_observer option have been removed. It is no longer possible to add a TaskObserver upon thread creation. (e501d62)
  • ⚠️ BREAKING The ocsp_result field has been removed from the public net::SSLInfo struct as part of the deprecation of the Expect-Staple feature. Code inspecting OCSP verification results via this field will no longer compile. (f366471)
  • ⚠️ BREAKING The FontService::ListFamilies() method has been removed from the public font_service.mojom interface. Clients using this IPC to enumerate system fonts will be broken. (bb555c3)
  • ⚠️ BREAKING On Windows, the public methods Emf::InitFromFile() and Emf::InitToFileForTesting() have been removed from the exported printing::Emf class. (2abeb2f)
  • ⚠️ BREAKING The rule_options member has been removed from the content_settings::RuleMetaData struct and its corresponding mojom definition, breaking any code that reads or writes to it. (e8b22be)
  • ⚠️ BREAKING The public operator== has been removed from the passage_embeddings::Embedding and history_embeddings::UrlData data structures, which will break tests or other code relying on direct equality comparison. (1aa8ca6)

API & Core Infrastructure

  • The TabGroup class in components/tabs now has an observer pattern, allowing clients to subscribe directly to group and visual data changes, simplifying UI updates. (9236242) (e74b83d)
  • As part of a large-scale refactoring, many components like ChromeTracingDelegate and SmartRestartMetrics were migrated away from the legacy BrowserList to use the new GlobalBrowserCollection and its observer. (6f905a1) (a9834f7) (59169c4)
  • Several components were updated from the legacy MemoryPressureListener to the new MemoryConsumer API for improved memory management. (5bf4747)
  • On Windows, dependencies on the legacy ATL framework were removed from UI automation, accessibility event waiters, and the updater in favor of modern WRL and base library equivalents. (c434495) (f3cfa32)
  • A new BrowserLaunchEvent proto was added to the enterprise reporting pipeline to capture browser launch information. (c6c9731)

Security & Privacy

  • To prevent a compromised renderer from spoofing audio activity, the browser process now verifies that a renderer is authorized to bypass the browser's audio service before marking a tab as 'audible'. (c1f3553)
  • A same-origin check is now enforced for main script fetches in Dedicated Workers to prevent a compromised renderer from exfiltrating assets from other origins. This is controlled by the kEnforceDedicatedWorkerSameOriginCheck feature flag. (7d3c950)
  • On Windows, a potential local privilege escalation vulnerability in Chrome Remote Desktop was fixed by using user impersonation when writing WebAuthn extension files. (98ca974)
  • For enterprise users, deep scanning for downloads now supports user-initiated cancellation. (d858c06)
  • The experimental ChallengeURL prototype was removed from the WebAuthn API as it was not on a standards track. (24b759e)

UI & Features

  • The WebUI home button now supports drag-and-drop for setting the homepage URL. (5bc04ab)
  • An implementation was added to automatically open tabs received via the Send-Tab-To-Self feature on desktop platforms, controlled by a new feature flag. (a77fda5)
  • On macOS, an issue causing the titlebar to get stuck in fullscreen when using vertical tabs has been resolved. (72c5f64)
  • Accessibility for the password generation popup has been improved for screen reader users by ensuring the popup widget is activated, allowing for proper navigation. (939b475)
  • The Windows updater UI has been polished with a smoother, anti-aliased progress bar and adjusted button colors. (64c765f) (79e3c49)

Testing & Tooling

  • Chrome for Testing (CfT) now supports a runtime configuration JSON file, specified by the --chrome-for-testing-config switch. The initial implementation adds an option to enable a virtual clipboard. (bca0df9)
  • The eventToPromise() test utility in WebUI has been updated to be strongly typed with TypeScript, improving test code quality and maintainability. (adadafb) (49febf0)
  • The Extreme Lightweight UMA Detector (ELUD) now embeds object and allocation sizes into its zap pattern, aiding in the diagnosis of memory corruption bugs from crash reports. (cbcea52)
