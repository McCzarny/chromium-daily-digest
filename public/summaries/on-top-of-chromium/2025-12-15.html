Chromium Digest: 2025-12-15

Overview

This summary covers 387 relevant commits out of 511 total on December 15, 2025, spanning from commit (596506f) to (726070a). The day was marked by an unusually high number of breaking changes impacting UI, security, and graphics APIs, requiring immediate attention from embedders. Other key updates include improvements to the Extensions framework, policy handling, and performance optimizations for WebUI loading.

Browser UI & Tabs

  • ⚠️ BREAKING BREAKING: The public TabDragContext interface was modified with a new virtual method for selecting a drag context. Embedders implementing this interface must add the new method to their derived classes. (b97f8fc) (2e55c6d)
  • ⚠️ BREAKING BREAKING: The class hierarchy of core UI components was altered, making VerticalTabStripRegionView inherit from TabStripRegionView. This public interface change may break embedders who rely on the previous class structure or use static_cast. (7eeb9f0)
  • ⚠️ BREAKING BREAKING: The public member tab (WeakPtr<TabInterface>) has been removed from the TabRendererData struct in chrome/browser/ui/tabs/tab_renderer_data.h. Code that accesses this member to get a reference to the tab will no longer compile. (9b973dd)
  • The tab strip continues to see improvements, especially for vertical tabs. An audio mute toggle has been implemented for vertical tabs, and a bug was fixed in the TabStripModel to ensure notifications are correctly propagated when moving tabs between pinned and unpinned states. (d8956f5) (3e16b51)
  • Several UI elements were refined, including a refreshed tab sharing infobar, an improved closing mechanism for the Download Bubble, and logic to ensure only one Reading Mode presentation is open at a time. (f4ad216) (6371608) (4e648e1)

Extensions

  • ⚠️ BREAKING BREAKING: The public class ExtensionsMenuViewPlatformDelegate has been removed. The extensions menu has been refactored to use an ExtensionsMenuViewModel::Observer pattern instead. Embedders with custom platform delegates must migrate to the new observer interface. (0ff49e3)
  • API correctness has been improved by moving validation earlier in tabs.create() and windows.create() calls to prevent creating UI elements for a failing API call. Additionally, the webRequest API schema was updated to correctly mark documentId as optional. (3d890c1) (6877424) (d6b5a3e)
  • A race condition that could prevent an extension's service worker from registering correctly during a rapid uninstall/reinstall cycle has been fixed. (2f44e11)

Security, Identity & Core APIs

  • ⚠️ BREAKING BREAKING: The deprecated crypto::HMAC class and its methods (Init, Sign, Verify) have been completely removed from crypto/hmac.h. Code must be migrated to the newer crypto::hmac::Sign* and crypto::hmac::Verify* functions. (06d7910)
  • ⚠️ BREAKING BREAKING: Support for "named interceptions" has been removed from the public Windows sandbox API (sandbox/win/src/interception.h). This is a feature removal from a core security interface. (80fad49)
  • ⚠️ BREAKING BREAKING: The UnexportableKeyService public interface was extended with a new GetKeyTag() method. Embedders who implement this service for hardware-backed keys must now provide an implementation for this new method. (be32c0a)
  • ⚠️ BREAKING BREAKING: The public FIDO constant kMaxInputSize was renamed to kMaxPRFInputSize in device/fido/public/fido_constants.h. Any code referencing the old constant name will fail to compile. (9efcd2d)
  • ⚠️ BREAKING BEHAVIORAL BREAKING CHANGE: The UseIssueTokenToFetchAccessTokens feature is now enabled by default. This alters the standard mechanism for fetching access tokens to always use the /IssueToken endpoint, which may break custom authentication flows or network configurations that rely on the previous behavior. (627b6aa)

Graphics (Viz)

  • ⚠️ BREAKING BREAKING: The viz::ReturnCallback type alias signature has been changed. It now expects a std::vector<ReturnedResourceViz> instead of std::vector<ReturnedResource>. Custom compositor clients (viz::HostFrameSinkClient) must update their callback signatures. (40df19a)
  • ⚠️ BREAKING BREAKING: The constructor for viz::ReturnedResource has been modified to require a gpu::SharedImageExportResult object instead of a raw gpu::SyncToken. Code that directly constructs viz::ReturnedResource objects must be updated. (082e1d7)

Services & Performance

  • ⚠️ BREAKING BREAKING: The constructor signature for TranslationDispatcher has been changed to facilitate dependency injection. Embedders directly instantiating this class will need to update their code to provide the new dependencies. (25c2d0a)
  • A new synchronous URL loader has been introduced for initial WebUI navigations (e.g., chrome://settings). This performance improvement will make browser UI pages appear faster by reducing asynchronous steps during startup. (6fc5621)
  • The ServiceWorkerAutoPreload feature has been disabled by default due to a bug that could cause blank pages. This will negatively impact the load performance of some pages that use service workers but improves reliability. (97e0cec)

Policy & Enterprise

  • When a URL is blocked by the IncognitoModeBlocklist enterprise policy, a new, specific error page (BLOCKED_IN_INCOGNITO_BY_ADMINISTRATOR) will now be displayed to the user for clearer feedback. (b1b02b4) (1e57d57)
  • The ExtensionInstallPolicyService was enhanced to fetch policies directly via the CloudPolicyClient, and the domain families.google has been added to the default allowlist for supervised user accounts. (1facebb) (d6f3370)

Testing & Infrastructure

  • On macOS, the toolchain was updated to use Xcode 17C52, and installer test utilities were refactored into a separate library for broader use. (8c7a5f3) (845f6d7)
  • The TestingBrowserProcess test API was improved to simplify test setup and teardown, and unsafe buffer usage in ElementIdentifier was fixed by migrating to std::string_view comparisons. (5b177ac) (dba3a4c)
