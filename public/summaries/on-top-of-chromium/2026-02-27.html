Chromium Digest: 2026-02-27

Overview

This digest covers 715 commits on the main branch from commit (b004ff2) to (a0f393d), with 507 identified as relevant to developers on Chromium-based projects. The day saw an unusual number of breaking changes to core public APIs in base, content, net, and url, requiring code updates for embedders. Notable feature work includes the chrome.serial API removal from desktop platforms, the stabilization of the refreshed Task Manager UI, and continued improvements to Vertical Tabs.

Breaking Changes

  • ⚠️ BREAKING The public API function url::EncodeURIComponent has been renamed to url::EncodeUriComponent. Code directly calling this function must be updated. (6c112d6)
  • ⚠️ BREAKING The method signatures for content::StoragePartition::ClearData and ClearDataForOrigin have changed, removing the quota_storage_remove_mask parameter. Embedders must update calls to these functions. (76d46af)
  • ⚠️ BREAKING The public class net::BackoffEntry is now marked final and its destructor is no longer virtual. Any code that subclasses net::BackoffEntry will fail to compile. (ff96c28)
  • ⚠️ BREAKING The chrome.serial API has been removed from Windows, Mac, and Linux builds as part of the Chrome Apps deprecation. The API is now only available on ChromeOS. (f39e3b6)
  • ⚠️ BREAKING The public API base::trace_event::TraceLog::SetArgumentFilterPredicate() has been removed. Privacy filtering for trace arguments is now handled within the tracing service, and custom filtering via this method is no longer supported. (7287a60)
  • ⚠️ BREAKING The method signature for views::WebView::TakeCrashedOverlayView has been changed to accept a callback for managing the overlay view's lifecycle. Code calling this method must be updated. (ff0a873)
  • ⚠️ BREAKING A static_assert has been added to the UmaHistogramEnumeration metric function, causing a compile-time failure if an enum's kMaxValue exceeds the maximum bucket count. Code using large enums with this function may now fail to build. (b717f55)
  • ⚠️ BREAKING The public static method LinearHistogram::FactoryGetWithRangeDescription() has been removed from base/metrics/histogram.h. Embedders must switch to an alternative factory method. (be046e7)
  • ⚠️ BREAKING The header file extensions/browser/api/device_permissions_prompt.h has been renamed to extensions/browser/api/usb_device_permissions_prompt.h. Includes of the old path will fail. (1c660ca)
  • ⚠️ BREAKING In the network service, the public type aliases network::OriginatingProcess and network::RendererProcess have been removed. Code must be updated to use network::OriginatingProcessId and network::RendererProcessId. (9de6c1e)
  • ⚠️ BREAKING A revert in the WebAudio API has removed the public method WebAudioBus::TryInitialize and related TryAllocate functions. Code that adopted these for non-crashing OOM handling will no longer compile. (3686b27)

APIs & Core Internals

  • A new SetFilter API was added to mojo::Remote<T>, allowing clients to inspect or modify outbound messages before they are sent. This mirrors existing functionality for incoming messages. (12ecb0a)
  • The internal "Legion" project has been renamed to "PrivateAI" across the codebase, and its crypto implementation was updated to use modern, stateless AEAD APIs. (2ceb181) (50e28f2) (f997bcc) (f5ce052)
  • The MemoryPressureListener in base/ now includes a sequence checker to enforce thread safety. (abf4312)

Extensions

  • The chrome.tabs.update() API now supports the highlighted property on desktop platforms, allowing extensions to programmatically add or remove a tab from the current selection. (f04cc6f)
  • The platform-agnostic TabsEventRouter will now handle the logic for tabs.onHighlighted and tabs.onHighlightChanged events, preparing for their implementation on other platforms. (f8bba03)
  • Foundational infrastructure was added for policy-driven filtering of extension activity logs, in preparation for enhanced enterprise telemetry. (4be68ed)

Browser & UI

  • The feature flag for the refreshed Task Manager UI has been removed, making it the default implementation. The old native macOS Task Manager code has been deleted. (5c16bf4) (9961485)
  • Significant work was done on the Vertical Tabs feature, including centralizing state management to resolve conflicts with immersive fullscreen mode and adding the ability to freeze the collapse state of tab groups. (b004ff2) (8f1266c) (7855d27)
  • The AutofillEnableVcn3dsAuthentication feature, which enables 3DS authentication for virtual card numbers, has been turned on by default. (e8f489a)
  • A new mechanism has been added to NavigationController::LoadURLParams to allow for "silent" scroll-to-text navigations, enabling features like cross-device scroll restoration without showing the standard visual highlight. (71f788d)
  • A new ui::ElementHighlighter helper has been introduced to generalize anchor highlights for ui::TrackedElements, which is now used by BubbleDialogDelegate. (c9fa491)

Security & Networking

  • The Device-Bound Sessions feature (kDeviceBoundSessions) is now enabled by default on Windows, removing the need for Origin Trial tokens. (067236d)
  • The fallback logic that attempted to verify SHA-1 signed certificates has been removed. Certificates using SHA-1 will now fail with a generic ERR_CERT_INVALID error. (180eeca)
  • A security fix was implemented to prevent the "thisisunsafe" keyword from bypassing enterprise block pages. (c09ea49)
  • A new public method, ClientSideDetectionHost::ReportUnsafeSite, was added to allow sending phishing reports that include screenshot data. This is paired with a new "Report Unsafe Site" menu item in the Help menu on macOS. (48ed90c) (5409480)
  • A bug fix in HostResolverManager::InvalidateCaches() addresses a potential crash by deferring the destruction of a probe's URLRequest to prevent re-entrant job modification. (dc86220)

Build & Code Health

  • Several modules were componentized to improve dependency management, including //chrome/browser/clipboard and //chrome/browser/collaboration/. (150bd48) (5f76bbc) (23278ae)
  • A bug in the Mojom C++ generator was fixed where forward-declaration headers could be generated without including <stdint.h>, even when types like uint32_t were used. (d4946a6)
  • Numerous calls to Vector::AppendVector() and Vector::AppendSpan() were migrated to the modern C++ append_range() method. (6b0ad3c) (b62e33f)
