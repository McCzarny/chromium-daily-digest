Chromium Digest: 2026-04-27

Overview

This summary covers 571 commits to the Chromium main branch on April 27, 2026, with 427 identified as relevant to developers. The day saw a significant number of breaking changes, particularly in security-related APIs like unexportable keys, extension behaviors, and core graphics and capture interfaces. Key commits range from 398bf4389292a344f5bdbfcc196ee3f723af36bc to 34d0ce4055b12f4b3049705d22eb7cfe5285250f.

Breaking Changes

  • ⚠️ BREAKING The Unexportable Keys API was significantly refactored, changing public method signatures and class hierarchies in the crypto library. This includes renaming DeleteSigningKeysSlowly to DeleteKeysSlowly, introducing UnexportableKey as a base class, and replacing AsStatefulUnexportableSigningKey() with AsStatefulKey(). Consumers of the Device Bound Session Credentials (DBSC) feature must update their code. (e94b66b) (2763d6c) (65a9d85)
  • ⚠️ BREAKING A strict same-origin policy is now enforced for SharedWorkers created by extensions and Isolated Web Apps. This security fix will terminate the renderer if violated and may break extensions that were incorrectly using cross-origin scripts. (0e9234d)
  • ⚠️ BREAKING The terminalPrivate.onProcessOutput() extension API event is now dispatched only to the sender renderer that opened the process, instead of being broadcast to all renderers. Extensions relying on the previous broadcast behavior will break. (4d9c2be)
  • ⚠️ BREAKING The chrome.virtualKeyboard and chrome.virtualKeyboardPrivate extension APIs have been removed from non-ChromeOS platforms (Windows, Mac, Linux). (ac2eccb)
  • ⚠️ BREAKING The viz::CopyOutputBitmapWithMetadata struct, returned by the widely used RenderWidgetHostView::CopyFromSurface method, has a new TrackedElementRects member and a modified constructor. Embedders using this API for surface capture must update their code. (3bf7300)
  • ⚠️ BREAKING The Web Feed (Following) feature has been completely removed from the browser, including the web_feed_subscriptions C++ module and the WEB_FEED_SORT feature flag. (3c759c0)
  • ⚠️ BREAKING Several DNS-over-HTTPS preference keys (e.g., kDnsOverHttpsMode, kDnsOverHttpsTemplates) have been removed from the common chrome/common/pref_names.h file and are no longer available on non-ChromeOS platforms. (be15c40)
  • ⚠️ BREAKING The public function CreateDetailedInstallDialogView has been removed from chrome/browser/ui/web_applications/web_app_dialogs.h. Code calling this function to create PWA installation dialogs will no longer compile. (b2ccbf3)
  • ⚠️ BREAKING The signature for PasswordStoreBackend::RemoveLoginsCreatedBetweenAsync has been changed, removing the sync_completion callback parameter. Embedders with custom PasswordStoreBackend implementations must update their method signature. (a0f57d1)
  • ⚠️ BREAKING The glRequestExtensionCHROMIUM command buffer API has been hardened and now validates requests against a strict allowlist. This may break clients that were requesting extensions not on the new pre-validated list. (7bdc81e)
  • ⚠️ BREAKING On Windows, the remoting::mojom::ChromotingHostServices interface implementation was moved to the daemon process and its BindChromotingHostServices method signature was changed, breaking direct integrations. (e882156)
  • ⚠️ BREAKING The signature for cc::ScrollUpdateEventMetrics::Create and cc::ScrollEventMetrics::Create in the compositor (cc) library has been changed to require a new scroll_begin_arrival_timestamp argument. (9dd4ef9)
  • ⚠️ BREAKING Obsolete variables and code related to non-Fluent scrollbars on Windows have been removed from the public header renderer_preferences.h. Embedders relying on these preferences will be affected. (07c1a24)

Security

  • History-related services like HistoryTabHelper and HistoryClustersService are now correctly disabled in Incognito profiles to prevent crashes and potential data leaks. (a05d583)
  • The UserInterventionForPasswordChange feature, which prompts users before a site-initiated password change, is now enabled by default. (ea0cf30)
  • Input handling in InputRouterImpl and PassthroughTouchEventQueue was hardened against Use-After-Free vulnerabilities that could occur during event processing. (33e1063)
  • Media Mojo services were hardened to handle invalid input from untrusted clients by using mojo::ReportBadMessage() instead of NOTREACHED(), preventing potential host process crashes. (3fc8eee)
  • The UseUnexportableKeyServiceInBrowserProcess feature is now enabled by default on Windows to reduce OAuth multilogin errors. (94c5447)

APIs and Extensions

  • A new asynchronous API, GetScreenshotsForTabs, was added to SessionSyncService to allow retrieval of screenshots for synced tabs from other devices. (af20922)
  • The Web Speech API now integrates the TinyGemma audio model to improve on-device speech recognition quality, controlled by the kOnDeviceWebSpeechSmallExpertModel feature flag. (ccd0ea2)
  • The omnibox will now prefer built-in starter pack keywords (e.g., @gemini) over conflicting extension-defined keywords to prevent crashes and improve user experience. (3f38d35)
  • The observer notification order in ToolbarActionsModel was fixed to ensure OnToolbarActionRemoved is called before OnToolbarPinnedActionsChanged, preventing observers from referencing already-removed actions. (3d269c1)

Browser UI and Features

  • Backend infrastructure was added to support importing and exporting flag configurations from chrome://flags. (b973615)
  • A new context menu item, "Listen to this page," has been added for desktop platforms, enabled by the kImprovedReadAloud feature flag. (021e219)
  • A bug was fixed where pinned toolbar actions, like the download button, would not appear correctly in popup windows. (fa8e625)
  • On macOS, right-clicking on Top-Chrome WebUIs now correctly makes the view the first responder, fixing focus-related issues with menus. (5aae030)
  • A crash that could occur when abruptly closing the Profile Picker window while the Managed User Profile Notice was displayed has been resolved. (6d57626)

Core Internals and Testing

  • ProcessMap and its callers were updated to use the strongly-typed content::ChildProcessId instead of raw integer IDs for improved type safety in process management. (ab62ab0)
  • A new interactive testing verb, MaybeEnterInteractiveMode(), was added to InteractiveViewsTest to allow developers to pause tests for UI inspection. (06548ff)
  • A thread-safety issue in base::FormatDouble was fixed by using base::ThreadLocalOwnedPointer to prevent data races. (4ee829d)
  • The persistent cache backend (SqliteBackendImpl) now provides more detailed error information on initialization failures, allowing callers to better handle database corruption or disk errors. (c426fc2)
  • Platform-independent source files for the Supervised User feature were moved from chrome/browser into a dedicated supervised_user component as part of a larger modularization effort. (6ba2a7b)
