Chromium Digest: 2026-01-14

Overview

This summary covers 436 relevant commits out of 618 total for the main branch on January 14th, 2026. This day saw several significant breaking changes to core UI and crypto APIs, alongside a major behavioral change for extension updates. Key improvements were also made to developer tooling, UI components like the Vertical Tab Strip, and core utilities in base. Breaking Changes

  • ⚠️ BREAKING Behavioral Change: Externally installed extensions (e.g., via enterprise policy) will now be automatically disabled if an update increases their required permissions. Administrators may need to manually re-enable these extensions and adjust deployment workflows. (b846067)
  • ⚠️ BREAKING UI API Removal: As part of a major layout refactoring, the virtual methods BrowserFrameView::GetBoundsForTabStripRegion() and BrowserFrameView::GetBoundsForWebAppFrameToolbar() have been removed. Embedders with custom frame view implementations will fail to compile and must adapt to the new centralized layout logic. (9c5016d) (8231cda)
  • ⚠️ BREAKING Crypto API Removal: Single-key deletion methods have been removed from cryptographic services. StatefulUnexportableKeyProvider::DeleteSigningKeySlowly is removed in favor of DeleteSigningKeysSlowly, and UnexportableKeyService::DeleteKey is removed in favor of the new batch method DeleteKeysSlowlyAsync. Callers must be updated to use the batch-oriented methods. (ed5561b) (a5c30f3) (7ec6152)
  • ⚠️ BREAKING Windows API Signature Change: Functions in base/win/winrt_storage_util.h, including GetPointerToBufferData and CreateIBufferFromData, now use base::span instead of raw pointers and length arguments. All call sites on Windows using these WinRT utilities must be updated. (e4201a2)
  • ⚠️ BREAKING Mojo API Signature Change: The webui_toolbar.mojom.PageHandler.ShowContextMenu method signature has changed. It now requires a ContextMenuType enum and passes coordinates differently, breaking integrations with this WebUI component. (02473f3)

Security & Privacy

  • A new public API, content::webid::IdentityCredentialSource, has been added to support FedCM. This allows embedders to directly query for FedCM sign-in accounts from identity providers, enabling more deeply integrated login UIs. (aa21ef9)
  • The URLBlocklist and URLAllowlist policies now use the standard sensitive policy protection mechanism, making their values visible (but redacted) on the chrome://policy page. (b74b53c)
  • A Time-of-check to time-of-use (TOCTOU) race condition in the Kiosk app external updater was fixed by copying the CRX file before validation. (a466b8c)

APIs & Core Internals

  • A new OnBeforeCutOrCopy hook has been added to views::TextfieldController, allowing embedders to intercept and handle cut and copy events, similar to the existing HandlePaste hook. (1b25d8d)
  • On Windows, base::StackTrace can now proactively probe for dbghelp.dll, enabling symbolization in some sandboxed processes where it previously failed. This improves debugging and crash analysis. (128ded4)
  • The PartitionAlloc memory allocator now supports multiple ThreadCache instances per thread, a foundational change aimed at improving performance and enabling future security enhancements. (0f545a6) (d7bfc3c)
  • The DirectML backend for the WebNN API is being deprecated, and its associated service unit tests have been removed. (b5f5069)
  • As part of Project Bedrock, DevToolsWindow has been migrated away from BrowserListObserver, reducing its dependencies on the Browser object and simplifying its lifecycle. (6fe5e1a)

Browser UI & Features

  • Significant progress was made on the new Vertical Tab Strip (VTS) feature, including animations for moving tabs between groups and ensuring the top of the toolbar acts as a drag handle. (1378fb4) (89c440f) (33c8a32)
  • The Reading Mode side panel will now use DomDistiller to generate and display content directly in its WebUI, and has received keyboard navigation improvements for its settings menus. (de56da7) (5f64c58)
  • On macOS, a bug was fixed where the Services submenu in context menus would have incorrect styling if the browser theme differed from the system theme. (c417103)
  • The avatar toolbar button's "identity pill" now animates smoothly when its text changes or when it expands and shrinks. (5cac7c2)

Extensions

  • The sidePanel.close() method is now exposed in the extensions API documentation. (4a89a29)
  • A bug has been fixed to ensure all disable reasons are preserved when an extension is reloaded, not just DISABLE_RELOAD. (4104c9d)

Networking

  • A potential crash in the SQL disk cache has been fixed. The crash could occur when an optimistic write failed for an entry that was simultaneously doomed by another operation. (5e4ce29)

Build System & Testing

  • The default TypeScript configuration (tsconfig_base.json) now uses moduleResolution: 'NodeNext', a preparatory step for future TypeScript migrations. This may require overrides in custom BUILD.gn files. (09eeb4e)
  • A large number of base::Contains() calls have been migrated to the C++20 standard library equivalent, std::ranges::contains(), for improved code health. (532ff61) (8e76c97) (2ad4bf7)
