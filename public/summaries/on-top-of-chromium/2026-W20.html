Chromium Weekly: 2026 Week 20

Overview

This week in Chromium development, out of 1922 total commits, 1452 were identified as relevant to platform developers. The week was dominated by a large number of breaking API changes aimed at modernizing core interfaces in the UI, networking, and memory management stacks. The tag for Chrome Apps was officially removed, a major milestone in their deprecation. Security was also a key focus, with significant hardening across the WebUSB/HID stacks, Site Isolation, and extension permissions.

Breaking API Changes

  • ⚠️BREAKING CHANGE The chrome.webviewTag API and the underlying <webview> tag have been removed from Chrome Apps on Windows, macOS, and Linux as part of the ongoing platform deprecation. (dfa28c4)
    ExtensionsBrowser
  • ⚠️BREAKING CHANGE Core content layer interfaces were updated with new pure virtual methods. Embedders must now implement GetWebContentsFromNativeWindow in content::ContentBrowserClient and OnChildWebContentsAttached/Detached in content::WebContentsObserver. (3221e4a) (4a7f0a4)
    BrowserAPI
  • ⚠️BREAKING CHANGE The views::Textfield control was modernized to use ui::ColorId. Public methods like set_text_color() and set_selection_text_color() have been removed in favor of their _id suffixed counterparts, requiring updates across UI code. (32b7f60) (f262b03) (f20aaa7)
    UI
  • ⚠️BREAKING CHANGE Core PartitionAlloc memory allocator headers have been moved into an internal/ subdirectory. This breaks embedders who directly include headers like partition_root.h for deep integration with Chromium's memory management. (cd836a8) (7c766eb)
    BaseBuild
  • ⚠️BREAKING CHANGE Numerous core interfaces received signature changes, requiring updates for custom implementations. Notable changes include net::ConnectJobFactory::NewConnectJob, URLLoaderThrottle::BeforeWillRedirectRequest, AutofillManager::OnFormsSeen, and the renaming of cc::MutatorHostClient to cc::MutatorHostDelegate. (6f90916) (8c41713) (97de7f3) (0394325)
    NetworkAutofillCompositor
  • ⚠️BREAKING CHANGE A recently added feature allowing extensions to modify the User-Agent request header via webRequest and declarativeNetRequest has been reverted and removed. (079b3a3)
    ExtensionsNetworkAPI

Security and Privacy

  • Service Workers can now intercept download requests initiated from browser UI actions like "Save As". This aligns behavior with web standards but is a significant change for embedders who relied on UI-initiated downloads bypassing Service Workers. (a2fbe85)
    ServiceWorkerDownloadsSecurity
  • Device interaction APIs were hardened. Fixes include mitigating a Use-After-Free in WebUSB on macOS, resolving a TOCTOU vulnerability in the USB stack by adding a lock, and adding bounds checks to the HID driver on Windows to prevent buffer over-reads. (a74fd42) (7e97805) (0bac4a7)
    USBSecurityWindows
  • Site Isolation was strengthened against bypass vulnerabilities. A compromised renderer is now prevented from redirecting input to cross-origin frames, and MHTML subframes are forced to commit with opaque origins. (1bdeadd) (e4b92a6)
    SecurityUINavigation
  • The network sandbox on Windows now brokers connections to all non-publicly-routable IP addresses. This routes local traffic through the browser process, fixing ERR_NETWORK_ACCESS_DENIED errors for users on certain VPN or virtualized network setups. (3129fd2)
    SecurityNetworkWindows
  • A bug in storage.onChanged that incorrectly leaked events from sources like Chrome Sync to untrusted extension content scripts has been fixed. Events now correctly respect the storage area's access level. (19a5003)
    ExtensionsStorageSecurity

Extensions

  • The alarms.create API was enhanced with a new optional persistAcrossSessions parameter, which defaults to true to maintain existing behavior while allowing for non-persistent alarms. (900c2b0)
    ExtensionsAPI
  • The stream pipeline for MIME handlers now caches the response body, allowing extensions to use chrome.mimeHandler.abortAndFallbackToNativeHandler without forcing a re-download of the content. (f4ddf8e)
    ExtensionsDownloadsNetwork
  • The long-deprecated and unused app_shell binary and its associated source files have been deleted from the codebase, cleaning up the extensions/ directory. (75cd2c6)
    ExtensionsBuild

Browser, UI, and Performance

  • ⚠️BREAKING CHANGE The navigator.clipboard.read() and readText() APIs now use asynchronous Mojo IPC. This is a behavioral change that prevents the renderer main thread from freezing while waiting for the OS to provide clipboard data. (a274f36)
    PerformanceAPIUI
  • A new feature allows users to drag a URL or file onto the toolbar area (excluding the omnibox) to open it in the current tab, improving user workflow. (2ce75bc)
    UIBrowser
  • Startup performance is improved by sending a not-yet-connected GPU channel pipe to the renderer during initialization. This allows the renderer to establish its GPU channel more quickly without a full roundtrip to the browser. (d98234c)
    NetworkGPUPerformance
  • The monolithic //chrome/browser:core build target has been modularized into smaller, more focused source_set targets. This architectural change improves build granularity and parallelism for developers. (a235e83)
    BuildInfraBrowser
