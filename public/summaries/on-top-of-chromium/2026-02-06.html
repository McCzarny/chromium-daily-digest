Chromium Digest: 2026-02-06

Overview

This digest summarizes 689 commits to the Chromium main branch on February 6th, 2026, from which 539 were identified as relevant to developers of Chromium-based projects. The changes span from the first commit (5f62b6d) to the last (265741b). This summary highlights multiple breaking changes to core browser interfaces, networking APIs, and metrics services, requiring developer action.

API & Core Interfaces

  • ⚠️ BREAKING BREAKING CHANGE: The public LocationBar interface has been updated with several new pure virtual methods, including GetBrowser() and IsDrawn(). Any custom implementation of this core UI interface must be updated to avoid build failures. (7665950)
  • ⚠️ BREAKING BREAKING CHANGE: The public AccountInfo struct no longer allows direct member access for full_name and given_name. Code must be updated to use the new accessor methods GetFullName() and GetGivenName(). (623d7f8)
  • ⚠️ BREAKING BREAKING CHANGE: Several public virtual methods have been removed from WindowController, including HasEditableTabStrip(), SupportsTabs(), and IsDeleteScheduled(). Callers must migrate to using helpers on TabListInterface or ExtensionTabUtil. (f706a1b) (8cc423f) (46847f4)
  • ⚠️ BREAKING BREAKING CHANGE: The skills::SkillsService public API has been modified. The AddOrUpdateSkillFromSync pure virtual function now requires a new description parameter, and the IsInitialized() method has been replaced with an enum-based GetServiceStatus() method. (b34d141) (c3d3652)
  • The MenuModel API now supports adding a "Preview" badge to menu items, in addition to the existing "New" badge, allowing for more descriptive UI hints. (e2dcab8)
  • A new virtual method, ConfirmInfoBarDelegate::GetButtonStyle(), was added to allow for custom button styling in ConfirmInfoBar, providing more UI control for embedders. (e0195fc)

Browser UI & Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The public utility function ExtensionTabUtil::GetEditableTabStripModel() has been removed. Embedders must update their code to get the TabStripModel directly from the Browser object. (854164e)
  • The window.setResizable(bool) API is now supported for standalone windowed PWAs that use tabs. Previously, this was restricted to tab-less PWAs. (c19b719)
  • A new OnAllTabsAreClosing method was added to the TabListInterfaceObserver to notify clients when a mass tab closure event begins, allowing for more timely reactions. (3649dd0)
  • The video Picture-in-Picture window now supports seeking forward and backward using the left and right arrow keys. (eba95f7)
  • On Windows, OS shortcut management for PWAs has been improved to filter by app_id, fixing bugs where shortcuts for a new app could be incorrectly deleted during app migrations. (aa4aa5e)

Networking & Security

  • ⚠️ BREAKING BREAKING CHANGE: All instances of "Private Network Access" (PNA) have been renamed to "Local Network Access" (LNA). This affects numerous public mojom interfaces (e.g., network::mojom::DevToolsObserver), classes, and constants, requiring significant updates for any code interacting with these network service APIs. (d28a1ec)
  • ⚠️ BREAKING BREAKING CHANGE: On macOS, a major refactor of keychain access within os_crypt has removed the public OSCrypt::SetKeychainForTesting method. Test setups must be migrated to use the OSCryptMocker utility instead. (158a2d2)
  • The GetToken endpoint will now include the Sec-CH-UA-Platform header, allowing the authentication server to make policy decisions based on the client's operating system. (2ed91e2)
  • Isolated Web Apps (IWA) key management now allows for "soft" key rotations using a previous_key field, preventing the app from appearing broken during an update cycle. (8d36a5d) (2b72489)

Autofill & Payments

  • ⚠️ BREAKING BREAKING CHANGE: The AutofillManager::Observer::OnFieldTypesDetermined method signature has changed. Implementations must add the new bool small_forms_were_parsed parameter to their method definition. (524d57f)
  • ⚠️ BREAKING BREAKING CHANGE: The public constant chrome::kYourSavedInfoSubPage has been removed and the function chrome::ShowYourSavedInfo has been renamed to chrome::ShowAutofill. Code navigating to this settings page must be updated. (0629950)
  • Internal Wallet/Payments APIs for UpsertPublicPass and UpsertPrivatePass have been migrated from JSON to Protobuf for network requests to improve efficiency and type safety. (f37b98b) (3b7648b) (883ab96)

Performance & Metrics

  • ⚠️ BREAKING BREAKING CHANGE: The Structured Metrics service has undergone a major threading refactor. StructuredMetricsRecorder is no longer ref-counted (now uses std::unique_ptr), and the synchronous EventStorage::TakeEvents method is now asynchronous with a callback. Embedders using this service must update their implementation. (8f66e3b)
  • ⚠️ BREAKING BREAKING CHANGE: The page_load_metrics.mojom interface has been altered by a large refactoring of Interaction to Next Paint (INP) metrics. This changes how EventTiming data is passed and will break consumers of this performance metrics API. (f5cd024)
  • The first set of features for the "PerfCombined2025" initiative has been enabled for Windows and Mac, including kPageAllocatorRetryOnCommitFailure and kAbslFlatMapInVariantMap, aimed at improving memory allocation and container efficiency. (17a08b0)
  • The thread pool's internal task sorting key now uses ThreadType instead of TaskPriority to represent importance. This is a foundational change for future task scheduling improvements. (2ece136) (3a96ae1)

Build & Code Health

  • A new command has been added to the DevTools UI bindings, allowing DevTools to trigger a browser restart. This is intended to support certain experiments that require a restart to take effect. (41808e6)
  • As part of an ongoing code health effort, several internal uses of std::unordered_map in components like PrefChangeRegistrar have been replaced with the more performant absl::flat_hash_map. (ea8759d) (355ee25) (138ca23)
