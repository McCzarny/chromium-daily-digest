This summary for the main branch on March 20, 2026, covers 431 relevant commits out of 583 total ((0939042)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((0939042)), (f08dfd5)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((f08dfd5))). The day was marked by numerous significant breaking changes affecting extensions, media handling, payments, and native event observation on Windows. A major security policy change was also landed, altering how Certificate Transparency is enforced.

Breaking Changes

  • ⚠️ BREAKING A major behavioral change to Certificate Transparency (CT) policy now causes Chromium to ignore Signed Certificate Timestamps (SCTs) delivered via OCSP responses by default. Websites relying solely on this mechanism for CT compliance may now trigger certificate trust errors. This is controlled by the CertificateTransparencyIgnoreOcspScts feature, which is now on by default. (4d1a2f7)
  • ⚠️ BREAKING Nine methods have been removed from the chrome.passwordsPrivate extension API, including deleteAllPasswordManagerData, showAddShortcutDialog, and isAccountStorageActive. Extensions must migrate to the new password_manager.mojom interface. (17f18ce)
  • ⚠️ BREAKING On Windows, the native event observer interface in base/message_loop/message_pump_win.h has been refactored. The Will/DidDispatchMSG methods are renamed to Will/DidRunNativeEvent and their signature now takes a uintptr_t identifier instead of a const MSG&. (fe915f0) (6be9b03)
  • ⚠️ BREAKING The release callback for media::VideoFrame has a new signature. It now receives an additional boolean lost_resource to indicate if the underlying shared image was lost, requiring updates for all custom VideoFrame producers. (e614ea8)
  • ⚠️ BREAKING IbanManager has been refactored from a profile-keyed service to a per-tab object, and IbanManagerFactory has been removed. Embedders must now retrieve the manager via PaymentsAutofillClient. (9ef7381)
  • ⚠️ BREAKING The kSynchronizedScrolling feature and ScrollHandlerState enum have been removed from the compositor. This changes the public API of the scheduler by altering the signature of Scheduler::SetTreePrioritiesAndScrollState in cc/scheduler/scheduler.h. (65b5613)
  • ⚠️ BREAKING The DeletePermission method overload that only takes a const url::Origin& has been removed from the public ActorLoginPermissionService interface. Callers must now use the overload that also requires a display_name. (8f184f3)
  • ⚠️ BREAKING The EvalJsResult::IsError() helper method has been removed from the public testing API in content/public/test/browser_test_utils.h. Tests should be updated to use EXPECT_FALSE(ExecJs(...)) or an equivalent matcher. (e812124)
  • ⚠️ BREAKING The recently added google_apis::GetCdmCertificate() function has been reverted and removed. Any code that adopted this new API will be broken. (2172bc8)

APIs & Extensions

  • A new method, CreatedBySessionRestore(), has been added to the public BrowserWindowInterface, allowing components to determine if a browser window was created as part of a session restore operation. (f200ba1)
  • The experimental TabStrip API was updated to ensure tabs closed via the API are correctly recorded in the browser's history as user-initiated actions. (c4338c9)

Security & Networking

  • DNS-over-HTTPS (DoH) requests will now use an empty IsolationInfo, which allows Alt-Svc information to be persisted and enables faster adoption of QUIC on subsequent connections. (6ca275c)
  • The SecurityPrincipal::IsWebUI() method now correctly identifies chrome-search:// as a WebUI scheme, aligning its behavior with site isolation checks and potentially affecting embedders relying on the previous definition. (2ba6492)
  • The networking stack now supports creating empty IsolationInfo and related key objects with a non-default partition. This allows non-web requests to be partitioned from regular requests without needing to be transient. (45a3637)

Browser UI & Features

  • The download bubble has been enhanced to display a "Checking for safety..." scanning animation while waiting for Safe Browsing content checks to complete, improving user feedback. (2cd8d6e)
  • A bug was fixed to ensure that infobars are correctly shown when a browser session is restored asynchronously. (dd30ff1)
