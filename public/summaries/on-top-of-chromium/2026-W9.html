Chromium Weekly: 2026 Week 9

Overview

This week was marked by an unusually high number of breaking changes across core libraries, UI, and networking APIs, requiring significant attention from embedders. Major feature developments include the removal of the chrome.serial API from desktop platforms, the launch of the refreshed Task Manager UI as the default, and the enabling of Device-Bound Sessions on Windows. This summary covers 38 relevant commits out of 2182 total across 4 days.

Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE: Core libraries saw significant disruption. base::VariantMap has been completely removed in favor of absl::flat_hash_map. In networking, net::BackoffEntry is now marked final and its destructor is no longer virtual. The URL utility url::EncodeURIComponent has been renamed to url::EncodeUriComponent. On Windows, the deprecated base/win/wincrypt_shim.h header has been removed, requiring direct inclusion of system headers. (e3feeef) (ff96c28) (6c112d6) (44a2896)
  • ⚠️ BREAKING BREAKING CHANGE: The TabListInterfaceObserver API was refactored, adding a new OnHighlightedTabsChanged method and changing the signatures of all existing methods to require a TabListInterface& parameter. Additionally, the content::StoragePartition::ClearData method signature was changed, and legacy UI views like PwaInstallView and OfferNotificationIconView were removed. (6a0e2c0) (aa6b880) (8cdbdb6) (76d46af) (7f0d235) (c83d0bf)
  • ⚠️ BREAKING BREAKING CHANGE: The remoting protocol was refactored, breaking implementations of SignalStrategy and consumers of SignalingMessage. In the network service, the network::mojom::NetworkContextClient::OnFileUploadRequested Mojo interface method signature was changed to use a strongly-typed ChildProcessId. (8e18de6) (ada63d3) (837710a)
  • ⚠️ BREAKING BREAKING CHANGE: APIs for metrics and tracing were modified. Several factory methods in base/metrics/histogram.h were removed or had their signatures changed, and a static_assert was added to UmaHistogramEnumeration that may cause build failures for large enums. For tracing, base::trace_event::TraceLog::SetArgumentFilterPredicate() has been removed, as privacy filtering is now handled internally. (0bc602b) (be046e7) (b717f55) (7287a60)

Extensions and APIs

  • ⚠️ BREAKING BREAKING CHANGE: As part of the Chrome Apps deprecation, the chrome.serial API has been removed from desktop platforms (Windows, Mac, Linux) and is now only available on ChromeOS. The Web Printing API has also been removed for Chrome Platform Apps. (f39e3b6) (7c68e46)
  • The chrome.tabs.update() API now supports the highlighted property on desktop, allowing extensions to programmatically add or remove a tab from the current selection. A new privacy API, autofill.other_datatypes_enabled, was also added to control non-payment/address autofill. (f04cc6f) (4dba39f) (0de8cdc) (a64e731)
  • The DevTools Protocol was enhanced with a new Extensions.listUnpackedExtensions command and an Emulation.setPrimaryScreen method for headless automation. (bc7e885) (2f1253d)

Security

  • The Device-Bound Sessions feature (kDeviceBoundSessions) is now enabled by default on Windows, binding authentication sessions to the device to mitigate cookie theft. (067236d)
  • Security was hardened in several areas: the allow-popups sandbox attribute is now enforced in the browser process, fallback logic for verifying SHA-1 signed certificates has been removed, and the "thisisunsafe" keyword can no longer bypass enterprise block pages. (a12e651) (180eeca) (c09ea49)

Browser UI

  • The refreshed Task Manager UI is now enabled by default across platforms, and the old native macOS Task Manager implementation has been removed. (5c16bf4) (9961485)
  • The Vertical Tabs feature received updates to centralize state management, resolving conflicts with fullscreen mode and tab group collapsing. In other UI changes, support for interactive doodles on the New Tab Page has been removed. (b004ff2) (8f1266c) (45ede36)

Core Internals and Testing

  • A new SetFilter API was added to mojo::Remote<T>, allowing clients to inspect or modify outbound messages before they are sent, mirroring existing functionality for incoming messages. (12ecb0a)
  • The testing infrastructure was improved with a new TestingBrowserProcessDeathTestMixin for browser death tests and a more ergonomic API for the Windows sandbox test runner. (99798d1) (f6e6490) (7b688cd)
