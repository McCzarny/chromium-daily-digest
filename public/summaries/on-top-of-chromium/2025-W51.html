Chromium Weekly Digest

Week 51 of 2025 (2025-12-14 to 2025-12-20)

Chromium Weekly: 2025 Week 51

Overview

This week was defined by an exceptionally high volume of breaking changes across the Chromium codebase, impacting UI, security, extensions, and core APIs. A major theme was the continued modernization of APIs, including the replacement of legacy systems like BrowserList and the migration of extension manifests to WebIDL. Across 6 days, 387 relevant commits were made out of 511 total, with significant refactoring in tab management, core utilities, and the graphics stack.

Browser UI & Tabs

  • ⚠️ BREAKING BREAKING CHANGE: A series of breaking changes refactored the public tab management APIs. The TabDragContext, TabDragDelegate, and TabStripRegionView interfaces were modified, the TabRendererData.tab member was removed, and TabCollection::MoveTab() was deleted. Embedders must update custom tab strip implementations. (b97f8fc) (7eeb9f0) (9b973dd) (c47f0fe) (a595e9f)
  • ⚠️ BREAKING BREAKING CHANGE: As part of the 'Project Bedrock' modernization effort, several public static methods for querying incognito state were removed from BrowserList in favor of functions in browser_finder.h. The unused BrowserList::size() method was also removed. (bde2c0b) (d4b2a8e) (64c605a)
  • ⚠️ BREAKING BREAKING CHANGE: The ui/views framework now enforces child list immutability during iteration in production builds via a CHECK. This will cause a hard crash if a View's children are mutated while being iterated over, exposing latent bugs in UI code. (4b55b0b)
  • The experimental 'Tab Organization' feature and its associated APIs in TabOrganizationService have been removed from the codebase. (e1c53e2)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE: Several extension delegate interfaces were broken. ExtensionsMenuViewPlatformDelegate was removed in favor of an observer pattern, ExtensionInstalledWaiter was replaced by ExtensionInstallWatcher, and the const qualifier was removed from several virtual methods in ExtensionFunctionDispatcher::Delegate. (0ff49e3) (d3615f2) (d6c7658)
  • The effort to modernize extension APIs continued, with oauth2, fileHandlers, crossOriginIsolation, chrome_url_overrides, web_accessible_resources, sharedModule, and protocolHandlers being converted from the legacy JSON schema to WebIDL. (3a20416) (53881ac) (df23b7a) (6d9ee98) (36bc943) (bc0d14a) (390c9b5)
  • API correctness was improved by moving validation earlier in tabs.create() and windows.create() calls, and a race condition that could prevent an extension's service worker from registering on rapid reinstall has been fixed. (3d890c1) (2f44e11)

Security & Privacy

  • ⚠️ BREAKING BREAKING CHANGE: Core crypto and authentication APIs were changed. The deprecated crypto::HMAC class was removed, the public UnexportableKeyService interface was extended with a new GetKeyTag() method, and the UseIssueTokenToFetchAccessTokens feature was enabled by default, altering auth flows. (06d7910) (be32c0a) (627b6aa)
  • ⚠️ BREAKING BREAKING CHANGE: The public safe_browsing::BinaryUploadService API was refactored. The Request class and the GetForProfile factory method were removed, requiring migration to the enterprise_connectors namespace. (afc8753) (5ee7771)
  • ⚠️ BREAKING BREAKING CHANGE: Support for 'named interceptions' was removed from the public Windows sandbox API (sandbox/win/src/interception.h), a feature removal from a core security interface. (80fad49)
  • ⚠️ BREAKING BREAKING CHANGE: The AreAllThirdPartyCookiesBlocked() method and its observer have been removed from the TrackingProtectionSettings API, and the GetTrackingProtectionSettings function was removed from the permissions::PermissionsClient interface. (3100d0f) (25d5686)
  • The enterprise policies IncognitoModeBlocklist and IncognitoModeAllowlist have been renamed to IncognitoModeUrlBlocklist and IncognitoModeUrlAllowlist respectively, changing the underlying preference names. (c78a55b) (43e427d) (3539075)

Core APIs & Performance

  • ⚠️ BREAKING BREAKING CHANGE: The core tracing API was refactored, removing the TraceLog::EnabledStateObserver and TraceLog::AsyncEnabledStateObserver interfaces. Embedders must migrate to the new per-session TraceSessionObserver. (8e0db64)
  • ⚠️ BREAKING BREAKING CHANGE: The base::MemoryPressureListenerRegistry is no longer a static singleton and now requires explicit instantiation. The base::LRUCache constructor was also changed to use std::optional for its max size. (83f4085) (119215c)
  • ⚠️ BREAKING BREAKING CHANGE: The Chrome DevTools Protocol (CDP) command Network.enable is now synchronous and its signature has changed. Automation clients must be updated to handle the new behavior. (407599e)
  • ⚠️ BREAKING BREAKING CHANGE: The kReduceUserAgentPlatformOsCpu command-line switch was removed. The reduced User-Agent string for platform and OS CPU is now the default and only behavior. (28d0394)
  • A new synchronous URL loader was introduced for initial WebUI navigations (e.g., chrome://settings) to improve the perceived performance of browser UI pages. (6fc5621)

Graphics & Media

  • ⚠️ BREAKING BREAKING CHANGE: The public Viz compositing API was modified. The viz::ReturnCallback signature now expects std::vector<ReturnedResourceViz>, and the viz::ReturnedResource struct was updated, breaking custom compositor clients. (40df19a) (082e1d7) (f04f79f)
  • ⚠️ BREAKING BREAKING CHANGE: The entire //ui/gfx/ipc/buffer_types directory has been removed, a major change for any code that relied on the IPC parameter traits for serializing graphics buffer types. (4292293) (eb22d98)
  • ⚠️ BREAKING BREAKING CHANGE: The public AudioBus::channel_span() method has been removed from media/base. Embedders must update to use AudioBus::channel() instead. (465fb43)

Build System & Testing

  • ⚠️ BREAKING BREAKING CHANGE: Build support for ARMv6 and ARM configurations with NEON disabled (arm_use_neon=false) has been completely removed. (e6b1ab2)
  • ⚠️ BREAKING BREAKING CHANGE: On Windows, the IProcessLauncher::LaunchCmdLine method in the Updater's public COM interface has been reverted to return E_NOTIMPL, breaking clients that rely on it to launch processes. (6684d0a)
  • Legacy testing setup methods BrowserProcess::CreateGlobalFeaturesForTesting and GlobalFeatures::Shutdown have been removed. Tests must now use the TestingBrowserProcess equivalents. (ced3c21)
