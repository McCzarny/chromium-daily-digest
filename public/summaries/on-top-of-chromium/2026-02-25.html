Chromium Digest: 2026-02-25

Overview

This daily summary for the main branch on February 25, 2026, covers 719 commits, with 521 identified as relevant to developers working on Chromium-based projects. This update features a significant number of breaking changes to core base, ui, and remoting libraries that will require code updates for embedders. Other key changes include new extension APIs for Autofill, enhancements to the DevTools Protocol, and important fixes to networking and UI components. The first relevant commit is (a4822bc) and the last is (8692434).

Breaking Changes

  • ⚠️ BREAKING The deprecated base/win/wincrypt_shim.h header has been removed. Projects on Windows must now include <windows.h> and <wincrypt.h> directly, which may require include order adjustments. (44a2896)
  • ⚠️ BREAKING The remoting protocol library has removed the virtual method SignalStrategy::SendStanza and changed the signature of IPseudoTcpNotify::TcpWritePacket to use base::span. Embedders who implement these interfaces must update their code. (8e18de6) (6338506)
  • ⚠️ BREAKING The constructor for base::MemoryConsumerRegistration and related registry methods now take an std::optional<MemoryConsumerTraits> instead of a direct object, requiring updates for all direct instantiations. (fad4543)
  • ⚠️ BREAKING In the core UI library, the public method ui::ColorVariant::IsSemantic() has been renamed to IsLogical(). Embedders using this method for custom UI components must update their code. (42552d9)
  • ⚠️ BREAKING The public function base::Histogram::InspectConstructionArguments in base/metrics/histogram.h now returns a ConstructionArgumentsValidity enum instead of void, breaking direct callers. (0bc602b)
  • ⚠️ BREAKING The Web Printing API is no longer available to Chrome Platform Apps. This feature removal will break any remaining apps that relied on this functionality. (7c68e46)
  • ⚠️ BREAKING The legacy OfferNotificationIconView and its feature flag kOfferNotificationPageActionIcon have been removed. Embedders explicitly using this UI component will be impacted. (c83d0bf)

APIs and Extensions

  • A new extension API, autofill.other_datatypes_enabled, has been added to the privacy settings. This allows extensions to control whether non-address and non-payment autofill data types are enabled. (4dba39f) (0de8cdc) (a64e731)
  • The DevTools Protocol has been enhanced. A new Extensions.listUnpackedExtensions command allows querying local unpacked extensions, and Emulation.setDeviceMetricsOverride now accepts a scrollbarType parameter. (bc7e885) (4471a6c)
  • A new method, GetAllTabsForProfile, has been added to the TabStripExperimentService API, allowing clients like Tab Search to fetch the tab hierarchy for all browser windows in a profile. (14274bc)
  • A bug in the extensions Web Request API has been fixed where event listeners for <webview> tags were incorrectly removed when a WebUI was moved between a tab and a side panel. (e91a60f)
  • In the experimental AI Language Model API, the onquotaoverflow event has been renamed to oncontextoverflow to more accurately reflect that it relates to the context window size. (6b82beb)

Browser and UI

  • On Windows, a new color ID kColorInfoBarButtonIconHovered has been introduced to fix a bug where the infobar close button icon would become invisible on hover when using custom dark themes. (51558c8)
  • Several Lens-related enterprise policies (e.g., LensCameraAssistedSearchEnabled) are now officially deprecated, with their functionality consolidated under the SearchContentSharing policy. (c711e48)
  • For enterprise reporting, a SaasUsageReportScheduler has been added to the ChromeBrowserCloudManagementController on desktop platforms to handle SaaS usage reporting. (b0ab60f)
  • The "flyover" animation for the Chrome Next UI is now enabled by default, changing the visual feedback for some UI interactions. (f413a5a)

Core Internals and Base Library

  • A new convenient overload has been added to DecodeUrlEscapeSequences in url/url_util.h that directly returns a decoded UTF-8 std::string. (f3aa498)
  • The session restore mechanism has been simplified by removing support for legacy session file paths used prior to Chrome version 85 (released in 2020). (c6f736a)
  • A critical crash in the sequence_manager has been fixed. The crash occurred when a task queue was unregistered while one of its tasks attempted to cancel another. (bc55c46)
  • The BrowserList class was refactored to remove its dependency on BrowserWindowInterface, an internal change preceding the eventual removal of BrowserList as part of the "bedrock" UI project. (df33d7f)

Networking and Security

  • On Windows, a WebRTC bug was fixed where ECN (Explicit Congestion Notification) options were not being set correctly on UDP sockets, which impacted L4S congestion control features. (f4d26f2)
  • The certificate verification process has been updated to support validity constraints based on a certificate's notBefore timestamp, allowing for more flexible, two-stage enforcement strategies. (15ed35d)
  • The connection allowlist is now enforced for link rel=preconnect to enhance security by preventing preconnections to origins not on the list. (1f13c15)

Testing and Infrastructure

  • A new TestingBrowserProcessDeathTestMixin has been introduced to properly initialize dependencies for death tests that rely on TestingBrowserProcess. (99798d1)
  • A MockActivationController class has been relanded for Windows and macOS to better emulate widget activation in tests, improving test reliability for UI components. (8a2462b)
  • The build system has been simplified by removing the disable_unknown_warning_option flag. Downstream projects are now expected to patch warnings locally. (9ba319a)
