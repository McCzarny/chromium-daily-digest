Notable updates also landed in security, networking, and browser UI, requiring careful review by developers building on Chromium.

API & Core Utilities

  • ⚠️ BREAKING BREAKING: The public ContentBrowserClient interface now requires embedders to implement the new pure virtual method IsFullscreenAllowedForUnfocusedWebContents to handle fullscreen requests from unfocused WebContents. (395a125)
  • ⚠️ BREAKING BREAKING: The core base::Thread utility was refactored, changing the implementation and potentially the thread-safety contract of public methods like IsRunning() and Stop(). The public method base::CurrentThread::SetTaskRunner was also removed entirely. (440b72b) (031d58e)
  • ⚠️ BREAKING BREAKING: The public content::WebUIController API has a new virtual method, GetDisplayDisposition, which subclasses must now implement to declare whether the UI is a regular page or an embedded element. (a24f112)
  • ⚠️ BREAKING BREAKING: The content::BluetoothAllowedDevices class is now a move-only type, which will break any embedder code that was previously copying instances of this class. (e56dd16)
  • ⚠️ BREAKING BREAKING: The pdf.mojom.PdfHost interface was changed, removing SaveUrlAs and replacing it with a new parameter-less SavePdf() method. This is a public Mojo API signature change. (97d9dd9)
  • ⚠️ BREAKING BREAKING: The PermissionService::AddPermissionObserver mojom API was updated to accept a PermissionStatusWithDetails struct instead of a PermissionStatus enum, allowing for richer permission state information. (3c88b7a) (eee991e)
  • ⚠️ BREAKING BREAKING: The protected virtual method for providing a locale in AimEligibilityService was changed from GetLocale() to GetLocaleImpl(). Subclasses must be updated. (1b9a035)
  • ⚠️ BREAKING BREAKING: The ui/ozone/public/ozone_platform.h public interface no longer contains the supports_color_picker_dialog property in OzonePlatform::PlatformProperties. (23eded4)
  • A new re-entrant iteration method, GetReentrantRange(), was added to base::ObserverList to support safe iteration patterns where observer methods might modify the list. (d5a7459)
  • The ui::Layer class now supports backdrop inversion via a new background_inverted property, allowing for more advanced visual effects. (5fd90ca)

Security & Identity

  • ⚠️ BREAKING BREAKING: Core sign-in API signatures have changed. Methods like AccountsMutator::AddOrUpdateAccount and GaiaAuthFetcher::StartAuthCodeForOAuth2TokenExchange now use a new signin::TokenBindingInfo struct or have new parameters, requiring caller updates. (72fba35) (12d9d6c)
  • ⚠️ BREAKING BREAKING: The DidChangeThemeColor and DidChangeBackgroundColor IPCs have been moved from the blink::mojom::LocalFrameHost interface to LocalMainFrameHost, preventing subframes from sending these messages and hardening against compromised renderers. (c4e43fe)
  • ⚠️ BREAKING BREAKING: In the Password Manager component, the PasswordChangeDelegate interface no longer has the OnPasswordFormSubmission method, and an enum value in ManagePasswordsReferrer was renamed from kChromeSettingsHomeOfTransactions to kChromeSettingsAutofillAndPasswords. (0720a68) (2199531)
  • Cookie parsing has been hardened to reject cookies with an empty name and an ambiguous value (e.g., =Foo=Bar) to mitigate potential cookie-shadowing attacks. This is controlled by the DisallowEmptyCookieName feature. (6e5ceda)
  • A security vulnerability in the Extensions API was fixed to correctly enforce webRequestBlocking permissions for non-webview requests, preventing a potential bypass by a compromised renderer. (2619b94)
  • The Sync scheduler logic was improved to attempt a sync cycle even with an expired access token, allowing the network layer to handle the token refresh instead of blocking the scheduler. (e486e98)

Build & Test Infrastructure

  • ⚠️ BREAKING BREAKING: The profile_manager.h header and related files have been moved from //chrome/browser to a new, more granular //chrome/browser/profiles:profile_manager target, which will break downstream projects that depend on the old target. (fa00b2f)
  • ⚠️ BREAKING BREAKING: The HarfBuzz dependency directory has been renamed from third_party/harfbuzz-ng to third_party/harfbuzz. This path change will break any build systems or code that directly reference the old path. (2f9f433)
  • ⚠️ BREAKING BREAKING: The header webstore_data_fetcher_delegate.h has been relocated from //chrome/browser/extensions to //extensions/browser, causing build failures for projects that include it from its previous location. (9ac1886)
  • The base::TaskEnvironment was updated to ensure mock time advances correctly in tests, fixing an issue where subclasses could have an incorrect time domain. (7506d50)
  • The extensions::test API was updated to enforce strict boolean typing for assertTrue and assertFalse, aligning with the W3C browser.test proposal. (85287c1)

Browser UI & Features

Extensions

  • A bug in EventListenerMap was fixed that could cause the removal of event listeners for all service worker instances of an extension when only one was being stopped. The logic now correctly targets only the specific worker's listeners. (05bf9a6)

Networking
