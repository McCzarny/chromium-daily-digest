Breaking Changes

  • ⚠️ BREAKING Browser Lifecycle Observation Removed: The BrowserListObserver class and its notification mechanism have been completely removed. Embedders must find a new way to monitor browser window creation and removal. The related utility function chrome::FindAllTabbedBrowsersWithProfile has also been deleted. (2ebf735) (4650246)
  • ⚠️ BREAKING Extension APIs Removed from Desktop: The chrome.usb and chrome.bluetooth* family of APIs have been removed from Windows and macOS builds, restricting them to ChromeOS only. Extensions relying on these APIs for hardware interaction on desktop platforms will no longer function. (1e44322) (32b3c6d)
  • ⚠️ BREAKING Core Networking Mojo Interfaces Changed: Opaque [Native] types in network_param.mojom (e.g., for HttpResponseHeaders, CertVerifyResult, SSLInfo) have been replaced with fully defined Mojo structures. This is a fundamental change requiring significant updates for any code directly interacting with these network service interfaces. (d825538)
  • ⚠️ BREAKING Browser UI API Removals: Several public methods on the BrowserWindow interface related to the bookmark bar (e.g., IsBookmarkBarVisible, FocusBookmarksToolbar) have been removed. Control has moved to the BookmarkBarController. Additionally, PinnedToolbarActions::GetCastButton() and SetActionElementIdentifier() were removed from the toolbar API. (aaed67f) (27ee3df) (9746e16)
  • ⚠️ BREAKING New Pure Virtual Methods Added to Public Interfaces: Embedders implementing content::DevToolsManagerDelegate must now provide an implementation for the new AllowInspectingTarget method. Similarly, implementers of AccessibilityAnnotatorBackend::Observer must add OnContentAnnotationsDeleted and OnContentAnnotationsCleared. (bfdcab0) (8add9cd)
  • ⚠️ BREAKING Public API Signature Changes: The content::URLDataSource::GetReplacements() method was removed, breaking WebUI data sources. The gfx::NativePixmapPlane constructor now uses uint32_t and uint64_t instead of int. The content::IdentityRequestDialogController::ShowModalDialog method added a new callback parameter. (7c2b6cb) (8c46973) (47ad706)
  • ⚠️ BREAKING Behavioral Security Change for HLS Media: HTTP Live Streaming (HLS) media can no longer use byte-range requests for cross-origin content that would taint the origin. This security fix may break applications that relied on this behavior for media playback. (d2e04a1)
  • ⚠️ BREAKING Behavioral Security Change for Extension Workers: A strict same-origin check is now enforced for DedicatedWorker main script fetches within Extensions and Isolated Web Apps. This may break extensions that were legitimately loading workers from a different origin. (8bde565)

Browser & UI

  • The chrome://policy page now correctly formats JSON values for policy conflicts, improving readability for administrators. (e5f168f)
  • A Use-After-Free vulnerability in the desktop FedCM (WebID) UI was fixed by adding WeakPtr checks to prevent access to destroyed UI objects during synchronous callbacks. (60dc402)
  • Accessibility on Windows has been improved by exposing heading levels to screen readers via UIA properties. (74f73f3)
  • A crash when creating a new profile has been fixed by sanitizing the avatar index read from user preferences. (df8537c)
  • The observer list for RenderViewContextMenuBase is now re-entrant, preventing crashes from nested observer notifications during menu construction. (5a97077)

Extensions

  • A new dictionary-based format for the mime_types_handler manifest key is being introduced behind the kApiMimeHandler feature flag. This will allow extensions to handle MIME types like application/pdf without plugins. (50bea86)
  • The Declarative Content API has been hardened against compromised renderers by validating CSS selector matches and ensuring the extension has host permissions before script injection. (3995e72)
  • A security vulnerability in the extension downloader was fixed; it no longer sends authentication headers on redirects to non-Google URLs. (c9de882)

Networking & Security

  • A Use-After-Free vulnerability in the File System Access API was patched. This introduces a new RemoveFileChangeObserver method to the SandboxFileSystemBackendDelegate API. (cfafd42)
  • A new enterprise policy, DataUrlInWebWorkerOpaqueOriginEnabled, was added to allow administrators to temporarily revert to legacy origin behavior for data: URL workers. (8d1a1e1)
  • The payment manifest cache key was changed from a hash to the full URL spec to fix a security issue. This will cause a one-time invalidation of existing cache entries. (cf568b2)
  • The net::HashValue::FromString function has been deprecated as part of a move towards a safer factory function pattern. (11fbf92)

Build, Test, & Developer Experience

  • Windows Build Performance: A new GN argument, use_lld_prefetch_inputs, was added for Windows builds to enable the /prefetch-inputs linker flag, significantly improving link times for large targets on cold builds. (8b450df)
  • Windows ATL Removal: A significant refactoring removed dependencies on legacy ATL security classes (e.g., CSecurityDesc) from the Updater and Enterprise Companion components, replacing them with modern equivalents from base::win. (9425684)
  • A new Clang flag, -fdiagnostics-show-inlining-chain, has been added to improve the clarity of compiler warnings and errors by showing the inlining context. (8f58188)
  • A new build flag was introduced to help developers identify performance costs associated with base::span bounds checks. (09ffe4b)
  • A centralized FakeSendTabToSelfModel was created to consolidate multiple test doubles into a single, maintainable fake for unit and browser tests. (4c6f0eb)
