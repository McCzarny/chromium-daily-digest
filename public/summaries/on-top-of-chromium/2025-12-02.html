Summary for 2025-12-02 on branch 'main'

Chromium Digest: 2025-12-02

Overview

This daily summary covers 677 commits on the main branch, with 523 deemed relevant to developers. The changes span from (6f8b887) to (b92eaaf). This edition is packed with numerous significant breaking changes across the GPU, Media, Security, and Identity APIs, requiring immediate attention from embedders. Other notable updates include behavioral changes to DevTools, UI refactoring, and the removal of several experimental features.

Breaking Changes: Core APIs & Identity

  • ⚠️ BREAKING The public field AccountInfo::hosted_domain has been removed from the core identity management API. Embedders must now use the new GetHostedDomain() accessor method, which returns a std::optional instead of using a sentinel string value. (bb0a00d)
  • ⚠️ BREAKING The public virtual method GetDomainsForDeferredCookieDeletion has been removed from the content::BrowsingDataRemoverDelegate interface. Custom implementations of this delegate will no longer compile. (102185f)
  • ⚠️ BREAKING The public method GetDeepScanningRequests has been removed from DownloadProtectionService. Code relying on this method to inspect active Safe Browsing deep scans will fail to build. (a3bc019)
  • ⚠️ BREAKING The constructor for privacy_sandbox::TrackingProtectionSettings has been changed, removing its dependencies on HostContentSettingsMap and ManagementService. Embedders who create or test this KeyedService directly must update their code. (a05228d)
  • ⚠️ BREAKING The base class of MemoryMappedRuleset has been changed from base::RefCountedThreadSafe to base::RefCounted, removing its thread-safety guarantees. This can cause build failures or race conditions for embedders using this class across multiple threads. (dbd3c69)
  • ⚠️ BREAKING The signature of the CombineCallbacks helper function in profile_management_types.h has been changed to accept a std::vector of callbacks instead of two individual callbacks. (eafd498)

Breaking Changes: GPU & Media

  • ⚠️ BREAKING The widely used media::VideoFrame class has had its public API significantly altered. The gpu_memory_buffer_ member and the static WrapExternalGpuMemoryBufferHandle() method have been removed, impacting how video frames with external GPU memory are handled. (9529e2b) (eb9adb2)
  • ⚠️ BREAKING Core GPU initialization APIs have changed. The enable_gpu_rasterization field was removed from gpu::mojom::RasterCreationAttribs, and the signatures for gpu::CommandBufferProxyImpl::Initialize and gpu::RasterDecoder::Initialize were modified. (8786a32) (f6ce63c)
  • ⚠️ BREAKING The public API of gfx::DisplayColorSpaces and its mojo interface have been refactored to use viz::SharedImageFormat instead of gfx::BufferFormat. This also affects the ui/display/manager/display_manager.h header. (e57f34c) (e37c199)
  • ⚠️ BREAKING On Windows, the return type of the GetCurrentFrame() method in the D3D12VideoEncodeDelegate class has been changed, affecting the public API for D3D12 video encoding. (484576f)

Breaking Changes: Build & File Paths

  • ⚠️ BREAKING Public headers for the WebAuthn (FIDO) API have been moved from //device/fido/ to a new //device/fido/public/ component. Embedders must update their include paths to avoid future build failures. (e3c9acb)
  • ⚠️ BREAKING The AutofillMessageController and its related files have been moved from chrome/browser/ui/autofill/payments/ to chrome/browser/ui/autofill/. This file path change will break builds for embedders that directly include this UI component. (f3b9fd4)

Developer Tools

  • ⚠️ BREAKING The behavior of the Target.exposeDevToolsProtocol method has been changed. The exposed binding is now restricted to the page's default execution context, which will break automation scripts that relied on its availability in other contexts like isolated worlds. (c81dc1b)
  • ⚠️ BREAKING The DevTools Fetch API can now override the Cookie header. This gives DevTools priority over the browser's internal cookie store, which may break testing and automation scripts that relied on the previous behavior. (04816f9)

Security & Privacy

  • The SyncTypesListDisabled policy has been updated. Disabling the tabs sync type will now also disable savedTabGroups and vice-versa, unifying their policy control. (a658296)
  • A new UI row for "Secure connections" (HTTPS-First Mode) has been added to the chrome://settings/security page as part of the "Bundled Security Settings" feature. (2706322)
  • The experimental Privacy Budget / Identifiability Study feature and its related code have been removed from the codebase. (b770092) (ded13cd) (f0e48fc)

Browser & UI

  • On Windows, a new method, StartCheckIsDefaultAndGetDefaultClientProgId, has been added to DefaultSchemeClientWorker to allow for more robust validation of the default browser state. (70ec72b)
  • The Omnibox will now limit suggestions to a single default match when a "fusebox" (contextual) attachment is present. (f9a9682)
  • Several use-after-free bugs were fixed, including one in OmniboxEditModel that could occur when popup observers invalidated autocomplete results and another in BrowserView during teardown. (b7986fe) (c07492b)

Base & Code Health

  • New public functions have been added in content/public/child/font_integration_init.h to provide a single entry point for initializing font services in utility processes. (86b1b7b)
  • New std::ranges-like sequence utilities (Concat, Reference, Singlet) have been added to media/base to provide iterable views over containers without copying data. (14b6360)
  • Build support was added for libdmg-hfsplus, a library used for creating DMG files, which will be used in the macOS packaging process. (4dc7d59)
  • Several expired or experimental feature flags were removed, including calculate-native-win-occlusion for Windows, tear-off-web-app-tab-opens-web-app-window for PWAs, and ListAllDisplayModes. (15bab8d) (0458527) (9200f54)
