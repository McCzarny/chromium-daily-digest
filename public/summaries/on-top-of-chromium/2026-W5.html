Week 5 of 2026 (2026-01-25 to 2026-01-27)

Chromium Weekly: 2026 Week 5

Overview

This week saw an unusually high volume of breaking changes across core APIs, extensions, and UI components. A significant churn occurred in the base::Value API, with types being refactored and then renamed back within days. Other key developments include numerous API modernizations in networking and extensions, and performance improvements to startup and process management. This summary covers over 550 relevant commits out of 715 total across 2 days.

Core APIs & Memory Management

  • ⚠️ BREAKING BREAKING CHANGE: A significant and disruptive churn occurred in the base::Value API. An initial large-scale refactoring replaced legacy base::DictValue and base::ListValue with modern base::Value::Dict and base::Value::List types. However, this change was reverted shortly after, with base::Value::Dict and base::Value::List being renamed back to base::DictValue and base::ListValue. Embedders will need to update their code to align with the final base::DictValue/base::ListValue naming. (f5a673b) (dddcb58)
  • ⚠️ BREAKING BREAKING CHANGE: base::BindOnce and base::BindRepeating are now marked with [[nodiscard]]. This will cause new compiler warnings or errors in projects that call these functions and discard the resulting callback, potentially breaking builds that treat warnings as errors. (6af8098) (dedcec8)
  • ⚠️ BREAKING BREAKING CHANGE: The PartitionAlloc memory allocator saw multiple breaking changes. PartitionRoot was refactored from a struct to a class, and low-level allocator shim function signatures were updated. These changes will break code that interacts directly with the allocator's internal structures. (4c9f31d) (150979b)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE: Several public extension APIs were modernized, requiring code updates. BrowserWindow::GetExtensionsContainer() was removed in favor of ExtensionsContainer::From(Browser&). The ExtensionsMenuDelegateDesktop API was also changed, and ExtensionTabUtil::GetTabIdForExtensions() now requires a WebContents& reference instead of a pointer. (6d09036) (ec3c934) (9c932e3)
  • The performance of extensions using content scripts at document_start has been improved by compiling the injected JavaScript on a background thread. (101c2fe)

Browser UI & APIs

  • ⚠️ BREAKING BREAKING CHANGE: The virtual method ShouldEnforceNewCanCommitUrlChecks has been removed from the public content::ContentBrowserClient interface. This was a temporary kill switch for stricter navigation security checks, and embedders must now remove any overrides. (26e2892)
  • ⚠️ BREAKING BREAKING CHANGE: Several UI-related APIs were refactored. ui::WebDialogUIBase was merged into ui::WebDialogUI, requiring updates to class inheritance. APIs for accessing the location bar in BrowserView and ToolbarView were changed, and the drag-and-drop method OSExchangeData::GetURLsAndTitles() was removed in favor of an updated GetURLs(). (3012c6e) (35c8239) (b503fa4)
  • ⚠️ BREAKING BREAKING CHANGE: The Autofill API received breaking changes. ChromeAutofillClient no longer inherits from content::WebContentsObserver, and the public method FormFieldData::IsFocusable() was renamed to is_focusable(). (ce9fc87) (321d1a0)

Networking & Security

  • ⚠️ BREAKING BREAKING CHANGE: Core networking Mojo interfaces were updated for improved type safety and clarity. The process_id field in network::mojom::URLLoaderFactoryParams is now a class instead of an int, and the has_user_gesture field in blink::mojom::CommonNavigationParams was renamed. (ae20224) (e340c57)
  • On Windows, security was enhanced by adding a SetIsolationSecurityAttribute method to the RestrictedToken class and by refactoring Code Integrity Guard (CIG) hooks to remove IPC usage. (37969c6) (46c377e)

Performance

  • Performance was a key focus, with several improvements enabled by default. A new policy (PriorityBoostLoadingBrowserNetworkPolicy) optimizes process priority during startup. Renderer process reuse features were enabled to reduce memory usage, and on Windows, a task was deprioritized to prevent loading certain DLLs on the critical startup path. (23517c6) (6e70d31) (054bd0b)

Build & Testing

  • ⚠️ BREAKING BREAKING CHANGE: A large number of test files were renamed to standardize suffixes (e.g., _unittests.cc to _unittest.cc). This may break custom build scripts or configurations that reference the old filenames. (29c9e32)
