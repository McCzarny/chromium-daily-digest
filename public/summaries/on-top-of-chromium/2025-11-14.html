Chromium Digest: 2025-11-14

Overview

On November 14, 2025, the Chromium project saw 694 commits, with 522 identified as relevant to developers. Key changes include the modernization of the extension installation UI, foundational work for the WebNN API on Windows, and the removal of several obsolete feature flags. Other notable updates focus on security hardening, networking improvements, and UI polish across the browser. The first commit of the day was `412b56f53d8e925be93291c767a49af09ac450bf` and the last was `c367a66a4b51943c55753961ff187afca4049025`.

Breaking Changes

  • ⚠️ BREAKING Several obsolete feature flags have been removed from the codebase. Projects that were enabling or disabling JourneysOmniboxAction, GCMDeleteIncomingMessagesWithoutTTL, ImageAnnotationServiceExperimental, or MinimumChromeVersion will need to update their configurations. (0de09dd) (412b56f) (063365f) (ddf4ba5)
  • ⚠️ BREAKING The deprecated codecs library has been removed from Python-based build and test scripts. Custom scripts must be updated to use open() with an explicit encoding where necessary. (4fffd50)

Browser UI & UX

  • The AI Model (AIM) omnibox popup received several updates, including gating it behind an eligibility service, refining its activation logic, and fixing UI bugs related to jitter, mouse handling, and input focus. (89213ac) (6499ec0) (a881217) (238d3a9) (cfca771)
  • On macOS, a bug was fixed where the first click on an inactive window would not properly bring it to the foreground, correcting a 'half-activation' state. (4b4dec9)
  • The signout confirmation prompt was updated to provide more context by showing the user the count of unsynced data items. (20f0fef) (43cd952)
  • A potential crash in TabStripControlButton has been fixed, which could occur when its background was updated during window destruction. (0c70bff)

Extensions

  • The extension download confirmation dialog has been modernized to use the ui::DialogModel framework, aligning it with other modern Chrome dialogs. (37365a1)
  • Several UI bugs in the extension installation dialog were fixed, improving layout consistency and color accuracy. (31b8f82) (cf409aa) (2d6ceb5)

Security

  • Support for PDF XFA forms is now controlled by an enterprise policy and a feature flag. The check has been moved from the renderer to the more secure browser process. (9045259)
  • A potential favicon spoofing vector has been mitigated. The logic for finding a fallback favicon now uses the most recently visited URL for a given host. (cc45d4e)
  • The long-running effort to improve memory safety continues, with many components converting from broad allow_unsafe_buffers pragmas to more granular UNSAFE_TODO markers for targeted auditing. (98360f7) (2bcb6e5) (fd570b2)
  • The use of rand() is now banned in new code submissions to enforce the use of more secure random number generation from base/rand_util.h. (71b37a4)

Web APIs

  • The WebNN API on Windows is now easier to use, as the browser will automatically install the required Windows App Runtime (WinAppRuntime) dependency on launch. (16ad7ec)
  • The WebXR Layers API has been enhanced to support stereo layouts (left-right and top-bottom) for quad, cylinder, and equirect layers, expanding capabilities for immersive experiences. (7dcfb19)
  • A deprecation warning for the Protected Audience APIs has been added and will be visible in Developer Tools, signaling future removal. (12d30b6)
  • A WebAuthn bug was fixed that could cause conditional passkey creation requests to fail due to an uninitialized member in PasskeyUpgradeRequestController. (0fe5016)

Networking

  • To mitigate cross-site tracking techniques, the TCP socket pool limit per proxy, per destination is being randomized. (3620e7c) (877cb61)

Developer Tooling & Infrastructure

  • A new internal debug page, chrome://contextual-tasks/internals, has been created for the Contextual Tasks feature. (651a85f)
  • Initial setup was added for PUMA (Private UMA), a new metrics collection system designed with enhanced user privacy. (23fdfe8) (1d3ac16)
  • The Chromium updater service is now instrumented with more detailed history event logging for update events and Omaha protocol requests. (254b651) (c420c75)
