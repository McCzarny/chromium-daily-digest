Chromium Digest: 2026-01-12

Overview

On January 12, 2026, the Chromium main branch saw 547 commits, with 384 being relevant to developers building on the platform. The day's changes included a significant number of breaking API removals and modifications in core services, networking, and developer tools. Notable work was also done to enhance security features like cache encryption and Secure Payment Confirmation. The first and last relevant commits were (a640371) and (c70144b).

APIs & Services

  • ⚠️ BREAKING BREAKING CHANGE: The public header base/strings/latin1_string_conversions.h and its function Latin1OrUTF16ToUTF16 have been removed. Embedders must find an alternative for this string conversion utility. (e4c0c41)
  • ⚠️ BREAKING BREAKING CHANGE: The public header ui/gfx/buffer_format_util.h has been removed as part of a migration from gfx::BufferFormat to SharedImageFormat. Downstream projects including this header will fail to build. (514bc7d)
  • ⚠️ BREAKING BREAKING CHANGE: The public virtual method ClearAllTabAssociationsForTask has been removed from the ContextualTasksService interface. Implementations or callers of this service will need to be updated. (c6eb642)
  • ⚠️ BREAKING BREAKING CHANGE: The public method IsSupervisedLocally has been removed from SupervisedUserService. This is part of a larger refactor to decouple profile dependencies. (279d9c4)
  • The signature for gpu::MappedMemoryManager::Alloc() was briefly changed to return a base::span<uint8_t> instead of void*, but this was reverted due to test failures. While not a persistent break, developers should be aware of this churn. (b9b01d3) (4c01cbb) (4caa0a3)
  • A new utility method, View::ConvertRectFromScreen(), has been added to the views framework, providing the inverse operation to ConvertRectToScreen(). (28cf934)
  • The Viz CopyOutputRequest API has been improved with a timeout mechanism and error codes to help diagnose surface capture failures. (21fc482)
  • An unused and potentially unsafe overload of base::File::ReadAtCurrentPosNoBestEffort has been removed to improve code safety. (b3a0fa5)

Networking & Security

  • ⚠️ BREAKING BREAKING CHANGE: A QUICHE dependency roll removes the public methods UsesHttp3() and UsesTls() from quic::ParsedQuicVersion and alters its constructor. Embedders must update to use IsIetfQuic() and the new constructor signature. (a00f081)
  • ⚠️ BREAKING BREAKING CHANGE: The extensions::ContentVerifier class now enforces destruction on the IO thread. This changes the lifetime management contract for this public, ref-counted class, potentially impacting embedders who manage its lifecycle. (78fb3ba)
  • ⚠️ BREAKING BREAKING CHANGE: The ValidateSignedWebBundleSignatures function for Isolated Web Apps no longer performs trust validation. Callers are now responsible for performing this check separately, representing a significant change in the API's contract. (d675064)
  • The Secure Payment Confirmation (SPC) feature using browser-bound keys (SecurePaymentConfirmationBrowserBoundKeys) has been promoted to stable for macOS and Windows, enabling more secure web payment flows. (56976fd)
  • The cache encryption system has been enhanced with a new preference to store the encrypted master key and by integrating chunked encryption for file reads and writes. (d80d8c1) (6d99411)
  • For enterprise users, the SafeBrowsingPasswordReuseEvent has been enriched with URL and referrer information to improve risk scoring. (57de1b2)
  • Data URL parsing in net/base now correctly preserves MIME type parameters, improving alignment with the Fetch Standard. (20d11f7)

Extensions

  • The ExtensionsMenuViewModel::OnHostAccessRequestRemoved method signature has been updated to include the index of the removed item, allowing for more efficient UI updates. (36e90ff)
  • The ExtensionApiCaptureTest suite has been modernized to be Manifest V3-only, with deprecated MV2 tests being removed. (399c988)

UI & Views

  • A crash on macOS when entering immersive mode with the vertical tabstrip enabled has been fixed. (40f1d6e)
  • A crash in the Picture-in-Picture window caused by child dialog destruction has been resolved. (dadc58c)
  • The ReloadButtonWebView has been refactored into a more generic WebUIToolbar to support additional controls in WebUI pages. (4651531)
  • On macOS, a custom image for the "Paste and Match Style" menu item was removed, as the operating system now provides a suitable default icon. (1c0cfb5)

Developer Tools

  • ⚠️ BREAKING BREAKING CHANGE: The Network.getRequestPostData DevTools protocol command has been updated. It now returns an additional base64Encoded boolean and sends binary data as a base64 string instead of a potentially corrupt UTF-8 string. Clients parsing binary request bodies must adapt to this new format. (e51443c)

Testing & Infrastructure

  • A new MockExtensionInstallPolicyService has been introduced to simplify unit testing of components that depend on extension installation policies. (5c05871)
  • The Universal Test Runner (UTR) now supports an --omit-default-test-args flag, giving developers more control over the test execution environment. (2d0ac29)
  • A new telemetry performance test has been added for the Omnibox AIM (Ask-It-To-Me) WebUI popup to track and improve its performance. (4f8c444)
