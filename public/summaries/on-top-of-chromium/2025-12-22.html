Chromium Digest: 2025-12-22

Overview

This summary covers 239 relevant commits out of 353 total from the main branch on December 22, 2025. The changes span from the first commit ((5606fc8)) to the last ((a140bc6)). This digest highlights several significant breaking API changes in privacy, UI, and extensions, alongside new enterprise policies and developer-facing improvements to the extensions platform and core base utilities.

Breaking Changes

  • ⚠️ BREAKING Privacy Sandbox: The public TrackingProtectionSettings class, its observer interface, and the TrackingProtectionSettingsFactory have been removed. Embedders relying on these for observing or managing tracking protection settings will be broken. (36e492c) (98049c0)
  • ⚠️ BREAKING UI Views: The virtual methods HandleWriteTextToClipboard and AllowStartDragEvent have been removed from views::TextfieldController. Subclasses overriding these methods to customize clipboard or drag-and-drop behavior will fail to build. (fbef187)
  • ⚠️ BREAKING Base Utilities: The public method base::TraceLog::IsEnabled() has been removed. Code must be updated to use base::TrackEvent::IsEnabled() to check if tracing is active. (5051d64)
  • ⚠️ BREAKING Extensions: The public method GetAssociatedWebContents() has been removed from extensions::ExtensionFunctionDispatcher. This is a public API removal that could impact embedders. (d74add7)
  • ⚠️ BREAKING Sign-in / Identity: Two public constructors have been removed from signin::AccessTokenFetcher. Embedders using these specific constructors to obtain OAuth2 access tokens will need to update their code. (352eb26)
  • ⚠️ BREAKING Security Policy: The public methods GrantReadRawCookies, RevokeReadRawCookies, and CanReadRawCookies have been removed from content/browser/child_process_security_policy_impl.h. (41b3b0a)

Security & Privacy

  • To prevent clickjacking, a Picture-in-Picture (PiP) occlusion tracker has been added to the download bubble. This will hide the bubble's controls if a PiP window overlaps them. (704eae4)
  • To improve security, the renderer process will now be terminated if it sends invalid arguments when creating an IndexedDB cursor, preventing a potential browser crash. (c903934)
  • The Safe Browsing OHTTP key service will now include an Accept header in its requests, aligning with RFC recommendations and allowing for future gateway extensibility. (3713ee7)

Extensions

  • The sidePanel.open API has been improved to support WINDOW_ID_CURRENT. This allows an extension to open a side panel in the currently active window without needing to query for its specific ID. (beede36)
  • Significant internal refactoring of the extensions menu has moved ownership of ExtensionActionViewModel to ExtensionsMenuViewModel, better aligning with an MVC architecture and simplifying data access. (1cef5c8) (32c7f8e) (ec9882c)

Browser UI & Views

  • The InitialWebUIManager will now only manage the display of NORMAL browser windows. This fixes an issue where floating DevTools windows would fail to appear on screen. (5815408)
  • The context menu order on the bookmarks bar has been updated to match the renderer's context menu, providing a more consistent user experience. (46e2514)
  • The "declutter" feature for inactive tabs is being deprecated, and its associated metrics have been removed from the codebase. (6234a37)

Policies & Enterprise

  • Two new enterprise policies, GeminiActOnWebAllowedForURLs and GeminiActOnWebBlockedForURLs, have been added. These provide administrators with granular, URL-based control over the "Act on this page" AI feature. (0c67554)
  • The ProxyOverrideRules policy now has enhanced validation logic, providing clearer error messages for invalid URL patterns or proxy values. (ed591fb)

Networking & Performance

  • On Windows, the google-chrome:// and chromium:// custom URI schemes will now be registered with the OS. This may affect how custom protocol handlers are resolved. (17d026d)
  • A new trigger has been added to initiate prewarming of network resources when a user interacts with the omnibox. This behavior is controlled by the PrewarmOnOmniboxInteraction feature flag. (ca37757)
  • The sampler for ExhaustedIntervalsPerThirtySeconds (related to memory pressure) has been moved to a dedicated thread. This ensures it can run even when the main thread is stalled, improving measurement accuracy. (c37aedf)

Developer Experience & Infrastructure

  • A large-scale refactoring has replaced numerous usages of base::to_underlying with the standard C++23 std::to_underlying, modernizing the codebase. (ad18a4c) (19cd5a1) (6f7da9c)
  • The LanguageDetectionAPI feature flag has been removed, enabling the functionality by default. (b513405)
  • As part of an ongoing refactoring, the BinaryUploadService has been moved from chrome/browser to the components directory and placed within the enterprise_connectors namespace. (7abbd96) (6e6df93)
