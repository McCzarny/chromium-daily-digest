Summary for 2025-11-21 on branch 'main'

Chromium Digest: 2025-11-21

Overview

This summary covers 627 commits on the main branch for November 21st, 2025, with 439 identified as relevant to developers working on Chromium-based projects. The day was marked by numerous significant breaking changes to core APIs and the build system. The changes span from commit (6fd82e2) to (3253015).

Breaking Changes

  • ⚠️ BREAKING The build system now requires C++20, as support for the use_cxx17 build variable has been removed. Downstream projects must update their toolchains and codebases from C++17. (e67ce2f)
  • ⚠️ BREAKING The public API setters in url::Replacements now accept std::basic_string_view instead of raw const CHAR* pointers for methods like SetUsername, SetPassword, and SetHost. Code using this widely-used URL manipulation class must be updated. (667164e)
  • ⚠️ BREAKING The logging::LoggingSettings struct has been moved from base/logging.h to its own header, base/logging/logging_settings.h. Projects including the old header to access this struct will fail to compile. (d7fb9d0)
  • ⚠️ BREAKING The libpng-based SkPngEncoder.h and SkPngDecoder.h have been removed from the public Skia API. Embedders must migrate to the Rust-based SkPngRustEncoder.h and SkPngRustDecoder.h alternatives. (32b26f0)
  • ⚠️ BREAKING The public method BrowserProcess::fingerprinting_protection_ruleset_service() has been removed as part of a larger component cleanup, breaking any code that calls this core browser API. (d035d43)
  • ⚠️ BREAKING The public static method content::ServiceWorkerContext::GetUpdateDelay() has been removed. Code calling this will no longer compile. (9c3fcc8)
  • ⚠️ BREAKING The ui::AXNodeData struct was changed by converting the bool_attributes member from a std::unique_ptr to a direct member. This breaks source and binary compatibility for code that used pointer-based access. (8488b1b)
  • ⚠️ BREAKING The public mojom enum optimization_guide::mojom::ModelBasedCapabilityKey was renamed to optimization_guide::mojom::OnDeviceFeature, breaking code that references the old name. (81de9ff)
  • ⚠️ BREAKING The base::MemoryPressureListener API will now dispatch MEMORY_PRESSURE_LEVEL_NONE notifications. Custom listener implementations may break if they do not expect this new notification level. (aef3061)
  • ⚠️ BREAKING On Windows, the IsValid() method was removed from base::win::GenericScopedHandle and then quickly restored in a revert, causing potential build breaks for code that had either not migrated or had just migrated away from it. (52ca070) (af344b9)
  • ⚠️ BREAKING The PasswordsProvider::GetActorLoginPermissions() method has been removed from desktop builds via conditional compilation, breaking any desktop-based consumers. (b549625)

Core & APIs

  • A new AccountInfo::Builder interface was introduced to provide a safer, more maintainable way to construct AccountInfo instances. (a37d5e3)
  • The ZipReader API in third_party/zlib was enhanced with a ReaderDelegate to allow reading from abstract data sources beyond just files or strings. (dc0e6de)
  • The Legion component's Encrypt, Decrypt, and SecureSession methods were made asynchronous in preparation for moving the implementation to a separate, more secure process. (1ce942f) (39e36b9) (30a3585) (ee210db)
  • The persistent cache database now supports being opened for a single, exclusive connection, improving performance. This is now used by the GPU shader cache. (4ddf154)
  • The last remnants of the legacy sigslot library were removed from the remoting protocol implementation, completing a long-term refactoring effort. (a5a759a)

Security & Privacy

  • New enterprise policies, IncognitoModeBlocklist and IncognitoModeAllowlist, were added. These take precedence over regular profile blocklists when in Incognito mode. (97821c4)
  • To support native SSO with Okta on managed devices, the browser will now conditionally spoof its user agent to appear as Safari when making requests to okta.com, controlled by a new feature flag. (445243e) (9f02006)
  • The ProxyOverrideRules enterprise policy now supports the scheme://host:port format in its rules. (3038c74) (0c00aa4)
  • The enterprise policies PrivacySandboxSiteEnabledAdsEnabled and PrivacySandboxAdMeasurementEnabled have been marked as deprecated. (151c91a) (bce66b1)
  • The PasskeyUnlockManager was refactored to use an interface (EnclaveManagerInterface), improving dependency injection and testability for passkey unlock flows. (48be92e)

Browser UI & UX

  • The logic for handling bookmark corruption has been improved. Instead of reassigning all node IDs, Chromium will now only reassign IDs involved in a collision, providing more stability for extensions that rely on bookmark IDs. (115ab85)
  • A bug was fixed in the Windows Text Services Framework (TSF) integration that could cause the last IME composition to be incorrectly committed instead of cancelled. (8697963)
  • A new pinnable toolbar button for a "Contextual Tasks" feature has been added. (8c04694)
  • For PWA windows, the tabstrip is now correctly positioned before the toolbar, and navigation buttons now use standard browser icons. (5a4159b) (fffe95d)
  • BubbleDialogDelegateView was updated to correctly calculate its anchor position when anchored to a WebUI element, fixing bugs where the bubble would not move with the main window. (4c73bcb)
  • The Omnibox WebUI popup was made more robust with logic to restart its renderer if it crashes, preventing the popup from becoming unresponsive. (1aa7817)

Extensions

  • New autofillPrivate APIs (getWalletablePassDetectionOptInStatus, setWalletablePassDetectionOptInStatus) were added to allow settings UIs to control the walletable pass detection feature. (7ad3a43)
  • ExtensionActionViewModel, a core component for managing toolbar extension icons, was updated to support multiple observers, improving testability and UI flexibility. (eb33936)
  • The crx_file::id_util::GenerateId() API was updated to use std::span for improved type safety and performance. (a82a18f)

Networking

  • A race condition in the File System Access API was fixed by removing a redundant user activation check in the browser process for ChooseEntries. This fixes a bug where the file picker could fail to open. (faf426c)
  • As part of moving connection-allowlist enforcement to the network service, a unique network restriction ID is now added to the URLLoaderFactory for each frame. (63c9638) (e0ef2cf)
  • Scaffolding was added for a ProxyingURLLoaderFactory to intercept network requests, in preparation for an upcoming Okta SSO extension integration. (9fc6e9f)

Build & Testing

  • The system_headers check, which helps enforce include-what-you-use (IWYU) principles, is now enabled for Windows builds. (2a37c48)
  • The EvalJs test helper function was documented to clarify that base::Value::Type::BINARY is not a supported argument type. (bc2fb97)
  • A new script was added to build/bench to run autoninja multiple times and collect performance metrics for analysis. (b4bafc0)
  • A clang+rust toolchain roll was relanded after a previous revert. (c9e8ad2)
