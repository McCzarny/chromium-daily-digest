Chromium Digest: 2026-03-10

Overview

This summary covers 509 relevant commits out of 707 total for the main branch on March 10, 2026. The day's changes include a significant number of breaking API and behavioral changes, particularly in graphics, autofill, and extensions. Key developments also include new APIs for UI components, security enhancements for networking protocols, and continued refactoring of core browser services. The first relevant commit was (d31051c) and the last was (39de2dc).

Breaking Changes

  • ⚠️ BREAKING The automatic fallback to the ANGLE D3D9 backend on Windows has been removed. If D3D11 initialization fails, Chromium will now use software rasterization instead of D3D9, which can significantly impact performance on older systems. A new feature flag, AllowANGLED3D9Fallback, can be used to temporarily re-enable the old behavior. (282348e)
  • ⚠️ BREAKING The public AutofillClient API has been changed. The method ShowAutofillAiFailureNotification(std::u16string message) is replaced by two new parameter-less methods: ShowAutofillAiSaveToWalletFailureNotification() and ShowAutofillAiFetchFromWalletFailureNotification(). Embedders must update their AutofillClient implementations. (b1627f2)
  • ⚠️ BREAKING The public methods set_storage_partition() on ServiceWorkerContextWrapper and ServiceWorkerProcessManager have been made private. Downstream code using these methods to configure service workers will need to be refactored, potentially using ServiceWorkerContextWrapperTestApi for test environments. (d0ba702)
  • ⚠️ BREAKING The signature for the public API extensions::EventFilter::RemoveEventMatcher has been changed as part of a performance optimization. Call sites must be updated, and a new RemoveEventMatchers method is available for batch removals. (cf6d1ed)
  • ⚠️ BREAKING The public method GetPersistentUsageRecordSessionSupport() has been removed from the media::KeySystemInfo interface. This will impact embedders implementing custom EME/CDM key systems. (acdadfe)
  • ⚠️ BREAKING The should_show_inactive field has been removed from the public tabs::TabDialogManager::Params struct. Code using this parameter to show dialogs without activating the parent window must be updated. (ddc789c)
  • ⚠️ BREAKING The H.265 video decoder is now stricter and will generate a decoding error for streams with configuration changes (e.g., resolution) on non-key frames. This may break playback for previously working but technically invalid content. (aca89b5)
  • ⚠️ BREAKING The signature for BrowsingDataQuotaHelper::FetchResultCallback has changed. The callback now receives a std::vector<QuotaInfo> instead of a const std::list<QuotaInfo>&, requiring updates to implementations that query quota data. (0f27d66)
  • ⚠️ BREAKING The command-line switch kEnableWebGLImageChromium is no longer available on Windows and macOS, as it has been restricted to ChromeOS. This removes a public flag used for specialized graphics overlay support. (9fccafe)
  • ⚠️ BREAKING The method signature for the pure virtual function BnplUiDelegate::UpdateBnplIssuerUi has been changed. Custom implementations of this interface must be updated to match the new signature. (5cc1bba)

APIs & Services

  • The Direct Sockets API has been extended to support Source-Specific Multicast (SSM) for UDP sockets, allowing isolated web applications to join and leave multicast groups with source filtering. (0302377)
  • The TabStrip API has been extended with last_active_time_ticks and an experimental ReplaceTabInSplit method to better support features like Tab Search. (ce0f022)
  • A new set of UI pages and backend logic for "Actor Login M2" has been introduced to manage Gemini/AI-related login permissions. (1e06b17) (8ddb886) (59ffaad) (ad2c924) (f3d7714)
  • The webAuthenticationProxy API is now restricted to Windows, macOS, and Linux, removing it from other platforms. (5112a53)
  • The PageContentAnnotationsService has been refactored to integrate with PageEmbeddingsService via a new bridge, decoupling the components. (9eb352a)
  • The obsolete url_param_filter directory and its remaining code have been removed from the tree. (6af3e35)

Extensions

  • A crash has been fixed that occurred when guest content in a <webview> attempted to open a window with an unsupported disposition, such as Picture-in-Picture. (1e10f80)
  • A UI bug was fixed to ensure the extension side panel correctly receives focus when it is shown. (e7a534e)
  • The logic for opening extension options pages has been unified into a single helper function. (152d856)
  • For improved diagnostics, additional DumpWithoutCrash calls have been added to the extension service worker lifecycle to help debug initialization issues. (2a7a57d)

UI & UX

  • A new API, WebUILocationBar::GetAnchorOrNull, has been added to support upcoming UI chips that will appear in the location bar. (e2081ee)
  • On macOS, a bug where double-clicking the omnibox failed to select a word if the omnibox popup was visible has been fixed. (ac47a60)
  • The Side Panel infrastructure has been refactored to be more platform-agnostic, including renaming SidePanelUtil to SidePanelHelper. (f1041bd) (2af1630) (1ddac02)
  • Accessibility for radio buttons that are not direct siblings has been improved to ensure screen readers announce their group position and size correctly. (467d3bf)

Networking & Security

  • To reduce latency, the SQLite cookie store can now be initialized early upon creation rather than waiting for the first load request. (24ddf87)
  • Link Previews are now restricted to only http and https schemes for improved security. (57482f5)
  • Logic has been implemented to support Trust Anchor ID intersection for Multi-Certificate Transparency (MCT) in the TAI retry flow, enhancing connection security. (be0a703)
  • An old and unused cookie database migration path (from v22 to v23) has been removed, simplifying the codebase. (08708d4)

Graphics & Media

  • On Windows, the Dolby Vision decoder library is now preloaded in the GPU process to ensure it is available after the sandbox is engaged. (1fb8969)

Base, Testing & Infrastructure

  • base::RangeAsRvalues() now correctly supports reverse iteration if the underlying range is reversible. (49ea398)
  • In Headless mode on Windows, DevTools screen emulation methods (Emulation.setDeviceMetricsOverride and Emulation.clearDeviceMetricsOverride) are now supported. (18bc6e9)
  • The testing API for views has been enhanced with an explicit require_visible parameter in ElementTrackerViews and BrowserElementsViews, allowing tests to query for non-visible views. (c00788d)
  • views::ScrollView has new test-only methods, IsHorizontallyOverflowing() and IsVerticallyOverflowing(), to check for content overflow. (925ab27)
