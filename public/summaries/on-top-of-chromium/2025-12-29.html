Chromium Digest: 2025-12-29

Overview

This report analyzes 100 commits from the Chromium repository, with 12 relevant changes identified after filtering for Windows and macOS platforms and focusing on browser APIs, extensions, security, and UI. Of these, 4 are marked as major breaking changes that may require action from embedders. Key themes include the removal of deprecated Safe Browsing and privacy-related APIs, refactoring of mojom interfaces, and ongoing cleanup of experimental features.

Security & Privacy

  • ⚠️ BREAKING [BREAKING] The virtual methods MaybeReportSafeBrowsingHit and CreateAndSendHitReport have been removed from the BaseUIManager and SafeBrowsingUIManager classes. This is part of a broader deprecation of hit reporting. Embedders that subclass these managers and override these methods will encounter build failures. (cbeca2e)
  • ⚠️ BREAKING [BREAKING] The public header components/privacy_sandbox/tracking_protection_prefs.h and its associated registration function tracking_protection::RegisterProfilePrefs have been deleted. Code that includes this header or calls the function will no longer compile. The remaining relevant preferences have been consolidated into privacy_sandbox_prefs. (dd66256)
  • Several feature flags and code paths related to the experimental Third-Party Cookie Deprecation (TPCD) have been removed as part of ongoing cleanup. This includes the TpcdExperimentEligibility class and various test-only feature settings. (1feabfd) (1ff3d48) (af24040) (aa33553)

APIs & Extensions

  • ⚠️ BREAKING [BREAKING] The searchbox.mojom interface has been refactored to remove the Stub suffix from several types, such as SearchContextStub (now SearchContext) and FileAttachmentStub (now FileAttachment). This directly changes the public API contract for the omnibox popup and will break any client code using the old type names. (1169c88)
  • ⚠️ BREAKING [BREAKING] The ContextualTasksContextController class, along with its factory and test support files, has been completely deleted from chrome/browser/contextual_tasks/. The commit message indicates it was a wrapper around ContextualTasksService. Any code depending on this class will be broken. (8858a40)
  • The GuestViewBase class, which powers <webview> tags for embedders, now handles the forwarding of draggable regions from the guest to the embedder. This internalizes logic that was previously handled by a custom WebContentsDelegate, simplifying the implementation. (9453180)
  • A dangling pointer crash in the extension's GinPort during service worker termination has been fixed. The fix invalidates ports when the context is destroyed to prevent use-after-free errors. (b84da97)
  • A null pointer crash in the developerPrivate extension API has been fixed. The AddPermissionsInfo function now correctly handles cases where an extension's granted permissions might be missing. (616d4c1)

Browser Internals & UI

  • The DeviceInfo class and its underlying sync proto (device_info_specifics.proto) have been updated with a new boolean field, desktop_to_ios_promo_receiving_enabled, to support a temporary cross-device promotional experiment. (3d9be32)
  • A null check has been added to BackForwardButton::NotifyClick to prevent a crash that could occur during browser shutdown or when no tab is active. (57b0396)
  • The feature flag kRestorePrimaryAccountInfo has been removed from the codebase, as the feature has been enabled by default since 2023. (dfc2862)
  • A new internal debug page has been added at chrome://unexportable-keys-internals to assist with the Device Bound Session Credentials (DBSC) feature. (814bcf8)
