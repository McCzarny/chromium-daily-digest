Chromium Digest: 2025-12-19

Overview

This summary covers 382 relevant commits out of 535 total, from commit (32fc2ca) to (adef7a7). The day was marked by an unusually high number of breaking changes across core, UI, and networking APIs, requiring significant attention from embedders. Other key areas of development include security enhancements, policy updates, and improvements to testing infrastructure.

Breaking API & Feature Changes

  • ⚠️ BREAKING BREAKING: The public virtual method ui::BrowserAccessibilityManager::FireBlinkEvent has been renamed to FireSourceEvent. Embedders who subclass BrowserAccessibilityManager must update their overrides. (65e8554)
  • ⚠️ BREAKING BREAKING: The deprecated Google Cloud Print feature has been removed, including the CloudPrintProxyEnabled policy and related preferences like kCloudPrintProxyEnabled. (8dd15b8)
  • ⚠️ BREAKING BREAKING: The public UI interface TabDragDelegate has been renamed to TabDragTarget. Components implementing this interface for tab dragging operations must be updated. (8ed9e83)
  • ⚠️ BREAKING BREAKING: The public method RenderWidgetHostView::CopyFromSurface now uses a base::expected<> in its callback to provide better error details, breaking the existing callback signature. (2fad3cf)
  • ⚠️ BREAKING BREAKING: The PictureInPictureWindowController::GetWindowBounds method was renamed to GetWindowBoundsInScreen and now returns the outer window bounds, a change in both name and behavior. (03aeeb0)
  • ⚠️ BREAKING BREAKING: Several base/ APIs, including base::trace_event::GetTraceBufferSizeInBytes, have changed their signatures to use base::ByteSize instead of ByteCount. (2bd833a)
  • ⚠️ BREAKING BREAKING: The public method HttpNetworkSession::SpdySessionPoolInfoToValue now returns base::Value directly instead of std::unique_ptr<base::Value>, altering the method signature. (2430c11)
  • ⚠️ BREAKING BREAKING: In google_apis/gaia/google_service_auth_error.h, the public enum value GoogleServiceAuthError::USER_NOT_SIGNED_UP has been renamed to ACCOUNT_NOT_FOUND. (86fce5b)
  • ⚠️ BREAKING BREAKING: The public API for DawnWireServices in WebGPU has been refactored, removing public getters like wire_client() in favor of new methods that manage internal locking. (31cbb39)

API & Core Internals

  • A new API, base::SysInfo::AmountOfTotalPhysicalMemory(), has been introduced to return memory size as a base::ByteSize object. The older AmountOfPhysicalMemory() which returned an int64_t is now deprecated. (4f269ef)
  • The legacy browser layout system (browser_view_layout_impl_old.cc) has been removed, finalizing the transition to the new layout manager introduced in the Chrome Refresh. (0660eaa) (3057a9f)
  • The feature flag kDeleteExpiredDownloads has been removed, making the automatic deletion of expired downloads a permanent, non-configurable behavior. (3198d2a)

Security

  • To prevent crashes from excessive memory allocation, a 256MB size limit has been imposed when reading text, HTML, and PNG data from the Windows clipboard. (600a5b3)
  • The certificate verification process has been updated to support Merkle Tree-based certificate verification. (43e6793)
  • For Device Bound Session Credentials (DBSC), garbage collection is now delayed by two minutes after session loading to improve startup performance. (2e87504)
  • The feature to redact passwords in screenshots submitted with user feedback has been enabled by default. (e4d33e7)
  • The Safe Browsing file analyzer will now use the RAR obfuscated analyzer for RAR files determined to be obfuscated. (c7121fb)

Networking

  • To improve load balancing, IP addresses for DNS-over-HTTPS (DoH) bootstrap lookups are now randomized. (6524fa3)
  • A CHECK in the proxy resolution request logic has been reverted to a DCHECK to prevent crashes in release builds caused by invalid traffic annotations. (9d6c2ed)

Browser UI & Features

  • The Omnibox popup now uses a mojom call to observe AI Model eligibility, allowing UI elements to update dynamically without a page reload. (adef7a7)
  • The kResetTamperedDefaultSearchEngine feature is now permanently enabled, automatically resetting the default search engine if it has been tampered with. (133ef7c) (7a97084)
  • A new function, GetPipWindowToExcludeFromScreenCapture, allows the system to exclude a Picture-in-Picture window from screen captures initiated by the same frame. (6cf22fc)
  • On macOS, several UI bugs have been fixed, including allowing Omnibox tooltips in non-key windows and correcting bubble anchoring in immersive fullscreen. (be6dea8) (95a45e9) (c04092a)

Extensions

  • A new extension disable reason, DISABLE_BLOCKED_BY_CLOUD_POLICY_CHECK, has been added to support the ExtensionInstallCloudPolicyChecksEnabled policy. (40752ba)
  • The developerPrivate API implementation has been simplified by merging platform-specific DeveloperPrivateFunctions classes into a single class. (cbbbe3a)

Policy & Enterprise

  • The chrome://policy page will now display a "Restart required" status for policies that have been changed but do not support dynamic refresh. (9435502)
  • A new SearchContentSharingPolicyHandler has been added to facilitate the deprecation of several older Lens-related policies. (818b9a9)
  • Warning logs will now be emitted before a user profile is forcibly deleted due to enterprise policy conflicts, providing more visibility for administrators. (26534df)

Testing

  • The EmbeddedTestServer can now be configured with a specific certificate and private key, aiding tests for features like Merkle Tree Certificates. (1f0b188)
  • Headless mode tests can now specify --enable-features and --disable-features via test metadata, allowing for better feature-specific testing. (d8ecf76) (f950b91)
  • New utility functions, browsertest_util::ExecuteUserScript*, have been added to simplify executing user scripts in browser tests. (a25757b)
