Week 13 of 2026 (2026-03-22 to 2026-03-28)

Chromium Weekly: 2026 Week 13

Overview

This week saw an exceptionally high volume of over 2,500 relevant commits out of more than 3,400 total. Development was dominated by a wave of breaking changes across core APIs, networking, and UI components, driven by large-scale refactoring efforts like 'Project Bedrock' aimed at modularizing the codebase. Key themes include API modernization, security hardening, and significant build system reorganization, requiring careful attention from downstream embedders.

API Modernization & Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE: As part of the 'Project Bedrock' initiative, numerous functions and class methods have been updated to accept a BrowserWindowInterface* instead of a Browser*, decoupling UI logic from the concrete Browser class. Embedders must update calls in areas like chrome/browser/ui/singleton_tabs.h and chrome/browser/ui/browser_commands.h. (7f61ab9) (14ee99d) (7af109e)
  • ⚠️ BREAKING BREAKING CHANGE: Numerous header files, components, and build targets were relocated, which will cause build failures for downstream projects. Notable moves include base/feature_list.h to base/feature.h, AccountSettingService to a new components/account_settings component, and profile_manager.h to a new //chrome/browser/profiles:profile_manager target. (5063835) (a1b05c5) (fa00b2f) (35ed0b5) (b5a1e23) (9ac1886)
  • ⚠️ BREAKING BREAKING CHANGE: The public content::ContentBrowserClient interface now requires embedders to implement the new pure virtual method IsFullscreenAllowedForUnfocusedWebContents. Additionally, Bluetooth-related methods were removed in favor of a new content::BluetoothDelegate implementation. (395a125) (3cb4ab6)
  • ⚠️ BREAKING BREAKING CHANGE: The behavior of base::ReplaceFile on Windows has been altered. It no longer fails if it cannot create a temporary backup file (e.g., due to a long path), removing the atomicity guarantee in that edge case. (36a258f)
  • ⚠️ BREAKING BREAKING CHANGE: The WebUI helper function cr.sendWithPromise() in ui/webui/resources/js/cr.ts now requires a generic type parameter specifying the expected return type (e.g., cr.sendWithPromise<string>('my-handler')), which will cause TypeScript compilation failures if omitted. (7c7425f)
  • ⚠️ BREAKING BREAKING CHANGE: The public gfx::HDRMetadata class was significantly refactored to use Skia-native types. Direct member access is no longer possible; embedders must migrate to new getter/setter methods. (8cb7cd0)

Security & Privacy

  • Experimental support for the post-quantum signature algorithm ML-DSA has been added to WebCrypto, controlled by the WebCryptoPQC runtime flag. (2da14fd)
  • ⚠️ BREAKING BREAKING CHANGE: RenderFrameHostImpl::EnterFullscreen() now strictly enforces the 'fullscreen' Permissions Policy at the browser level, rejecting requests that previously might have succeeded without the policy. (23da98a)
  • The browser has been hardened against compromised renderers by verifying user gesture signals from the browser process for permissions, preventing subframes from sending theme change IPCs, and adding state validation to SubresourceProxyingURLLoader. (89632d8) (c4e43fe) (5ee8fb4)
  • The kLocalNetworkAccessPromptDirectSockets feature is now enabled by default, which will cause users to be prompted for permission when a site requests local network access via the Direct Sockets API. (d96e669)
  • Cookie parsing has been hardened to reject cookies with an empty name and an ambiguous value to mitigate potential cookie-shadowing attacks, controlled by the DisallowEmptyCookieName feature. (6e5ceda)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The chrome.test.runTests() API is now promise-based, changing its signature from synchronous to asynchronous. Existing extension test suites will need to be updated to use await or .then(). (d0c2f20)
  • The browser.* API namespace is now enabled by default for all extensions via the kExtensionBrowserNamespaceAndPolyfillSupport feature, greatly improving cross-browser compatibility with Firefox and other browsers. (6f12cd1)
  • The WebstoreInstaller has been moved from //chrome to //extensions to improve modularity, requiring new interfaces to plumb dependencies from factories in //chrome. (cb457f6) (9ac1886)

Networking

  • ⚠️ BREAKING BREAKING CHANGE: Navigation queueing is now permanently enabled with the removal of the kQueueNavigationsWhileWaitingForCommit feature flag. This represents a fundamental behavioral change in how concurrent navigations are handled. (369b2d9)
  • ⚠️ BREAKING BREAKING CHANGE: Core networking APIs were updated for upcoming multi-network support, including adding a network_restrictions_id parameter to network.mojom.NetworkContext.CreateWebSocket and a target_network parameter to the net::ClientSocketPool::GroupId constructor. (0f1662a) (52a2dea)
  • A new enterprise policy, MaxConnectionsPerProxyForWebSocket, has been added, allowing administrators to control the WebSocket proxy connection limit for more granular control over network resources. (1c530cb)

Browser UI & Features

  • A new Gemini toolbar button was introduced, controlled by the kToolbarGeminiButton feature flag. Implementation and UX fixes were added for both horizontal and vertical tab strips. (00caeeb) (20e8cdc) (262b31e)
  • The Send-Tab-To-Self feature was enhanced to support sending a tab's full back/forward navigation history, not just the current URL. (19645de)
  • The experimental Vertical Tabs feature received numerous refinements, including an 'Expand on Hover' option available in settings and various fixes for animations and focus management. (62ce586) (885030e) (c3f3459) (5d9dd66)
  • Branding was updated from 'GPay' to 'Google Wallet' across various Autofill and payments dialogs and icons. (ce95852) (da0e34b) (019be57) (9b0f418)
