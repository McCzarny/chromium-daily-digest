Week 50 of 2025 (2025-12-07 to 2025-12-13)

Chromium Weekly: 2025 Week 50

Overview

This week saw an exceptionally high volume of breaking changes across Chromium's core APIs, impacting browser lifecycle, UI, networking, and numerous components like Autofill and Identity. A total of 1009 relevant commits out of 1313 were analyzed. Key developments include the enablement of an out-of-process PDF viewer for enhanced security, the introduction of structured cloning in extension messaging, and the removal of the Media Galleries API on Windows and macOS.

Widespread Breaking API Changes

  • ⚠️ BREAKING BREAKING CHANGE: The core browser startup and shutdown sequence was refactored. The Init() and Shutdown() methods on GlobalFeatures and BrowserProcess are replaced with a new multi-phase approach, breaking the lifecycle for embedders and requiring updates to tests using TestingBrowserProcess. (24ba371)
  • ⚠️ BREAKING BREAKING CHANGE: A massive refactoring of the Autofill component removed or changed numerous public methods like AutofillClient::DidFillForm and AutofillManager::GetCachedFormAndField, requiring significant updates for embedders using this component. (0e0c6aa) (658a93b) (6ab0209) (1a8b684) (2d6f99f)
  • ⚠️ BREAKING BREAKING CHANGE: Multiple core utility APIs were modified. Functions in base/system/sys_info.h now return the base::ByteSize class instead of integer types, and deprecated callbacks using std::unique_ptr<std::string> were removed from SimpleURLLoader in favor of std::optional<std::string>. (d310efd) (c7e3442) (558b2e7) (31e0c06)
  • ⚠️ BREAKING BREAKING CHANGE: Several public UI and Content APIs were broken. TabStripModel::selection_model() now returns a TabStripModelSelectionAdapter, ContentBrowserClient::RegisterWebUIControllerInterfaceBrokers has a new signature, and content::RenderProcessHost::FastShutdownIfPossible gained a new parameter. (58e491f) (2b5bd6d) (1095311)
  • ⚠️ BREAKING BREAKING CHANGE: The generic network error net::ERR_DNS_SERVER_FAILED has been removed and replaced by five more specific error codes, requiring embedders to update error handling logic. (0b88079)

Feature Removals and Behavioral Changes

  • ⚠️ BREAKING BREAKING CHANGE: The Media Galleries API (chrome.mediaGalleries) has been restricted to ChromeOS, effectively removing this feature from Windows and macOS and breaking any extensions or applications that relied on it. (2e27c61)
  • ⚠️ BREAKING BREAKING CHANGE: The network service sandbox is no longer enabled by default on Windows, reverting a recent change. This alters the default security posture for embedders on Windows. (e1e1886)
  • ⚠️ BREAKING BREAKING CHANGE: The web-app-manifest-protocol-handlers feature has been removed, disabling the ability for Progressive Web Apps to register as protocol handlers. (5c7e4d6)
  • ⚠️ BREAKING BREAKING CHANGE: The public EnterpriseIdentityService and its factory have been removed. Embedders must migrate away from this service for managed account identification. (5d6bd7b)

Extensions

  • Support for structured cloning has been added to extension messaging APIs like runtime.sendMessage. This allows for richer data types (e.g., Map, Set, Date) to be passed in messages, augmenting the previous JSON-only serialization. (39b5b05) (88e3bae) (69363e6)
  • The declarativeNetRequest API was enhanced to expose RuleConditionKeys, allowing extensions to perform feature detection for supported rule conditions at runtime. (ab42e9f)
  • Reliability of persisted webRequest listeners was improved with fixes for race conditions during extension updates and more efficient synchronous cleanup when an extension is unloaded. (2836073) (d33b324) (022ce91)

Security and Privacy

  • The PDF viewer now runs in an out-of-process iframe (OOPIF) by default on Windows and macOS. This improves security and stability by isolating PDF content from the main browser process. (b7cbb1e)
  • The ForcedDiceMigration feature has been enabled by default, completing the migration to the DICE sign-in architecture and potentially changing sign-in flows for some users. (ea4b1a4)
  • Local Network Access (LNA) now invalidates cached resources if the network configuration changes (e.g., connecting to a VPN), ensuring the browser re-fetches resources to apply correct IP-based permission checks. (a9b4054)

Browser Core and Performance

  • Native support for the Matroska media container format (.mkv, .mka) has been added, allowing these files to be played directly in the browser without extensions. (d081300)
  • A significant performance issue causing UI lag when pasting very large amounts of text into a views::Textfield has been fixed via a new OnBeforePaste hook. (81aebdb)
  • The V-Sync mechanism was improved on both macOS (using the modern CADisplayLink framework) and Windows (better multi-GPU support) to enhance graphics performance and smoothness. (75e50bd) (6187f20) (b8f19a5)
  • The Split View (Side-by-Side) feature is now unconditionally available on supported platforms, with its controlling feature flag removed. (1b9e069) (0fc0517)
