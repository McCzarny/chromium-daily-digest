Chromium Digest: 2026-05-14

Overview

This summary covers 445 relevant commits out of 597 total for the main branch on May 14th, 2026. This digest focuses on changes impacting developers working on Chromium-based projects, with a special emphasis on breaking changes, APIs, security, and browser-level features for Windows and macOS. The day's activity started with commit (5e8bd89) and concluded with (3eec1c7).

Breaking Changes

  • ⚠️The variations::VariationsServiceClient interface has been modified. The methods RemoveGoogleGroupsFromPrefsForDeletedProfiles and RemoveEnterpriseGroupsFromPrefsForDeletedProfiles have been removed, and embedders must now implement the new pure virtual method GetAllProfilesKeys. (07c8aa0)
    APIBrowserEnterprise
  • ⚠️The virtual method URLLoaderThrottle::BeforeWillRedirectRequest signature has changed. It now takes a const net::RedirectInfo& instead of a pointer, and parameters for modifying request headers have been removed. Overriding classes must be updated. (8c41713)
    NetworkAPI
  • ⚠️The mojo interfaces for Service Workers have been updated to prevent a race condition. blink::mojom::EmbeddedWorkerInstanceHost::RequestTermination and blink::mojom::ServiceWorker::AddKeepAlive now require a uint64_t sequence number, breaking integrations that use these interfaces. (d92cc43)
    ServiceWorkerAPI
  • ⚠️The remoting component has been cleaned up. The public classes JitterBuffer and InMemoryFifoBuffer have been deleted. Additionally, the signature of AudioInjector::SetSampleInfo has changed to take a base::OnceCallback<void(bool success)>, requiring updates for custom audio injection implementations. (6ab855d)
    APIMedia
  • ⚠️The ui/views dialog API has been modified. DialogDelegate::set_frame_margins now accepts a FrameMarginsParams struct instead of FrameMargins. Embedders creating custom dialogs must update their code. (651cf21)
    UIAPI
  • ⚠️A new pure virtual method, GetExpectedDisplayTime(), has been added to the public blink::WebVideoFrameSubmitter interface. Embedders with custom implementations must provide this method. (94af377)
    MediaAPI
  • ⚠️The cc::ImageAnimationController::Client nested class has been renamed to cc::ImageAnimationController::Delegate. Implementations of this interface must be updated. (4a9f742)
    CompositorAPI
  • ⚠️In the cc::BrowserControlsOffsetManagerClient interface, the RenderedDeviceScaleFactor() method has been renamed to DeviceScaleFactor(). Implementations must be updated. (c923203)
    CompositorUIAPI
  • ⚠️The BackgroundTracingManager has been moved from content/ to services/tracing/. Embedders using this system must update include paths and call the new factory function content::CreateBackgroundTracingManager. (0be4281)
    APIPerformance
  • ⚠️The build visibility of the third-party XML parser, //third_party/expat:expat, has been restricted. Downstream projects with a direct dependency may see build failures and need to be added to an allowlist. (4eaa1b7)
    Build
  • ⚠️Headers for the PartitionAlloc memory allocator have been split into public and internal sets. This may break builds for embedders who were including headers now considered internal. (7c766eb)
    BaseBuild
  • ⚠️The device::mojom::SensorProvider::GetSensor method signature has changed to include a new SensorConnectionWatcher remote. Clients calling or implementing this mojom interface will need to be updated. (19452b0)
    APISecurity

APIs & Core Interfaces

  • A new aura::Window::GetWeakPtrAsWindow() method was added to simplify lifetime management and encourage using base::WeakPtr over WindowTracker for single windows, helping prevent use-after-free bugs. (d81edbd)
    UIBase
  • A new WindowFeatureController class has been introduced to centralize checks for which window features a Browser instance supports, as part of a larger refactoring effort. (9527174)
    BrowserUI
  • The experimental DeclarativePerformanceObserver feature, which allows websites to record telemetry from navigation start, continues development with the addition of an HTTP header parser and a skeleton PageLoadMetricsObserver. (2d4c09d) (de01842)
    PerformanceAPI
  • The sql::Database transaction methods have been hardened to better handle cases where the database is closed during a transaction, preventing crashes and improving testability. (1ee70dc) (9574a48)
    StorageBase
  • A virtual destructor has been added to MessageMapInterface in ui/gfx/win/window_impl.h to prevent potential resource leaks on Windows when custom implementations are used. (83e1b6f)
    UIWindows

Security

  • To mitigate a Site Isolation bypass, validation for MHTML subframes has been tightened to enforce that they commit with opaque origins, preventing a compromised renderer from spoofing its origin. (e4b92a6)
    SecurityNavigation
  • Missing UUID allowlist checks were added to bluetooth::Adapter's RegisterAdvertisement and CreateLocalGattService methods, preventing compromised sandboxed processes from spoofing BLE advertisements or GATT services. (08e384b)
    BluetoothSecurity
  • The browser will now terminate a renderer process if it supplies invalid keys to IndexedDB, hardening the browser against compromised renderers. (206286e)
    StorageSecurity
  • The Chromium updater has been hardened by improving temporary directory selection and sanitizing manifest paths to block directory traversal attacks. (58d2139)
    UpdaterSecurity
  • The Incognito mode URL blocklist policy implementation was refactored and moved to a more central location at components/policy/core/browser/url_list/. (c9cc012)
    PolicySecurityPrivacy

Browser & UI

  • A new feature allows users to drag a URL or file onto the toolbar area (excluding the omnibox) to open it in the current tab, improving user workflow. (2ce75bc)
    UIBrowser
  • A new surface synchronization mechanism has been introduced for WebUI to align the initial rendering of embedded toolbars with the top-level browser window, controlled by the kBypassOutdatedSurfaceActivation feature flag. (98d27c8)
    UICompositor
  • On macOS, the browser frame's glass effect will now use NSGlassEffectView on newer OS versions to better align with platform UX designs. (06e2d61)
    UImacOS
  • A potential race condition during browser shutdown on macOS has been fixed by ensuring async profile-load callbacks correctly handle cases where a profile is no longer available. (3a9ac96)
    BrowsermacOSProfiles
  • A toast notification will now be displayed to confirm when Chrome has been successfully set as the default browser. (2bdf836)
    UIBrowser

Extensions

  • A recent change that broke PDF printing from extensions has been reverted. The SetUpBeforeUnloadControl() function was found to be necessary for this functionality. (e0194c7)
    ExtensionsPrinting
  • The behavior of the search engine override dialog is now configurable via a Finch flag, allowing for testing of a less intrusive version that appears only once per session. (63bc242)
    ExtensionsBrowser
  • The main extensions menu will now automatically close when a context menu is opened from one of its items, improving UI consistency. (9979b04)
    ExtensionsUI

Build & Testing

  • Several components, including chrome_xr_integration_client and browser_activity_watcher, were refactored to have header-only source_set targets. This allows downstream targets to include their headers without depending on the large //chrome/browser:browser monolith. (aebb18d) (aee6563) (066fd15)
    BuildBrowser
  • Test-only functions like SetAutoAcceptWebAppDialogForTesting have been replaced with modern, RAII-style scoped helpers (ScopedAutoAcceptWebAppDialogs) to prevent state leakage between tests. (43fb711)
    TestingBrowser
  • The build toolchain detection logic was updated to recognize the "Insiders" preview channel for Visual Studio 2026. (1550583)
    BuildWindows
  • The build process for Protobuf's well-known-types is being updated to prefer generating code at build time, helping resolve conflicts between "full" and "lite" protobuf variants. (0360593)
    BuildInfra
