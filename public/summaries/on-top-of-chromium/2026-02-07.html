After filtering for changes relevant to Windows and macOS browser development, this summary highlights key updates in Security, Browser UI, Extensions, Networking, and Web Applications. One major breaking change was identified, disabling Widevine hardware-secure codecs on Windows, which may impact protected media playback for some services.

Security & Privacy

  • ⚠️ BREAKING BREAKING: Widevine hardware-secure decryption (HW_SECURE_ALL) has been disabled by default on Windows. This is a significant behavioral change to the stable Encrypted Media Extensions (EME) API. Services that rely on this specific robustness level for content protection will need to adapt, as requestMediaKeySystemAccess will no longer grant access for it. This change is part of a shift towards using PlayReady SL3000 for hardware DRM on Windows. (ea1b3b8)
  • Several internal security enhancements were made, including removing unsafe buffer usage in the Windows sandbox policy, ensuring the ChildProcessSecurityPolicy's SecurityState is kept alive during asynchronous file system path checks to prevent crashes, and refactoring the setup of Secure DNS preferences. (28ae1cc) (5698ea6) (29918fb)

Browser UI & Features

  • The 'Start Chrome with Windows' feature received several updates, including the wiring up of its opt-in/out infobar, UI updates to the settings toggle button, and new localizable strings. (58bd717) (c7683f5) (3ac9521)
  • The behavior of startup prompts for setting the default browser and pinning to the taskbar has been decoupled. An experiment will now show one or the other, but not a combined prompt, to better evaluate user engagement with each flow. (a8bbe5f)
  • A new 'Back to opener' item has been integrated into the back/forward navigation menu, allowing users to quickly return to the tab that opened the current one. This involved refactoring BackForwardMenuModel to support the new entry. (d5b9c16)
  • The Vertical Tabs feature saw several bug fixes and reversions to address crashes related to tab closing and dragging, while also updating separator padding. A change to render hovercards for vertical tabs was reverted due to a dangling pointer crash. (7629ff8) (956ab52) (7f161fd) (8ecc009)
  • A new WebUI surface chrome://feedback/report-unsafe-site has been added to provide a user-facing dialog for reporting unsafe sites. (2d0a217)
  • On macOS, the main menu bar's 'Tab' menu text will now update dynamically based on whether the user is in Right-to-Left (RTL) mode or has Vertical Tabs enabled. (8a3a16d) (f16cdcb)

Extensions

  • The offscreen.createDocument and offscreen.closeDocument API callbacks are now optional, improving developer ergonomics for contexts that do not use Promises. (dca5915)

Networking & Cache

  • The SQL disk cache backend has been significantly optimized. Changes include implementing lazy entry creation to avoid immediate database writes, deferring the actual insertion until data is written or the entry is closed. This reduces unnecessary database transactions for short-lived cache entries. (3ef364b) (2288d17) (a3b60c0)
  • The prefetching service underwent internal refactoring to clarify dependencies and improve code structure, such as moving the creation of the isolated URLLoaderFactory. (3c44de9) (aa792f6)

Omnibox

  • The internal protocol for configuring the Omnibox AIM (Advanced Intelligence in the Omnibox) feature has been updated. This includes adding new tool modes like kGeminiPro and kAimGenPrompt, and adding support for aim_url_params to ToolConfig and ModelConfig to allow for more dynamic URL generation. (435974d)
  • The SearchProvider will now restrict suggestions to only verbatim matches when in specific experimental tool modes, such as 'typed nano banana' or 'deep search'. (bbf2654)

Web Applications (PWAs)

  • The Web Install API, which powers the experimental <install> HTML element, has updated its UMA/UKM metrics to differentiate between different API versions and entry points. (72cea34)
  • Internal logic for finalizing a PWA installation and handling the PWA update dialog has been refactored for better code structure, moving logic into dedicated jobs and controllers. (788ce5c) (1aabde1)

PDF

  • The 'Save to Drive' feature for PDFs is now enabled on ChromeOS builds and is controllable via the RestrictPdfSaveToGoogleDriveAccountsToPattern enterprise policy. (10dada4)
  • A PDFium roll includes the removal of the internal CFX_FontMgr::NewFixedFace() function. Additionally, a fix was added to prevent a nullptr crash in the Windows PDF font mapper. (6841251) (25f63e8)

Developer Utilities & Internals

  • The ELF reader in base/debug has been updated to support larger BuildIDs, which is necessary for compatibility with newer linkers like wild that use larger hashes. (383ff55)
  • The ObserverLists for WindowTreeHost and FocusController have been explicitly marked to allow re-entrancy, preventing potential issues during complex event notifications like a window being destroyed during an activation change. (0596395) (f009f4d)
