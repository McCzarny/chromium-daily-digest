Chromium Digest: 2026-01-27

Overview

This digest covers 680 commits on the main branch from January 27, 2026, with 535 identified as relevant to developers working on Chromium-based projects. The day saw an unusually high number of breaking API changes across core utilities, extensions, and UI components. Other key areas of focus include performance optimizations and security enhancements. The full range of commits is from (6bc3790) to (37969c6).

APIs and Core Internals

  • ⚠️ BREAKING The widely used base::Value class has been refactored. The nested types base::Value::Dict and base::Value::List have been renamed to base::DictValue and base::ListValue respectively. This is a fundamental change that will require updates in any code that directly uses or casts to the old types. (dddcb58)
  • ⚠️ BREAKING The gfx::BufferFormat enum and related utilities have been removed from the graphics stack. Embedders interacting with graphics buffers must migrate to viz::SharedImageFormat. (abf19e4)
  • ⚠️ BREAKING In the drag-and-drop API, OSExchangeData::GetURLsAndTitles() has been removed. Code must be updated to use GetURLs(), which now returns a struct containing both URL and title information. (b503fa4)
  • ⚠️ BREAKING The ui::WebDialogUIBase class has been merged into ui::WebDialogUI, removing the base class. Embedders inheriting from WebDialogUIBase must update their code to inherit directly from ui::WebDialogUI. (3012c6e)
  • ⚠️ BREAKING base::BindOnce and base::BindRepeating are now marked with [[nodiscard]]. This may cause new compiler warnings or errors in projects that call these functions and discard the result, potentially breaking builds that use -Werror. (6af8098) (dedcec8)
  • ⚠️ BREAKING The public API for BackgroundTaskScheduler now accepts TaskIds enum types directly instead of int. Callers must update their method calls to pass the enum. (2dab51d)
  • ⚠️ BREAKING The virtual method MakeBitstreamOutputStream has been removed from the public AudioManagerBase class. Its creation is now guarded by the ENABLE_PASSTHROUGH_AUDIO_CODECS build flag, effectively removing it from default builds. (17535aa)
  • ⚠️ BREAKING In the PartitionAlloc memory allocator, PartitionRoot has been refactored from a struct to a class and its member variables renamed. This will break any code that directly accesses members of PartitionRoot. (4c9f31d)
  • ⚠️ BREAKING The low-level allocator shim API in base/allocator/partition_allocator/ has been updated. Allocation function signatures now accept an additional size_t alloc_token argument, breaking embedders who use this shim layer directly. (150979b)
  • The ProfileManager now defers asynchronous profile loading until after the initial profile is created to prevent race conditions during startup. (7c0f1c6)

Extensions and Web Apps

  • ⚠️ BREAKING The method BrowserWindow::GetExtensionsContainer() has been removed from the public BrowserWindow interface. Embedders must now use the static method ExtensionsContainer::From(Browser&) to access the extensions container. (6d09036) (b2aefa0)
  • ⚠️ BREAKING The public API for ExtensionsMenuDelegateDesktop has been changed. GetActiveWebContents() was removed, and the OnActiveWebContentsChanged observer method no longer passes a content::WebContents* parameter. (ec3c934)
  • ⚠️ BREAKING The signature for ExtensionTabUtil::GetTabIdForExtensions() was changed from taking a WebContents* pointer to a WebContents& reference, requiring updates for all callers. (9c932e3)
  • ⚠️ BREAKING Public headers for the Split Tab feature were moved from components/tabs/public/ to new locations in components/browser_apis/tab_strip/ and components/split_tabs/, requiring embedders to update include paths. (5ae8298) (d9e6fad)
  • Performance of extensions using content scripts at document_start is improved by compiling the injected JavaScript on a background thread. (101c2fe)
  • The Web App Launch API for file handling was updated to align with spec changes, adding a targetURL attribute to launch parameters. (9e287c6)

Networking and Security

  • ⚠️ BREAKING The PrivateNetworkRequestPolicyOverride enum in content::ContentBrowserClient was renamed to LocalNetworkAccessRequestPolicyOverride. (c335bed)
  • ⚠️ BREAKING The core networking Mojo interface network::mojom::URLLoaderFactoryParams was changed. The process_id field is now a network::OriginatingProcess class instead of a raw int, improving type safety. (ae20224)
  • ⚠️ BREAKING In the public blink::mojom::CommonNavigationParams Mojo struct, the has_user_gesture field has been renamed to has_possibly_filtered_user_gesture to better reflect its behavior. (e340c57)
  • On Windows, a new method SetIsolationSecurityAttribute was added to the RestrictedToken class, allowing a token's default DACL to block access from processes without a specific security attribute. (37969c6)
  • On Windows, the sandbox hooks for Code Integrity Guard (CIG) were refactored to remove the use of IPC, improving security and performance. (46c377e)
  • A new DevTools Protocol domain, SmartCardEmulation, was added to allow emulation of the PC/SC backend for testing the Web Smart Card API. (6066af9)
  • Auto-revocation of permissions for "spammy" notifications is now enabled by default. (a269e9e)

Browser UI and Features

  • ⚠️ BREAKING APIs in BrowserView and ToolbarView have been refactored to make the distinction between LocationBar and LocationBarView more explicit. This changes method signatures for accessing the location bar and will break embedder code that interacts with the omnibox container. (35c8239)
  • ⚠️ BREAKING The chrome://user-actions WebUI page and its handler have been moved from chrome/browser/ui/webui/ to components/webui/user_actions/. This change of file paths will break builds that directly include the old headers. (69880ac)
  • The browser now handles google-chrome:// scheme navigations internally, improving robustness and performance by bypassing OS-level protocol handlers. (e440cf0)
  • On macOS, the maximum size for an application's Info.plist data was increased, fixing installation failures for PWAs with large icon sets. (89eb883)
  • The feature to display a user-actionable error when the bookmark sync limit is exceeded (kSyncShowBookmarksLimitExceededError) has been enabled by default. (41a8a5e)

Autofill

  • ⚠️ BREAKING ChromeAutofillClient no longer inherits from content::WebContentsObserver. Code that relied on this inheritance for casting or observing WebContentsObserver events directly from a ChromeAutofillClient instance will break. (ce9fc87)
  • ⚠️ BREAKING The public method FormFieldData::IsFocusable() in the Autofill API has been renamed to is_focusable(), affecting any code that inspects form field properties. (321d1a0)

Performance

  • A new PriorityBoostLoadingBrowserNetworkPolicy was implemented to improve startup performance. It disables process priority boosting on startup and then dynamically enables it for the network process and renderers only when a page is actively loading. (23517c6)
  • Two features supporting renderer process reuse (kTrackEmptyRendererProcessesForReuse and kProcessPerSiteUpToMainFrameThreshold) have been enabled by default to improve performance and memory usage. (6e70d31)
  • To improve startup performance on Windows, the task runner for BatteryLevelProviderWin was changed to BEST_EFFORT priority, preventing it from loading certain DLLs during the critical startup path. (054bd0b)

Build and Test Infrastructure

  • ⚠️ BREAKING A major file renaming effort was undertaken to standardize test file suffixes from _unittests.cc and _browsertests.cc to _unittest.cc and _browsertest.cc, respectively. This will break custom build configurations or scripts that explicitly reference the old filenames. (29c9e32)
  • The Sync browser test framework was updated to use a real EmbeddedTestServer with a FakeServer backend, enabling more comprehensive end-to-end testing of the network stack. (0304860) (0b52517)
  • On Windows, the installer's logic for moving and deleting directories was made more robust by using a new FileConductor to perform pseudo-transactional operations, increasing resilience against file-locking issues. (cde7f49)
