Chromium Digest: 2026-03-12

Overview

This summary covers 489 relevant commits out of a total of 692 on the main branch for March 12, 2026. The changes span from commit (156499f) to (115d9a2). This digest highlights multiple breaking changes to core networking, UI, and service APIs, alongside significant behavioral changes in extension messaging and the Viz display service. Developers building on Chromium should review the breaking changes section carefully.

Breaking Changes

  • ⚠️ BREAKING The return type for HttpStream::GetTotalReceivedBytes() and GetTotalSentBytes() has been changed from int64_t to net::ByteSize. This API signature change in net/http/http_stream.h will cause build failures for any code calling these methods. (5895f46)
  • ⚠️ BREAKING The signature for net::ClientSocketPool::PreconnectCompletionCallback has changed to include a std::unique_ptr<ClientSocketHandle> as its second argument. Embedders using this networking callback must update their implementations. (88f44d9)
  • ⚠️ BREAKING A revert has removed the MutableNetworkTrafficAnnotationTag parameter from several core networking functions, including ClientSocketPool::RequestSocket and ClientSocketHandle::Init. This is a signature change to stable, public networking APIs that will break builds. (5ca41b4)
  • ⚠️ BREAKING The macOS-specific NativeFrameViewMacClient class has been removed. Embedders on macOS must migrate from this class to the new generic non_client_hit_test_callback_ on views::FrameView to handle draggable regions in frameless windows. (5a2ee1d)
  • ⚠️ BREAKING Extension messaging will now throw an error when a SharedArrayBuffer is included in a message, where it previously failed silently. This is a significant behavioral change that will break extensions relying on the old behavior. (7c8c8fb)
  • ⚠️ BREAKING The public API for getting the bookmark clipboard format, bookmarks::BookmarkNodeData::GetBookmarkFormatType(), has been removed. Consumers must now use ui::ClipboardFormatType::BookmarkEntriesType() for clipboard and drag-and-drop operations. (dfec277)
  • ⚠️ BREAKING The type field in the public sessions::tab_restore::Window struct has been renamed to window_type. Code accessing this field from the tab restore service will fail to compile. (4b2975c)
  • ⚠️ BREAKING The instance_ids member has been removed from the public apps::LaunchResult struct. Code inspecting this struct after an app launch will need to be updated. (42535d5)
  • ⚠️ BREAKING The Viz display service's LayerContextImpl::DoUpdateDisplayTree method no longer returns an error for invalid browser controls parameters (e.g., negative heights). It now silently clamps the values. Embedders relying on the error for state validation will experience a behavioral break. (d22dd57)
  • ⚠️ BREAKING Several constants have been removed from public headers, which will cause build failures if referenced. This includes the kSyncBookmarksBatchUploadSelectedItems switch, the kWebPermissionAPI feature flag, and the kNewFileSystemInfoCollector and kNewWinSignalsCollector enterprise feature flags. (bc0f0d2) (b62aea4) (83b1fa8)

Networking & Security

  • The ServiceWorkerAutoPreload feature is now enabled by default. This performance optimization may alter resource loading behavior for web applications that rely on Service Workers. (d500868)
  • Security has been hardened by adding a browser-side check to enforce the allow-modals sandbox attribute. This prevents a compromised renderer from showing alert(), confirm(), or prompt() dialogs if the sandboxed frame lacks the permission. (7e25766)
  • A new helper function, net::IPAddress::IsMulticast(), has been added to the networking library to check for multicast IP ranges. (5749767)
  • The network stack now includes a MIME type mapping for markdown (.md) files to text/markdown. (ade6ca0)
  • A race condition in TcpConnectJob has been fixed, improving networking stability by blocking progress while the job may be pending asynchronous destruction. (a1d8d86)
  • Policies related to "Chrome Frame," a technology deprecated for over a decade, have been completely removed from the codebase. (0fa4bed)

APIs & Extensions

  • A new media pipeline error code, PIPELINE_ERROR_HARDWARE_CONTEXT_RESET, has been introduced. This provides more specific feedback for unrecoverable hardware-related playback failures when a hardware context reset fails to recover. (bbfb0b5)
  • The PageActionObserver interface has been extended with new methods (OnAnchoredMessageShown, OnAnchoredMessageHidden) to allow observation of anchored message visibility changes. (5d8fda4)
  • The browser is now more resilient to messages from stale extension service workers. Instead of crashing, the browser will now drop IPC messages from outdated worker instances. (de506e6) (10acabd)
  • A new "Back-To-Ad" history intervention has been implemented behind a feature flag. When enabled, it modifies back/forward navigation to skip entries that are likely ad-related. (61ed18a)
  • The FormFieldData struct used by Autofill was updated with a new selected_option_text_ member. This allows Autofill to more accurately fill <select> elements by matching on the option's visible text in addition to its value. (95b08b3)

Browser & UI

  • The VerticalTabStripScrollBar has been refactored into a more generic views::RoundedScrollBar for broader use across the UI toolkit. (bb86127)
  • A bug on macOS that caused right-clicking to deselect all text in a text field has been fixed. (2c0f959)
  • The Side Panel's core UI class, SidePanelUIBase, was refactored to depend on abstract interfaces like BrowserWindowInterface instead of concrete classes, improving modularity. (f1b9f68) (bb785b0)
  • A potential dangling pointer in the global accelerator listener for macOS was fixed, preventing crashes when an accelerator is pressed after its listener has been destroyed. (9f60e90)

Testing & Infrastructure

  • As part of a broad modernization effort, base/logging.cc was updated to use std::span, improving type safety. Numerous unit tests were also updated to use modern utilities like base::test::TestFuture for cleaner asynchronous testing. (ae4a06c) (1693de3)
  • Ongoing code modularization efforts continued, with components like chrome/browser/site_isolation/, chrome/browser/status_icons/, and the undo manager being moved into their own build targets. (9c845fc) (c132b3d) (3401c7c)
  • To resolve build timeouts on macOS, vectorizer passes have been temporarily disabled during the Mach-O ThinLTO link step. (f4d0bed)
  • A large number of internal API tests for extensions, including permissions, declarativeNetRequest, nativeMessaging, and omnibox, were updated from Manifest V2 to Manifest V3. (be9c464) (b2f7a35) (868a121)
