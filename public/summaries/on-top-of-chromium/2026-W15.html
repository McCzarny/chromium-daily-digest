Week 15 of 2026 (2026-04-05 to 2026-04-11)

Chromium Weekly: 2026 Week 15

Overview

A brief overview paragraph (2-4 sentences). Mention: 2327 relevant commits out of 3009 total across 7 days. This week was dominated by a massive API refactoring effort known as 'Project Bedrock', introducing numerous breaking changes by replacing Browser pointers with a new BrowserWindowInterface. Other major themes include the removal of third-party cookie deprecation APIs, significant security hardening against Use-After-Free vulnerabilities, and the introduction of a new configuration system for Chrome for Testing.

Major Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE The 'Project Bedrock' refactoring replaced raw Browser* pointers with the narrower BrowserWindowInterface* in many public APIs. Widely used functions like chrome::FindBrowserWithWindow and chrome::GetBrowserCount were removed in favor of a new GlobalBrowserCollection singleton, requiring significant updates for embedders interacting with browser instances. (78da6a2) (d9ac938) (ebcb57e) (2a4a86d)
  • ⚠️ BREAKING BREAKING CHANGE As part of the Third-Party Cookie Deprecation (3PCD) cleanup, core APIs and features were removed. This includes the deletion of TpcdMetadataManager, the public method net::CookieSettingsBase::IsExemptFromThirdPartyCookieBlocking, and the kTpcdMetadata... feature flags. (b7f05fa) (c9934fd) (708fe41)
  • ⚠️ BREAKING BREAKING CHANGE Several core interfaces now require implementation updates. A new pure virtual method HasSavedFrame() was added to content::RenderWidgetHostView, and NotifySwappedRWHVChildFrameFromRenderManager was added to content::RenderFrameHostManager::Delegate. Additionally, base::ObserverList now disallows re-entrant iteration by default, which may cause runtime failures in existing code. (1e42bd2) (b83f5c5) (83dc1d0)
  • ⚠️ BREAKING BREAKING CHANGE The extension downloader will no longer follow HTTP redirects when fetching CRX files, a major behavioral change that will break update servers relying on redirects. Separately, the extensions::RuntimeAPIDelegate::OpenOptionsPage method is now asynchronous, requiring a signature change from returning bool to taking a callback. (04fad69) (011e693)
  • ⚠️ BREAKING BREAKING CHANGE Core networking and graphics APIs were modified. Methods in net::HttpTransaction and net::URLRequestJob that return byte counts now use the type-safe net::ByteSize class instead of int64_t. The public API for creating GPU contexts, viz::ContextProviderCommandBuffer::CreateForRaster, also had its signature changed. (79bbb97) (72bf5dc) (b7eadee)

Security

  • Widespread Use-After-Free (UAF) vulnerabilities were fixed across the codebase. Patches were applied to Service Workers, WebSockets, Password Forms, WebContents, and Chrome Remoting, primarily by improving lifetime management with WeakPtr and safer callback patterns. (d17fe11) (5d8f582) (c542194) (899e77f) (174583c)
  • The browser was hardened against compromised renderers and network attacks. This includes validating TextureQuad data during Viz deserialization, enforcing a same-origin check for main script fetches in Dedicated Workers, and blocking forbidden HTTP methods (TRACE, TRACK) for no-cors requests. (acafd83) (7d3c950) (d513cd2)
  • The WebCrypto API now supports post-quantum key encapsulation mechanisms ML-KEM-768 and ML-KEM-1024, preparing for future cryptographic standards. (b582009)

Browser, UI & APIs

  • A new TabStrip API is being introduced for programmatic tab strip updates. In parallel, the TabGroup class in components/tabs now has an observer pattern, allowing clients to subscribe directly to group and visual data changes. (ea91004) (4a588dd) (9236242)
  • The BrowserContext::UniqueId() method is now deprecated. A new, more secure BrowserContext::UniqueToken() method returning a base::UnguessableToken should be used instead to uniquely identify a profile. (4f81ffc) (f2112b5)
  • The views::WebView class now includes a SetOwnedWebContents() method, allowing it to take ownership of an externally created WebContents instance, simplifying lifetime management for embedders. (a203cc3)
  • The "Send to your devices" feature has been improved. The context menu option is now a dynamic submenu that directly lists available target devices, and a new feature allows tabs received via this mechanism to be opened automatically on desktop. (1399633) (a77fda5)

Networking

  • Detailed DNS resolution information is now propagated through the network stack. The net::ResolutionDetails struct was expanded to include data like the source of the result (e.g., cache, DoH, system resolver), which is available in LoadTimingInternalInfo for performance analysis. (2e629dc) (e3c1d6a)
  • ⚠️ BREAKING BREAKING CHANGE The factory method PrePrefetchService::Create in content/public/browser/pre_prefetch_service.h was updated to support pre-calculating request headers on the UI thread, changing its signature and requiring embedders to update their calls. (508c306)
  • The experimental ServiceWorkerAutoPreload feature for subresources has been removed. This includes the removal of the kAutoPreload enum value from the public blink.mojom.ServiceWorkerFetchHandlerBypassOption interface. (2bb2c50)

Testing & Infrastructure

  • Chrome for Testing (CfT) now supports a runtime configuration JSON file, specified via the --chrome-for-testing-config switch. The initial implementation adds an option to enable a virtual clipboard for automated tests. (f908610) (bca0df9)
  • A new InteractionTestUtilSimulatorWebUI was implemented, allowing automated tests to interact with tracked WebUI elements using the same APIs as for Views and WebContents, improving the testability of WebUI components. (3609c24)
  • On Windows, a large effort to remove dependencies on the legacy Active Template Library (ATL) continues, with components like UI automation, the updater, and content_main being migrated to modern WRL and base library equivalents. (c434495) (fc1e63c) (cc9e302)
