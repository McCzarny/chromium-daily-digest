This digest covers 419 relevant commits out of 589 total for the main branch on December 1st, 2025, from commit (377f5e8)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((377f5e8)) to (cc49735)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((cc49735)). The day was marked by several significant breaking changes to core public APIs in base, content, and url, requiring immediate attention from embedders. Other notable changes include the removal of the experimental IP Protection feature, UI updates to tab reloading, and API modernization efforts.

Major API & Breaking Changes

  • ⚠️ BREAKING BREAKING: The public content::ContentBrowserClient interface has changed. The virtual method GetIpProtectionProxyBypassPolicy() was removed, and IsInitialWebUIScheme was renamed to IsInitialWebUIURL. Embedders overriding these methods must update their implementations to avoid build failures. (a402ef2) (f9d507d)
  • ⚠️ BREAKING BREAKING: Several public APIs in the base library were changed. A char* overload of base::File::ReadNoBestEffort was removed in favor of a base::span version. Additionally, base::LockFreeBloomFilter was converted to a template class, and the kDisableBRP trait was removed from the public base::RawPtrTraits enum. These changes will require code updates for any project using these APIs. (9cdc48c) (b05b958) (344b8ff)
  • ⚠️ BREAKING BREAKING: The public class url::StringViewReplacements has been removed from url/gurl.h. Its functionality was merged into the url::Replacements class, and any code directly using the removed class will no longer compile. (d170193)
  • ⚠️ BREAKING BREAKING: Public API signatures for MIDI handling have been updated. Methods in MidiHost and ChildProcessSecurityPolicyImpl now require a strongly-typed content::ChildProcessId instead of a primitive int, which will break existing calls to these interfaces. (cee62ba)

Security & Privacy

  • The experimental IP Protection (IPP) feature has been largely removed from //chrome/browser and //services/network in preparation for its complete deletion. This includes the removal of the chrome/browser/ip_protection directory and related feature flags. (a402ef2) (41bfe9a)
  • The Identifiability Study (Privacy Budget) feature has been removed. This cleanup removes the experimental feature's mojo interfaces, metrics providers, and core implementation, reducing codebase complexity. (db00ab2) (0d71dc3) (52c1810)
  • App-bound tracked preference encryption (kEnableEncryptedTrackedPrefOnEnterprise) is now enabled by default for Windows enterprise devices, enhancing security for managed browsers. (a6d5573)
  • A new error enum, PERMISSION_DENIED_BY_SYSTEM, has been added to media_stream.mojom. This allows for more granular error reporting when media capture is blocked by system-level policies like Data Loss Prevention (DLP). (36de4cd)

Browser UI & User Experience

  • The behavior for reloading tabs has been updated. When multiple tabs are selected, pressing the reload button or Ctrl+R will now reload all selected tabs, restoring a previously available functionality. (368c4c2)
  • A new "Account and network protection" section has been added to the Security settings page (chrome://settings/security). (6164dce)
  • As part of the ongoing Chrome visual refresh, a new layout for popup and DevTools browser windows is being introduced behind the kChromeRefresh2023NTB feature flag. (28589c6)

Developer Experience & Sign-in

  • Direct access to AccountInfo::hosted_domain is now deprecated and will be made private. Developers must update their code to use the GetHostedDomain() accessor method to prevent future build failures. (0cc5bbc)
  • The OneTimeTokenRetrievalError enum has been moved from one_time_token_service.h to its own header. Builds including the old header to access this enum will fail and must be updated to include one_time_token_retrieval_error.h. (14191ba)
  • The ScopedAnimationDurationScaleMode testing utility was moved from ui/compositor to ui/gfx. This is a breaking change for downstream test suites, which will need to update their include paths. (f4ebc25) (419e322)
  • The callback signature for SimpleURLLoader is being updated from std::unique_ptr<std::string> to std::optional<std::string> across the codebase, aligning with modern C++ practices. (5711954)
