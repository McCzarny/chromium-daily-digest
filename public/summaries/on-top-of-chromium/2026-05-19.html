Chromium Digest: 2026-05-19

Overview

This digest covers 704 commits on May 19, 2026, with 530 commits filtered as relevant to developers. The changes span from the first commit 57f11432f6cb94138958a5923a980f0d68009f8f to the last d96a1c977af36e723288493f119c68e6672d1914. This summary highlights a significant number of breaking API changes affecting media, UI, networking, and extensions. Developers should pay close attention to signature changes in the core AudioManager, the removal of the FullscreenObserver pattern, and updates to the winhttp and accessibility APIs. Other notable changes include security hardening on Windows and macOS, and the removal of policy support in the headless shell.

Browser & Core APIs

  • ⚠️BREAKING: The public content::FrameConnector interface has two new pure virtual methods, SetKeepSurfaceAlive and IsKeepingAlive. Embedders who implement this interface must now provide implementations for these new methods to support capturing background tabs. (9675bd8)
    BrowserUI
  • ⚠️BREAKING: Support for applying Chrome policies in the headless shell has been completely removed. The headless_use_policy GN flag and all associated logic are gone, which will affect embedders who used policies for configuration. (fcdf7e8)
    BrowserPolicy
  • ⚠️BREAKING: The last_active_time_ticks and last_active_elapsed_text properties have been removed from the tabs_api.mojom.Tab Mojo interface. Clients of the TabStrip API using these fields will need to be updated. (5f89e9c)
    BrowserTabsUI
  • The winhttp::NetworkFetcher was refactored to enforce a strict single-use contract to prevent Use-After-Free vulnerabilities. Reusing an instance will now trigger a CHECK failure. (98cf28d)
    NetworkWindowsSecurity
  • A significant build system refactoring was performed on //chrome/browser to break up the monolithic :browser target into smaller, header-only source sets, improving dependency management. (a3e448d)
    BuildBrowser

UI & Accessibility

  • ⚠️BREAKING: The FullscreenObserver class and its associated AddObserver/RemoveObserver methods have been removed from FullscreenController. Consumers must migrate to the new RegisterOnFullscreenStateChanged method, which uses a base::CallbackListSubscription pattern. (00d519b)
    UIBrowser
  • ⚠️BREAKING: The ActionFlags enum and the flags field have been removed from the public ui::AXActionData structure. This is an API-breaking change for embedders using Chromium's accessibility interfaces. (da04fe8)
    AccessibilityUI
  • A new, safer API for creating bubble widgets, views::BubbleDialogDelegateView::CreateBubble, has been introduced. It returns a std::unique_ptr<Widget> to improve memory management and clarify ownership. (0b4629d)
    UI
  • A new method, SetShouldAllowKeyEventsDuringInputProtection, was added to ui::DialogModel::Builder to allow dialogs to block key events like Enter and Esc during the initial input protection period, preventing keyjacking. (e64e0f3)
    UISecurity

Media

  • ⚠️BREAKING: The signatures for AudioManager::GetAudioInputDeviceNames() and GetAudioOutputDeviceNames() have been changed to return a bool indicating success instead of void. This is a major breaking change for any embedder that implements a custom AudioManager or calls these methods directly. (d1b707c)
    Media
  • ⚠️BREAKING: The constructor for media::AudioDiscardHelper has been modified, removing the delayed_discard boolean parameter. This will break custom audio decoders or media pipeline components that instantiate this class. (9877688) (e7794c6)
    Media
  • A security vulnerability in the Media Foundation CDM on Windows was patched to prevent a site isolation bypass by correctly migrating permissions on existing CDM store roots. (4e5395b)
    MediaSecurityWindows

Extensions

  • ⚠️BREAKING: A legacy compatibility layer for upgrading New Tab Page extension permissions has been removed. Extensions relying on the old, automatic upgrade path will now be disabled upon update, requiring developers to update their manifest and installation flow. (81cc594)
    Extensions
  • The default Content Security Policy (CSP) for MV3+ extensions is now also applied to extensions of type kUserScript for greater consistency and security. (6de2ae4)
    ExtensionsSecurity

Networking

  • ⚠️BREAKING: The winhttp::NetworkFetcher completion callback on transport-level failures now receives the exact HRESULT error code instead of an HTTP 200 status. Consumers must update their error handling logic to interpret HRESULT values. (2720391)
    NetworkWindows
  • The stable feature flag kForceSecureDnsDohFallback has been removed. Configurations or embedders relying on this flag to force DNS-over-HTTPS fallback will see a behavioral change. (d495b25)
    NetworkPrivacy
  • Search engines can now opt-in to receiving the X-Geo header by setting a new send_x_geo_header field in their prepopulated engine data, allowing for location-aware search results. (a197dda)
    NetworkBrowser

Security

  • On Windows, PWA installation now sanitizes app names that are reserved device names (e.g., PRN, COM1) to prevent potential denial-of-service or unauthorized device I/O vulnerabilities. (1981552)
    SecurityWindowsBrowser
  • PartitionAlloc hardening, including MiraclePtr, is now correctly enabled in the macOS app-shim process to mitigate potential exploits from untrusted renderer IPCs. (68319a1)
    SecuritymacOSBase
  • A use-after-free vulnerability in LegacyRenderWidgetHostHWND on Windows has been fixed by adding guards against re-entrant calls that could destroy the object while it was still in use. (c50e173)
    SecurityWindowsUI
  • A dangling pointer crash in DevToolsRemoteServerInfobarDelegate has been resolved by fetching the active browser on-demand instead of caching a raw_ptr that could become invalid. (47edc8f)
    SecurityDevToolsBrowser

Testing & Infrastructure

  • A new MojoBrowserProxyFactory class has been added to ui/webui/resources/mojo to simplify the creation of test browser proxies for Mojo-based WebUI, reducing boilerplate code for feature authors. (f500fe8)
    TestingUI
  • The --ignore-gpu-blocklist command-line switch was extended to accept a comma-separated list of entry IDs, allowing for more granular debugging of specific driver workarounds. (6f26990)
    GPUTesting
