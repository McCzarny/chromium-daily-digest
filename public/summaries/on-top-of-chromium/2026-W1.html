Week 1 of 2026 (2026-01-01 to 2026-01-03)

Chromium Weekly: 2026 Week 1

Overview

This week was marked by a significant number of breaking changes across several core APIs, impacting security, privacy, and the UI framework. Out of 122 commits across 3 days, 12 were highlighted for their direct impact on embedders and extension developers. Key updates include the removal of the aura::WindowEventDispatcherObserver, multiple breaking changes in security-related services, and enhancements to the Extensions platform.

Security & Privacy

  • ⚠️ BREAKING BREAKING CHANGE: Several service interfaces received breaking changes, requiring updates in downstream implementations. affiliations::AffiliationService now requires an AsWeakPtr() method, EnclaveManagerInterface methods were renamed to PascalCase, and the privacy_sandbox::MaybeSetRollbackPrefsModeB function signature was simplified. (2b5dedb) (36fb252) (1bcd25c)
  • ⚠️ BREAKING BREAKING CHANGE: The safe_browsing::PingManager has removed support for hit reporting. The public method ReportSafeBrowsingHit and the HitReport struct are now deleted, affecting embedders who reported when Safe Browsing warnings were shown. (bb63549)
  • Security was further hardened by resolving a memory safety issue in the Windows USB device handler and refining Safe Browsing's client-side detection logic to improve metrics and performance. (d5d92a2) (137a3f7) (6f1b149)

UI & APIs

  • ⚠️ BREAKING BREAKING CHANGE: The aura::WindowEventDispatcherObserver interface and its registration methods on aura::Env have been removed. Projects using this public API to monitor window events must migrate to ui::PlatformEventObserver. (47e1cd3)

Extensions

  • The Extensions platform has been modernized with the introduction of Promise support for the runtime.onMessageExternal event, allowing listeners to return a Promise for asynchronous responses. (3061069)
  • An experimental feature flag, kContentVerifierCacheIncludesExtensionRoot, was added to test a potential fix for extension corruption caused by hash mismatches. (dd0f836) (621959d)

Networking

  • A bug in Local Network Access (LNA) was fixed, restoring the ability for fetch() calls within shared workers in Chrome Apps to access private network resources. (f061a97)

Testing

  • ⚠️ BREAKING BREAKING CHANGE: In test utilities, the pure virtual method PlatformSupportsLowEnergy() was removed from the device::BluetoothTestBase class. Downstream test classes must remove their implementations of this method. (b64652f)
