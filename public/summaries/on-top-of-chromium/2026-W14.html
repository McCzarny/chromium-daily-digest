Week 14 of 2026 (2026-03-29 to 2026-04-04)

Chromium Weekly: 2026 Week 14

Overview

A busy week in Chromium development saw 1334 relevant commits out of 1793 total. The period was marked by an exceptionally high number of breaking API changes, driven largely by the ongoing "Bedrock" UI refactoring project which aims to decouple core browser logic from UI implementations. Other major themes include significant security hardening across the navigation and media pipelines, the graduation of the SharedWorker API to a stable feature, and continued modularization of the browser's build system.

Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE The "Bedrock" UI refactoring project has altered numerous core browser functions. APIs like chrome::NewTab, chrome::CloseWindow, and chrome::OpenEmptyWindow now accept or return a BrowserWindowInterface* instead of a Browser*. Embedders directly calling these functions must update to use the new abstract interface. (6fdf39d) (d3e3cfb) (4357394)
  • ⚠️ BREAKING BREAKING CHANGE The base::Thread constructor has been overhauled, removing the simple Thread(const std::string& name) version. Embedders must now use new signatures that explicitly pass a delegate or a Restartable tag, breaking all existing thread creation sites. (7d91034)
  • ⚠️ BREAKING BREAKING CHANGE The content::ContentBrowserClient interface has been trimmed. The public virtual methods GrantCookieAccessDueToHeuristic and UpdateEnabledBlinkRuntimeFeaturesInIsolatedWorker have been removed, requiring embedders to delete their overrides. (ac62164) (b01d62a)
  • ⚠️ BREAKING BREAKING CHANGE The public permission.mojom interface was updated. The user_gesture parameter was removed from RequestPermission, and several methods now return a more detailed PermissionStatusWithDetails struct instead of a simple status, breaking Mojo clients. (5168b89) (7052b70)
  • ⚠️ BREAKING BREAKING CHANGE The signin::IdentityManagerProvider class has been removed. Embedders must now use IdentityManagerFactory directly to access the IdentityManager. Additionally, many public OAuth2 scope constants were made private, breaking sign-in integrations. (4db6a2e) (8b1ec69) (4fd8668)
  • ⚠️ BREAKING BREAKING CHANGE The global ServiceProcessHost::Observer interface has been removed. Code must now use the per-instance ServiceProcessHost::Options::WithObserver() pattern to monitor service process events. (ca7b044)
  • ⚠️ BREAKING BREAKING CHANGE The ui::Widget class now enforces a CHECK to prevent its own destruction from within its callback stack. Code relying on this incorrect synchronous destruction pattern will now crash and must be updated to defer destruction. (6b98320)
  • ⚠️ BREAKING BREAKING CHANGE For security, the content::MediaSessionPlayerObserver::OnResume signature now includes a bool triggered_by_user parameter. All implementations of this observer must be updated. (75237fb)

Security & Privacy

  • Core navigation logic was significantly hardened by upgrading numerous DCHECK assertions to CHECKs in NavigationRequest and RenderFrameHostManager. This ensures that internal state inconsistencies will cause a safe crash in release builds, preventing potential vulnerabilities. (662ea60) (9acb98e) (bf94625)
  • ⚠️ BREAKING The media pipeline was hardened by rejecting EncryptionScheme::kUnencrypted during Mojo deserialization of media::DecryptConfig. This behavioral change prevents a compromised renderer from bypassing subsample validation. (878ee24)
  • The File System Access API on Windows now blocks write actions to paths matching */.git/hooks, mitigating a potential remote code execution vector. (fb916b2) (5ada3cd)
  • A critical vulnerability in the chrome.debugger API was fixed, which previously allowed an extension to bypass web_accessible_resources checks and navigate subframes to private extension pages. (5baca06)
  • The Disruptive Notification Revocation feature, part of Safety Hub, is now enabled by default on desktop, allowing the browser to automatically revoke permissions from sites sending disruptive notifications. (963d707)

Browser UI & Features

  • Significant progress was made on a new WebUI-based toolbar, with new Mojo APIs added for managing action states and element identifiers. This is part of a long-term effort to replace the native Views-based toolbar. (fcb45c1) (5b3d5f4) (26d42e6)
  • On macOS, mouse movement event coalescing has been disabled to allow for a higher polling rate. This change provides more responsive mouse input, which is particularly beneficial for games and other high-precision applications. (80792e0)
  • A major regression on Windows was fixed where opening URLs from external applications would not respect the current workspace, instead always opening in the last active browser window across all workspaces. (0dfe151)
  • The experimental Vertical Tabs feature received numerous bug fixes and improvements, including restoring the ability to rapidly close consecutive tabs and fixing drag-and-drop lag on macOS. (4d470ec) (5c294f0) (276d08e)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE The WebAuthn API for extensions now enforces stricter validation. Host permissions must now be for the specific origin matching the Relying Party (RP) ID, which may break extensions that relied on the previous, more lenient subdomain matching. (ecf43dd)
  • The chrome.test.assertEq API has been updated to use Object.is() for primitive equality checks instead of ===. This is a behavioral change that may require updates to tests, as Object.is(-0, 0) is false. (94def1d)
  • Several core extension components, including CWSInfoService and webstore_data_fetcher, have been moved from //chrome/browser/extensions to the more generic //extensions/browser, continuing the effort to modularize the extensions system. (d57d493) (e029d19)

APIs & Core Internals

  • The SharedWorker API is now enabled by default on all platforms, graduating from its experimental status to become a stable web platform feature. (6f4c113)
  • The major effort to break down the monolithic //chrome/browser build target continued, with components like offline_pages, plugins, printing, and spellchecker being moved into their own build files to improve dependency management and parallelism. (b87be23) (dc7f841) (0790c8f) (0234876)
  • For developer builds integrating Rust, a Rust panic! will now trigger a LOG(FATAL) in C++, providing more detailed crash reports that include C++, Java, and task runner stack traces. (89012c0)
