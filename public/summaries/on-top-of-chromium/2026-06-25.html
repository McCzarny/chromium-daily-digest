Chromium Digest: 2026-06-25

Overview

Across 740 commits, 555 were identified as relevant to desktop embedders on Windows and macOS, ranging from (845cd0e) to (86833e1). This summary highlights architectural refactors, API updates, and security hardening relevant to Chromium-based applications.

API & Interface Changes

  • ⚠️The PasswordManager internal interfaces CreateFormManager and ProcessAutofillPredictions have been transitioned from pointers to references to enforce non-null safety, requiring updates to call sites. (39c5428)
    PasswordManagerBrowser
  • ⚠️PasswordManagerClient::IsSavingAndFillingEnabled now requires a url::Origin object instead of a GURL to improve security handling for opaque origins. (64e2f47)
    PasswordManagerSecurity
  • ⚠️The BrowserInfoBarManager::Show method signature was altered to require an explicit WebContents parameter, breaking existing implicit tab resolution logic. (d6e5f29)
    BrowserUI
  • ⚠️Language tag utilities were relocated to the base::i18n namespace. Projects must update code references to avoid compilation failures. (34246c2)
    BaseI18n

Security & Networking

  • Hardened macOS sandbox profiles for the network process and on-device model execution process by restricting access to user cache/temp directories. (39a2dda) (1883a1d)
    SecuritymacOSNetwork
  • Fixed a security flaw in PasteAllowedRequest by keying cache requests to specific RenderFrameHost instances to prevent cross-frame privilege escalation. (4e8515b)
    SecurityBrowser
  • Enabled FedCmMultipleRequests by default, activating the new FederatedRequestService and Mojo-based architecture. (9728c52)
    BrowserSecurity

UI & Component Architecture

  • Enhanced Document Picture-in-Picture window parity on macOS, including support for rounded corners and improved bounds/resize calculations. (b41d84d) (25f0da2)
    UImacOS
  • Modularized chrome/test and chrome/browser/ui to reduce monolithic compilation overhead, which may require BUILD.gn adjustments for some embedders. (9fe8da5) (c4ce96a)
    BuildUITesting
  • Fixed a critical browser crash on Windows by migrating services/webnn/host to a component build, ensuring singletons are correctly shared across DLL boundaries. (ea7c9fb)
    WindowsBuild
