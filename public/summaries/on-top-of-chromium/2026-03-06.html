Chromium Digest: 2026-03-06

Overview

This summary covers 565 relevant commits out of 741 total commits on the main branch for March 6th, 2026. The day saw a large number of breaking changes across core APIs, UI components, and testing infrastructure, requiring significant updates for embedders. Notable improvements include the ability for WebSockets to use HTTP/3 connections and the introduction of a Rust-based security policy component. The first relevant commit was (2b3bc52) and the last was (485e8be).

Core APIs & Libraries

  • ⚠️ BREAKING BREAKING: The raw pointer overloads for base::File::Read and base::File::Write have been made private. Code must be updated to use the base::span overloads for file I/O operations. (2f6529b)
  • ⚠️ BREAKING BREAKING: The callback for the public FedCM API content::webid::FederatedEmbedderLoginRequest::Set has been changed from a base::RepeatingCallback to a base::OnceCallback. Callers must be updated as the continuation state is now considered terminal. (2d27c87)
  • ⚠️ BREAKING BREAKING: The deprecated kPepper value has been removed from the public media::MediaContentType enum. Any remaining code referencing this value will fail to compile. (57f8280)
  • ⚠️ BREAKING BREAKING: The kEnableWebGLImageChromium command-line switch has been moved from content/public to third_party/blink/public, and the kDisableWebGLImageChromium switch has been removed entirely. Embedders referencing these switches will need to update their paths or remove usage. (e327feb) (8854092)
  • A new Rust implementation of ChildProcessSecurityPolicy has been introduced behind a feature flag. This is a major step in migrating core security components to Rust for improved memory safety. (5440cf2)

Browser & UI

  • ⚠️ BREAKING BREAKING: The public TabRendererData class has been completely removed. Custom UI components and embedder code must be updated to use the TabData struct instead. (bf0fd12) (af1897a)
  • ⚠️ BREAKING BREAKING: The --launch-isolated command-line switch for Windows browser isolation has been removed. This feature is now configured via the Windows Registry, and a new C++ API chrome::SetIsolationState has been introduced for programmatic control. (03a70ad)
  • The "Auto Tab Groups" feature has been removed from the Tab Search WebUI. (549edd6)
  • A new command-line flag, --translate-target-language, has been added to automatically translate pages to a specified language upon loading. (d23ab77)
  • The bookmarks overflow menu will now remain open after deleting an item, improving user workflow. (a324040)
  • On macOS, a stability issue with re-entrant calls to TextInputClientMac has been fixed by returning a dummy value instead of crashing. (7ab3e5c)

Security, Sync & Identity

  • ⚠️ BREAKING BREAKING: The signature of the virtual method syncer::DataTypeController::GetPreconditionState() has been changed. Embedders with custom DataTypeController implementations must update their method signature to GetPreconditionState(const PreconditionContext& context). (37a2d3f)
  • ⚠️ BREAKING BREAKING: The public DiceResponseParams::SigninInfo structure has been modified to support multiple accounts. Embedders who parse DICE sign-in responses or interact directly with this data structure will need to update their code. (740a888)
  • ⚠️ BREAKING BREAKING: The signature of the public Safe Browsing method ClientSideDetectionHost::ReportUnsafeSite has changed. It no longer accepts raw screenshot data and now requires a single SkBitmap object. (a0748d0)
  • Data Controls policies have been extended to cover custom clipboard formats, allowing enterprise policies to block or modify data during paste operations. (1a34b1d)
  • A potential Out-Of-Memory (OOM) crash in the History Embeddings feature was fixed by adding a size check before decompressing gzip data, preventing issues with corrupted files. (3dcee0a)
  • On macOS, a bug causing Device Bound Session Cookies (DBSC) to hit quota limits after system sleep was fixed by migrating quota tracking from base::TimeTicks to base::Time. (25b61d1)

Extensions

  • ⚠️ BREAKING BREAKING: The signature for ExtensionTabUtil::IsTabStripEditable() has been changed to IsTabStripEditable(Profile& profile). All callers must be updated to pass the relevant profile. (8bf2b94)
  • ⚠️ BREAKING BREAKING: The public static method ExtensionTabUtil::OpenOptionsPageFromAPI() has been removed. Its logic has been moved into the chrome.runtime API implementation, breaking any external code that called this utility function directly. (1567ebe)
  • ⚠️ BREAKING BREAKING: The structure of the RequestFileSourceResponse dictionary in the chrome.developerPrivate extension API has changed. Source code details are now nested within a source object, which will break tools that parse the previous flat structure. (5dd40e6)
  • A heap-use-after-free crash in the declarativeContent.removeRules API has been fixed. The crash occurred when duplicate rule identifiers were provided. (98a65b0)

Networking & Developer Tools

  • ⚠️ BREAKING BREAKING: The Audits.checkContrast command has been removed from the DevTools protocol. Developer tools or automation scripts that use this command for accessibility checks will no longer function. (f0f700c)
  • A new WS_OVER_H3 job type was added to HttpStreamFactory, allowing WebSockets to reuse existing QUIC (HTTP/3) connections. This can significantly improve connection performance and reduce latency. (9c623b4)
  • A memory regression in the Network Service has been fixed by correcting the memory accounting logic for Durable Messages in DevTools to use actual decompressed byte counts. (ff728ac)
  • A new "Export to JSON" button has been added to the chrome://safe-browsing page for easier debugging and data analysis. (5ed885e)

Remoting

  • ⚠️ BREAKING BREAKING: The remoting signaling API has been refactored. FTL (Federated Transfer Layer) specific methods have been moved from the generic SignalStrategy public interface to a new, dedicated FtlSignalStrategy class. (f154eb3)
  • ⚠️ BREAKING BREAKING: A method in the public remoting::SignalStrategy::Listener interface has been renamed from OnSignalStrategyStateChange to OnSignalingStateChanged. All implementations must be updated. (88d4a31)

Testing & Infrastructure

  • ⚠️ BREAKING BREAKING: The BrowserActivator test utility class has been removed. Tests that relied on this class for managing browser window activation must be updated to use views::test::MockWidgetActivator. (b6ec3b4)
  • ⚠️ BREAKING BREAKING: The ui/base/interaction API has been refactored to use a strongly-typed UniqueIdentifier class instead of the ElementIdentifier alias. This changes multiple public function signatures in ElementTracker and related classes, breaking embedders who use this UI testing framework. (3ac974d) (fe6930d)
