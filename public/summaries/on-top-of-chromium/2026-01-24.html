Chromium Digest: 2026-01-24

Overview

This digest covers 89 commits from the Chromium repository, filtered to focus on changes relevant to developers building on top of the browser for Windows and macOS. This analysis excludes platform-specific changes for Android, iOS, and ChromeOS, as well as internal Blink/V8 modifications. Key updates include two breaking changes in core utilities and APIs, security hardening for the clipboard, and new enterprise policies. The first commit is (af71659) and the last is (752be1b).

Core & Base Utilities

  • ⚠️ BREAKING BREAKING: The signature for base::ToVector has been changed. The two-argument overload no longer has a default projection parameter. Code making single-argument calls like base::ToVector(my_range) will now resolve to a new, more efficient overload and may require updates if the previous behavior with a default projection was expected. (a2039f6)
  • The google-chrome:// URL scheme is now correctly handled on macOS when opened via Apple Events, stripping the prefix before navigation to align with behavior on other platforms. This logic was extracted into a reusable google_chrome_scheme_util utility. (d033255)
  • The webui_url_utils files have been moved from chrome/browser/ui to chrome/common to allow access from components that cannot depend on the browser UI layer. This is a non-functional refactoring. (a3db8af)
  • The signature for media::PaintCanvasVideoRenderer::ConvertVideoFrameToRGBPixels has been updated to use base::span<uint8_t> instead of a raw void* for the pixel buffer, improving type safety. (dcf4400)

Security & APIs

  • ⚠️ BREAKING BREAKING: Support for the "Preview" protocol has been completely removed from the Digital Identity API. Calls attempting to use this protocol will now fail. Only the openid4vp protocol remains for bypassing the user interstitial. (a40eb38)
  • Security has been enhanced for the clipboard API. The methods ReadAvailableCustomAndStandardFormats() and ReadUnsanitizedCustomFormat() now require renderer paste permissions via IsRendererPasteAllowed(), preventing compromised renderers from reading custom clipboard formats without user activation. (67ed1f7)
  • A new enterprise policy, RestrictPdfSaveToGoogleDriveAccountsToPattern, has been added. This allows administrators to restrict which Google accounts can be used for the 'Save to Drive' feature in the PDF viewer by specifying a regex pattern. (c86d1da)

Extensions

  • A new command-line flag, --permit-dnr-rules-on-webviews, has been introduced to allow Declarative Net Request rules to apply to <webview> elements, which is disabled by default. (bfcea3d)
  • A policy validation error in M145 related to Extension install policies has been fixed. The creation of the CloudPolicyStore for this purpose is now disabled for Chrome versions below M146. (af71659)

Browser UI & Features

  • The hide delay for media session notifications has been significantly reduced from 2.5 seconds to 0.5 seconds. This change aims to more quickly hide notifications for media that is no longer controllable. (0f27921)
  • A crash in the WebUI Toolbar has been fixed by adding a null check for the BrowserWindowInterface before use. (a6d3f63)
  • In Vertical Tabs, a placeholder title will now be correctly shown for tabs that are still loading and do not yet have a title. (9c249d5)
  • A bug causing crashes in incognito mode has been fixed by disabling bitmap prefetching for the search aggregator feature, which is not active in incognito. (57efe55)

Testing & Infrastructure

  • Several UI browser tests have been made more robust by explicitly waiting for browser window visibility (f9e4c09b) and ensuring a deterministic activation order for created browser windows (2fbf651b). (f9e4c09) (2fbf651)
  • A flaky test for the Access Code Cast feature has been re-enabled after UI migration to Lit, which is expected to resolve the flakiness. (e992002)
