Chromium Digest: 2025-12-17

Overview

This summary covers 499 relevant commits out of a total of 645 on the main branch for December 17th, 2025. A significant number of breaking changes were landed, impacting core APIs, extensions, and the DevTools Protocol. Major themes include ongoing refactoring of tab management, modernization of extension APIs, and significant changes to core utilities like memory management and IPC graphics types. The first relevant commit was (a868dde) and the last was (11544a8).

Browser, Core APIs & UI

  • ⚠️ BREAKING BREAKING: The base::MemoryPressureListenerRegistry is no longer a static singleton. It now requires explicit instantiation, and calls to base::MemoryPressureListenerRegistry::Get() will fail without an active instance in scope. Embedders must update their code to manage the registry's lifecycle. (83f4085)
  • ⚠️ BREAKING BREAKING: The entire //ui/gfx/ipc/buffer_types directory has been removed. This is a major change for any code that relied on the IPC parameter traits defined within for serializing graphics buffer types. (4292293) (eb22d98)
  • ⚠️ BREAKING BREAKING: The public method AutofillManager::form_structures() has been removed. Embedders relying on this method must find alternative ways to access form structure data. (3d6df2f)
  • ⚠️ BREAKING BREAKING: The TabDragDelegate interface has been modified. The OnTabDragUpdated method now returns a TabDragContext* instead of void, requiring implementations to be updated. (c47f0fe)
  • ⚠️ BREAKING BREAKING: The public method TabCollection::MoveTab() has been removed from components/tabs/public/tab_collection.h. Code using this method for direct tab manipulation must be refactored. (a595e9f)
  • ⚠️ BREAKING BREAKING: The NoState Prefetch API has been changed with the removal of NoStatePrefetchManager::AddSameOriginSpeculation() and NoStatePrefetchNavigationThrottle. Integrations must be updated to use StartPrefetchingFromLinkRelPrerender(). (6fc1e40)
  • ⚠️ BREAKING BREAKING: The unused public method BrowserList::size() has been removed from the API. (64c605a)
  • Significant work continues on tab management, especially for vertical tabs. In addition to the breaking change in TabDragDelegate, the TabDragController was updated to better calculate window offsets, and a new VerticalTabDragHandler was introduced to manage detaching tabs. (3725750) (fec8df0)
  • As part of the "Project Bedrock" initiative, multiple components were migrated away from the legacy BrowserList and BrowserListObserver towards the more modern BrowserWindowInterface and BrowserCollectionObserver. (75eda64) (1f2b8ee) (e4aff77) (868cd21)
  • On Windows, window icons are now explicitly resized to system metrics before being set, preventing scaling artifacts from incorrectly sized bitmaps. (6359740)
  • For custom UI developers, the ColorProviderBrowserHelper now exposes its underlying ColorProviderSource, providing more direct access to the color source for a browser instance. (70117c9)

Extensions

  • ⚠️ BREAKING BREAKING: The ExtensionInstalledWaiter class has been removed and replaced by ExtensionInstallWatcher. Code waiting for extension installations must be updated to use the new class, which is now tied to the browser window lifecycle. (d3615f2)
  • The runtime.sendMessage API schema was updated to clarify promise support and add a missing parameters field to the sendResponse function definition. (467a3a8) (da18619)
  • Internal refactoring continued to separate browser finding/creation from tab opening logic within OpenTabHelper and TabsCreateFunction, improving code organization. (55b825b) (5189a3b) (e06e718)
  • The migration of extension APIs from JSON to the WebIDL format continues, with oauth2, fileHandlers, crossOriginIsolation, and chrome_url_overrides being converted. (3a20416) (53881ac) (df23b7a) (6d9ee98)

Developer Tools & Performance

  • ⚠️ BREAKING BREAKING: The Chrome DevTools Protocol (CDP) command Network.enable has been changed from asynchronous to synchronous. Its signature was altered to remove the callback. Automation clients must be updated to handle the new synchronous behavior and use the new Network.configureDurableMessages command for durable message functionality. (407599e)
  • A new CDP command, Tracing.getTrackEventDescriptor, has been added to allow Perfetto to display more complete information about tracing categories. (09edb3e)
  • A new trace event, InteractionToActualNavigationStart, has been added to navigation timelines to help visualize the time from user input to the start of a navigation. (c9f531b)
  • The chrome://traces page is now accessible without requiring the debug UI preference, simplifying the process for users to capture and share traces. (19f75a0)

Networking & Security

  • A crash in ChromePasswordProtectionService has been fixed. The service now uses a more robust method to get content size, preventing a nullptr dereference in WebUI-based browser environments. (6c58127)
  • The behavior of proxy override rules for DNS probes has been changed. Insecure schemes (e.g., http://) are now stripped from hostnames before DNS resolution to prevent failures, while secure schemes are preserved. (d53f993)
  • The Web Speech API now correctly uses the language from the accept-languages header as a default, instead of always falling back to en-US. (dd62ba9)
  • For enterprise use cases, the SafeArchiveAnalyzer Mojo interface was updated to support the analysis of obfuscated RAR archives. (6392e4b)

Media & Remoting

  • ⚠️ BREAKING BREAKING: The public method AudioBus::channel_span() has been removed from media/base. Embedders must be updated to use AudioBus::channel() instead. (465fb43)
  • ⚠️ BREAKING BREAKING: The signature of the public method remoting::SignalingAddress::Parse() has changed. The std::string* error output parameter was removed, requiring call sites to be updated. (ddb0437)
  • On macOS, IPC support was added to enable the use of CADisplayLink (hardware VSync) from the browser process, allowing VSync events to be forwarded to the GPU process. (3bf69c9) (b7e6bac)
  • The Symphonia media decoder (#use-symphonia-decoder) is now enabled by default on all desktop platforms (Windows, Mac, Linux). (83c0d54)

Build & Testing

  • The legacy testing methods BrowserProcess::CreateGlobalFeaturesForTesting and GlobalFeatures::Shutdown have been removed. Downstream tests should now use TestingBrowserProcess::SetUpGlobalFeaturesForTesting() and TearDownGlobalFeaturesForTesting(). (ced3c21)
  • For C++/Rust interoperability, circular includes are now allowed between cxx targets and their Rust counterparts, and Rust crate names are shortened in the build using string_hash. (c9f2e92) (9d5b686)
  • ExtensionServiceTestBase was updated to allow the use of TestSyncService instead of the default SyncService, simplifying test setup for sync-dependent features. (8569334)
