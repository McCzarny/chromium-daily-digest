Chromium Digest: 2026-01-16

Overview

This daily summary covers 415 relevant commits out of 615 total, focusing on changes impacting developers building on Chromium for Windows and macOS. Notable non-breaking changes include a new WebUI histogram API and the integration of a 'back-to-opener' navigation feature. Breaking Changes

  • ⚠️ BREAKING The base::MemoryPressureListener interface has been updated. The OnMemoryPressure method signature now receives (old_level, new_level) instead of a single pressure level, breaking all existing implementations of this core listener. (8c3705f)
  • ⚠️ BREAKING The Permissions API has been significantly refactored. PromptOptions is no longer part of PermissionRequest and is now passed as a direct parameter to methods like Accept(), Deny(), and OnPromptResolved(). This breaks method signatures on the public permissions::PermissionRequestManager and permissions::PermissionsClient interfaces. (7688ee1) (5d3a4e1) (b47e74e)
  • ⚠️ BREAKING A new virtual method, GetEndpointForDrag, has been added to the public content::ContentBrowserClient interface. Embedders must implement this method to ensure correct enforcement of drag-and-drop security policies. (f29965a)
  • ⚠️ BREAKING The public WebNavigationParams API has been changed. It now uses a new AgentClusterKey structure instead of several booleans, breaking embedders who construct or inspect these parameters during navigation. (c5a1fcd)
  • ⚠️ BREAKING The gpu::GLHelper class and the corresponding public API method GetGLHelper() have been removed from blink::WebGraphicsContext3DProvider. Embedders relying on this for GPU context access will be broken. (e67fa28)
  • ⚠️ BREAKING Support for the Versatile Video Coding (VVC) codec has been removed. This includes the removal of VVCPROFILE_* enumerators from the public media::VideoCodecProfile and gpu::VideoCodecProfile APIs. (bdb67c3)
  • ⚠️ BREAKING The legacy TabLoader-based session restore mechanism and its associated feature flag BackgroundTabLoadingFromPerformanceManager have been completely removed. All tab restoration now uses the Performance Manager path. (cf33d9c) (3f0d5d7)
  • ⚠️ BREAKING The static ExtensionAssetsManager::GetInstance() method has been removed. Embedders must now use ExtensionsBrowserClient::Get()-&gt;GetAssetsManager() to access this functionality, requiring code changes. (9a7acb2)
  • ⚠️ BREAKING Several public virtual functions have been removed from the TabDragContext interface (chrome/browser/ui/views/tabs/dragging/tab_drag_context.h), which will break custom tab dragging implementations. (78933f9)
  • ⚠️ BREAKING The public methods TabStripModel::SetTabGroupNeedsAttention and TabStripModelObserver::OnTabGroupNeedsAttentionChanged have been removed as part of a refactoring. (e6961ef)
  • ⚠️ BREAKING The constructor signature for TabStripControlButton and its subclasses (e.g., NewTabButton) has changed. It now requires a BrowserWindowInterface* instead of a TabStripController*. (291681b)
  • ⚠️ BREAKING The public header content/common/process_visibility_tracker.h has been renamed to content/common/process_priority_tracker.h, which will break any code that includes the old file path. (24075b8)
  • ⚠️ BREAKING On Windows, the base::win::AccessToken API in access_token.h has changed. The method GetSecurityAttributeString has been removed and replaced with the more flexible GetSecurityAttribute method. (dbc59bc)
  • ⚠️ BREAKING The deprecated helper function base::MakeCheckedNum&lt;&gt;() has been removed from base/numerics/checked_math.h. Use C++17 Class Template Argument Deduction with base::CheckedNumeric instead. (419f713)
  • ⚠️ BREAKING The public virtual method AliasRequiresProxyOverride has been removed from the net::ProxyDelegate interface as part of a feature cleanup. (ee52776)
  • ⚠️ BREAKING The constructor signature for TurnSyncOnHelper has changed, removing the SigninManager* parameter. This breaks code that instantiates this helper for sign-in flows. (101c6d7)
  • ⚠️ BREAKING Method signatures in the public extensions::InstallApproval class have been changed to use content::BrowserContext* instead of Profile*. (6c18838)
  • ⚠️ BREAKING The sandboxed SQLite VFS has been moved to a new component (//components/sqlite_vfs), which modified and broke the associated persistent_cache.mojom interface. (38cb111)
  • ⚠️ BREAKING The protobuf message for enterprise SaaS usage reporting was renamed and its structure was altered, which will break data ingestion pipelines that parse this report. (f9e2d26)

Browser UI & Features

  • The "back-to-opener" feature is now integrated with the main browser back button. The button will be enabled and trigger this action if there is no other navigation history to go back to. (9bdc317)
  • The TabListInterfaceObserver has been extended with new methods: OnTabMoved to notify of tab reordering and OnTabListDestroyed to allow for proper cleanup. (bc04ad2) (ee2eb14)
  • A skeleton for a new chrome://skills dialog and page handler has been added, laying the groundwork for a future feature. (e97ce33)
  • Functionality for launching Chrome in the foreground on Windows startup has been implemented behind the kLaunchOnStartup feature flag. (a5d750f)
  • On macOS, zoom gestures like pinch-to-zoom are now disabled for the WebUI toolbar to prevent unintentional resizing of the UI. (1ce290d) (cd0ceae)
  • A user action is now logged when a user switches between horizontal and vertical tab strip modes, improving usage metrics. (28a80f6)

APIs & Extensions

  • A new, lightweight chrome.histograms API has been introduced for WebUI pages to log UMA histograms. This API does not require chrome.metricsPrivate or extra Mojo interface registration. (9044a5b)
  • The immutability of an extension's manifest is now strictly enforced by promoting DCHECKs to CHECKs in Extension::SetManifestData, enhancing security. (40393e3)
  • Metrics have been added to chrome://inspect to track the usage of various remote debugging features. (1e0ca04)
  • The documentation for the sidePanel.close() method was updated to clarify its behavior when only a global side panel is open. (a559130)

Networking & Security

  • Enterprise Data Controls policies are now enforced for drag-and-drop operations on desktop platforms, preventing sensitive data from being dragged out of the browser against policy. (f29965a)
  • HTTP cache encryption logic has been improved. Ownership of the CacheEncryptionProvider was moved to ProfileNetworkContextService to manage a master encryption key stored in profile preferences. (2fd5f4d)
  • A new histogram (Settings.SearchEngines.GoogleIconMismatches) has been added to help quantify the prevalence of third-party search engines that spoof the Google icon in settings. (1c9edd4)
  • The QUICHE library was rolled, bringing updates to QUIC and HTTP/2 session handling. (9126550)
  • Several features related to automated password changes have been enabled by default, improving the user experience for password security. (24d48c4)

Core Internals & Build System

  • The Rust toolchain on Windows was updated to pass flags via response files (.rsp) to avoid hitting command-line length limits during builds. (b6fec29)
  • The PageContentScreenshotService was moved from //chrome/browser to the more generic //components/page_content_annotations as part of a larger modularization effort. (6ec6966)
  • On Windows, the BrowserUpdaterClient was updated to use a Mojo proxy to query updater state and policies, improving robustness. (e3464dd)
  • An unsafe buffer operation in traced_value_proto_writer.cc was fixed, and an unsafe wcslen call was replaced with wcslen_s in Windows IME code to improve memory safety. (543aa22) (988a7a6)
  • CSS lint checks have been integrated into the standard WebUI build process, enforcing code quality for UI components. (c2b3d5f)
