Chromium Digest: 2026-05-16

Overview

This digest covers 55 commits from the Chromium repository. After filtering based on relevance to browser embedders on desktop platforms (Windows/macOS) and ignoring internal web platform, OS-specific, and minor changes, this report focuses on key updates in GPU, Security, UI, and PDF functionalities. One major breaking change was identified in the GPU control list API, alongside several important security hardenings for device interactions and extensions.

GPU

  • ⚠️The GpuControlList::MakeDecision method signature has been changed, removing the default value for the ignored_entries parameter. Callers must now explicitly provide a std::vector<uint32_t>, which may require updates for embedders using this GPU blocklisting interface. (c7427fc)
    GPUAPI
  • The helper function IOSurfaceIsWebGPUCompatible() has been moved from //media/base/mac to //ui/gfx/mac to resolve a dependency conflict where //gpu could not depend on //media. This is a code organization refactor primarily affecting macOS builds that interact with WebGPU and IOSurfaces. (59ad18d)
    GPUmacOSMedia

Security

  • The WebUSB implementation on macOS has been hardened against a potential Use-After-Free (UAF) vulnerability. The fix prevents claiming a USB interface if it advertises an endpoint address already claimed by another interface, mitigating risks from maliciously crafted USB devices. (a74fd42)
    USBSecuritymacOS
  • A potential shortcut hijacking vulnerability in the extensions' Global Shortcuts portal has been fixed. The change ensures that suggested accelerators for manifest-defined commands are cleared if they are not globally active, preventing malicious extensions from suggesting hijacks of reserved browser keys. (39567c3)
    ExtensionsSecurity
  • A bounds check was added to the HID connection logic on Windows to prevent a potential buffer over-read. This hardens the driver against devices that could cause bytes_transferred to exceed the buffer size after internal processing. (0bac4a7)
    SecurityWindows

UI

  • Native UI rendering via gfx::RenderText has been improved to correctly display text-default emoji codepoints followed by a variation selector (VS-16) in color. This fixes an issue on Windows and macOS where characters like '©️' or '™️' would render as monochrome glyphs in tab titles and other native UI surfaces, aligning their appearance with web content. (1c45c78)
    UIFontsmacOS

PDF

  • The internal implementation for PDF Ink Signatures continues to be built out. Recent changes add backend logic for managing text annotations, including showing, editing, deleting, and handling undo/redo states. This work is part of an ongoing feature development within the PDF viewer. (f05ce42) (b1809ed) (04cd755) (4f6437d) (feed479) (f880d93) (f469ee2)
    PDFUI

Web Apps

  • The Web App launch process has been refactored to use the central browser_navigator, consolidating navigation logic. Additionally, the chrome://web-app-internals debug page's JSON structure was normalized from an array to a dictionary, improving frontend type safety and maintainability. (fe7265c) (f058bd2)
    BrowserUI
