Chromium Digest: 2025-11-12

Overview

This summary covers 426 relevant commits out of 610 total commits from the Chromium main branch on November 12, 2025. Key changes include the removal of the deprecated `NavigationHandle::SetCorsExemptRequestHeader()` API, ongoing migration away from `BrowserListObserver`, and continued development of AI features in the omnibox. First commit: (9fd20c1), Last commit: (2d035cb).

Breaking Changes

  • ⚠️ BREAKING The deprecated and unused API NavigationHandle::SetCorsExemptRequestHeader() has been removed. Downstream projects relying on this function will need to remove its usage. (9cc9245)
  • ⚠️ BREAKING As part of Project Bedrock, browser tests continue to be migrated away from the deprecated BrowserListObserver. Projects should use BrowserList::AddObserver and BrowserList::RemoveObserver instead. (038899c)
  • ⚠️ BREAKING The internal diagnostics page chrome://audio has been removed from the browser. (d368131)
  • ⚠️ BREAKING The unlaunched experimental "Partitioned Popins" feature has been removed, including its controller, permissions policies, HTTP headers, and IDL definitions. (1a2cbc1) (1ce0cb5) (1b14dc3) (8c2b7da) (c9d54f0)
  • ⚠️ BREAKING The internal Autofill API AutofillDriver::ExtractForm() was refactored to ExtractFormWithField(), changing its signature to use a FieldGlobalId instead of a renderer-specific form ID. (83d041c)

Browser UI & Features

  • A Mojo interface for the AI Model (AIM) popup in the omnibox was added, facilitating communication between the browser and the WebUI. (65727d6) (d274bf2) (6ff1965)
  • The "Save" and "Cancel" buttons in the "Save PDF to Drive" dialog on Windows have been swapped to better align with platform conventions. (5ad18b8)
  • Picture-in-Picture windows can now be excluded from screen recordings on macOS via a new proxy for the screen capturer. (2abe340)

Extensions

  • Additional crash keys have been added to help diagnose a rare issue where an extension's site access toggle is off but its access level is incorrectly set to "On all sites". (7a55650)

Security

  • Fixed a bug on Windows where Chrome could hold a critical policy section indefinitely, ensuring the section is always exited to prevent potential deadlocks. (1046b03)
  • On macOS, the key deletion logic for Device Bound Session Credentials (DBSC) was updated to use an application tag for more precise key identification. (60f88a6)

Developer Tools

  • DevTools can now collect font-related @rules for pseudo-elements like ::first-letter, allowing for more accurate style inspection. (0f24f72)

Testing & Infrastructure

  • A large number of browser tests were updated to use valid certificates configured on the test server, removing the need for the --ignore-certificate-errors command-line switch and improving test correctness. (64d5855) (7b97b4a)
  • Introduced ProcessPerformanceScenarioObserverList to centralize the management of performance scenario observers, unblocking future optimizations. (c72888f)
  • The third-party abseil-cpp library has been rolled to a newer version. (6d6c83c)

2025-11-11

Chromium Digest: 2025-11-11

Overview

On 2025-11-11, the Chromium main branch saw 576 commits, with 406 considered relevant for browser developers. This day's changes include several significant API and architectural updates, such as the removal of the Partitioned Popins feature, performance boosts to networking key structures, and continued work under Project Bedrock to decouple core components. First commit: 96bff48d29d12cab7f5d380a5cc5a35a65e9a750, Last commit: 089db6cd18f69288a49ebb1fc988c0d1d2976c19.

Architecture & APIs

  • ⚠️ BREAKING As part of the 'Project Bedrock' initiative, several services including session_service and idle_service were migrated away from observing BrowserList. This architectural change reduces dependencies on the global Browser object and may require updates for downstream code that interacts directly with these components. (0e5b5d1) (1f82c9e) (643ce22)
  • ⚠️ BREAKING The Partitioned Popins feature, which was in a DevTrial, is being removed. This initial step removes the enabling IPC call, effectively disabling the feature globally. (154f95a)
  • ⚠️ BREAKING In the Autofill component, the function ExtractForm() has been renamed to ExtractFormWithField() to better clarify its purpose. Code calling the old function will need to be updated. (5063b6a)
  • ⚠️ BREAKING The feature flag and kill switch for a previously launched postMessage optimization have been removed, finalizing the feature's integration. (3e8c3dd)
  • A new reason, kBackgrounded, has been added to the SidePanelHideReason enum. This helps differentiate when a side panel is hidden due to a tab switch versus other user actions. (074a009)

Performance & Networking

  • The core networking classes IsolationInfo, NetworkIsolationKey, and NetworkAnonymizationKey were refactored to use ref-counted data members. This makes them significantly cheaper to copy, reducing overhead in network requests. (d9614f9)
  • A race condition in the GPU process cache was fixed by moving the PersistentCache to its own dedicated 'GPUPersistentCache' directory, improving stability. (69c0a1a)

Browser UI

  • The Omnibox 'Answer in Omnibox' (AIM) and 'Composebox' features received several updates, including support for file drag-and-drop and new UI states for errors like file limits. (29a75ee) (589885e) (e3d64b2)
  • The HID device chooser on Windows has been improved to correctly find device product names. It now checks all available device info entries before falling back to a generic name, fixing issues where some devices appeared unnamed. (a34ca6d)
  • The state management for the omnibox popup (OmniboxPopupView) was refactored for better consistency and reliability. (937004d)

Testing

  • ⚠️ BREAKING The --ignore-certificate-errors flag has been removed from a large number of browser tests to improve security and reliability. Downstream projects may need to update their test environments to use a proper HTTPS test server. (55032fa) (8f56585) (ce0ad21) (59f6448) (798cbae) (56b1829)
  • A new command-line switch, --attempt-form-filling-tool-skips-ui, was added to allow the form-filling test tool to bypass the data selector UI, streamlining automated testing. (a352613)

2025-11-10

Chromium Digest: 2025-11-10

Overview

A total of 482 commits were made to the 'main' branch. After filtering, 322 relevant commits were analyzed. The day's changes span from commit (9231b16) to (a37032d).

Breaking Changes

  • ⚠️ BREAKING As part of the "Project Bedrock" initiative to decouple browser components, the global BrowserListObserver is being replaced. New interfaces, ProfileBrowserCollection and BrowserCollectionObserver, have been introduced to provide an observable collection of browser windows on a per-profile basis. Code that previously relied on observing the global list of browsers will need to migrate to this new, more structured API. (886bfce) (9fbc578) (2577a13)
  • ⚠️ BREAKING The OnUpdateError method has been removed from the update_client::CrxInstaller interface in components/update_client/update_client.h. This method was found to be uncalled across the codebase, and its removal cleans up the component installer API. External projects implementing this interface will need to remove the method override. (d279473)
  • ⚠️ BREAKING Delegate methods in TlsStreamAttempt::Delegate have been renamed for clarity. WaitForServiceEndpointReady() is now WaitForTlsHandshakeReady(), and GetServiceEndpoint() is now GetServiceEndpointForTlsHandshake(). Projects using these net component APIs will need to update their method calls. (38579c3)

Security

  • The Secure DNS (DNS-over-HTTPS) feature in "Automatic" mode now supports a fallback to well-known DoH providers. A new preference, kDnsOverHttpsAutomaticModeFallbackToDoh, allows Chromium to use a hardcoded list of providers (e.g., Google DNS) if upgrading the system's DNS provider fails, preventing a fallback to insecure DNS. As part of this work, the non-functional AlekBergNl provider was also removed from the list. (76cab8c) (984ddf8) (c5225eb)
  • Two new enterprise policies, GeolocationAllowedForUrls and GeolocationBlockedForUrls, have been introduced. These policies give administrators fine-grained control to allow or block geolocation access for specific URL patterns, complementing the existing DefaultGeolocationSetting policy. (4f84a5b)

Networking

  • Navigations that are in progress when a tab is closed will now correctly report a network error of net::ERR_ABORTED. This change improves the accuracy of navigation lifecycle events and resolves race conditions for observers of NavigationHandle, particularly for external protocol navigations. (01c95b3)

Extensions

  • A large-scale refactoring has renamed ToolbarActionViewController to ToolbarActionViewModel throughout the extensions UI codebase. This change better reflects the Model-View-ViewModel (MVVM) architecture and impacts numerous files related to the toolbar and extension action presentation. (f1115cf) (4f2f7ae) (b9f3429)

UI/UX

  • The legacy UI controls for video Picture-in-Picture have been completely removed, along with the associated feature flag kVideoPictureInPictureControlsUpdate2024. This finalizes the transition to the updated 2024 UI and simplifies the overlay implementation in chrome/browser/ui/views/overlay/. (272e694)
  • The Bookmarks Bar folder menu has been enhanced with 'Open all' and 'Open all in new tab group' buttons, providing functionality previously only available in the context menu. (0e87e22)
  • A new API has been added to the Glic service (chrome/browser/glic/host/glic.mojom) that allows it to open the Password Manager settings page programmatically. (da2ce6c)

Bug Fixes

  • Fixed a crash on macOS that could occur when entering immersive fullscreen mode while a popover (NSPopover) was visible. The workaround in components/remote_cocoa/app_shim/immersive_mode_controller_cocoa.mm proactively closes open popovers before the transition. (b68f959)
  • Corrected a race condition that could leave an empty managed app list on chrome://management. The UI handler now ensures that WebAppPolicyManager is fully initialized before displaying its data. (1a0886f)

Infrastructure

  • The internal plugin loading mechanism in content/browser/plugin_list.cc has been significantly simplified. Several internal methods were consolidated, and the LoadingState enum was replaced with a boolean, reducing complexity in this legacy component. (269b749) (1b93c96) (293add8)

Chromium Digest: 2025-11-09

Overview

A total of 107 commits were made to the 'main' branch. After filtering for changes relevant to projects built on Chromium, 9 commits were analyzed. The day's changes span from commit (96c4358) to (cfa4cb1).

Security

  • Introduces a "Site Familiarity" check that can defer navigation to determine if a site is new to the user. This mechanism, which checks browsing history and a high-confidence allowlist, is used to decide whether to enable V8 optimizations. This change primarily affects process selection logic within chrome/browser/site_protection. (9beb380)
  • The logic for enabling image sharing (IsShareImageEnabledForProfile) in chrome/browser/glic/public/glic_enabling.cc now includes a check for managed accounts, enhancing policy enforcement for enterprise users. (903dd22)

UI/UX

  • A null-check was added for the expandedSearchboxShowVoiceSearch flag in the omnibox component (cr_components/searchbox/searchbox.ts), preventing a potential browser crash. (f519a52)

Developer Tools

  • A new command-line switch, --webnn-ort-ep-device, has been added to allow developers to force the WebNN service to use a specific ONNX Runtime execution provider (e.g., a particular GPU), which is useful for testing and debugging. (98e6bd5)
  • The UI compositor's debugging utilities (ui/compositor/debug_utils.cc) can now print the background color of a layer when dumping the layer tree, providing more context for developers debugging UI rendering. (905c682)

Infrastructure

  • The Local Storage implementation now uses a dedicated WriteMetadata() method within LocalStorageLevelDB to handle writing metadata for last accessed and modified times. This refactoring improves the clarity and maintainability of the DOM Storage service. (64b770d)
  • A large-scale cleanup removed unnecessary includes of base/files/file_util.h across the codebase, including in core files like base/base_paths_mac.mm and base/cpu.cc. (502b6c6)

Bug Fixes

  • A null pointer access in RendererBlinkPlatformImpl has been fixed, resolving a crash that occurred when running in single-process mode (--single-process) due to a missing RendererMemoryCoordinatorPolicy. (7b5a317)

Testing

  • A new helper method, ButtonTestApi::NotifyMouseClick, was added to the views test framework. This simplifies UI tests by allowing developers to trigger a button click without manually constructing a MouseEvent. (0db0398)
