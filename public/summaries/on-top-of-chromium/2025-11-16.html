Summary for 2025-11-16 on branch 'main'

Chromium Digest: 2025-11-16

Overview

A total of 134 commits were made to the 'main' branch. After filtering, 16 relevant commits were analyzed. The day's changes span from commit (4012d80) to (003bbee).

API

  • ⚠️ BREAKING The allow_any_cors_exempt_header_for_browser parameter has been removed from network::mojom::NetworkContextParams. This parameter, previously used by the now-defunct WebLayer, is no longer consumed by any part of the codebase. This change simplifies the network context configuration. (5575443)
  • ⚠️ BREAKING Memory allocation methods in the GPU command buffer client API have been updated to return base::span instead of raw pointers. Specifically, TransferBufferInterface::Alloc(), TransferBufferInterface::AllocUpTo(), and RingBuffer::Alloc() now provide safer, bounds-checked memory spans. Projects directly interacting with these low-level GPU APIs will need to update their call sites. (c9131c0)

UI/UX

  • A bug in the Chromium Task Manager that showed multiple "Renderer" processes with a PID of 0 has been fixed. The task provider now correctly waits for renderer processes to be fully launched before creating task entries, ensuring accurate process information is displayed. (4449050)
  • Corrected alignment, colors, and focus states for the composebox feature within the omnibox popup, ensuring a more consistent look and feel with the latest UI specifications. (4012d80)

Infrastructure

  • The Node.js dependency used in Chromium's toolchain has been upgraded from v22.11.0 to v24.11.1 LTS. (8a75cf3)
  • Multiple changes were made to improve build hygiene by reducing header dependencies. This was achieved by switching to base/functional/callback_forward.h and using forward declarations more frequently in components like media, base, and ui, which helps reduce transitive includes and improve compilation times. (81c1f92) (1fdff46) (9ec1ad7) (d208977)
  • A refactoring within the Autofill component has removed the form_event_logger_ member from CreditCardAccessManager. The logger is now accessed via a method on BrowserAutofillManager, simplifying the class's constructor and dependencies. (304a475)
  • Buffer safety has been improved across the remoting/ component by migrating from broad, file-level #pragma allow_unsafe_buffers suppressions to more granular, inline UNSAFE_TODO markers for specific unsafe operations. (1fa10c7)
