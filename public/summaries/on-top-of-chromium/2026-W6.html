Chromium Weekly: 2026 Week 6

Overview

This week saw an unusually high number of breaking API changes, requiring significant developer action across core interfaces, networking, metrics, and testing. A major behavioral change disables Widevine hardware-secure decryption on Windows in favor of PlayReady. Other key developments include real-time certificate store monitoring on Windows, optimizations to the disk cache, and new capabilities for Progressive Web Apps. This summary covers 36 relevant commits out of 1512 total across 3 days.

API & Core Interfaces (Breaking Changes)

  • ⚠️ BREAKING BREAKING CHANGE: A broad effort to adopt base::span has impacted several public APIs. base::File::Read/Write, gin::ArrayBuffer, and components/webui/flags/feature_entry.h now require base::span instead of raw pointers and size arguments, necessitating updates for file I/O, V8 interoperability, and feature flag definitions. (dd02263) (e490adb) (d570054) (2e89d3b)
  • ⚠️ BREAKING BREAKING CHANGE: The public LocationBar interface has new pure virtual methods (GetBrowser(), IsDrawn()), and AccountInfo now requires using GetFullName() and GetGivenName() accessors instead of direct member access. (7665950) (623d7f8)
  • ⚠️ BREAKING BREAKING CHANGE: Access to the tab strip model has been refactored. Several virtual methods were removed from WindowController, and ExtensionTabUtil::GetEditableTabStripModel() was also removed. Code must be updated to retrieve the TabStripModel directly from the Browser object. (f706a1b) (8cc423f) (46847f4) (854164e)
  • ⚠️ BREAKING BREAKING CHANGE: The AutofillManager::Observer::OnFieldTypesDetermined method signature has changed, requiring implementations to add a new bool small_forms_were_parsed parameter. (524d57f)
  • ⚠️ BREAKING BREAKING CHANGE: The Structured Metrics service was refactored for threading. StructuredMetricsRecorder is no longer ref-counted, and the EventStorage::TakeEvents method is now asynchronous, breaking existing implementations. (8f66e3b)
  • ⚠️ BREAKING BREAKING CHANGE: The page_load_metrics.mojom interface was significantly altered to refactor Interaction to Next Paint (INP) metrics, which will break consumers of this performance metrics API. (f5cd024)

Networking & Security

  • ⚠️ BREAKING BREAKING CHANGE: Widevine hardware-secure decryption (HW_SECURE_ALL) has been disabled by default on Windows via the EME API. Services relying on this robustness level for protected content must adapt, as the platform shifts towards using PlayReady SL3000 for hardware DRM. (ea1b3b8)
  • ⚠️ BREAKING BREAKING CHANGE: 'Private Network Access' (PNA) has been renamed to 'Local Network Access' (LNA) across the codebase. This affects numerous public mojom interfaces, classes, and constants in the network service, requiring significant updates for dependent projects. (d28a1ec)
  • On Windows, the browser can now monitor the OS certificate store for changes in real-time, allowing for automatic synchronization of the trust store without a browser restart. (515c9bd)
  • The SQL disk cache backend received significant optimizations, including improved write buffering and lazy entry creation, which defers database writes until data is added. This reduces unnecessary I/O and improves performance for short-lived cache entries. (6048204) (767da05) (3ef364b) (2288d17) (a3b60c0)

Browser UI & Web Applications

  • The window.setResizable(bool) API is now supported for standalone windowed PWAs that use tabs, expanding a feature previously restricted to tab-less PWAs. (c19b719)
  • On macOS, a bug was fixed to allow drag-and-drop of downloads from PWA windows by introducing a new Mojo interface to handle the drag operation in the app shim process. (f36a3d1)
  • A new 'Back to opener' item has been added to the back/forward navigation menu, allowing users to quickly return to the tab that opened the current one. (d5b9c16)
  • The TabListInterfaceObserver has a new OnAllTabsAreClosing method to notify clients when a mass tab closure event begins, allowing for more timely reactions. (3649dd0)

Extensions

  • Critical stability fixes were landed, addressing a use-after-free crash in OneTimeMessageHandler and another crash caused by using a v8::Context after it had been invalidated during page navigation. (c3abe5e) (2740712)
  • The offscreen.createDocument and offscreen.closeDocument API callbacks are now optional, improving developer ergonomics for non-Promise-based use cases. (dca5915)

Testing

  • ⚠️ BREAKING BREAKING CHANGE: The hang-prone testing utility ui_test_utils::WaitForBrowserToClose() has been removed. Test suites must be migrated to use the more robust ui_test_utils::BrowserDestroyedObserver. (fef9f36) (1eeffe6) (11ba8dd)
  • ⚠️ BREAKING BREAKING CHANGE: On macOS, the public OSCrypt::SetKeychainForTesting method has been removed. Tests must now use the OSCryptMocker utility to mock keychain access. (158a2d2)
