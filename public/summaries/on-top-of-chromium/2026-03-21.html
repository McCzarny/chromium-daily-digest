Highlights include a breaking change to HDCP 2.3 reporting on Windows for PlayReady content, a new performance optimization for the WebRequest API, and several UI and security enhancements.

Media & DRM

  • ⚠️ BREAKING On Windows, the Encrypted Media Extensions (EME) API will now report HDCP 2.3 as unsupported for PlayReady-protected content. Previously, queries for HDCP 2.2 and 2.3 were treated identically. This change corrects the capability reporting, as the underlying Windows OS does not yet differentiate between the two versions. Content providers relying on the previous behavior for HDCP 2.3 may need to adjust their logic. (87eee40)

Extensions

  • A new feature flag, kOptimizeWebRequestProxy, has been introduced to change how network requests are proxied for extensions. When enabled, it avoids globally routing all subresource requests through the browser process just because a <webview>-using extension is installed. Instead, proxying is more targeted, applying only to requests from actual <webview> frames or when an extension with webRequest or declarativeNetRequest permissions is active. This is expected to improve performance by allowing more direct renderer-to-network requests. (95c9427)
  • A new histogram has been added to track the length of text set on extension action badges (chrome.action.setBadgeText). This data will inform a future decision on whether to impose a length limit to prevent performance issues from excessively long badge strings. (73e8489)
  • Internal refactoring continues to decouple extension-related code from the main //chrome module. Several constants and preference keys related to the Chrome Web Store (kAppsGalleryDownloadURL, kCWSInfoTimestamp) have been moved into the //extensions component. (1320e33) (83b14c7)

Security & Networking

  • As part of ongoing security hardening, validation for the Origin header on navigation requests has been tightened. Logging for unexpected Origin headers is now restricted to renderer-initiated navigations, fixing an issue where browser-initiated navigations (like going 'Back' to a POST redirect) were incorrectly flagged. (bbbb03f)
  • The internal ServiceWorker framework has been refactored to use the type-safe content::ChildProcessId wrapper instead of raw integers for representing process IDs, improving code clarity and safety. (0e161e8)
  • The internal logic for checking Certificate Transparency (CT) requirements has been refactored for efficiency. The delegate is no longer checked for policy overrides if the Signed Certificate Timestamps (SCTs) already comply with the policy. (5ade5a4)

UI & UX

  • A new context menu item, 'Ask Gemini' (or 'Summarize this page with Gemini'), has been added. This feature, part of the Glic (Gemini in Chrome) integration, is being introduced via an experiment. (8df2c9f)
  • The UI for the Saved Tab Groups "Resumption Rail" has been updated to correctly handle focus traversal, ensuring the 'more' button is focused in the correct order relative to its parent item. Additionally, the 'everything button' is now correctly hidden for new user profiles that are in a grace period for the feature's IPH promo. (ac30f15) (4ac6e41)
  • The Certificate Manager subpage (chrome://settings/certificates) has been migrated from Polymer to Lit as part of the ongoing WebUI modernization effort. (082abf4)

Infrastructure & Bug Fixes

  • A crash has been fixed in the internal ActorKeyedService framework. The crash occurred when a task completion callback tried to query the list of active tasks. The fix ensures the completed task is removed from the active list *before* callbacks are run. (24300df)
