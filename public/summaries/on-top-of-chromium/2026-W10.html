Week 10 of 2026 (2026-03-01 to 2026-03-07)

Chromium Weekly: 2026 Week 10

Overview

This week was marked by an exceptionally high number of breaking changes across the Chromium codebase, requiring significant attention from embedders. A total of 1092 relevant commits were merged out of 1453 total. Major themes include the migration of the entire ui::Clipboard API to be fully asynchronous, the introduction of a Rust-based security policy component, and significant API modernizations in the //base library. Notable new features include support for WebSockets over HTTP/3 and a new SurfaceEmbedConnector API for advanced UI composition.

Core APIs & Infrastructure

  • ⚠️ BREAKING BREAKING CHANGE: The ui::Clipboard API has been made fully asynchronous. Synchronous methods like ReadText, ReadData, and GetStandardFormats have been removed or converted to require a callback. This major change affects all consumers of the clipboard API, including context menus and extension functions. (1d23bcd) (34bc314) (c6bd1aa) (a1527fc) (14c828a)
  • ⚠️ BREAKING BREAKING CHANGE: The //base library continued its modernization by making raw pointer overloads for base::File::Read, base::File::Write, and base::File::ReadAtCurrentPos private, requiring the use of base::span. Additionally, the deprecated non-span overload for Pickle::WriteBytes was removed. (2f6529b) (3b61a3b) (18a2af4) (4fda9b2)
  • ⚠️ BREAKING BREAKING CHANGE: The base::MemoryPressureMonitor class has been removed. Embedders must migrate to using the MemoryPressureListener API. In a related change, base::AvailablePhysicalMemoryMonitor was renamed to base::AvailableMemoryMonitor. (1acb6aa) (dad691e) (0ed195d)
  • A new public API, SurfaceEmbedConnector, was introduced in content/public/browser. It allows a child WebContents to be embedded within a parent without a traditional frame hierarchy, enabling advanced UI compositions for features like the WebUI Browser Surface. (a01932c) (70e95fb)

Security

  • In a significant move towards memory safety, a Rust-based implementation of ChildProcessSecurityPolicy has been introduced. This is a major milestone in the effort to migrate critical security components from C++ to Rust and is currently controlled by a feature flag. (5440cf2)
  • ⚠️ BREAKING BREAKING CHANGE: As part of the ongoing effort to deprecate SHA-1, support for enabling SHA-1 signed certificates for local trust anchors has been completely removed. The VERIFY_ENABLE_SHA1_LOCAL_ANCHORS flag and the has_sha1 field in net::CertVerifyResult have been deleted from the net library and its Mojo interfaces. (0533a59) (5b62b90) (70a730f)
  • ⚠️ BREAKING BREAKING CHANGE: The behavior of fullscreen mode has been altered. Fullscreen will now be exited if a subframe that requested it (or contains the fullscreen element) performs a cross-document navigation. Previously, only main frame navigations would exit fullscreen, so this may impact applications relying on the old behavior. (9f026d2)
  • ⚠️ BREAKING BREAKING CHANGE: The Federated Credential Management (FedCM) API underwent several refactorings. The callback for FederatedEmbedderLoginRequest::Set was changed to a base::OnceCallback, and a new method was added to the content::webid::IdentityCredentialSource interface, breaking existing implementations. (5c44bb8) (2d27c87)

Networking

  • WebSockets can now be established over existing QUIC (HTTP/3) connections. A new WS_OVER_H3 job type was added to HttpStreamFactory, which can significantly improve connection performance and reduce latency by reusing connections. (9c623b4)
  • ⚠️ BREAKING BREAKING CHANGE: The networking stack saw several breaking API changes. The signature of net::FileStream::Seek was updated to use base::expected, and the return type of net::TransportConnectJob::Factory::Create was changed to std::unique_ptr<ConnectJob>. (e6d8aba) (da27f3d)
  • The kMulticastInDirectSockets feature flag has been removed, making multicast support in the Direct Sockets API a stable, default-enabled feature. (85b4f29)

Extensions & Platform Apps

  • ⚠️ BREAKING BREAKING CHANGE: As part of the ongoing deprecation of Chrome Apps, the chrome.browser API has been removed for all non-ChromeOS platforms (Windows, Mac, Linux). Any remaining apps on these platforms that rely on this API will no longer function. (d1392f5)
  • ⚠️ BREAKING BREAKING CHANGE: Several extension-related C++ APIs were broken. IconImage::Observer and ExtensionPrefsObserver were updated to inherit from base::CheckedObserver. Additionally, utility functions like ExtensionTabUtil::CreateTabList() and ExtensionTabUtil::OpenOptionsPageFromAPI() were removed. (95f9642) (98f3bdd) (1567ebe)
  • ⚠️ BREAKING BREAKING CHANGE: The stable chrome.tabs.update extension API will now throw an error if an extension attempts to un-highlight all tabs in a window by passing an empty array to the highlighted property. This was previously ignored silently and may break extensions that relied on that behavior. (ebae198)

Browser & UI

  • ⚠️ BREAKING BREAKING CHANGE: The core content::ContentBrowserClient interface was modified multiple times. Methods related to Isolated Web App permissions were replaced, and CreateURLLoaderHandlerForServiceWorkerNavigationPreload was renamed, requiring embedders to update their implementations. (fc322bb) (66509a9)
  • ⚠️ BREAKING BREAKING CHANGE: The public TabRendererData class has been completely removed from the UI toolkit. Embedders and custom UI components must now use the TabData struct. A new TabDataManager class was also introduced to centralize observation of tab data changes. (bf0fd12) (af1897a) (56faad1)
  • Several features were removed from the codebase, including the "Tab Declutter" feature and its associated command IDs, the "Auto Tab Groups" feature from the Tab Search UI, and the WebAuthn CaBLE v1 protocol implementation. (5bbd8d1) (549edd6) (02514ec)
