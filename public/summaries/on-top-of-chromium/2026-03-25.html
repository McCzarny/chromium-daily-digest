Notable changes include a behavioral shift in base::ReplaceFile on Windows, major refactors in the enterprise content analysis and HDR metadata APIs, and multiple signature changes to observer interfaces.

Breaking Changes

  • ⚠️ BREAKING The public virtual method OnKeywordModeChanged() has been removed from the OmniboxClient interface. Embedders subclassing this client must remove their implementations. (f22d35c)
  • ⚠️ BREAKING The public gfx::HDRMetadata class has been significantly refactored to use Skia-native types. Direct member access is no longer possible; embedders must migrate to new getter/setter methods like HasMDCV() and SetCLLI(). (8cb7cd0)
  • ⚠️ BREAKING The behavior of base::ReplaceFile on Windows has been altered. It no longer fails if it cannot create a temporary backup file (e.g., due to a long path), and will instead proceed without a backup. This changes the success/failure contract and removes the atomicity guarantee in this edge case. (36a258f)
  • ⚠️ BREAKING Multiple enterprise content analysis APIs have been refactored. RequestHandlerBase was moved to //components with a modified constructor, ContentAnalysisDelegate::Data was moved and renamed, and the CalculateEventResult helper function was relocated. Embedders using these deep scanning interfaces must update paths, type references, and constructor calls. (76f05db) (ab4c102) (3b6e122)
  • ⚠️ BREAKING The signatures for SharedWorkerService::Observer::OnWorkerCreated and DedicatedWorkerService::Observer::OnWorkerCreated have changed. The worker_process_id parameter is now a strongly-typed content::ChildProcessId instead of an int, requiring updates to observer implementations. (12222ff) (57d5b4f)
  • ⚠️ BREAKING The signature for AutofillClient::EntityImportPromptResultCallback has been updated to include a new const EntityImportUIContext& parameter. Embedders using this callback for features like ShowEntityImportBubble must update their function signatures. (d130440)
  • ⚠️ BREAKING The behavior of net::DataURL::BuildResponse has changed. The mime_type output parameter now only contains the MIME type essence (e.g., "application/pdf"), while any additional parameters are moved to the headers output. Code that previously parsed the mime_type string for parameters will break. (7ec7459)
  • ⚠️ BREAKING Two core interfaces have new virtual methods with default implementations. content::ContentBrowserClient now has IsPopupBypassAllowed() for security, and content::DelegatedFrameHostClient on macOS has GetResizeDeadlinePolicy() for PWA window resizing. Embedders implementing these interfaces should add the new methods. (3b4c638) (6b18bb3)
  • ⚠️ BREAKING The public method SetAllPromosSnoozed() has been removed from the NtpPromoController API in components/user_education, removing the ability to programmatically snooze New Tab Page promos. (0fa7983)
  • ⚠️ BREAKING The public method PointerLockRequested() has been removed from the ClientSideDetectionHost class in the Safe Browsing API. (0fbb68f)
  • ⚠️ BREAKING Support for high-channel-count audio layouts was reverted. This removes the public function GetConcurrentMaxChannels() from media/base/channel_layout.h and reverts the kMaxConcurrentChannels constant to 8. (ce520a1)
  • ⚠️ BREAKING On macOS, the obsolete google_toolbox_for_mac dependency and the associated public Cocoa UI class FirstRunDialogViewController have been removed. (90cd586)

Extensions

  • ⚠️ BREAKING Behavior Change: The feature to control user script execution via a per-extension toggle is now enabled by default. This replaces the previous model where it was tied to developer mode, and migration logic for the old preference has been removed. (9ffa658)

Security

  • The PermissionServiceImpl now verifies the true transient activation state from the browser process before trusting a user_gesture signal from the renderer, preventing a compromised renderer from spoofing user gestures for permissions. (89632d8)
  • Security has been hardened in the Windows printing system by adding sanity checks for record sizes in PostScript metafile playback. (33d6bfa)
  • Bounds checking was added to the VP8 accelerator to prevent a potential unsigned integer underflow when decoding malformed video streams. (42ff1bc)

Networking

  • The kLocalNetworkAccessPromptDirectSockets feature is now enabled by default. Users will be prompted for permission when a site requests access to local network devices via the Direct Sockets API. (d96e669)
  • For improved debugging, NetLog will now track pre-existing WebSocket connections by creating synthetic "alive" events when a log capture begins. (89a10e5)
  • The kAddAutomaticWithDohFallbackMode feature for DNS-over-HTTPS is now enabled by default, acting as a killswitch to decouple the DoH fallback mechanism from its UI settings. (ab42a0a)

Browser & APIs

  • Observation support has been added to the AccountSettingService, allowing embedders to monitor for account setting changes. (7b12fa9)
  • A memory leak in the FrameSessionTracker for speech recognition was fixed, where ended sessions were not being properly unregistered. (5f349d8)
  • A bug was fixed in the D3D12 video decoder on Windows where it could read stale bitstream data from previous frames, correcting potential visual corruption. (5b73734)
  • For WebView2, the static library WebView2LoaderStatic.lib has been added to the distribution. (a631df7)

Base & Core Utilities

  • To improve performance, many calls to GURL::GetContent() were replaced with the more efficient GURL::GetContentPiece() to avoid unnecessary string allocations. (642c40e)
  • A bug in base::RepeatingCallback::operator= was fixed to prevent incorrect behavior when using WithBoundArguments. (f7483ce)
  • Re-entrancy is now explicitly allowed for ObserverList in PrintJob::Observer and DisplayManager::Observer, improving stability in observer patterns. (088627e) (1186933)

Testing & Infrastructure

  • A new test-only method, SetUrlFamiliarForTesting, was added to the SiteFamiliarityFetcher to allow browser tests to override site familiarity verdicts for more consistent test environments. (82c8f69)
  • InProcessBrowserTest now better handles asynchronous browser activation to prevent race conditions in tests for WebUI-based features. (430b88a)
