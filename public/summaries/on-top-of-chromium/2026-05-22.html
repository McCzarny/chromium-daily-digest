Chromium Digest: 2026-05-22

Overview

This summary covers 703 commits on the main branch for May 22, 2026, with 507 identified as relevant to developers. Key changes include major API refactorings, the finalization of Manifest V2 deprecation steps, and significant updates to the build system. View the full log from (e5b651c) to (c14e0f1).

Extensions

  • ⚠️A BREAKING CHANGE as the kExtensionManifestV2Disabled feature flag has been removed. This forces all Manifest V2 extensions into the "disable" phase, removing the intermediate "warning" stage. Embedders relying on this flag to delay the transition must update. (e768ae9)
    ExtensionsBrowser
  • Security for extension background pages was hardened with per-frame deduplication and strict activity matching for lazy keepalive decrements, preventing premature termination by a compromised renderer. (4a7596d)
    ExtensionsSecurity
  • Fixed a null-dereference crash in the bookmarkManagerPrivate.drop API. The code now safely handles cases where drag data is missing, preventing a potential browser crash. (9d324e9)
    ExtensionsBookmarks

Browser & Core APIs

  • ⚠️BREAKING CHANGE: The core Tab Strip service API has been moved from chrome/browser/ui/tabs/tab_strip_api to a new component at components/browser_apis/tab_strip. This changes build dependencies and header include paths for embedders using this public API. (d10436b)
    BrowserTabsAPI
  • ⚠️BREAKING CHANGE: The public method AccountTrackerService::Initialize has been removed. Its logic is now in the constructor, requiring PrefService* and base::FilePath on instantiation. Embedders must update to a single-step initialization. (c0a736f)
    SyncProfilesAPI
  • ⚠️BREAKING CHANGE: The public method Browser::is_delete_scheduled() has been removed. Code must be updated to use BrowserWindowInterface::IsDeleteScheduled() instead. (1353217)
    BrowserAPI
  • ⚠️BREAKING CHANGE: The behavior of WebContents::GetFocusedWebContents() has changed. It can now return nullptr in scenarios involving embedded WebContents, which may cause null pointer dereferences in embedder code that previously assumed a non-null return value. (1ad0580)
    BrowserAPIUI
  • ⚠️BREAKING CHANGE: The base::FileTracing::Provider interface has been removed. File tracing is now handled internally within base::File using Perfetto, removing the public API for supplying a custom file tracing provider. (6855e06)
    BaseInfra
  • A new IcuBridge class has been introduced in base/i18n to abstract away direct calls to the ICU library. This is in preparation for supporting different ICU backends. (d829192)
    BaseI18n

Build System

  • ⚠️BREAKING CHANGE: The build system for WebUI has been significantly modularized. Files from the monolithic //chrome/browser/ui:ui target have been moved to new, smaller source sets (e.g., //chrome/browser/ui/webui/). Build systems with direct dependencies on old file locations will break. (9694359)
    BuildUIInfra
  • ⚠️BREAKING CHANGE: fuzzable_proto_library targets now produce shared libraries (components) instead of static libraries in component builds. This change, made to fix ODR violations, will break custom build configurations that relied on static linking. (0fcbba9)
    BuildTestingInfra

UI & UX

  • ⚠️BREAKING CHANGE: The components/eye_dropper/EyeDropperView has been refactored to use a controller pattern and no longer inherits from views::WidgetDelegateView. This changes the public API's ownership model and structure, breaking embedders that use this component. (23a8d51)
    UIAPI
  • ⚠️BREAKING CHANGE: Support for the "borderless" value for the display manifest member and display-mode CSS media feature has been removed. It was an alias for "unframed", which must now be used exclusively. (8df25b6)
    UICSS
  • A new feature for global Most-Recently-Used (MRU) tab switching with Ctrl-Tab has been implemented. Controlled by the kCtrlTabMru flag, it allows users to toggle between the two most recent tabs across all windows. (2e673d0)
    UITabsBrowser
  • On modern macOS keyboards, the text placeholder (fn) for function key shortcuts is now replaced with the proper monochrome globe symbol (🌐︎), aligning Chrome's UI with native platform appearance. (9917a2e)
    UImacOS
  • The tab loading throbber animation is now compositor-driven, eliminating a repeating timer to improve rendering performance. This may result in a slight visual change to the animation's 'sweeping arc'. (cca3271)
    UIPerformanceCompositor

Graphics & Printing

  • ⚠️BREAKING CHANGE: For security hardening, MapBufferRange, UnmapBuffer, and FlushMappedBufferRange have been removed from the public gpu::gles2::GLES2Interface. This breaks embedders who use this low-level graphics interface directly. (7cc1fc6)
    GPUAPISecurity
  • ⚠️BREAKING CHANGE: The printing::mojom::PrintManagerHost Mojo interface has been refactored. UpdatePrintSettings was renamed to GetPrintPreviewParams and its method signature was altered, breaking the IPC contract for print preview. (ea38e74)
    PrintingAPI

Networking & Security

  • A new option, server_padding, has been added to net::SSLServerConfig. This allows embedders to enable TLS server-side padding as a defense against traffic analysis. (6eb3ab2)
    NetworkSecuritySSL
  • On Windows, support has been added to force Address Space Layout Randomization (ASLR) for sandboxed child processes, making them harder to exploit. Embedders can override this via the new ContentBrowserClient::ShouldForceASLR() method. (09eaeae)
    SecurityWindows
  • Download path validation was updated to prevent symlink traversal attacks by ensuring the final resolved file path is strictly contained within the intended download directory. (3cf6610)
    SecurityDownloads
  • Fixed a use-after-free crash in the ProxyResolver service that could occur when multiple resolvers shared a V8 isolate. The fix ensures stale cleanup tasks are properly pruned. (69cbd06)
    NetworkV8

Autofill

  • As part of the Email Verification Protocol (EVP) feature, the AutofillClient interface was updated with a new ShowEmailVerificationPopup method to display a permission prompt before sharing a verification token. (e4c8c19)
    AutofillUI

Testing

  • The 'Kombucha' interactive testing framework was updated to support non-blocking mouse gestures. Test APIs like MoveMouseTo now use a callback chain instead of nested run loops, a significant change for authors of interactive UI tests. (735d345)
    TestingUI
  • New GTest matchers (FormFieldDataEq, FormDataEq) were introduced for comparing Autofill data structures. They provide more detailed property-level mismatch messages, improving the clarity of test failures. (0611378)
    TestingAutofill
