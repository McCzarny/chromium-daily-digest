Chromium Digest: 2026-02-26

Overview

A significant day with 682 commits, of which 492 were deemed relevant to developers building on Chromium. The day was marked by a large number of breaking API changes across the base, extensions, ui, and network components, requiring careful review by embedders. Changes spanned from core utility removals to significant refactoring of observer and network interfaces. The first relevant commit was (76aa3b0)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((76aa3b0)) and the last was (db2d44a)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((db2d44a)).

Breaking Changes

  • ⚠️ BREAKING The TabListInterfaceObserver API was broken in multiple commits. A new virtual method OnHighlightedTabsChanged was added, and later, the signatures of all observer methods (e.g., OnTabAdded, OnTabRemoved) were changed to require a TabListInterface& as the first parameter. Implementations will fail to compile until updated. (6a0e2c0) (aa6b880) (8cdbdb6)
  • ⚠️ BREAKING The base::VariantMap class has been completely removed from the codebase. All usages have been replaced with absl::flat_hash_map. Any project relying on base::VariantMap will break. (e3feeef)
  • ⚠️ BREAKING The extensions::ContentHashReader class was removed from the public header extensions/browser/content_hash_reader.h and replaced with a free function. This is a breaking API change for code that directly used this class for extension content verification. (881021b)
  • ⚠️ BREAKING Multiple public network and content APIs have had their signatures changed: - The Mojo interface network::mojom::NetworkContextClient::OnFileUploadRequested changed its process_id parameter from int to the strongly-typed ChildProcessId. - The function content::service_worker_client_utils::NavigateClient now requires the worker's security state to be passed for Local Network Access (LNA) checks. (837710a) (ffbeba9)
  • ⚠️ BREAKING The remoting::SignalingMessage variant has been refactored to hold a JingleMessage instead of an XmlElement, breaking consumers of the remoting API who construct or inspect these messages. (ada63d3)
  • ⚠️ BREAKING The legacy PwaInstallView class and its associated feature flag kPageActionsMigrationPwaInstall have been removed from chrome/browser/ui/views/page_action/, affecting embedders who may have used this UI view directly. (7f0d235)
  • ⚠️ BREAKING The Compositor (cc) public API has been changed: - The PaintOpBuffer::DeepCopyAsRecord method signature in cc/paint/paint_op_buffer.h is now const, and its behavior was modified. - The public method drawElementImagePlaceholder has been removed from the cc::RecordPaintCanvas class. (d35f1ea) (4588ad6)
  • ⚠️ BREAKING The PersistentCache API in components/services/storage/ has been updated. The key parameter for Find and Insert methods was changed from std::string_view to base::span<const uint8_t>, requiring callers to adapt to the new binary data type. (8a8c079)
  • ⚠️ BREAKING The command-line switch kEnableBenchmarking has been moved from components/variations/variations_switches.h to base/base_switches.h. Code referencing the old path will fail to compile. (7a6bb4c)
  • ⚠️ BREAKING The AccessibilityAnnotationSpecifics sync proto has been updated. Fields representing dates as int64 unix timestamps have been removed and replaced with a new NaiveDate message type, breaking components that directly serialize this sync entity. (0f43be9)

Core Internals & Tooling

  • The raw pointer overloads for base::File::Read and base::File::Write were briefly made private but the change was reverted due to build failures. Developers should be aware of the intent to migrate to base::span-based APIs. (ad342bc) (0c81119)
  • A new, more ergonomic one-shot AEAD encryption/decryption API was added to crypto/aead.h. (9ad1480)
  • The reentrancy policy for several observer classes, including RenderFrameObserver and DevToolsAgentHostObserver, was explicitly set to kAllowed to formalize existing behavior and prevent future crashes. (009262e) (58dc881)

Browser UI & Features

  • On macOS, Picture-in-Picture (PiP) windows will now correctly move to fullscreen spaces, improving the user experience. (af7cb90)
  • Support for interactive doodles on the New Tab Page has been removed. The underlying Mojo interfaces and handler logic have been deleted, simplifying the NTP implementation. (45ede36)
  • A use-after-free crash in MenuController was fixed by replacing a raw pointer with a ViewTracker, improving UI stability. (8edca3f)
  • A crash in Chrome for Testing and Headless Mode when running with --disable-infobars has been fixed. (e0f7f07)

Networking & Services

  • P2PSocketUdp on Windows was enhanced to retry sending packets after an ERR_NO_BUFFER_SPACE error, improving reliability for WebRTC. (be46c60)
  • The experimental SQL-based disk cache backend received new feature parameters (SqlDiskCacheReleaseMemoryAfterWrites, SqlDiskCacheCacheSize) to allow for fine-tuning of its SQLite configuration. (f8128f4)
  • The TcpConnectJob has been made more robust by ensuring it fully shuts down upon completion to prevent dangling pointers. (1d6f53a)

Extensions

  • The bluetoothPrivate API was converted from the legacy JSON schema format to the modern WebIDL format as part of ongoing modernization efforts. (48b52a5)
  • A new install prompt type, EXTENSION_PARENT_APPROVAL_PROMPT, was added for the supervised user extension installation flow. (5d0acf8)
  • The responsibility for persisting and restoring webRequest event listeners has been migrated to the central EventRouter, replacing a legacy ad-hoc implementation. (223feef)

Security

  • To prevent potential phishing, main-frame navigations to chrome://image are now blocked by a navigation throttle. This URL is intended only for subresource loading in WebUI. (ef26f22)
  • A new kill switch, kRevokeTokensWithInvalidCharactersOnLoad, was added to validate and clear invalid refresh tokens on load to prevent later crashes. (fad8b7e)
  • A crash in TokenBindingHelper was fixed that occurred when an empty binding key was passed to the crypto backend. The code now correctly handles this case. (6323f50)

Developer Tools

  • A new DevTools protocol method, Emulation.setPrimaryScreen, was added for headless automation, allowing tests to simulate different primary screen configurations. (2f1253d)
  • Cookie handling logic has been refactored to centralize permission checks into a new static method, NetworkHandler::CanAccessCookie, providing a unified and more secure way to manage cookie access for the Network and Storage domains. (9a6fcf2)
