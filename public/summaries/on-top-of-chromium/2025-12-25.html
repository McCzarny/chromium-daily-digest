This summary highlights a significant breaking change in the Task Manager API, along with updates to the browser UI, core components, and testing infrastructure.

APIs and Extensions

  • ⚠️ BREAKING Introduces a breaking change to the task_manager::TaskManagerInterface. Several methods, including GetMemoryFootprintUsage, GetGpuMemoryUsage, and GetSqliteMemoryUsed, have had their return types changed from base::ByteCount to std::optional<base::ByteSize>. Other methods like GetV8Memory now use base::ByteSize for output parameters. Embedders or extensions using this interface to query process memory information will need to update their code to handle the new types and optional return values. (3010a9b)

Browser UI

  • Adds a preliminary implementation for updating loading animations in the Vertical Tabs UI via the UpdateLoadingAnimations interface. This is part of the ongoing development of the vertical tabs feature. (1267e59)
  • Enhances print preview metrics by adding tracking for the 'actual size' PDF scaling option. A corresponding UI label has also been added to the menu. (b13ef07)

Core

  • Fixes a crash in base::FeatureList that could be triggered by malformed input to the --enable-features command-line parameter. This improves browser stability during startup when using feature flags. (762aea6)
  • Adjusts the experimental WebNN API to use BEST_EFFORT thread priority for background ONNX Runtime (ORT) session creation. This performance tweak aims to prevent long-running WebNN tasks from blocking higher-priority browser tasks. (7a08eaa)

Build & Test Infrastructure

  • General updates to infrastructure include automated test sharding configuration, cleanup of internal team mentions in metadata files, and an adjustment to a presubmit script related to MiraclePtr. These changes do not affect browser functionality. (8388d6b) (7df688c) (7bc784a)
