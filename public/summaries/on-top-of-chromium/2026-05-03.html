Chromium Digest: 2026-05-03

Overview

This digest covers 22 commits, with a focus on changes relevant to Windows and macOS embedders. After filtering out platform-specific (iOS, Android, Ash) and internal rendering engine changes, several key updates were identified in Autofill, browser internals, and networking. The most notable change is a breaking API modification in the Autofill component, which removes a virtual method from a public class.

Autofill

  • ⚠️ BREAKING The BrowserAutofillManager::DidShowSuggestions() method was changed from virtual to non-virtual. This is a breaking API change for any downstream code that inherits from BrowserAutofillManager and overrides this method, as it will now cause a compilation failure. The intended pattern is now to use the AutofillManager::Observer::OnSuggestionsShown() callback. (bc91d7f)

Browser Internals

  • The UnloadController, which manages the browser closing process (e.g., beforeunload events), was refactored. It is no longer a direct member of the Browser class but is now owned by BrowserWindowFeatures. Access patterns have been updated to use UnloadController::From(browser). While a significant internal change, the public API of the Browser class for closing remains functionally unchanged. (beaae0f)
  • A bug fix in the Windows touch keyboard implementation now correctly respects the autocorrect="off" HTML attribute, preventing unwanted text replacements. This aligns behavior with web standards. (02d88f7)

Networking & Storage

  • Internal optimizations were made to the Off-Main-Thread Prefetch (OMTPrefetch) mechanism to deduplicate in-flight tasks and improve the header cache refresh logic, enhancing performance and reliability of preloading. (13008e3) (c83f29a)
  • A bug in the sandboxed SQLite Virtual File System (VFS) was fixed. The VFS now correctly treats zero-length files as non-existent, aligning with the behavior of built-in VFSes and preventing issues with journal and write-ahead log file handling. (c14174a)

Infrastructure

  • Minor updates include a comment clarification in ui/views/scoped_paint_lock.h and a routine update to internationalization string resources. (5e2ce33) (8f666cc)
