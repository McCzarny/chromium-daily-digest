Chromium Digest: 2026-04-20

Overview

This summary covers 440 relevant commits out of 601 total, from commit 46394ace988541e7481f9446e9472fb3ea420dac to bbad371561e34913370ececf50bc29959838b7f7. The day was marked by a significant number of breaking changes impacting core networking, file utilities, Autofill, Remoting, and the Privacy Sandbox. Other notable updates include API enhancements for extensions, security fixes, and continued efforts to improve code health through modularization and refactoring.

Breaking Changes

  • ⚠️ BREAKING The core networking class net::URLRequest has changed method signatures. Methods that previously returned byte counts as int64_t now return the unsigned type net::ByteSize. Embedders will need to update their code to handle the new type. (8ade05b)
  • ⚠️ BREAKING The behavior of the core file utility base::DeletePathRecursively has been altered on Windows. It will no longer follow reparse points (e.g., directory junctions, symlinks), aligning its behavior with POSIX systems. This could silently break code that relies on the previous behavior of deleting linked content. (4ca74d8)
  • ⚠️ BREAKING The public virtual method TriggerPlusAddressUserPerceptionSurvey has been removed from the autofill::ContentAutofillClient interface. Embedders who subclass this client and override the method will encounter a build failure. (2f6d1f8)
  • ⚠️ BREAKING A large number of public header files and some method overloads related to the legacy Chromotocol protocol have been removed from remoting/protocol. This includes headers for PseudoTCP, datagram sockets, and various channel factories, significantly impacting any embedder using these legacy components. (053d46b) (76fc706) (59bb995)
  • ⚠️ BREAKING APIs related to the Privacy Sandbox have been modified. The public enum PrivacySandboxService::PromptSuppressedReason has been removed. Additionally, the custom_key_commitment and custom_issuer fields have been removed from the public Mojo interface network::mojom::TrustTokenParams. (41e4d73) (e4f0243)

APIs & Extensions

  • The chrome.debugger API has been updated to expose the WebMCP domain, allowing developers to experiment with the feature directly from extensions. (71a14cd)
  • The TabStrip API was enhanced to support context menus, and the Glic component extension now supports glic.invoke. (b796e06) (d1bdfa4)
  • Several deprecated or unused extension APIs have been removed, including the webrtcAudioPrivate API and the chrome.appviewTag API on desktop platforms. (6f72765) (a7c9dd8)
  • A bug was fixed where the userVisibleOnly: false setting for silent push notifications was not persisted across browser restarts. The setting is now correctly saved in user preferences. (dd5be66)
  • Internal PWA logic was refactored to better decouple installation and launch logic by exposing more app control to the WebContentsManager. (062e13d)

Networking & Security

  • A new net::CryptoPrivateKey class has been introduced to facilitate migrating away from BoringSSL-specific APIs for private key operations, improving cross-platform compatibility. (d0a78a5)
  • A security fix prevents a potential leak of device attributes to regular HTTPS origins that are listed in an Isolated Web App's scope_extensions. (8e46d53)
  • A bug in the Happy Eyeballs v3 implementation that was breaking TLS 1.2 renegotiation for client certificate authentication has been fixed. (51bda44)
  • The enterprise policy PrivacySandboxPromptEnabled has been deprecated and its enforcement logic removed, as the associated user prompt is being removed from the browser. (ff025fc)
  • A bug was fixed to prevent sensitive IBANs from being stored in the autocomplete history table. (245dd79)

Browser UI & Features

  • The refreshed Infobar UI and its prioritization logic have been enabled by default. (e6f0526)
  • The ToolbarButtonProvider has been updated to support UnownedUserData, allowing clients to depend on it without a direct dependency on BrowserView or Browser, improving component decoupling. (d37c85a)
  • A performance issue in TranslatePrefs that caused unnecessary disk writes on every navigation was fixed by adding an early-out and optimizing preference updates. (b27227f)
  • A use-after-free crash in the TabStripActionContainer was resolved by using a WeakPtr for an animation callback. (213ddd4)

Autofill & Passwords

  • The Credential Management API (navigator.credentials.get()) has been updated to correctly respect the user's biometric re-authentication setting. (6b43fb9)
  • The internal PasswordStore implementation has been refactored to use the modern StoredCredential struct instead of the legacy PasswordForm, as part of an ongoing modernization effort. (1b95806)
  • A crash was fixed that could occur when merging Autofill profiles that contained phone numbers with invalid country codes. (333c435)

Core Internals & Testing

  • A significant modularization effort was completed for the //chrome/browser/device_reauth component, moving it out of the monolithic //chrome/browser target to improve dependency boundaries and build times. (dab3955)
  • A widespread refactoring effort replaced repeated parsing of the New Tab Page URL with cached GURL objects across many components to improve performance. (ec0f7b5) (9e92c17) (9abdf92) (910f0e9)
  • Support for Google FuzzTest has been enabled for Windows builds, expanding the availability of this modern fuzzing engine. (be6f9db)
  • Several obsolete classes from the legacy audio input path have been removed, including AudioInputStreamHandle, MojoAudioInputStream, and AudioInputDelegate. (a3de08e)
