Summary for 2026-02-09 on branch 'main'

Chromium Digest: 2026-02-09

Overview

This summary covers 615 commits on the main branch from February 9th, 2026, with a focus on 451 changes relevant to developers building on Chromium. The day was marked by numerous breaking API changes across UI, networking, security, and core components, requiring code updates for embedders. The first relevant commit was (bcb84ec) and the last was (97b3f24).

Breaking Changes

  • ⚠️ BREAKING API Removal: The public virtual method ViewAccessibility::OverrideFocus has been removed from ui/views. Embedders must migrate to the new aria-activedescendant pattern by using ViewAccessibility::SetActiveDescendant. (30d8cde)
  • ⚠️ BREAKING API Removal: The public methods BrowserList::CloseAllBrowsersWithProfile() and BrowserList::CloseAllBrowsersWithIncognitoProfile() have been removed. Embedders must implement alternative logic for closing browsers associated with a specific profile. (d29ce19)
  • ⚠️ BREAKING API Removal: The public virtual method WindowController::CanClose() has been removed from the Extensions API. Embedders must now implement this check directly. (6c171b9)
  • ⚠️ BREAKING API Signature Change: The AutofillClient interface now includes a new pure virtual method, ShowAutofillAiFailureNotification. Embedders with custom AutofillClient implementations must add this method to avoid build failures. (095b4dd)
  • ⚠️ BREAKING API Signature Change: The ui::Clipboard methods ExtractCustomPlatformNames and ReadAvailableStandardAndCustomFormatNames are now asynchronous and require a callback to receive results. (63af498)
  • ⚠️ BREAKING API Signature Change: The DnsTransactionFactory::CreateTransaction method signature has changed. The bool secure parameter is replaced by a new AttemptMode enum to specify DNS or DNS-over-HTTPS. (17be678)
  • ⚠️ BREAKING API Signature Change: The UnexportableKeyService methods DeleteAllKeys and DeleteAllKeysSlowlyAsync (and their Mojo equivalents) no longer accept a priority parameter. (efa2d3b)
  • ⚠️ BREAKING API Signature Change: The PasswordChangeServiceInterface::IsPasswordChangeSupported method now accepts a const password_manager::PasswordForm& instead of a const GURL&. (4deeba3)
  • ⚠️ BREAKING API Signature Change: The core allocator setup function base::allocator::PartitionAllocSupport::ReconfigureAfterFeatureListInit now accepts a FeatureListConfiguration struct instead of multiple boolean parameters. (14484c4)
  • ⚠️ BREAKING Component & API Change: The ResumableUploadRequest class has moved to components/enterprise/connectors/core and its constructor/factory methods now require a scoped_refptr<base::SequencedTaskRunner>, breaking existing instantiation patterns. (72adb91)

Browser UI and Core

  • A new menu item, "Report an unsafe site," has been added to the main application menu, with command handling in BrowserCommandController. (f5b26be)
  • The User Education system now prevents feature promos from appearing while an automated "Agent" is actively controlling a tab. (dd23914)
  • The Tab Search data model is now prevented from updating while it is being preloaded but not visible, improving performance. (43d2930)
  • The First Run Experience (FRE) features a refreshed design for the "set as default browser" screen. (ce66f9f)
  • A bug was fixed in the side panel toolbar that caused it to persist incorrectly after switching modes due to a use-after-move. (e8a9b02)

Security and Privacy

  • A new SearchIntegrity component has been introduced to monitor search engine configurations against an allowlist of trusted providers. (bf13717)
  • A new policy, WebRtcDiagnosticLogCollectionAllowedForOrigins, has been added to allow enterprise control over the WebRTC diagnostic logging API. (22400f0)
  • For improved privacy, the mechanism for tracking deleted profiles now uses relative basenames instead of full, absolute paths to avoid leaking system information. (95b8d19)
  • The PermissionServiceImpl now validates that incoming permission requests are not empty, hardening the service against malformed requests from a renderer process. (3258f06)

Networking

  • The RenderDocument feature and its SkipRendererCancellationThrottle optimization have been enabled by default, which may alter navigation cancellation behavior for improved performance. (b4d0474)
  • The rollout of NetworkServicePerPriorityTaskQueues was reverted due to a suspected bug. (03e9c1a)

Base, Performance, and Core Utilities

  • On Windows and macOS, the memory suppression features kSuppressMemoryListeners and kSuppressMemoryMonitor are now enabled by default to reduce memory pressure notifications. (bed11e0)
  • On macOS, file quarantine logic in base/mac/mac_util.mm now uses the official NSURLQuarantinePropertiesKey API instead of a private extended attribute. (0b330d2)
  • A new experimental CPU Performance API has been implemented to provide performance information based on CPU manufacturer, core count, and model. This is not yet exposed to web content. (1b46924)

Testing and Infrastructure

  • The base::test::TestFuture utility has been enhanced with a kQueue mode, allowing it to queue multiple values for testing scenarios with multiple asynchronous callbacks. (0223ed4)
  • An issue where DevTools clients would fail to interact with unloaded tabs has been addressed by forcing the WebContents to render when a client auto-attaches. (216858a)
  • The Chromium Updater will now upload its history log in crash reports to aid in debugging client-side update issues. (ad48b67)
  • The signin_util in end-to-end tests was updated to support signing into the browser without enabling full sync, improving test scenarios for features like supervised users. (3efcdcf)
