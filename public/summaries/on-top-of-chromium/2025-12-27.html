Core APIs & Components

  • ⚠️ BREAKING BREAKING: The UserManager::Observer and UserSessionStateObserver interfaces now inherit from base::CheckedObserver and provide default empty implementations for their virtual methods. Previously, these methods were declared without default implementations, forcing observers to implement them to avoid link-time errors. This change in the observer contract means existing code may compile but exhibit different runtime behavior if it was implicitly relying on the old contract. (063bace)
  • ⚠️ BREAKING BREAKING: The ui::FormatBytes utility function, along with related functions like ui::FormatSpeed, has been migrated to use the strongly-typed base::ByteSize class instead of primitive types like int64_t or the base::ByteCount helper. This is a public API signature change that will cause build failures for embedders calling these functions directly. Code must be updated to use the new base::ByteSize type for formatting byte values. (eae5d29)

Refactoring & Internal Improvements

  • The TabStripService, which provides the backend for the Tab Strip API, has undergone significant internal refactoring. This work introduced a centralized validation framework to improve code clarity, maintainability, and testability without altering the public API surface. (fa1d9f8) (49383bd)
  • A minor refactoring in base/synchronization/waitable_event_win.cc now uses base::span to fix an UNSAFE_TODO. This improves type safety for this low-level, Windows-specific synchronization primitive. (306590f)

Bug Fixes

  • Several fixes have been applied to the experimental "Contextual Tasks" feature. These changes address issues with data fetching from URLs, incorrect navigation handling, and UI animations in the zero-state overlay. (76d90be) (1bed1ea)
