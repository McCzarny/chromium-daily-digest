Chromium Digest: 2026-04-01

Overview

This report covers 684 commits on the main branch, with 554 deemed relevant to developers building on Chromium. The day's changes include a significant number of breaking API changes across core content, media, UI, and permissions systems, requiring action from embedders. Key themes include security hardening in drag-and-drop and navigation, the graduation of the SharedWorker API to a stable feature, and ongoing modularization of the codebase. Breaking Changes

  • ⚠️ BREAKING The core drag-and-drop API has been changed for security reasons. The signature of content::RenderWidgetHostImpl::StartDragging now requires a RenderFrameHost* as a direct parameter. The content::DragDownloadFile constructor signature was also significantly modified. Embedders using these APIs must update their call sites. (988ac5c)
  • ⚠️ BREAKING The permission.mojom interface was updated. The return types for RequestPermissions, RevokePermission, and AddPermissionObserver have been changed from a simple status to a PermissionStatusWithDetails struct, breaking clients of this Mojo API. (7052b70) (4e04747) (36e8c6e)
  • ⚠️ BREAKING As a security measure, the Presentation API has been hardened to only permit http:// presentation URLs for localhost. This is a significant behavioral change that prevents its use for discovering arbitrary devices on a local network over insecure HTTP. (52c5e88)
  • ⚠️ BREAKING Multiple public C++ API signatures were changed, requiring updates in derived classes or call sites: - media::DataSource::IsStreaming() and WouldTaintOrigin() are now const. - BrowserAutofillManager::FillOrPreviewForm signature was modified. - AutofillClient::GetDeviceAuthenticator is now const. - views::Widget::InitParams no longer has the force_system_menu_for_frameless parameter on Windows. (b156700) (6d25e4a) (109dc40) (7c5af3a)
  • ⚠️ BREAKING Several public APIs were removed or renamed: - GetHDRMetadata was removed from the media::AcceleratedVideoDecoder interface. - AccessibilityAnnotationService was renamed to AccessibilityAnnotatorService. - blink::WebString::ContainsOnlyASCII() was renamed to ContainsOnlyAscii(). - The Mojo Rust FFI crate was moved to mojo/public/rust/c_mojo_api, a build-breaking path change. (72e2fc5) (d133671) (e798b84) (84d3aff)

Security

  • Core navigation code was significantly hardened by upgrading numerous DCHECK assertions to CHECKs in NavigationRequest and RenderFrameHostManager. This ensures that internal state inconsistencies will cause a safe crash in release builds, preventing potential security vulnerabilities from corrupted states. (662ea60) (9acb98e) (bf94625)
  • The DevTools protocol was hardened by preventing Input.dispatchDragEvent from bypassing file access restrictions and by restricting experimental page commands to trusted clients only. (59d7444) (d283fa5)
  • A vulnerability was fixed in ui/accessibility where a compromised renderer could bypass bounds clipping on the root web area, potentially allowing it to spoof accessibility focus at arbitrary screen locations. (847f299)
  • The javascript: scheme is now disallowed when setting a home page URL via drag-and-drop to the home button, preventing a potential misuse vector. (b920b51)

APIs and Extensions

  • The SharedWorker API is now enabled by default on all platforms, graduating from its previous experimental status to become a stable web platform feature. (6f4c113)
  • The WebDialogDelegate interface was updated to allow dialogs to specify whether they should process keyboard accelerators. This fixes a regression and improves behavior for components like the file selection dialog. (3d18890)
  • The DICE SIGNIN header parser was updated to support a new hierarchical multi-account format, improving multi-account handling for identity services while maintaining backward compatibility. (d203523)
  • The ExtensionContentVerificationUsesExtensionRoot feature, which alters how extension content integrity is verified, has been enabled by default after a successful experiment. (2aa5824)

Browser UI and Features

  • On macOS, mouse movement event coalescing has been disabled to allow for a higher polling rate. This benefits applications that require high-frequency input, such as games, by providing more responsive mouse input. (80792e0)
  • A significant regression on Windows was fixed where opening URLs from external applications would not respect the current workspace, instead always opening in the last active browser across all workspaces. (0dfe151)
  • Several tab dragging bugs were resolved, including a deadlock related to deferred window visibility and an issue on macOS where dragging the last tab from a fullscreen window would incorrectly move the entire window. (0cdc11e) (9ed9fc9)
  • The logic for theme garbage collection was improved to prevent the incorrect removal of a user's local theme when it is temporarily overridden by a synced theme. (8ab097c)

Media

  • On Windows, a use-after-free bug in MediaFoundationVideoEncodeAccelerator was fixed by decoupling the encoder's lifetime from OS callbacks, improving stability during video encoding. (70f35b0)
  • On macOS, stricter validation was added to the ScreenCaptureKit implementation to prevent unconsented fallback screen capture when using the native picker, enhancing user privacy. (f77c6f1)

Build System and Internals

  • The effort to break down the monolithic //chrome/browser target continued. The offline_pages, plugins, printing, and spellchecker components were moved into their own build files, improving build dependency management and parallelism. (b87be23) (dc7f841) (0790c8f) (0234876)
  • As part of the "Bedrock" UI refactoring initiative, multiple call sites were migrated from the legacy Browser object to modern abstractions like BrowserWindowInterface and GlobalBrowserCollection. (524e16b) (765e765)
  • Test reliability was improved by replacing a flaky RunUntilIdle() call in accessibility tests with a more deterministic wait helper, FlushPendingViewsAXUpdates(). (03b5ae6)
