Chromium Digest: 2025-11-29

Overview

This digest covers 7 commits from the Chromium repository. After filtering for changes relevant to Windows and macOS embedders and excluding platform-specific updates (Android, iOS, ChromeOS), internal refactoring, and routine updates, the key changes focus on UI enhancements, enterprise policy additions, and core stability improvements. No major breaking changes were identified.

Browser UI & Features

  • An experimental 'Open all profiles' button has been added to the profile picker UI. This feature is controlled by the kOpenAllProfilesFromProfilePickerExperiment flag and allows users with fewer than five profiles to open all of them simultaneously. The implementation touches the profile picker's WebUI and Views components. (ba97eda)

Enterprise & Policy

  • Enterprise reporting capabilities have been enhanced to include policy information in certificate authority authorization (CAA) signals reports. This change affects the ProfileReportGenerator and related services, adding more data to security reports without altering existing APIs. (ea46643)

Core & Internals

  • A stability improvement was made in the MemoryPressureListenerRegistry by converting a CHECK macro, which would cause a crash, into a conditional check. This prevents potential crashes in production under certain memory pressure conditions. (1d4aaff)

Build & Infrastructure

  • Routine updates were made to translation files (.xtb) across various components to align with the latest source strings. (8d3531d)
