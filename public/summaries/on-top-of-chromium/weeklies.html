Chromium Weekly: 2025 Week 48

Overview

A turbulent week in Chromium development saw 1933 relevant commits out of 2680 total, introducing a vast number of breaking changes across the codebase. The most significant development was the enabling of the Network Service Sandbox by default on Windows, a major security milestone. Multiple core APIs in networking, identity, and UI were modified, requiring significant attention from embedders. Instability was also a theme, with a major refactor of the AccountInfo identity struct being partially reverted after causing crashes.

Security

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The Network Service Sandbox is now enabled by default on Windows. This is a major behavioral change that enhances security by restricting the network process's access to the system, but it may break embedders who rely on the previous, more permissive environment. (c6bd7f0)
  • The remote debugging server has been hardened with a new "approval mode" controllable via the kDevToolsRemoteDebuggingEnabled preference and the kDevToolsRemoteDebuggingAllowed enterprise policy. This requires user approval for new connections, improving security for developers. (2e576ad) (2748580)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The core WebAuthn public API AuthenticatorCommon::MakeCredential has been updated to include a new payment_options parameter, requiring embedders to update their calls. (5d55cd3)
  • An initial implementation of a proxied Unexportable Key Service was added, laying the groundwork for the network process to interact with hardware-backed keys (like the macOS Secure Enclave) via the browser process. (269bac8) (a6cfb36)

Core APIs & Internals

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The AccountInfo identity struct underwent a volatile refactoring to privatize the hosted_domain field. The changes, which required all consumers to use a new GetHostedDomain() accessor, caused crashes and were partially reverted late in the week. Embedders should monitor this area closely for instability. (da1dfa4) (d85bfc8) (3428bfe) (659bbd1)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The callback for the fundamental networking utility network::SimpleURLLoader::DownloadToString now receives a std::optional<std::string> instead of a std::unique_ptr<std::string>, requiring updates to all client code. (a19f517)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE A new pure virtual method, GetOAuthConsumerFromId, was added to the SigninClient interface. Embedders with custom SigninClient implementations must add this method to avoid build failures. (5657390)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE Transitive includes for ui/gfx/geometry/rect.h and rect_f.h have been removed from many public headers like content/public/browser/web_contents.h. Code that relied on these headers to implicitly include gfx::Rect or gfx::RectF will fail to build without adding the includes directly. (5ac7f34)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The base::MemoryPressureListener API will now dispatch a notification when memory pressure returns to MEMORY_PRESSURE_LEVEL_NONE. Custom listeners must be updated to handle this new notification to avoid unexpected behavior. (d139d7a)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The entire components/fingerprinting_protection_filter component has been deleted from the codebase. Projects with a dependency on this component will no longer build. (b6d76c1)

Browser UI & Features

  • The new tabbed browser layout, part of the "Chrome Next" UI refresh and supporting the Toolbar Height Side Panel (THSB), is now enabled by default. (76dd43c) (838d53b)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The experimental "Tab Scrolling" feature (scrollable-tabstrip) has been completely removed from the codebase. The tabstrip will now revert to the default behavior of shrinking tabs when the strip is full. (ef29bab)
  • Work continues on the "Tab Group Focusing" feature, with a new OnTabGroupFocusChanged method added to the TabStripModelObserver interface to notify listeners of group focus state changes. (de85415) (c7b09ea)

Extensions & Developer Tools

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The ability for extensions to delete passwords via chrome.browsingData.removePasswords has been removed. Calls to this function are now a no-op and will not affect the password store. (9fb257e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The Chrome DevTools Protocol (CDP) has moved the StyleSheetId type from the CSS domain to the DOM domain. Automation clients referencing CSS.StyleSheetId must be updated to use DOM.StyleSheetId. (23a0a9f)
  • The remote debugging server can now automatically find an available port if the default port 9222 is in use. The active server address is now displayed on chrome://inspect. (a4e85e3) (8a418d9)
  • The WebRequestPersistFilteredEvents feature is now enabled by default. This may alter the behavior of extensions using the chrome.webRequest API by changing how event listeners are handled across navigations. (49be6c3)

Enterprise

  • A new enterprise policy, SearchContentSharingSettings, was added to allow administrators to control content sharing with AI features, independent of other AI-related settings. (6e7bf35)
  • A new StaticStorageQuotaEnabled enterprise policy has been added, allowing administrators to override the default storage quota estimation behavior. (4ff2cdf)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The value savedTabGroups has been removed as a valid option from the SyncTypesListDisabled enterprise policy. Configurations using this value to disable tab group syncing are no longer valid. (1746a94)

Chromium Weekly: 2025 Week 47

Overview

This week saw over 2600 commits, dominated by a major push for API modernization that introduced numerous breaking changes across the `content`, `extensions`, `views`, and `base` libraries. A significant milestone was reached as the build system now mandates C++20. Key security enhancements include a new approval mode for remote debugging and a framework for on-disk cache encryption.

Build System

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The build system now requires C++20. The use_cxx17 build variable has been removed, and downstream projects must update their toolchains and codebases to be C++20 compliant. (e67ce2f)

Core APIs & Breaking Changes

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The public classes content::ResourceContext and content::BrowserOrResourceContext have been removed. Embedders must now use BrowserContext on the UI thread for all profile-related operations, as direct IO-thread access is no longer possible. (4752211) (954d496)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The extensions::ExtensionsBrowserClient interface has two new pure virtual methods: GetExtensionManagement() and GetExtensionActionRunner(). Embedders who implement this client must provide implementations for these new methods. (bc9fccb)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The public API for the persistent_cache component has been refactored. The BackendParams class and Backend::Initialize method have been removed, requiring embedders to adapt to a new BackendStorage and PendingBackend model. (da81816)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE Setters in the widely-used url::Replacements class now accept std::basic_string_view instead of raw const CHAR* pointers for methods like SetUsername, SetPassword, and SetHost. (667164e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The content::PluginService public API has been significantly altered. GetPluginInfo() was renamed to HasPlugin(), and the asynchronous GetPluginsAsync() method was removed in favor of the synchronous GetPlugins(). (7be86a6) (958d69c) (ced68fd)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The views::DialogDelegate public API has changed. The SetTitleMargins() and SetFootnoteMargins() methods have been replaced by a new SetFrameMargins() method. (3645cfd)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The extensions::Message struct has been converted to a class. Direct member access (e.g., message.data) must be replaced with accessor methods (e.g., message.data()). (6724f6a)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The base::MemoryPressureListener API will now dispatch MEMORY_PRESSURE_LEVEL_NONE notifications. Custom listener implementations may break if they do not expect this new notification level. (aef3061)

Security & Privacy

  • A new "approval mode" is being added to the remote debugging server. When enabled via a feature flag, new WebSocket connections will require user confirmation via a dialog and infobar, enhancing security for remote sessions. (3d94716) (cd88f7e) (7c12e9d)
  • A new framework for on-disk cache encryption has been introduced. A Mojo interface, network.mojom.CacheEncryptionProvider, allows the network service to encrypt the disk cache using OS-level cryptographic keys. (354e930) (01a412d)
  • Private Network Access (PNA) checks are now enabled by default for workers, as the kPrivateNetworkAccessForWorkers feature flag has been removed. This enforces stricter security rules for network requests initiated from workers. (a56961c)
  • The "Ask-before-HTTP" dialog, which prompts users before navigating from an HTTPS to an insecure HTTP page, is now enabled by default, improving user awareness of insecure connections. (93a5e10)
  • New enterprise policies, IncognitoModeBlocklist and IncognitoModeAllowlist, were added to give administrators more granular control over URL access specifically within Incognito mode. (97821c4)

Extensions

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The chrome.history API and related internal queries will now include "actor" visits by default. This is a significant behavioral change that will alter the results of history queries for extensions and internal UI components. (4fc6703)
  • A new feature flag, WebRequestPersistFilteredEvents, has been introduced to mitigate a race condition on startup. When enabled, it persists filtered webRequest event listeners in preferences to ensure they are loaded before a service worker can miss initial network requests. (2fdf5bd) (b604380)
  • The sidePanel.onClosed event is now officially exposed in the public extension API documentation, standardizing its availability for developers. (b2ce95e)

Networking & Performance

  • The BrowserInterfaceBrokerImpl is now created lazily on RenderFrameHostImpl instances. This optimization, controlled by the kLazyBrowserInterfaceBroker flag, can save 50-100 KiB of memory per background tab at startup. (82396a0)
  • The SharedDictionaryCache, used for caching compression dictionaries for dcb and dcz content encodings, is now enabled by default to improve performance on subsequent resource loads. (5af54ff)
  • The network stack now supports proxy override rules via the ProxyOverrideRules enterprise policy. These rules are evaluated before other proxy settings, allowing for more granular control over traffic routing. (956f59c) (3038c74)

Browser Core & UI

  • A new framework for managing the default browser setting has been introduced on Windows, featuring a DefaultBrowserSetter interface and an observable service to monitor for changes. (c588f44) (537f46f)
  • A new infobar prioritization system, enabled by the kInfobarPrioritization flag, prevents high-priority infobars (e.g., security warnings) from being queued behind less critical ones. (9945c13)
  • A new experimental DevTools command, Page.getAnnotatedPageContent, has been introduced to allow fetching of AI-annotated page content. This adds a new virtual method to the WebContentsDelegate public interface. (a2401fd)

Chromium Weekly: 2025 Week 46

Overview

This week saw 868 relevant commits out of 1218 total. The most significant theme was the ongoing 'Project Bedrock' initiative, which introduced breaking changes by replacing the global `BrowserListObserver` with a new per-profile API. Other major developments include the complete removal of the experimental 'Partitioned Popins' feature, the introduction of new enterprise security policies for geolocation, and performance improvements to core networking classes and the base library.

Breaking Changes & API Removals

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE As part of 'Project Bedrock', the global BrowserListObserver is being replaced by the new ProfileBrowserCollection and BrowserCollectionObserver interfaces. This major architectural change requires downstream projects to migrate from observing a global list of browsers to a per-profile collection. (886bfce) (9fbc578) (2577a13) (0e5b5d1) (1f82c9e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The experimental 'Partitioned Popins' feature has been completely removed from the codebase. This includes its controller, permissions policies, associated HTTP headers, and IDL definitions. (154f95a) (1a2cbc1) (1ce0cb5) (1b14dc3) (8c2b7da)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE Several APIs and features were removed, including the unused update_client::CrxInstaller::OnUpdateError method, the deprecated NavigationHandle::SetCorsExemptRequestHeader() API, and the internal chrome://audio diagnostics page. (d279473) (9cc9245) (d368131)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The Autofill feature for suggesting plus-based email addresses has been removed, along with the In-Product Help (IPH) feature for the 'Customize Chrome' refresh. (69b5460) (f8581d1)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The internal Autofill API AutofillDriver::ExtractForm() was renamed to ExtractFormWithField() and its signature was updated, requiring changes in any downstream code that calls it. (5063b6a) (83d041c)

Security & Privacy

  • Two new enterprise policies, GeolocationAllowedForUrls and GeolocationBlockedForUrls, were introduced, giving administrators fine-grained control to allow or block geolocation access for specific URL patterns. (4f84a5b)
  • Secure DNS (DNS-over-HTTPS) in 'Automatic' mode now supports falling back to a hardcoded list of well-known DoH providers if upgrading the system's DNS provider fails, preventing a downgrade to insecure DNS. (76cab8c) (984ddf8) (c5225eb)
  • The Navigation Gating feature, which displays a confirmation dialog before navigating to potentially sensitive URLs, has been enabled by default to enhance user safety. (b0bc944)

Browser UI & Features

  • The legacy UI controls for video Picture-in-Picture have been completely removed, finalizing the transition to the updated 2024 UI and simplifying the overlay implementation. (272e694)
  • Development on Omnibox AI features continued with the addition of a Mojo interface for the 'Answer in Omnibox' (AIM) popup, facilitating communication between the browser and its WebUI. (65727d6) (d274bf2) (6ff1965)
  • The Bookmarks Bar folder menu was enhanced with 'Open all' and 'Open all in new tab group' buttons, adding functionality previously only available in the context menu. (0e87e22)

Base Library & Performance

  • The core networking classes IsolationInfo, NetworkIsolationKey, and NetworkAnonymizationKey were refactored to use ref-counted data members, making them significantly cheaper to copy and reducing overhead in network requests. (d9614f9)
  • Two new high-performance helpers were added to the base library: base::GetAvailablePhysicalMemory() for a platform-agnostic way to estimate available memory, and base::LazyStringBuilder for building strings with fewer intermediate allocations. (d42f011) (6f3703c)

Testing Infrastructure

  • A significant effort was made to improve test security and correctness by removing the --ignore-certificate-errors command-line switch from a large number of browser tests, requiring them to use a properly configured HTTPS test server instead. (55032fa) (8f56585) (ce0ad21) (64d5855) (7b97b4a)
