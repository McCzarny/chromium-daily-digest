Chromium Weekly: 2025 Week 52

Overview

A significant week of API modernization and cleanup, this summary covers 529 relevant commits out of 852 total across 7 days. The week was dominated by a large number of breaking changes aimed at improving type safety and removing obsolete code, particularly within the Identity Manager, Task Manager, and Privacy Sandbox components. Other notable updates include functional improvements to the Extensions platform and continued development of the Vertical Tabs UI.

API Modernization & Breaking Changes

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The task_manager::TaskManagerInterface API has been modernized to use base::ByteSize and std::optional. Methods like GetMemoryFootprintUsage and GetProcessTotalNetworkUsage now return std::optional<base::ByteSize> instead of using sentinel values, requiring embedders to update their code to handle the new types. (c89f2a8) (3010a9b)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: A major refactoring of the Identity Manager API has made several members of AccountInfo private, requiring the use of new getter methods. Additionally, the components/signin/public/identity_manager/scope_set.h header and specific signin::AccessTokenFetcher constructors have been removed. (d2acdce) (454a9ad) (66263c5) (352eb26)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Several obsolete Privacy Sandbox classes have been removed, including TrackingProtectionSettings, its observer, and the TrackingProtectionOnboarding class. Embedders relying on these for managing tracking protection settings or onboarding flows will be broken. (36e492c) (98049c0) (18e568e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Core browser and UI utilities have been updated. BrowserList::empty() is removed in favor of GlobalBrowserCollection, ui::FormatBytes now requires base::ByteSize, and base::TraceLog::IsEnabled() is replaced by base::TrackEvent::IsEnabled(). The C++23 std::to_underlying now replaces the base:: equivalent. (cef649e) (eae5d29) (5051d64) (271bd17)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The UserManager::Observer contract has changed to provide default empty implementations for its methods, which may alter runtime behavior for existing observers. Separately, the return type of PrerenderHostRegistry::CreateAndStartHost has been changed from FrameTreeNodeId to PrerenderHostId. (063bace) (8eacbc5)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public methods for raw cookie access, such as GrantReadRawCookies, have been removed from ChildProcessSecurityPolicyImpl. In UI views, the virtual methods HandleWriteTextToClipboard and AllowStartDragEvent have been removed from views::TextfieldController. (41b3b0a) (fbef187)

Extensions

  • The sidePanel.open API now supports WINDOW_ID_CURRENT, allowing an extension to open a side panel in the currently active window without querying for its ID. Additionally, the runtime.onMessage event listener schema was updated to officially support returning a Promise for asynchronous responses. (beede36) (0d1975a)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public method GetAssociatedWebContents() has been removed from extensions::ExtensionFunctionDispatcher, which may impact embedders that use it. (d74add7)
  • Work continues to align input.ime.onKeyEvent key values with the DOM specification (e.g., "Ctrl" to "Control"). The change is being developed behind the InputImeKeyboardEventDataToDOMSpec feature flag to allow for testing before it becomes default behavior. (e3e9b21) (1473840)

Browser UI & Features

  • The Vertical Tabs feature now supports drag-and-drop for reordering tabs. The underlying interface for updating loading animations has also been implemented. (96affd4) (1267e59)
  • The Print Preview dialog has been enhanced with an "actual size" option in the PDF scaling dropdown, allowing users to print documents without any scaling. (abdd4a8) (b13ef07)
  • For Progressive Web Apps on Windows, the Window Controls Overlay (WCO) toggle state is now managed on a per-window basis, fixing a bug where the state was incorrectly shared across all windows of the same app. (8e2fb48)

Security, Networking & Enterprise

  • To improve stability, the renderer process will now be terminated if it sends invalid arguments when creating an IndexedDB cursor. Renderer hang detection has also been made more accurate with a new PingMainThread IPC probe. (c903934) (e74ee98)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: A dependency roll of the QUICHE library has removed several deprecated methods from the public quic::ParsedQuicVersion API in favor of a single, unified IsIetfQuic() method. (c5b2270)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The SafeBrowsingUIHandler has been componentized into an abstract base class and a concrete implementation, requiring embedders to update instantiation from SafeBrowsingUIHandler to SafeBrowsingContentUIHandler. (7341663)
  • Two new enterprise policies, GeminiActOnWebAllowedForURLs and GeminiActOnWebBlockedForURLs, have been added to provide administrators with URL-based control over the "Act on this page" AI feature. (0c67554)

Chromium Weekly: 2025 Week 51

Overview

This week was defined by an exceptionally high volume of breaking changes across the Chromium codebase, impacting UI, security, extensions, and core APIs. A major theme was the continued modernization of APIs, including the replacement of legacy systems like BrowserList and the migration of extension manifests to WebIDL. Across 6 days, 387 relevant commits were made out of 511 total, with significant refactoring in tab management, core utilities, and the graphics stack.

Browser UI & Tabs

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: A series of breaking changes refactored the public tab management APIs. The TabDragContext, TabDragDelegate, and TabStripRegionView interfaces were modified, the TabRendererData.tab member was removed, and TabCollection::MoveTab() was deleted. Embedders must update custom tab strip implementations. (b97f8fc) (7eeb9f0) (9b973dd) (c47f0fe) (a595e9f)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: As part of the 'Project Bedrock' modernization effort, several public static methods for querying incognito state were removed from BrowserList in favor of functions in browser_finder.h. The unused BrowserList::size() method was also removed. (bde2c0b) (d4b2a8e) (64c605a)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The ui/views framework now enforces child list immutability during iteration in production builds via a CHECK. This will cause a hard crash if a View's children are mutated while being iterated over, exposing latent bugs in UI code. (4b55b0b)
  • The experimental 'Tab Organization' feature and its associated APIs in TabOrganizationService have been removed from the codebase. (e1c53e2)

Extensions

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Several extension delegate interfaces were broken. ExtensionsMenuViewPlatformDelegate was removed in favor of an observer pattern, ExtensionInstalledWaiter was replaced by ExtensionInstallWatcher, and the const qualifier was removed from several virtual methods in ExtensionFunctionDispatcher::Delegate. (0ff49e3) (d3615f2) (d6c7658)
  • The effort to modernize extension APIs continued, with oauth2, fileHandlers, crossOriginIsolation, chrome_url_overrides, web_accessible_resources, sharedModule, and protocolHandlers being converted from the legacy JSON schema to WebIDL. (3a20416) (53881ac) (df23b7a) (6d9ee98) (36bc943) (bc0d14a) (390c9b5)
  • API correctness was improved by moving validation earlier in tabs.create() and windows.create() calls, and a race condition that could prevent an extension's service worker from registering on rapid reinstall has been fixed. (3d890c1) (2f44e11)

Security & Privacy

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Core crypto and authentication APIs were changed. The deprecated crypto::HMAC class was removed, the public UnexportableKeyService interface was extended with a new GetKeyTag() method, and the UseIssueTokenToFetchAccessTokens feature was enabled by default, altering auth flows. (06d7910) (be32c0a) (627b6aa)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public safe_browsing::BinaryUploadService API was refactored. The Request class and the GetForProfile factory method were removed, requiring migration to the enterprise_connectors namespace. (afc8753) (5ee7771)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Support for 'named interceptions' was removed from the public Windows sandbox API (sandbox/win/src/interception.h), a feature removal from a core security interface. (80fad49)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The AreAllThirdPartyCookiesBlocked() method and its observer have been removed from the TrackingProtectionSettings API, and the GetTrackingProtectionSettings function was removed from the permissions::PermissionsClient interface. (3100d0f) (25d5686)
  • The enterprise policies IncognitoModeBlocklist and IncognitoModeAllowlist have been renamed to IncognitoModeUrlBlocklist and IncognitoModeUrlAllowlist respectively, changing the underlying preference names. (c78a55b) (43e427d) (3539075)

Core APIs & Performance

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The core tracing API was refactored, removing the TraceLog::EnabledStateObserver and TraceLog::AsyncEnabledStateObserver interfaces. Embedders must migrate to the new per-session TraceSessionObserver. (8e0db64)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The base::MemoryPressureListenerRegistry is no longer a static singleton and now requires explicit instantiation. The base::LRUCache constructor was also changed to use std::optional for its max size. (83f4085) (119215c)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The Chrome DevTools Protocol (CDP) command Network.enable is now synchronous and its signature has changed. Automation clients must be updated to handle the new behavior. (407599e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The kReduceUserAgentPlatformOsCpu command-line switch was removed. The reduced User-Agent string for platform and OS CPU is now the default and only behavior. (28d0394)
  • A new synchronous URL loader was introduced for initial WebUI navigations (e.g., chrome://settings) to improve the perceived performance of browser UI pages. (6fc5621)

Graphics & Media

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public Viz compositing API was modified. The viz::ReturnCallback signature now expects std::vector<ReturnedResourceViz>, and the viz::ReturnedResource struct was updated, breaking custom compositor clients. (40df19a) (082e1d7) (f04f79f)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The entire //ui/gfx/ipc/buffer_types directory has been removed, a major change for any code that relied on the IPC parameter traits for serializing graphics buffer types. (4292293) (eb22d98)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public AudioBus::channel_span() method has been removed from media/base. Embedders must update to use AudioBus::channel() instead. (465fb43)

Build System & Testing

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Build support for ARMv6 and ARM configurations with NEON disabled (arm_use_neon=false) has been completely removed. (e6b1ab2)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: On Windows, the IProcessLauncher::LaunchCmdLine method in the Updater's public COM interface has been reverted to return E_NOTIMPL, breaking clients that rely on it to launch processes. (6684d0a)
  • Legacy testing setup methods BrowserProcess::CreateGlobalFeaturesForTesting and GlobalFeatures::Shutdown have been removed. Tests must now use the TestingBrowserProcess equivalents. (ced3c21)

Chromium Weekly: 2025 Week 50

Overview

This week saw an exceptionally high volume of breaking changes across Chromium's core APIs, impacting browser lifecycle, UI, networking, and numerous components like Autofill and Identity. A total of 1009 relevant commits out of 1313 were analyzed. Key developments include the enablement of an out-of-process PDF viewer for enhanced security, the introduction of structured cloning in extension messaging, and the removal of the Media Galleries API on Windows and macOS.

Widespread Breaking API Changes

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The core browser startup and shutdown sequence was refactored. The Init() and Shutdown() methods on GlobalFeatures and BrowserProcess are replaced with a new multi-phase approach, breaking the lifecycle for embedders and requiring updates to tests using TestingBrowserProcess. (24ba371)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: A massive refactoring of the Autofill component removed or changed numerous public methods like AutofillClient::DidFillForm and AutofillManager::GetCachedFormAndField, requiring significant updates for embedders using this component. (0e0c6aa) (658a93b) (6ab0209) (1a8b684) (2d6f99f)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Multiple core utility APIs were modified. Functions in base/system/sys_info.h now return the base::ByteSize class instead of integer types, and deprecated callbacks using std::unique_ptr<std::string> were removed from SimpleURLLoader in favor of std::optional<std::string>. (d310efd) (c7e3442) (558b2e7) (31e0c06)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: Several public UI and Content APIs were broken. TabStripModel::selection_model() now returns a TabStripModelSelectionAdapter, ContentBrowserClient::RegisterWebUIControllerInterfaceBrokers has a new signature, and content::RenderProcessHost::FastShutdownIfPossible gained a new parameter. (58e491f) (2b5bd6d) (1095311)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The generic network error net::ERR_DNS_SERVER_FAILED has been removed and replaced by five more specific error codes, requiring embedders to update error handling logic. (0b88079)

Feature Removals and Behavioral Changes

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The Media Galleries API (chrome.mediaGalleries) has been restricted to ChromeOS, effectively removing this feature from Windows and macOS and breaking any extensions or applications that relied on it. (2e27c61)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The network service sandbox is no longer enabled by default on Windows, reverting a recent change. This alters the default security posture for embedders on Windows. (e1e1886)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The web-app-manifest-protocol-handlers feature has been removed, disabling the ability for Progressive Web Apps to register as protocol handlers. (5c7e4d6)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE: The public EnterpriseIdentityService and its factory have been removed. Embedders must migrate away from this service for managed account identification. (5d6bd7b)

Extensions

  • Support for structured cloning has been added to extension messaging APIs like runtime.sendMessage. This allows for richer data types (e.g., Map, Set, Date) to be passed in messages, augmenting the previous JSON-only serialization. (39b5b05) (88e3bae) (69363e6)
  • The declarativeNetRequest API was enhanced to expose RuleConditionKeys, allowing extensions to perform feature detection for supported rule conditions at runtime. (ab42e9f)
  • Reliability of persisted webRequest listeners was improved with fixes for race conditions during extension updates and more efficient synchronous cleanup when an extension is unloaded. (2836073) (d33b324) (022ce91)

Security and Privacy

  • The PDF viewer now runs in an out-of-process iframe (OOPIF) by default on Windows and macOS. This improves security and stability by isolating PDF content from the main browser process. (b7cbb1e)
  • The ForcedDiceMigration feature has been enabled by default, completing the migration to the DICE sign-in architecture and potentially changing sign-in flows for some users. (ea4b1a4)
  • Local Network Access (LNA) now invalidates cached resources if the network configuration changes (e.g., connecting to a VPN), ensuring the browser re-fetches resources to apply correct IP-based permission checks. (a9b4054)

Browser Core and Performance

  • Native support for the Matroska media container format (.mkv, .mka) has been added, allowing these files to be played directly in the browser without extensions. (d081300)
  • A significant performance issue causing UI lag when pasting very large amounts of text into a views::Textfield has been fixed via a new OnBeforePaste hook. (81aebdb)
  • The V-Sync mechanism was improved on both macOS (using the modern CADisplayLink framework) and Windows (better multi-GPU support) to enhance graphics performance and smoothness. (75e50bd) (6187f20) (b8f19a5)
  • The Split View (Side-by-Side) feature is now unconditionally available on supported platforms, with its controlling feature flag removed. (1b9e069) (0fc0517)

This week was defined by a monumental number of breaking changes across the Chromium codebase, requiring significant attention from downstream projects. Build System & Toolchain

(9529e2b) (42fcda8) (db2f79e) (fdbf02d)

Chromium Weekly: 2025 Week 48

Overview

A turbulent week in Chromium development saw 1933 relevant commits out of 2680 total, introducing a vast number of breaking changes across the codebase. The most significant development was the enabling of the Network Service Sandbox by default on Windows, a major security milestone. Multiple core APIs in networking, identity, and UI were modified, requiring significant attention from embedders. Instability was also a theme, with a major refactor of the AccountInfo identity struct being partially reverted after causing crashes.

Security

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The Network Service Sandbox is now enabled by default on Windows. This is a major behavioral change that enhances security by restricting the network process's access to the system, but it may break embedders who rely on the previous, more permissive environment. (c6bd7f0)
  • The remote debugging server has been hardened with a new "approval mode" controllable via the kDevToolsRemoteDebuggingEnabled preference and the kDevToolsRemoteDebuggingAllowed enterprise policy. This requires user approval for new connections, improving security for developers. (2e576ad) (2748580)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The core WebAuthn public API AuthenticatorCommon::MakeCredential has been updated to include a new payment_options parameter, requiring embedders to update their calls. (5d55cd3)
  • An initial implementation of a proxied Unexportable Key Service was added, laying the groundwork for the network process to interact with hardware-backed keys (like the macOS Secure Enclave) via the browser process. (269bac8) (a6cfb36)

Core APIs & Internals

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The AccountInfo identity struct underwent a volatile refactoring to privatize the hosted_domain field. The changes, which required all consumers to use a new GetHostedDomain() accessor, caused crashes and were partially reverted late in the week. Embedders should monitor this area closely for instability. (da1dfa4) (d85bfc8) (3428bfe) (659bbd1)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The callback for the fundamental networking utility network::SimpleURLLoader::DownloadToString now receives a std::optional<std::string> instead of a std::unique_ptr<std::string>, requiring updates to all client code. (a19f517)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE A new pure virtual method, GetOAuthConsumerFromId, was added to the SigninClient interface. Embedders with custom SigninClient implementations must add this method to avoid build failures. (5657390)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE Transitive includes for ui/gfx/geometry/rect.h and rect_f.h have been removed from many public headers like content/public/browser/web_contents.h. Code that relied on these headers to implicitly include gfx::Rect or gfx::RectF will fail to build without adding the includes directly. (5ac7f34)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The base::MemoryPressureListener API will now dispatch a notification when memory pressure returns to MEMORY_PRESSURE_LEVEL_NONE. Custom listeners must be updated to handle this new notification to avoid unexpected behavior. (d139d7a)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The entire components/fingerprinting_protection_filter component has been deleted from the codebase. Projects with a dependency on this component will no longer build. (b6d76c1)

Browser UI & Features

  • The new tabbed browser layout, part of the "Chrome Next" UI refresh and supporting the Toolbar Height Side Panel (THSB), is now enabled by default. (76dd43c) (838d53b)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The experimental "Tab Scrolling" feature (scrollable-tabstrip) has been completely removed from the codebase. The tabstrip will now revert to the default behavior of shrinking tabs when the strip is full. (ef29bab)
  • Work continues on the "Tab Group Focusing" feature, with a new OnTabGroupFocusChanged method added to the TabStripModelObserver interface to notify listeners of group focus state changes. (de85415) (c7b09ea)

Extensions & Developer Tools

  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The ability for extensions to delete passwords via chrome.browsingData.removePasswords has been removed. Calls to this function are now a no-op and will not affect the password store. (9fb257e)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The Chrome DevTools Protocol (CDP) has moved the StyleSheetId type from the CSS domain to the DOM domain. Automation clients referencing CSS.StyleSheetId must be updated to use DOM.StyleSheetId. (23a0a9f)
  • The remote debugging server can now automatically find an available port if the default port 9222 is in use. The active server address is now displayed on chrome://inspect. (a4e85e3) (8a418d9)
  • The WebRequestPersistFilteredEvents feature is now enabled by default. This may alter the behavior of extensions using the chrome.webRequest API by changing how event listeners are handled across navigations. (49be6c3)

Enterprise

  • A new enterprise policy, SearchContentSharingSettings, was added to allow administrators to control content sharing with AI features, independent of other AI-related settings. (6e7bf35)
  • A new StaticStorageQuotaEnabled enterprise policy has been added, allowing administrators to override the default storage quota estimation behavior. (4ff2cdf)
  • ‚ö†Ô∏è BREAKING BREAKING CHANGE The value savedTabGroups has been removed as a valid option from the SyncTypesListDisabled enterprise policy. Configurations using this value to disable tab group syncing are no longer valid. (1746a94)

Overview

Build System
