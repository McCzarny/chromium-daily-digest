Chromium Digest: 2026-05-25

Overview

This digest summarizes 163 development commits from the Chromium repository, focusing on key changes relevant to Windows and macOS embedders while excluding platform-specific code (Android, iOS, ChromeOS), internal rendering engines (Blink, V8), and experimental features. The analyzed changes highlight progress in browser architecture modularization, enterprise security hardening, and infrastructural type-safety improvements. Spanning commits refs/heads/main@{#(1635567)} to refs/heads/main@{#(1635861)}, these updates emphasize long-term maintainability without introducing breaking changes to the stable public API surface.

Enterprise & Security

  • Extended WebAuthn VDI affiliation checks to Windows, macOS, and Linux. This security restriction requires managed profile and device affiliation for remote desktop scenarios using enterprise_util::IsProfileAffiliated(). (cc6a021)
  • Implemented stricter PaymentRequest validation for Secure Payment Confirmation (SPC). The browser will now terminate the renderer process if SPC is combined with multiple other payment methods. (58edc61)
  • Added RestrictBackgroundFetchFromServiceWorkerEnabled enterprise policy to allow administrators to manage Service Worker background fetch permissions. (2bfe6d2)

Architecture & Modularization

  • Continued decoupling of the monolithic Browser class by extracting session service and window control logic into dedicated helpers like SessionServiceBrowserHelper. (8564649) (9017aa6)
  • Ported MachineLevelUserCloudPolicyStatusProvider to Mojo, facilitating the migration of chrome://policy WebUI to modern interfaces. (e5f5b3e)

Performance & Infrastructure

  • Refactored network and memory components to use the ByteSize helper class instead of raw signed integers, improving type safety and preventing potential overflows. (c96a0c0) (eedc8b1)
  • Annotated WaitableEvent::WaitMany with WILL_BLOCK to allow the ThreadPool to dynamically scale worker counts and prevent potential UI hangs. (387ee2d)

UI & Browser Features

  • Added omnibox chip signals for top-level MIME handler extensions (e.g., PDF viewers), improving user transparency regarding which extension is processing content. (dd51786)
  • Fixed profile-isolation issues in the HTTP cache where per-profile field trial experiments were incorrectly resetting based on global local-state preferences. (2868aa4)
