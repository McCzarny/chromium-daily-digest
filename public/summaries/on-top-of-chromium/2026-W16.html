Chromium Weekly: 2026 Week 16

Overview

This week saw 546 relevant commits out of 763 total, introducing an exceptionally high number of breaking changes across the Chromium codebase. Major themes include a fundamental refactoring of browser window lifecycle management APIs, the complete removal of the synchronous OSCrypt backend, significant security hardening in the extensions platform, and continued modularization of the monolithic //chrome/browser build target. Embedders will need to address numerous API removals and signature changes to adapt to these modernizations.

APIs & Core Interfaces

  • ⚠️ BREAKING BREAKING CHANGE: Browser lifecycle observation has been fundamentally refactored. The BrowserListObserver class and numerous utility functions like chrome::FindTabbedBrowser, chrome::FindBrowserWithProfile, and chrome::FindLastActive have been removed. Embedders must migrate to the new ProfileBrowserCollection and GlobalBrowserCollection APIs, which often return a BrowserWindowInterface* instead of a Browser*. (c6268a4) (c7811f0) (2ebf735) (4650246) (2ac4e0f)
  • ⚠️ BREAKING BREAKING CHANGE: The entire synchronous OSCrypt backend, including the components/os_crypt/sync directory and OSCryptMocker for testing, has been removed. All consumers must migrate to the asynchronous os_crypt_async APIs, replacing calls like OSCrypt::Init with os_crypt_async::Init on Windows. (a4cab4a) (21b80f4) (51493b3)
  • ⚠️ BREAKING BREAKING CHANGE: The LoginDatabase interface in the password manager has been updated to use the new StoredCredential struct instead of PasswordForm. Code interacting directly with the password database must adopt the new data structure. (fa3099d) (167b1b4) (6b01163)
  • ⚠️ BREAKING BREAKING CHANGE: Several core interfaces have new pure virtual methods, requiring implementation by embedders. This includes ShouldUseFirstPartyCookieContext on content::ContentBrowserClient and AllowInspectingTarget on content::DevToolsManagerDelegate. (9f52b02) (bfdcab0)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The chrome.usb and chrome.bluetooth* APIs have been removed from Windows and macOS builds, restricting them to ChromeOS. Desktop extensions relying on these hardware interaction APIs will no longer function. (1e44322) (32b3c6d)
  • ⚠️ BREAKING BREAKING CHANGE: The development workflow for installing extensions via the Chrome DevTools Protocol (CDP) has been altered. Extensions installed this way are now non-persistent and will be removed on browser restart, impacting automated testing setups. (dddf05b) (0333892)
  • ⚠️ BREAKING BREAKING CHANGE: The extensions platform has been hardened. chrome.scripting.executeScript with allFrames: true now requires host permissions for every frame, Declarative Net Request redirects involving file:// URLs require file access permission, and programmatic calls to permissions.add/remove now require a user gesture. (e05c69e) (53caa53) (1aca864)

Networking & Security

  • ⚠️ BREAKING BREAKING CHANGE: Core networking Mojo interfaces in network_param.mojom have been refactored to remove deprecated [Native] types for structures like SSLInfo and HttpResponseHeaders. Embedders must update their code to use the new, fully-defined Mojo structures. (c565345) (d825538)
  • ⚠️ BREAKING BREAKING CHANGE: The legacy ICE-based connection classes (IceConnectionToClient, IceConnectionToHost) have been removed from the Remoting component. All connections now use WebrtcConnectionToClient, marking a major step in deprecating the old Chromotocol. (14c775e) (e8ee844)
  • ⚠️ BREAKING BREAKING CHANGE: On Windows, the os_crypt component no longer supports decrypting data encrypted with pre-M79 versions of DPAPI. This may cause data loss for users with very old profiles that have not been opened since late 2019. (db40741)
  • Security has been hardened across the browser. This includes blocking the applescript: scheme on macOS, adding recursion depth limits to bookmark deserialization to prevent stack overflows, and verifying the main script URL in the browser process for Service Workers to prevent policy downgrades. (71c38a1) (0e3f531) (0f7954e)

Browser UI & Features

  • ⚠️ BREAKING BREAKING CHANGE: The text autosizer feature, which automatically inflated text sizes, has been completely removed. The corresponding text_autosizing_enabled and device_scale_adjustment properties have been removed from the public WebPreferences API. (4797e6b)
  • ⚠️ BREAKING BREAKING CHANGE: The ToolbarButtonProvider interface has been refactored to return abstract interfaces (AppMenuControl, AvatarToolbarButtonInterface) instead of concrete button classes, breaking embedders who directly accessed the concrete views. (0800caa) (6c9ccc1)

Build System & Testing

  • ⚠️ BREAKING BREAKING CHANGE: Major build system modularization continues, with code for networking, enterprise features, password management, and more being extracted from the monolithic //chrome/browser:browser library into new, more granular targets. Embedders will experience build failures until they update their BUILD.gn files to explicitly depend on the new targets. (888790c) (8a22588) (92b2a6c) (3ef188b)
  • ⚠️ BREAKING BREAKING CHANGE: The deprecated testing utility TestBrowserClosedWaiter has been removed. Downstream tests must be migrated to use ui_test_utils::BrowserDestroyedObserver. (dd0cca1)
