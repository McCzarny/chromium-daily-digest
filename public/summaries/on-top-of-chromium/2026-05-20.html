Chromium Digest: 2026-05-20

Overview

This summary covers 523 relevant commits out of 692 total on the main branch for May 20, 2026, from commit a285b7adff3d... to c354f0a09ec5.... The focus is on changes impacting developers of Chromium-based projects, with a strong emphasis on breaking changes, new APIs, and updates to the browser UI, security, and networking subsystems for Windows and macOS.

Breaking Changes

  • ⚠️The public C++ interface RenderFrameHostImpl::AddResourceTimingEntryForFailedSubframeNavigation has a new signature. The method now takes a SubframeResourceLengths struct and a completion_time instead of a URLLoaderCompletionStatus object, breaking embedders who call this method directly. (a6a9a9f)
    BrowserAPINetwork
  • ⚠️Several public helper functions have been removed from chrome/browser/ui/browser_finder.h, including chrome::GetIncognitoBrowserCount() and chrome::GetGuestBrowserCount(). Embedders relying on these functions will need to find alternative ways to query browser state. (686390b)
    BrowserAPI
  • ⚠️The public API media::VideoCaptureDevice::Client::OnIncomingCapturedImage now requires a new const gfx::Size& natural_size parameter. Embedders implementing this client interface for video capture must update their method signatures. (636ff60)
    MediaAPI
  • ⚠️The audio::SoundsManager and audio::AudioStreamHandler APIs have been changed to accept a resource ID (int) instead of raw data (std::string_view). Embedders using this API to play sounds from memory must now package audio as a resource. (2788a58)
    MediaAPI
  • ⚠️The PinnedToolbarAction enum in the public toolbar_ui_api_data_model.mojom interface has been simplified. Several media-related enums are consolidated into kRouteMedia, requiring embedders to pass the icon separately. (a3798e5)
    UIAPI
  • ⚠️The ConfirmInfoBar class is now an abstract base class. Direct instantiation with std::make_unique<ConfirmInfoBar> will fail; use the new static factory method ConfirmInfoBar::Create() instead. (0cda611)
    UIBanner
  • ⚠️The public virtual method ui::TouchEditable::IsCommandIdEnabled has a new bool can_paste parameter. Subclasses must update their implementation to match the new signature. (b57bb78)
    UI
  • ⚠️The initially_focused_view parameter in views::Widget::InitParams has changed from std::optional<View*> to std::unique_ptr<ViewTracker>. Code that directly constructs widgets will need to be updated. (5c153ab)
    UI
  • ⚠️Public enum values in the split_tabs::SplitTabLayout API have been renamed for clarity. kVertical is now kSideBySide and kHorizontal is now kStacked. (4abe9bf)
    UITabs
  • ⚠️The SupervisedUserBlockInterstitialV3 feature flag and the public function supervised_user_error_page::GetBlockMessageV2 have been removed, making the V3 UI permanent. Embedders calling the removed function will have build failures. (4622302)
    BrowserAPI

API & Extensions

  • The DownloadManager interface now includes asynchronous versions of GetAllDownloads() and GetDownload(). This is a non-breaking addition that allows for deferred initialization of the download system to improve startup performance. (8cbf419)
    DownloadsAPIPerformance
  • An experimental embedderData field has been added to TargetInfo in the DevTools protocol. The new DevToolsManagerDelegate::GetEmbedderDataForTarget hook allows embedders to populate this field with custom metadata. (5aa804a)
    DevToolsAPI
  • A new enterprise policy, HttpHeaderInjection, has been defined. It will allow administrators to inject custom HTTP headers into requests for specified URL patterns. The network interceptor implementation is forthcoming. (a6a8295)
    PolicyNetworkEnterprise
  • The WebUI Omnibox now sends the cursor position to the AutocompleteController, enabling more context-aware suggestions, such as completing a word in the middle of a query. (1ad530f)
    UIAPI
  • A cleanup mechanism was added to ExtensionPrefs to deduplicate sub-event filters (e.g., for webRequest) that may have accumulated in user preferences over time. (6e05d4d)
    ExtensionsStorage

UI & Views

  • On Windows, a new feature allows views::Widget instances to be excluded from screen capture via SetExcludeFromScreenCapture(). This is intended for Picture-in-Picture windows. (7d0d0a7)
    UIWindows
  • The revamped "Clear Browsing Data" dialog is now permanent, as the kDbdRevampDesktop feature flag has been removed. Embedders relying on the old UI will now see the new version. (30ea8b7)
    UIPrivacy
  • The Windows system menu can now use a Views-based implementation, controlled by the kMenuSimplification feature flag, as part of ongoing UI modernization efforts. (2ae8ddc) (7aee783)
    UIWindows
  • On macOS, a bug where the decorative "Glass" frame could render on top of web content and intercept clicks has been fixed by correctly sorting the NSGlassEffectView in the view hierarchy. (5dc59c8)
    UImacOS
  • Native UI text rendering on Windows and macOS now correctly honors the VS-15 (U+FE0E) variation selector, forcing emoji-default characters to render in their monochrome text presentation. (4c82c62)
    UIFonts
  • The First Run Experience (FRE) has been updated to include a "Feature Showcase" step to introduce users to new capabilities. (7219a0c)
    UIBrowser

Security & Privacy

  • A potential origin spoofing vulnerability in Progressive Web Apps (PWAs) has been fixed. When a PWA is uninstalled, its windows are now forcibly closed, bypassing beforeunload handlers to prevent the window from remaining open without its custom UI. (d47269c)
    SecurityBrowser
  • On Windows, trust verification for the remote WebAuthn native messaging host has been hardened by using PathService and verifying the binary is signed by Google. (448fb48)
    SecurityWindows
  • The macOS renderer sandbox was hardened to deny the ability to post distributed notifications, controlled by the new kMacSandboxDistributedNotifications feature flag. (cd96f38)
    SecuritymacOS
  • Navigations initiated from the Reading Mode side panel are now more secure. The browser now verifies user gestures and downgrades CURRENT_TAB navigations to NEW_FOREGROUND_TAB to prevent a compromised renderer from hijacking the active tab. (a30257f)
    SecurityNavigation
  • Device-Bound Session Credentials (DBSC) registration will now use the final URL after any redirects to perform validation checks, improving the security of the registration process. (159631e)
    SecurityNetwork

Performance & Infrastructure

  • LazyBrowserInterfaceBroker is now enabled by default, which is expected to save browser memory by deferring the creation of some interfaces. (bdccc71)
    PerformanceBrowser
  • New histograms, Startup.FirstWebContents.FirstContentfulPaint and Startup.FirstWebContents.LargestContentfulPaint, have been added to provide more standardized performance insights for the initial tab. (a05b3e4)
    Performance
  • The HTTP cache initialization process has been updated to detect and delete empty cache directories on startup, reclaiming disk space. (6dff202)
    NetworkStoragePerformance
  • A bug in ZstdSourceStream that could violate the FilterSourceStream contract has been fixed. The stream will no longer return zero output bytes while leaving input unconsumed. (9aba30f)
    Network
  • The [[nodiscard]] attribute has been added to the RenderFrameHost::FrameIterationAction enum to help prevent bugs where its return value is ignored during frame traversal. (64f1be3)
    BaseBrowser

Testing

  • A suite of testing flags for web app install dialogs has been unified into a standard RAII pattern with dedicated helper classes like ScopedAutoAcceptWebAppDialogs, centralizing logic and cleaning up test code. (fff5596)
    TestingBrowser
  • The fuzzer_test build target now supports a dict_deps attribute, allowing fuzzer dictionaries to be generated as part of the build process. (c5f7aef)
    TestingBuild
