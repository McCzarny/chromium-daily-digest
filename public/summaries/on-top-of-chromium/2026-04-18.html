Chromium Digest: 2026-04-18

Overview

This digest covers 48 commits from the Chromium repository, with a focus on changes relevant to Windows and macOS embedders. Key updates include breaking changes in the Remoting and URL components, significant refactoring of Windows security object handling, and various improvements to UI, extensions, and accessibility. Remoting

  • ⚠️ BREAKING BREAKING: The legacy ICE-based connection classes IceConnectionToClient and IceConnectionToHost have been removed. ChromotingHost now exclusively uses WebrtcConnectionToClient. This is a significant step in the removal of the legacy Chromotocol. (14c775e)
  • ⚠️ BREAKING BREAKING: Following the removal of the ICE connection classes, the underlying legacy transport classes IceTransport and IceTransportChannel, along with their dependencies like ChannelMultiplexer, have also been deleted. (e8ee844)

Networking & URL

  • ⚠️ BREAKING BREAKING: The deprecated, unsafe url::IsStandard(const char* spec, const Component& scheme) overload has been removed from url/url_util.h. Callers must migrate to the IsStandard(std::optional<std::string_view> scheme) overload. (83cace3)
  • WebSocket handshakes over HTTP/3 will now correctly populate QUIC version information and delegate to the session handle for QUIC-specific error details, aligning its behavior with QuicHttpStream. (e7db959)

Base & System (Windows)

  • A significant refactoring has removed dependencies on ATL security objects (like CSecurityDesc, CDacl, CSid) from chrome/updater and chrome/enterprise_companion. These have been replaced with their modern base::win equivalents (base::win::SecurityDescriptor, base::win::Sid). As part of this, base::win::SecurityDescriptor gained a new SetDaclEntry overload that accepts a base::win::AccessToken. (c7f06b9)

UI & Accessibility

  • A bug on Windows has been fixed where the context menu item to pin/unpin the tab search button to the toolbar was not functional. The command is now correctly handled. (95f3647)
  • Accessibility on macOS for spin buttons (<input type="number">) has been improved. The browser now sends specific increment/decrement event intents, preventing VoiceOver from misinterpreting value changes as text edits and ensuring it announces the full new value. (9bfcec8)

Extensions

  • A new GlicPrivate.invoke endpoint has been implemented to allow fetching prompts by prompt_id. This is part of the ongoing development for the Glic feature. (0845cc7)

Internal Refactoring & Cleanup

  • Unused PWA manifest update utilities (manifest_update_utils.h/.cc) and an associated UMA histogram have been removed, cleaning up code orphaned by the migration to a modern job-based architecture for manifest updates. (9338d92)
  • As part of a large-scale refactor, consumers of OptimizationGuideModelProvider within the segmentation platform are being migrated to use the OptimizationGuideGlobalState API. This consolidates the model provider into a globally managed asset. (63b7b2d)
