Chromium Digest: 2025-11-15

Overview

A total of 185 commits were made to the 'main' branch. After filtering, 77 relevant commits were analyzed. The day's changes span from commit (ce24490) to (a8e68db).

API

  • ⚠️ BREAKING The method signature for RequestToShowUserConfirmationDialog has been updated across the ActorTaskDelegate C++ interface and the actor_webui.mojom interface. A new boolean parameter, for_blocklisted_origin, was added to allow the UI to display different dialogs for origins on the optimization guide blocklist. (fc202bb)
  • An observer pattern has been introduced to the Autofill CreditCardAccessManager. A new CreditCardAccessManager::Observer class in components/autofill/core/browser/payments/credit_card_access_manager.h now allows clients to monitor credit card fetch events like OnCreditCardFetchStarted and OnCreditCardFetchSucceeded. (1ea624c) (cfee778) (48a322d)

Extensions

  • Comprehensive tests have been added for JSON serialization in Extensions messaging APIs, including runtime.sendMessage, tabs.sendMessage, and long-lived connections via connect. The new tests cover a wide range of types and edge cases to ensure stable communication between content scripts and background pages. (510da73)
  • A new browser test ensures that ExtensionActionViewModel methods do not crash if called after the associated extension has been uninstalled, improving the robustness of the extensions UI. (55358c7)

UI/UX

  • Significant visual and functional enhancements were made to the Omnibox and New Tab Page 'Composebox'. This includes new SVG-based voice input animations, layout fixes for compact mode, and improved cancel/escape key logic. The '+' context menu icon is also now hidden when the Omnibox is in keyword mode. (93ca825) (f53d9a3) (c113dad) (3cd3a37) (6d71861)
  • The Omnibox context menu has been refined, setting a maximum width of 400px to prevent layout issues. Additionally, menu items for 'Deep Search' and 'Create Images' are now conditionally displayed based on user eligibility. (1f901e0) (ce24490)
  • The New Tab Page layout for Most Visited sites has been fixed to prevent the 'Show less' button from improperly wrapping to a new row. Filtering logic for AI Action chips was also improved to exclude search result pages and invalid URLs. (14e20bf) (2c05c1b)

Bug Fixes

  • A pure virtual function crash in OmniboxPopupPresenterBase has been resolved. The WidgetDestroyed method, which was being called during destruction, was changed from a pure virtual to a standard virtual function to prevent the crash. (fad5c6f)
  • Fixed an issue where the Omnibox WebUI popup would not appear. The RoundedOmniboxResultsFrame was incorrectly clamping its height to zero, which caused the WebContent to be hidden. It now clamps to a 1x1 pixel size to keep the widget active. (80aa43d)

Security

  • Safe Browsing's download protection logic in chrome/common/safe_browsing/download_type_util.cc was updated to classify .apkm files as the ANDROID_APK download type, enhancing security for Android-related file downloads. (2c5127b)

Base

  • The base::CheckedNumeric library in base/numerics/checked_math.h was enhanced with new IsValidAnd() and IsInvalidOr() helper methods, allowing for more concise chaining of validity checks. (465fc12)

Infrastructure

  • Several recent changes related to the Omnibox composebox were reverted due to build failures and regressions, including the relocation of the ComposeboxHandler and the addition of new page classifications. This indicates instability in this area during a period of active refactoring. (8635d70) (f8cbc49)
