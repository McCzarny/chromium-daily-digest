Chromium Digest: 2026-04-30

Overview

This digest covers 745 commits on the main branch from commit (084453c) to (5186844), with 556 identified as relevant to developers. This cycle introduces numerous significant breaking changes to the Remoting component, network observers, PWA manifest handling, and several core component APIs. Other notable updates include enabling Vertical Tabs by default, modernizing the extension MIME handler architecture, and patching several security vulnerabilities.

Breaking Changes

  • ⚠️ BREAKING Remoting: The SessionConfig and CandidateSessionConfig classes have been removed, enforcing WebRTC as the sole transport protocol. This removes public APIs like SessionManager::set_protocol_config() and ConnectionToHost::config(), requiring embedders to update their remoting implementations. (32d22ac)
  • ⚠️ BREAKING Networking: A new method, OnPlatformLocalNetworkPermissionRequired, has been added to the URLLoaderNetworkServiceObserver mojom interface. Embedders must implement this method to handle new OS-level permission prompts for the Local Network Protection (LNP) feature, otherwise local network requests may fail. (3218389)
  • ⚠️ BREAKING Web Applications (PWAs): The webapps::ManifestId type has been changed from a GURL alias to webapps::ValidManifestId, which wraps an std::optional<GURL>. This enforces validation at the type level, requiring code that uses ManifestId to handle potential invalid IDs and access the GURL via .value(). (73319ea)
  • ⚠️ BREAKING Enterprise Policy: The tie-breaking logic for the PrefURLListMatcher component has been altered. It now prioritizes the longest host *before* the longest path when evaluating multiple matching URL patterns. This can change how existing enterprise policies are applied. (806b0e9)
  • ⚠️ BREAKING Compositor: Several public methods, including CreateUIResourceFromImportedResource, have been moved from the base class cc::LayerTreeHostImpl to the derived class viz::VizLayerTreeHostImpl. Embedders calling these methods on the base class will need to update their code. (a0e7321)
  • ⚠️ BREAKING Components: The PageContentExtractionService::Observer::OnPageContentExtracted method signature has changed. It now passes a page_content_annotations::PageContent variant instead of a scoped_refptr. Existing observers must be updated to handle the new type. (2f7a652)
  • ⚠️ BREAKING Extensions: The webstorePrivate API permission and its associated feature definitions have been completely removed. This API was used by the legacy Chrome Web Store hosted app and is no longer accessible. (de56b30)
  • ⚠️ BREAKING Base Library: The public function base::DateIntervalFormat and the associated base::DateFormat enum have been removed from base/i18n/time_formatting.h. Code relying on this for formatting date intervals must be updated. (1c49b4d)
  • ⚠️ BREAKING Components: The public method SendTabToSelfModel::AddEntry has been renamed to SendEntry. Embedders using the components/send_tab_to_self API must update their method calls. (2576012)
  • ⚠️ BREAKING Sync: The device_info_specifics.proto protocol buffer has been modified, replacing the glic_experimental_triggering_opted_in boolean field with an enum. This is a breaking change for any client that directly parses this sync proto. (2beaaac)
  • ⚠️ BREAKING Mojom: The signature of the public mojom interface FrameWidgetInputHandler::PasteFromImageBytes has been changed, removing its bool success callback. Clients calling this method will need to be updated as it no longer provides a result. (6f76ae0)
  • ⚠️ BREAKING Components: The public methods Normalize() and Magnitude() have been removed from the passage_embeddings::Embedding class. Consumers of this components/passage_embeddings API must now ensure embeddings are normalized before use. (9d9fbd3)

Browser & Core Components

  • A new dictation component has been added to the browser, introducing a SessionController for managing dictation sessions. (3d5eec5)
  • The migration from the deprecated base::MemoryPressureListener to the new base::MemoryConsumer API continues, with ProgramCache and OnDeviceTailModelService being updated. This gives memory management policies more flexibility. (1c39e4f) (6c95781)
  • The logic for Page Actions has been refactored out of Browser and into a new, more modular chrome/browser/ui/page_actions component. (4f093f4)
  • The 'Bedrock' initiative continues with large-scale refactorings to migrate code from Browser* to the new BrowserWindowInterface, improving modularity. (dc981f9) (5ed392f) (b5fd10b)

UI & UX

  • The Vertical Tabs feature is now enabled by default on Windows and macOS. Follow-up changes improve drag-and-drop visuals and behavior. (20e810a) (bee36ee) (35a0a9a)
  • A new utility, views::ScopedPaintLock, has been introduced. It allows developers to temporarily block and defer paint updates within a View and its children, which is useful for preventing unnecessary paints during complex UI updates. (9e609b6)
  • On macOS, the compositor is now 'warmed up' when a window is about to become visible, reducing latency and the potential for hangs during window activation. (ae5a92b)
  • On Windows 11, PWA window caption buttons now support Snap Layouts on hover by returning the correct HTMAXBUTTON hit-test value. (9ceea0d)
  • For accessibility, text selection events for native views are now fired on the node itself rather than as a document-level event, improving the experience for assistive technologies. (4b19c4f) (0525c9b)

Extensions

  • The architecture for how extensions handle MIME types is being modernized. A new chrome.mimeHandler.getStreamInfo() API has been defined and internal delegates were updated to route generic MIME handlers through the OOPIF path, improving security and performance. (d680baa) (e8aa93f) (db42fd6)
  • The stable Tab Strip API has been enhanced with new moveTab and moveCollection methods. Concurrently, the experimental WebUI-based Tab Strip feature and its associated context menu API were removed. (f7ccd6c) (17de746)
  • A Content Security Policy (CSP) bypass in the Manifest V2 enforcer was fixed. The sanitizer now correctly processes CSP Level 3 directives like script-src-elem, preventing insecure inline script execution. (99e2591)
  • To prevent cross-renderer interference, webRequest listener removals are now scoped to the originating renderer process. (be668fb)
  • The metricsPrivate.getHistogram API has been restricted via an allowlist, effectively making it a private API not intended for general use. (7446ed8)

Networking & Security

  • A security flaw in the Web Share API was patched. An HTTPS page can no longer use a <base href="file:///"> tag to bypass scheme restrictions and share file: URLs. (65fe959)
  • A bug was fixed on macOS where DRM-protected video could be screen-recorded. The video layer's preventsCapture property is now correctly updated when it is recycled to display protected content. (51a8376)
  • A logic error in insecure download blocking was fixed. The code was incorrectly using bitwise operations on PageTransition core types, which could allow some malicious downloads to bypass mixed-content blocking. (0b7e6fe)
  • The Out-Of-Renderer CORS implementation was updated to allow the Sec-GPC (Global Privacy Control) request header with a value of "1", aligning with privacy standards. (95e3466)
  • A crash in the WebAuthn Google Password Manager (GPM) flow was fixed, improving the stability of passkey recovery on desktop platforms. (cddebef)

Testing & Build System

  • A new test helper, ScopedGlobalFeaturesOverride, was added to simplify the management of global features within tests. (f6607e7)
  • The GlobalBrowserCollection is now mockable, simplifying tests that have a dependency on it. (5507db0) (fa31f88)
  • The Inspector Protocol, which powers DevTools, was rolled to a new version. This can potentially impact third-party tools and automation scripts that rely on it. (ca1f40f)
