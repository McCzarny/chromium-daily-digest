Chromium Digest: 2025-12-04

Overview

This summary covers 453 relevant commits out of 612 total, from commit (bf2bcad) to (1489f15). The day was marked by a major toolchain upgrade to C++23 and numerous breaking changes to core browser, UI, and content APIs, requiring significant updates for downstream projects.

Build System & Toolchain

  • ⚠️ BREAKING The default C++ standard for the Chromium build has been upgraded to C++23. This is a fundamental toolchain change that will require downstream projects to update their build configurations to be C++23 compatible. (5047a65)
  • ⚠️ BREAKING The legacy recovery component has been deleted, removing its public API header recovery_component_installer.h and associated functions like RegisterRecoveryComponent. (d060f5d)
  • ⚠️ BREAKING An unused const char* overload for base::File::WriteAtCurrentPosNoBestEffort has been removed from the public file API. (6a0870c)

Browser & Core APIs

  • ⚠️ BREAKING The core content::ContentBrowserClient embedder interface has been modified, removing the virtual method GetUserAgentBasedOnPolicy. Embedders must now use the simpler GetUserAgent() method. (18b17a9)
  • ⚠️ BREAKING The virtual method IsUnpartitionedStorageAccessAllowedByUserPreference has been removed from the ContentBrowserClient public interface as part of a feature cleanup. (fcea483)
  • ⚠️ BREAKING A new pure virtual method IsBlocked() has been added to the tabs::TabInterface. All custom classes inheriting from this interface must now provide an implementation for this method. (1503703)
  • ⚠️ BREAKING The virtual method ShouldDisplayFavicon has been removed from the public TabStripModelDelegate interface. Favicon display logic is now centralized elsewhere. (cc15891)
  • ⚠️ BREAKING The deprecated AllTabContentsesList class and AllTabContentses() function have been removed from tab_contents_iterator.h. Code iterating over all tabs must use alternative methods. (4fd923f)
  • ⚠️ BREAKING The hosted_domain member of the public AccountInfo struct has been made private. Code must be updated to use the GetHostedDomain() accessor method instead of direct field access. (cca5b70)
  • ⚠️ BREAKING The signature for blink::Platform::DecodeAudioFileData has changed. It now returns a std::unique_ptr<WebAudioBus> instead of taking an output parameter. Embedders implementing this virtual function must update their code. (f889d4d)
  • The feature flag for the built-in HLS media player (kBuiltInHlsPlayer) has been removed, making the feature permanently enabled by default. (f100e37)

UI & Views Framework

  • ⚠️ BREAKING The primary tab strip UI class TabStripViewInterface has been renamed to TabStripRegionView. Embedder code that interacts with the browser's tab strip view will need to be updated. (16f1c19)
  • ⚠️ BREAKING The core graphics struct viz::TransferableResource has been refactored. It no longer contains individual fields like mailbox and size, which are now accessed through a scoped_refptr<gpu::ClientSharedImage>. This breaks any code that directly constructs or accesses the previous members. (bbb801f)
  • ⚠️ BREAKING In ui/views, view builder factory logic has been moved from the core ui/views/view.h header into ui/views/metadata/view_factory.h. This will break code that relied on these functions being available via the common view.h include. (a878963)
  • ⚠️ BREAKING The virtual method OnBubbleWidgetPaintAsActiveChanged() has been removed from the public BubbleDialogDelegate API in ui/views. (000952e)
  • ⚠️ BREAKING The public method views::Combobox::ArrowButtonPressed() has been made private, removing it from the public API. (abc6da4)
  • The ComposeboxHandler for WebUI has been relocated to a new directory under cr_components/composebox/, which will require include path updates for projects using it. (9255297)

Security & Networking

  • ⚠️ BREAKING Public methods in the Safe Browsing ClientSideDetectionHost::IntelligentScanDelegate interface have been renamed (e.g., InquireOnDeviceModel to StartIntelligentScan), breaking custom implementations. (6f6d03b)
  • ⚠️ BREAKING The public WebAuthn header cable_discovery_data.h has been moved to a new path under device/fido/public, breaking existing include paths. (8735bd5)
  • ⚠️ BREAKING As part of the third-party cookie deprecation (3PCD) cleanup, the public API methods IsCookieDeprecationLabelAllowed and IsCookieDeprecationLabelAllowedForContext have been removed from content::ContentBrowserClient and components::privacy_sandbox::PrivacySandboxSettings. (26c4ae9)
  • The network service's client certificate cache will now be flushed when the AutoSelectCertificateForUrls policy changes, allowing new rules to take effect without a browser restart. (5a79971)
  • The UnexportableKeyProvider on macOS was updated to retrieve all available hardware-bound keys, improving key management capabilities. (a506a14)
  • The Local Network Access (LNA) policy LocalNetworkAccessRestrictionsEnabled has been removed, as the feature has now fully launched. (31ef206)

Extensions

  • Extension service worker registration will now be retried in more transient error cases, such as kErrorDiskCache, improving reliability. (152316e)
  • Logic has been added to clear persisted filtered listeners from preferences upon startup, aligning their behavior with other registered events. (13aeb38)

Testing & Developer Tools

  • A potential crash in WebContentsDevToolsAgentHost has been fixed for scenarios where a page is attached to itself more than once. (690fdc4)
  • A regression test was added for a ChromeDriver send_keys bug that occurred when sending very long strings. (ad67ca6)
