Chromium Weekly: 2026 Week 17

Overview

This week saw a massive number of breaking changes, most notably the complete removal of the core BrowserList API, requiring significant refactoring for any embedder managing browser windows. The ongoing modularization effort continued to break up the monolithic //chrome/browser target, while major parts of the legacy remoting protocol were deleted. Across 7 days, there were 929 relevant commits out of 1247 total.

Core Browser Architecture & Lifecycles

  • ⚠️ BREAKING BREAKING CHANGE The fundamental BrowserList class and many static helper functions in browser_finder.h have been completely removed. This was the primary API for iterating over open Browser instances. The widely-used global function chrome::GetBrowserCount() was also removed, requiring migration to the new profile-specific ProfileBrowserCollection API. (499cfae) (ba141b8) (af10f4e) (22369dd)
  • ⚠️ BREAKING BREAKING CHANGE The public BrowserWindow interface has been significantly slimmed down. Methods related to sharing, "Send Tab to Self", DevTools docking, and QR code generation have been removed and relocated to new dedicated controllers, breaking embedders who implement or call these methods. (c20a9e0) (ab08725) (c9a8aa7) (d383d4e)
  • ⚠️ BREAKING BREAKING CHANGE The ongoing modularization effort has extracted several components from the monolithic //chrome/browser target. Embedders must now explicitly depend on new targets for browser lifetime management (//chrome/browser/lifetime), sessions (//chrome/browser/sessions:sessions), and signin (//chrome/browser/signin) to access their APIs. (94c8534) (b904862) (3f51165)
  • ⚠️ BREAKING BREAKING CHANGE The core base::DeletePathRecursively file utility has changed its behavior on Windows to no longer follow reparse points (e.g., symlinks), aligning with POSIX behavior. This could silently break code that relied on the previous behavior of deleting linked content. (4ca74d8)

Other Major API Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE The legacy (non-WebRTC) remoting protocol stack has been almost entirely removed. This includes public headers and classes for the Chromotocol protocol, PseudoTCP, datagram sockets, and the legacy video pipeline, significantly impacting any embedder using these components. (053d46b) (76fc706) (2b28768) (9cc0984)
  • ⚠️ BREAKING BREAKING CHANGE Core networking APIs were modified. net::URLRequest methods that returned byte counts now use the unsigned type net::ByteSize instead of int64_t. The net::HttpStreamKey constructor now requires a target_network parameter, breaking direct instantiations. (8ade05b) (6b8a801)
  • ⚠️ BREAKING BREAKING CHANGE The Identity and Sign-in APIs have been refactored to decouple from Sync. This removes ConsentLevel.SYNC from the IdentityManager API and the revokeSyncConsent() method from IdentityMutator, requiring updates for embedders managing primary accounts. (b71b759) (960d5db)
  • ⚠️ BREAKING BREAKING CHANGE The content::DirectSocketsDelegate interface was simplified, removing three virtual methods related to Private Network Access checks. Embedders who implement this delegate must update their code. (c86e354)
  • ⚠️ BREAKING BREAKING CHANGE Multiple Autofill and Password Manager interfaces were broken. ContentAutofillClient lost TriggerPlusAddressUserPerceptionSurvey, PasswordManagerDriver lost ShowPasswordSuggestions(), and several Plus Address APIs were removed entirely. (2f6d1f8) (ff574ae) (8635789)

Security

  • Mojo IPC on Windows has been hardened against PID-reuse and named pipe squatting attacks. The system can now validate the caller's image path and verifies that the server process has an integrity level equal to or higher than the client. (3d3c54b) (0e083d3)
  • Post-quantum cryptography (PQC) support has been enabled by default for WebRTC's DTLS handshake via the WebRtcPqcForDtls feature, improving security against future cryptographic threats. (274fc7f)
  • A security vulnerability in the File System Access API was fixed by adding an origin check to prevent a compromised renderer from monitoring files outside its permitted scope. (12f2a15)
  • A JSON injection vulnerability in the Live Caption Translation feature was fixed by replacing an unsafe base::StringPrintf call with the base::WriteJson API to ensure proper escaping. (a1ab0fb)

Extensions & Enterprise

  • ⚠️ BREAKING BREAKING CHANGE The --enable-unsafe-extension-debugging command-line flag has been removed. Automation or testing infrastructure that relied on this flag to install extensions via the Chrome DevTools Protocol will no longer function. (3e47d0a)
  • A new enterprise remote command, ExtensionUpdateCheckJob, has been implemented, allowing administrators to remotely trigger an update check for all installed extensions. (527aa2f)
  • Pages containing iframes with a chrome-extension:// source will now be ineligible for the Back-Forward Cache (BFCache). This behavioral change improves security and stability at the cost of potential performance for affected pages. (ea39006)
