Chromium Digest: 2025-12-24

Overview

This digest analyzes 56 commits from the Chromium repository, filtering down to 11 changes relevant to Windows and macOS embedders. The selection focuses on APIs, networking, and core browser functionalities, excluding platform-specific and internal rendering changes. This batch contains four major breaking changes that affect the Supervised User service, Task Manager, and Networking APIs, requiring attention from developers building on top of Chromium. APIs & Services

  • ⚠️ BREAKING Breaking Change: The supervised_user::SupervisedUserServiceObserver interface has been modified, removing the OnSearchContentFiltersChanged() and OnBrowserContentFiltersChanged() methods. This refactoring centralizes safe search preference observation. Embedders using this observer to monitor changes in supervised user content filters will need to adapt to the new mechanism, which involves observing the preference changes directly. (d6e3c60)
  • ⚠️ BREAKING Breaking Change: The public API for the Task Manager has been updated to use modern, type-safe C++ constructs. Key methods in TaskManagerInterface and Task now use base::ByteSize for network statistics instead of the deprecated base::ByteCount. Notably, GetProcessTotalNetworkUsage() now returns a std::optional<base::ByteSize> instead of using a negative number as a sentinel value, and the unused method GetCumulativeProcessTotalNetworkUsage() has been removed entirely. These changes require embedders who interact with the Task Manager to update their method calls and type handling. (c89f2a8)
  • ⚠️ BREAKING Breaking Change: In the URL ranking components, the public enum TabEventTracker::TabSelectionType has been renamed to TabEventTracker::TabSelectionCause. This change affects the public header components/visited_url_ranking/public/url_grouping/tab_event_tracker.h. Any code that uses this enum to track the cause of tab selections must be updated to use the new name. (070a08b)
  • The Windows elevation service, used for recovery operations, will now delete the ChromeRecovery directory upon execution. This enhances cleanup by removing leftover files from the recovery component. (07b2d30)
  • A behavioral change in the supervised user service now ensures incognito tabs are closed immediately when the related preference is updated, rather than as a side-effect of other changes. (3a0d3c1)

Networking

  • ⚠️ BREAKING Breaking Change: A dependency roll of the QUICHE library has removed several deprecated methods from the public quic::ParsedQuicVersion API. Methods such as UsesTls(), UsesHttp3(), SupportsGoogleAltSvcFormat(), and KnowsWhichDecrypterToUse() have been removed in favor of a single, unified IsIetfQuic() method. Projects that directly interact with Chromium's QUIC implementation must update their code to use this consolidated API. (c5b2270)

UI & UX

  • A recent feature for asynchronous hit-testing during drag-and-drop operations on macOS has been reverted. This was done to fix a crash observed in Canary builds, restoring the previous synchronous behavior until the root cause is identified. (93bedac)
  • The print preview dialog has been enhanced to include an "actual size" option in the PDF scaling dropdown menu. This allows users to print PDF documents without any scaling or centering applied. (abdd4a8)
  • A bug fix in the Aura windowing system prevents a crash that could occur when a window's parent was changed during a transient window restacking operation. (8a92bb3)

Refactoring & Infrastructure

  • Mojo C++ bindings have been modernized to use C++20 concepts instead of std::enable_if for template metaprogramming. This is a large-scale internal refactoring that improves code readability and compiler diagnostics without changing API behavior. (f1db470)
  • Various internal components and tests have been updated, including a migration of storage components to new crypto APIs, updates to presubmit scripts, and refactoring of Windows-specific test code. (ed186c6) (f718d2c) (5276a73)
