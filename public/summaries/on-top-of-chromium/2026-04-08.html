Chromium Digest: 2026-04-08

Overview

This summary covers 793 commits on the main branch, with 622 identified as relevant to developers working on Chromium-based projects. The changes span from commit (fea075a) to (1572e92). This was an exceptionally active day for API refactoring, resulting in a large number of breaking changes across core utilities, browser interfaces, and networking. Other notable changes include ongoing security hardening and UI refinements to context menus and side panels.

Breaking Changes: Core APIs & Interfaces

  • ⚠️ BREAKING base::ObserverList now disallows re-entrant iteration by default on desktop platforms. The default policy has been changed from kAllowReentrancy to kDisallowReentrancy. Code that relies on nested observer notifications will now trigger DCHECK failures and must be updated to explicitly use the kAllowReentrancy policy. (83dc1d0) (cf7e006)
  • ⚠️ BREAKING The BrowserWindow public interface has been significantly refactored. The pure virtual method ZoomChangedForActiveTab() has been removed, and numerous utility functions in signin, sync, and web_applications now accept a BrowserWindowInterface* instead of a Browser*. Custom BrowserWindow implementations must be updated. (0501de8) (c7fb45c)
  • ⚠️ BREAKING The method for querying browser instances has been changed. The free functions chrome::GetTotalBrowserCount() and chrome::GetBrowserCount() have been removed. Callers must migrate to the new GlobalBrowserCollection and ProfileBrowserCollection APIs. (ebcb57e)
  • ⚠️ BREAKING The extensions::RuntimeAPIDelegate::OpenOptionsPage method is now asynchronous. Its signature has changed from returning a bool to returning void and taking a base::OnceCallback<void(bool)>. Embedders with custom implementations must update to the new signature. (011e693)
  • ⚠️ BREAKING The content::RenderWidgetHostView public interface has a new pure virtual method, HasSavedFrame(). Embedders with custom implementations of this interface must add an implementation for this new method to avoid compilation errors. (1e42bd2)
  • ⚠️ BREAKING The public virtual method ContentBrowserClient::IsIsolatedContextAllowedForUrl has been removed. Embedders must now implement the new DirectSocketsDelegate::AreDirectSocketsAllowed to grant access to APIs like Direct Sockets. (5697236)
  • ⚠️ BREAKING The public constructor for AuthenticatorWin now requires a gfx::NativeWindow. Code that relied on the previous constructor, which automatically found the active browser window, must be updated to explicitly provide the native window handle. (3df2ead)
  • ⚠️ BREAKING The RecordReplayDriver class has been refactored into a pure abstract interface. Embedders directly using the old concrete class must update their code to use the new ContentRecordReplayDriver implementation and associated factory patterns. (560f67a)
  • ⚠️ BREAKING The static method TranslateLanguageList::DisableUpdate() has been removed. Tests that used this method to prevent server-side language list updates must be updated to use the new constructor that allows injecting a mock TranslateUrlFetcher. (6aef37c)
  • ⚠️ BREAKING Multiple base and content APIs have been changed. This includes replacing std::map with base::FieldTrialParams in feature list APIs, removing GetIgnoredDuplicateNavigationCount() from content::NavigationHandle, and refactoring base::trait_helpers to use base::ParameterPack. (9aa846e) (2950468) (0b91f88)

Breaking Changes: Networking & Mojo

  • ⚠️ BREAKING Significant Third-Party Cookie Deprecation (3PCD) logic has been removed from the network stack. This includes the removal of the kTpcdMetadataStageControl and kTpcdMetadataGrants features, TPCD_METADATA and TPCD_HEURISTICS content setting types, and related DevTools parameters. Embedders relying on these for 3PCD integration will need to adapt. (708fe41)
  • ⚠️ BREAKING APIs for secure cookie exemptions have been migrated from scheme/host-based to origin-based. This changes method signatures in content_settings::CookieSettings and net::CookieAccessDelegate, requiring embedders to update how they configure cookie exemptions. (1575beb)
  • ⚠️ BREAKING The ContentSettingsManager::AllowStorageAccess mojo interface signature has been changed. The origin, site_for_cookies, and top_frame_origin parameters were removed, as this information is now derived from the RenderFrameHost on the browser side. (8c04824)
  • ⚠️ BREAKING The chrome.mojom.PluginInfoHost.GetPluginInfo() Mojo interface has been changed. The origin input parameter was removed to prevent the renderer from supplying an incorrect origin. Consumers and implementers of this interface must update to the new signature. (3c9de1b)

Security

  • Multiple security vulnerabilities were addressed, including a Use-After-Free in PasswordFormManager, a state leak in the File System Access API, a UAF in WebSocketSpdyStreamAdapter, and a potential out-of-bounds write in the GPU command buffer. (c542194) (a4c6a71) (5d8f582) (dc5e20c)
  • WebAuthn "Immediate Mode" with passwords and passkeys has been enabled by default. (7b93b74)
  • A potential browser crash during printing has been fixed. The browser now safely rejects an invalid memory handle from the renderer instead of crashing. (7d72d49)

APIs & Extensions

  • The views::WebView class now includes a SetOwnedWebContents() method, allowing it to take ownership of an externally created WebContents instance. (a203cc3)
  • The PageContentExtractionService has new asynchronous methods (Get...Async) that wait for initial content extraction to complete, preventing race conditions on page load. (9bd9cbe)
  • Service worker registration for extensions will now be retried on a transient kErrorNotFound error, improving reliability in certain enterprise environments. (0c3687a)
  • The network::DataElement public API was enhanced with a TryAs<T> helper to simplify type checking and data access in a single operation. (09fd4dc)

Browser UI & Features

  • The "Send to your devices" option in context menus has been converted into a dynamic submenu that directly lists available target devices, controlled by the kSendTabToSelfShowTargetsInContextMenus feature. (1399633)
  • The "Tabs From Other Devices" side panel now has an entry point in the 3-dot menu and its availability is now restricted to regular profiles (excluding incognito and guest). (0964b4e) (33653e7)
  • On macOS, support for the AXTextOperation parameterized attribute was implemented, enabling bulk text operations like find-and-replace for accessibility clients. (dea0b52)
  • The TabStrip WebUI playground at chrome://tab-strip/, an early testing ground for the feature, has been removed as it is now obsolete. (5e4e689)

Testing & Infrastructure

  • ChromeDriver was optimized to avoid redundant focus script calls if an element is already active, which could previously cause issues with contenteditable elements. (a310a5c)
  • A large-scale refactoring migrated numerous keyed service factories from the Singleton pattern to the preferred NoDestructor pattern for improved memory management and shutdown behavior. (6d036b4)
