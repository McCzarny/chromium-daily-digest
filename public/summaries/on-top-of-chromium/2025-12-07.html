Chromium Digest: 2025-12-07

Overview

This report analyzes 12 commits, focusing on changes relevant to Windows and macOS browser development. After filtering out platform-specific (Ash, iOS) and minor changes, one major breaking change was identified in the content public API. Other updates include additions to base utilities, fixes in policy handling, and ongoing development of experimental UI features.

Browser Core & APIs

  • ⚠️ BREAKING A breaking change was introduced to the core content::RenderProcessHost public interface. The virtual method FastShutdownIfPossible now includes a new optional parameter, use_outermost_main_frame_check. Embedders who inherit from this class and override this method must update their function signatures to match, otherwise their code will fail to compile. (1095311)

Base Utilities

  • New helper functions have been added to base/byte_size.h to facilitate conversion from the deprecated base::ByteCount type to the modern base::ByteSize and base::ByteSizeDelta types. This encourages migration to the newer, safer types. (1bd1f4e)

Policy

  • A potential double-free error was fixed in CloudPolicyClient within the OnPromotionEligibilityDetermined method, improving stability for enterprise policy handling. (bd8ae4f)

UI & Features

  • Development continues on the experimental 'Contextual Tasks' feature, with changes to the compose box layout and the addition of voice input functionality. These changes are behind a feature flag and do not affect stable builds. (fb175d8) (b158c41)

Housekeeping

  • Routine updates include the ingestion of new translations and the removal of obsolete TODO comments from various parts of the codebase, including safe browsing tests. (a51642e) (725c483) (3a3a8a0)
