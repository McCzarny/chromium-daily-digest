Chromium Digest: 2026-05-07

Overview

This digest covers 776 commits to the Chromium main branch on May 7th, 2026, with 573 commits identified as relevant to developers building on the platform. The day was marked by an unusually high number of breaking changes across core APIs, extensions, networking, and the build system, requiring careful attention from embedders. The first commit of the day was 175b21588a85f119f3590b162d3f82eca74b21e6 and the last was 81c154bee3c796d05d993b89e44bf43e3ac92bd9.

Breaking API & Build Changes

  • ⚠️Multiple core browser interfaces have been modified. content::WebContents::ForSecurityDropFullscreen now returns std::optional<base::ScopedClosureRunner> to fix a UAF. TabListInterface::MoveTabGroupToWindow now returns a bool to indicate success. The bookmarks::BookmarkClient interface requires a new pure virtual method implementation: GetBookmarkFormFactor(). A recently added CreateBubble method on BubbleDialogDelegateView was reverted. (058c2d6) (daea58c) (85f5cc3) (275e4e3)
    BrowserAPIBookmarks
  • ⚠️The extensions::WebRequestInfo struct has been changed, replacing int render_process_id and int frame_routing_id with a strongly-typed content::GlobalRenderFrameHostId. Additionally, the chrome.mimeHandler.abortAndFallbackToNativeHandler() extension API was removed shortly after its introduction. (0f97ab9) (18dd6a9) (176ea56)
    ExtensionsAPI
  • ⚠️The net::URLRequest class no longer has the set_force_ignore_site_for_cookies() method; this logic is now centralized in the NetworkDelegate. In the File System API, the origin parameter has been removed from the blink::mojom::FileSystemManager::Open Mojo interface. (c0edd27) (5c6dc2f)
    NetworkStorageAPI
  • ⚠️Graphics, Media, and XR APIs saw several breaking changes. cc::LayerTreeHostSingleThreadClient was renamed to cc::LayerTreeHostSingleThreadDelegate. The tex_coord_rect field was removed from viz render pass quads. MediaRemoter::Client requires a new CreateRemotingDataStreamSender method. The content::XrInstallHelper::EnsureInstalled callback now uses an enum instead of a boolean. On macOS, the VideoCaptureProvider interface for screen capture was updated. Security & Privacy

  • A new enterprise policy, IsolatedModeSettings, has been introduced, allowing administrators to replace the standard Incognito mode with a new "Isolated mode" profile type. (b2955dd)
    EnterprisePolicySecurity
  • A vulnerability in the PWA LaunchQueue was fixed. The system now ensures a navigation has committed before delivering File System Access handles, preventing a malicious page from intercepting them. (2375aea)
    SecurityStorage
  • A logic flaw was corrected where HTTPS-First Mode exemptions were incorrectly being applied to users with Advanced Protection enabled. (46126f9)
    SecurityNetwork
  • A Use-After-Free vulnerability when handling redirects to chrome-extension:// URLs was fixed by ensuring the Extension object is kept alive via scoped_refptr during security checks. (79f10bb)

API & Extensions

  • A new convenience function, SimpleMenuModel::SetIconForCommandId, has been added. This provides a more robust way to set menu item icons by command ID instead of relying on brittle index-based access. (0b02dcb)
    UIAPI
  • The SoundsManager has been moved from a ChromeOS-specific location to the more general //services/audio directory. This refactoring allows it to be reused for browser audio playback on desktop platforms like Windows and macOS. (608ac09)
    BrowserMediaAPI
  • For WebUI In-Product Help (IPH), a new containerElement option allows developers to specify a custom parent for help bubbles, which is particularly useful for displaying help within modal dialogs. (be0c292)
