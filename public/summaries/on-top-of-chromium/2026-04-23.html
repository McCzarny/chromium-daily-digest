Chromium Digest: 2026-04-23

Overview

This summary covers 489 relevant commits out of 646 total from the main branch on April 23, 2026. The changes span from (c2744eb) to (6523f9d). This digest highlights significant API modifications, build system refactoring, and security enhancements impacting developers of Chromium-based projects. Multiple breaking changes were landed, affecting core browser interfaces, identity management, security APIs, and the build system.

Build System & Infrastructure

  • ⚠️ BREAKING BREAKING: The //chrome/browser/sessions component has been modularized into separate public (:sessions) and implementation (:impl) targets. Embedders must now explicitly add a dependency on //chrome/browser/sessions:sessions to access session management APIs, as they are no longer part of the main //chrome/browser:browser target. (b904862)
  • ⚠️ BREAKING BREAKING: The Scripting API has been moved from //chrome/browser/extensions/api/scripting to //extensions/browser/api/scripting. This is a build-breaking change requiring updates to include paths for any project that uses this API. (1b72b88)
  • A libc++ dependency roll has removed its custom, non-standard public header <float.h>. Builds that directly include this header will now fail. (ffd3856)

APIs: Core, Identity, and Sign-in

  • ⚠️ BREAKING BREAKING: The core BrowserWindow interface has been significantly slimmed down. Methods related to DevTools docking (UpdateDevTools, CanDockDevTools) and QR code generation (ShowQRCodeGeneratorBubble) have been removed. Embedders must now use the new internal DevtoolsUIController and qrcode_generator::QRCodeWindowController classes for this functionality. (c9a8aa7) (d383d4e)
  • ⚠️ BREAKING BREAKING: As part of a major refactor to decouple sign-in from sync, ConsentLevel.SYNC has been removed from the IdentityManager and IdentityMutator APIs. Embedders managing primary accounts or sync consent must migrate to the new APIs that no longer use this consent level. (b71b759)
  • ⚠️ BREAKING BREAKING: The is_child_account field has been removed from the public GaiaAuthConsumer::ClientOAuthResult struct. Code that relies on this field for supervision signals must be updated to use Account Capabilities instead. (8165306)
  • ⚠️ BREAKING BREAKING: The StartupTracingController is no longer a singleton accessible via content::StartupTracingController::GetInstance(). The class has been moved to services/tracing/public/cpp, and embedders must now instantiate and manage its lifecycle, likely via BrowserMainLoop. (db2fbde)
  • ⚠️ BREAKING BREAKING: The public function chrome::FindBrowserWithGroup has been removed from browser_finder.h. Any code that relied on it to locate a browser associated with a tab group will break. (cf0a15d)
  • ⚠️ BREAKING BREAKING: The public method SetMemoryLimit on DiscardableSharedMemoryManager has been renamed to SetMaxBytes. (bb585b3)
  • ⚠️ BREAKING BREAKING: Several public functions have been removed from TemplateURLTableModel as part of a refactor of the search engine settings. Clients should access the TemplateURLService directly instead. (0bd9372)

Security & Permissions

  • ⚠️ BREAKING BREAKING: The public UnexportableKeyService mojo interface has been updated. Methods like Sign now use a new role-specific UnexportableSigningKeyId, and the garbage collection method was renamed to GetAllKeysForGarbageCollectionSlowlyAsync. Clients of this API must update method calls and type usage. (c84ea43) (234b233)
  • ⚠️ BREAKING BREAKING: The permissions::PermissionRequestData struct has been refactored to be a data-only struct, removing the PermissionResolver member. Call sites must adapt to the new way of constructing permission requests. (35ad82c)
  • ⚠️ BREAKING BREAKING: The WebSensorProvider::GetSensor Mojo interface signature has been changed to include a user_gesture boolean, which is now required to gate permission prompts on user activation. (5fbd61a)
  • ⚠️ BREAKING BREAKING: The LeakDetectionCheck::Start method in the Password Manager has changed its signature to accept a callback, breaking existing integrations with the password leak detection service. (43a05b0)
  • A security vulnerability in the File System Access API has been fixed. An origin check was added to prevent a compromised renderer from monitoring files outside its permitted scope. (12f2a15)

Networking & Page Metrics

  • ⚠️ BREAKING BREAKING: The net::HttpStreamKey constructor signature has changed. It now requires a target_network parameter to support multi-network capabilities, breaking any code that directly creates instances of this core networking key. (6b8a801)
  • ⚠️ BREAKING BREAKING: The signature of viz::GpuHostImpl::EstablishGpuChannel has been changed to require a mojo::ScopedMessagePipeHandle. This change, part of a GPU channel initialization optimization, will break existing callers. (221b0ba)
  • ⚠️ BREAKING BREAKING: The PageLoadMetricsObserver interface has been changed by renaming the virtual method OnMainFrameIntersectionRectChanged to OnMainFrameRectChanged. All implementations of this observer must be updated. (b269479)
  • The certificate verification service can now factor in network time uncertainty, which is plumbed through its Mojo interface to improve verification accuracy. (823df87)

Extensions & Mime Handling

  • ⚠️ BREAKING BREAKING: The MimeHandlerStreamManager has been relocated from chrome/browser/pdf/ to extensions/browser/mime_handler/. Its public methods have also been renamed to be generic (e.g., IsPdfExtensionHost is now IsExtensionHost), breaking embedders who interact with this system. (35639b0) (371cec5)
  • The --enable-unsafe-extension-debugging command-line flag has been removed. Automation or testing infrastructure that relied on this flag to install extensions via the Chrome DevTools Protocol will no longer function. (3e47d0a)
  • A race condition that could cause duplicate service workers to be tracked for a single extension has been fixed by comparing worker version IDs. (efa90ae)

UI & Features

  • On macOS, icon handling has been temporarily removed from menus to fix an issue where many menu items were unexpectedly showing icons. This restores the previous UI behavior. (c59a04f)
  • On macOS, an internal resampling mechanism was added to the CoreAudio TAP (CATap) loopback implementation to ensure consistent audio capture when system and stream sample rates differ. (06b4b92)
  • A bug causing duplicate pinned tabs to appear on startup has been resolved. (82325a5)
