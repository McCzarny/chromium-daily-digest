Chromium Digest: 2026-04-26

Overview

This digest covers 17 commits, with 9 relevant to desktop browser development after filtering for platform and scope. The changes focus on UI performance, feature cleanup, and infrastructure improvements. No major breaking changes to stable public APIs were identified in this set of commits. The analysis ranges from commit (c708812) to (27ab255).

Browser UI & Features

  • The Omnibox popup's performance has been improved by deferring its display until visually ready and keeping its WebContents attached on hide to prevent renderer stalls. This behavior is controlled by new feature flags (kOmniboxAimDetachWebContentsOnHide, kOmniboxWebUIDetachWebContentsOnHide). The internal OmniboxPopupPresenterBase::ShouldDeferUntilVisualStateReady() method signature was changed from returning a bool to std::optional<base::TimeDelta> to simplify the interface. (c708812)
  • Minor cleanups and refactors were made to several browser features. The feature flag kIPHPlusAddressFirstSaveFeature for an In-Product Help bubble was removed. Internal code for the 'Send Tab to Self' feature was refactored for clarity. Additionally, metrics for NTP voice search were migrated to a unified component to support future deprecation of legacy code. (82f1866) (4901bca) (a2652ff)
  • Several minor UI tweaks were applied to the experimental 'Contextual Tasks' feature, including adjustments to composebox animations and CSS margins. (ecf2c1f) (27ab255)

Infrastructure & Tooling

  • The Windows installer logic for detecting duplicate file hierarchies (IsIdenticalFileHierarchy) has been improved with additional logging and a bug fix to correctly return false on enumeration errors. Separately, a performance refactor in the accessibility code (ui/accessibility) reduces unnecessary string copies by using string_view and base::JoinString. (9d390f2) (6de91e4)
