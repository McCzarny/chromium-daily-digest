Chromium Digest: 2026-03-24

Overview

On March 24, 2026, a total of 687 commits were made to the Chromium main branch, with 519 being relevant to developers building on top of the browser. This day saw a significant number of breaking API changes across core components, networking, and UI, requiring careful review and code updates for downstream projects. Key themes include major refactoring for modularity, API hardening for type safety, and the removal of several deprecated features and build flags. The first relevant commit was (bb7e88b) and the last was (7c7425f).

Core & Build System

  • ⚠️ BREAKING BREAKING: The core feature declaration macros (BASE_DECLARE_FEATURE, BASE_FEATURE) have been moved from base/feature_list.h to a new base/feature.h header. Files that include the former to declare a feature will now fail to compile and must be updated. (5063835)
  • ⚠️ BREAKING BREAKING: The public method SessionCommand::size() has been removed as part of an interface formalization. Embedders must now use SessionCommand::contents().size() to get the size of the command's contents. (364c522)
  • ⚠️ BREAKING BREAKING: The public interface of syncer::DeviceInfo has been refactored to remove dependencies on internal sync_pb protocol buffer types. Clients must now use native C++ types (e.g., syncer::DeviceInfo::SharingFeature) instead of protobuf enums. (165bf33)
  • ⚠️ BREAKING BREAKING: Rust build targets for sequencing and task-related code have been moved from //mojo to //base. Projects with Rust code must update their build configurations to depend on the new //base:base_rust meta-target. (e70725a)
  • ⚠️ BREAKING BREAKING: The //third_party/material_design_icons dependency has been removed from the repository. Any downstream project that implicitly relied on its presence will fail to build. (a2b222e)
  • The PostTaskAndReplyWithResult utility in base/task now supports reply callbacks with multiple arguments by allowing the task function to return a std::tuple. (b791274)
  • The unmaintained build_with_tflite_lib build option has been removed from several components, including omnibox, permissions, autofill, and media. TensorFlow Lite is now consistently enabled where used. (711d2fc) (c907c62) (16a600b) (16777dd)

Networking

  • ⚠️ BREAKING BREAKING: The public constructor and comparison operators for net::ClientSocketPool::GroupId have been modified to include a target_network parameter. This is a core networking change for multi-network support that will break any code that constructs or compares GroupId objects directly. (52a2dea)
  • ⚠️ BREAKING BREAKING: The kSkipTpcdMitigationsForAds feature flag has been removed. This enforces Third-Party Cookie Deprecation mitigations for all users, which may break systems that relied on this flag to maintain old cookie behavior. (5414c90)
  • Work has begun to plumb a target_network handle through the networking stack, starting with URLRequest and HttpRequestInfo. This is a foundational step towards enabling true multi-network support. (87a4047) (337a3df)
  • The SystemProxyResolutionService in net/proxy_resolution was refactored into a platform-agnostic base class to reduce code duplication and simplify future implementations. (883a3b6)

Browser UI & Features

  • ⚠️ BREAKING BREAKING: The WebUI helper function cr.sendWithPromise() in ui/webui/resources/js/cr.ts now requires a generic type parameter specifying the expected return type (e.g., cr.sendWithPromise<string>('my-handler')). Calls without this parameter will fail TypeScript compilation. (7c7425f)
  • ⚠️ BREAKING BREAKING: Methods in BrowserTabStripController have been changed to no longer accept a TabData parameter. Embedders or tools that subclass or directly call methods on this controller to create or manage tabs must update their calls. (74da1f8)
  • ⚠️ BREAKING BREAKING: Core tab-related enums, including TabAlertState and TabNetworkState, have been moved from various chrome/browser/ui locations to a new public component at components/tabs/public/. Embedders must update their #include paths. (b5a1e23) (6b531e1) (a9634cc)
  • ⚠️ BREAKING BREAKING: Enums related to the deprecated Tab Organization feature have been removed from the stable chrome/browser/ui/webui/tab_search/tab_search.mojom interface. Clients using these enums will fail to compile. (d479cf0)
  • The FindBrowserWithActiveWindow() function has been refactored to return a BrowserWindowInterface* instead of a Browser*. This change is part of an initiative to decouple UI from the core browser object. (7af109e)
  • New observer notifications, OnWidgetUserDragStarted and OnWidgetUserDragEnded, have been added to views::WidgetObserver to allow clients to distinguish between window dragging (moving) and resizing on Windows and macOS. (b4fe7f8) (070e93c)
  • A new animation controller has been added at chrome/browser/ui/animation to help orchestrate complex, multi-element animations within the browser UI. (547a110)

Autofill & Payments

  • ⚠️ BREAKING BREAKING: The AutofillSharedStorageHandler class has been removed, and the constructor for PersonalDataManager has been updated to no longer accept a shared_storage_handler parameter. Embedders who construct this class manually must update their code. (d7b4885)
  • ⚠️ BREAKING BREAKING: The public function autofill::ToSafeFieldType has been changed to return std::optional<FieldType> instead of FieldType. Callers must be updated to handle the optional return value, for example by using .value_or(). (b834d1a) (31cce64)
  • ⚠️ BREAKING BREAKING: The AccessibilityQueryService::Query public API has changed. The callback signature was updated and the MemorySearchResult struct was refactored to support structured metadata. (6619ea2)

Security & Privacy

  • ⚠️ BREAKING BREAKING: RenderFrameHostImpl::EnterFullscreen() now strictly enforces the 'fullscreen' Permissions Policy at the browser level. Calls that previously might have succeeded without the policy will now be rejected, which is a significant behavioral change for embedders. (23da98a)
  • ⚠️ BREAKING BREAKING: The method signature for device::usb::DeviceImpl::HasControlTransferPermission has been changed to include a direction parameter to support new WebUSB telemetry. Direct callers of this service API must be updated. (e30b44e)
  • A new feature flag, kIgnoreSearchProviderOverrides, has been added. When enabled, Chrome will disregard the kSearchProviderOverrides preference, preventing unwanted changes to the default search provider. (24ef1aa)
  • On Windows, clipboard handling was improved to correctly process top-down DIB images that have a negative height value, preventing failures when reading such images from the clipboard. (a824c19)

Extensions

  • The kExtensionBrowserNamespaceAndPolyfillSupport feature is now enabled by default. This is a major improvement that enables the browser.* API namespace for all extensions, greatly improving cross-browser compatibility with Firefox and other browsers that use this standard. (6f12cd1)
  • The WebstoreInstaller has been moved from //chrome to //extensions to improve modularity. This required introducing new interfaces to plumb dependencies like InstallVerifier from factories in //chrome. (cb457f6)

Enterprise

  • ⚠️ BREAKING BREAKING: The InitializeRequest method in enterprise_connectors::ContentAnalysisInfoBase is now a pure virtual method. Any embedder implementing this base class for content analysis must now provide an implementation. (2d6e854)
  • A new HasActiveScreenshotRule method has been added to RulesServiceBase for Data Controls, providing a convenient way for UI components to check if any active rules restrict screenshots. (26fe1a8)

Testing & Developer Tools

  • A new webmcp (Web Machine-learning Control Plane) CDP domain handler has been added on the browser side to host tool call log events. (c4dd9ad)
  • The ui/base/interaction framework for interactive tests was enhanced with the ability to declare, set, and get 'temporary' values that can be shared between test steps. (8d32c08)
