Chromium Digest: 2025-11-13

Overview

This digest summarizes 442 relevant commits out of 608 total, from commit a176b703fd9f to 271992280b0a. The day's highlights include breaking changes removing plus address suggestions and an In-Product Help feature, significant improvements to the base library with new helpers like `GetAvailablePhysicalMemory` and `LazyStringBuilder`, and impactful UI changes to tab management and security features.

Breaking Changes

  • ⚠️ BREAKING In-browser suggestions for creating plus addresses in Autofill have been removed as part of the feature's deprecation. (69b5460)
  • ⚠️ BREAKING The In-Product Help (IPH) feature related to the "Customize Chrome" refresh, controlled by the kIPHDesktopCustomizeChromeRefreshFeature flag, has been removed from the codebase. (f8581d1)

Browser UI & Features

  • Support for vertical and WebUI tab strips has been re-integrated into the new browser layout, ensuring custom UI projects can continue to leverage these features. (e38dca2) (495914b)
  • Page action icons can now be colored using accent colors, providing more flexibility for UI theming beyond the default foreground color. (444d85d)
  • The "Tabs from other devices" section is now hidden if a user has disabled history or tab sync, more accurately reflecting user privacy choices. (0450618)
  • A new feature parameter has been added to the Desktop Zoom feature, allowing for Finch experimentation with different default scaling factors. (d3c2f4a)
  • Favicons are now pre-loaded when manually restoring tabs from a session, improving perceived performance. (9b8ef9b)

Security & Privacy

  • The Navigation Gating feature, which displays a confirmation dialog before navigating to potentially sensitive URLs, has been enabled by default. (b0bc944)
  • The Passkeys UI now checks for the availability of User Verifying (UV) keys before displaying an error, improving the user experience. (2896dfc)
  • A new PasskeyUnlockManager feature flag has been introduced to separate passkey state management from its UI, allowing for more granular feature control. (846932a)
  • Enterprise Data Loss Prevention (DLP) checks for blocking images now apply to cloud-based scans when pasting clipboard content, extending protection beyond local scans. (fe8ea79)

Developer Tools & Testing

  • DevTools can now be opened directly to a specific panel via a URL parameter. For example, using panel="performance" will open the Performance panel directly. (096d042)
  • The "Actor" automation framework now dispatches a mousemove event before a click event to more closely simulate real user interaction in tests. (fe03c46)
  • An improvement to headless mode on macOS was reverted due to test failures, indicating ongoing work in this area. (a0e2946)

Base & Core Improvements

  • A new platform-agnostic helper, base::GetAvailablePhysicalMemory(), has been added to provide a consistent way to estimate available memory. (d42f011)
  • A new high-performance base::LazyStringBuilder class was introduced to build strings by retaining references to components, avoiding intermediate memory allocations. (6f3703c)
  • The codebase continues to be modernized for memory safety by replacing raw pointers and sizes with std::span in multiple areas, including bookmarks and tab state serialization. (132358c) (cdd3e5c) (4238934)
  • The Crashpad dependency was updated on Windows, fixing a potential nullptr reference and improving crash reporting stability. (2719922) (faabd5e)
