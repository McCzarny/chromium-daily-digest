Overview

This summary covers 527 relevant commits out of 712 total commits to the Chromium main branch on March 4, 2026. The changes span from the first commit (5ce3994) to the last commit (91ea801).

Breaking Changes

  • ⚠️ BREAKING The core clipboard API ui::Clipboard::GetStandardFormats in ui/base/clipboard/clipboard.h has been converted from a synchronous function to an asynchronous one that requires a callback. This is a major signature change that will break any code directly calling this method. (14c828a)
  • ⚠️ BREAKING The public virtual method CreateURLLoaderHandlerForServiceWorkerNavigationPreload in content/public/browser/content_browser_client.h has been renamed to CreateURLLoaderHandlerForServiceWorkerInitiatedNavigationRequest. Embedders overriding this method must update to the new name. (66509a9)
  • ⚠️ BREAKING The default behavior of the public API WebContents::IgnoreInputEvents has changed. The single-argument version now blocks accessibility input events by default, which may break embedders relying on the previous behavior. (c208fa7)
  • ⚠️ BREAKING The deprecated, non-spanified Pickle::WriteBytes(const void* data, size_t length) overload has been removed from the public API in base/pickle.h. Code must be updated to use the span-based overload. (18a2af4) (4fda9b2)
  • ⚠️ BREAKING Public virtual methods in ContentRendererClient and the ServiceWorkerContext::StartWorkerCallback signature have changed to include a blink::ServiceWorkerToken. Embedders implementing ContentRendererClient or using StartWorkerForScope must update their method signatures. (39a3c85)
  • ⚠️ BREAKING The public API for side panel content creation in chrome/browser/ui/side_panel/side_panel_entry.h has been changed. GetContent() and CacheView() now use a SidePanelNativeView type alias instead of std::unique_ptr<views::View>, requiring updates for consumers. (e769dc6)
  • ⚠️ BREAKING The public method DataProtectionUrlLookupService::DoLookup now requires a SessionID and no longer accepts a content::WebContents*. Consumers of this enterprise security service must update their calls. (9736f5a)
  • ⚠️ BREAKING On Windows, the chrome::IsolatedBrowser class has been removed from the public API and replaced with a free function chrome::LaunchIsolatedBrowser, which has a different signature and return type. (18f98f7)
  • ⚠️ BREAKING The FedCM API has been refactored, moving FederatedActorLoginRequest to content/public/browser and renaming it. A new method was also added to the public content::webid::IdentityCredentialSource interface, breaking existing implementations. (5c44bb8)
  • ⚠️ BREAKING The public method RemovePartialSplits in chrome/browser/ui/tabs/tab_strip_model.h has been renamed to MaybeRemoveSplitsForUpdate. Projects interacting with the tab strip model for split tabs must adopt the new name. (efeb792)
  • ⚠️ BREAKING The Tab Declutter feature has been removed, including its associated public command IDs (IDC_DECLUTTER_TAB, IDC_DECLUTTER_MENU) from chrome/app/chrome_command_ids.h. This is a public API removal. (5bbd8d1)
  • ⚠️ BREAKING The kOpenSplitView command has been removed from the public mojom interface ui/webui/resources/js/browser_command/browser_command.mojom. Any WebUI or feature relying on this command will break. (98d1b49)

APIs and Core Infrastructure

  • A new TabDataManager class has been introduced to centralize observation of tab data changes, simplifying how UI clients can listen for and react to updates. (56faad1)
  • In Autofill, a new method Autofill[Driver|Agent]::ScrollFieldIntoView() was added to programmatically bring a form field into view. (0176466)
  • On macOS, openURL has been made asynchronous to prevent blocking the main thread when launching external applications, improving browser responsiveness. (3588647)
  • Third-party dependencies were updated, including zlib to version 1.3.2, abseil-cpp, and QUICHE. (2b1b62f) (2383140) (c849809)
  • On Windows, a new feature flag has been added to retry CreateFileMapping upon ERROR_COMMITMENT_LIMIT failures, potentially improving stability under memory pressure. (9c86acc)

Browser UI and Features

  • Vertical tabs received several improvements, including better contrast in dark mode, drop target link arrows for drag-and-drop, and animations for adding/removing tabs. (bc90d82) (45ea30f) (fc40d95)
  • A use-after-free crash on macOS related to rendering tab group icons in the system menu has been fixed by removing a cached pointer to the ColorProvider. (f6620ae)
  • The profile creation flow has been updated with a visually refreshed customization bubble and a new sign-in promo for the avatar button in the toolbar. (fb5b7ad) (5cd1c18) (1802dbe)
  • A bug in 'Send Tab To Self' that caused devices with generic names (e.g., "iPhone") to be improperly deduplicated has been fixed, ensuring all target devices are displayed. (be489af)
  • The old 'Delete Browsing Data' settings page has been completely removed from the codebase, finalizing the transition to the newer UI. (ea17fef)

Extensions

  • Extensions can now set the Origin header in fetch requests. This behavior is controlled by the kBypassRequestForbiddenHeadersCheck feature flag and is not enabled by default. (0970f34)
  • A new enterprise policy, ExtensionDOMActivityLoggingEnabled, has been defined to control telemetry for extension DOM activity and script injection. (20089d3) (06c68e0)

Networking and Security

  • A new feature flag, kDohFallbackAllowedWithLocalNameservers, has been added to allow DNS-over-HTTPS to fall back to insecure DNS even when local nameservers are configured, intended for testing purposes. (1e172fc)
  • Safe Browsing's Client-Side Detection (CSD) logic was updated to bypass allowlists and report limits for user-initiated reports, ensuring users can always report a suspicious page. (82a0029)
  • A response header monitor has been implemented for prerendered pages, introducing new methods to PrerenderHandle to track prewarm progress. (86eb608)

Testing

  • The HistogramTester utility has a new helper method, GetTotalCountForPrefix, for easier testing of related histograms. (f4020c3)
  • A new method, SetWaitForNextFrame, was added to RenderFrameSubmissionObserver to provide a more reliable way to wait for the next frame in browser tests. (eb37403)
  • The test-only MockActivationController was relanded to better emulate widget activation in tests on Windows and macOS, improving test reliability. (8d74846)
