Chromium Digest: 2026-04-07

Overview

On April 7, 2026, the Chromium main branch saw 582 commits, with 502 identified as relevant to developers. This day was marked by a large number of significant breaking changes, primarily driven by the 'Project Bedrock' refactoring which replaces Browser* pointers with a new BrowserWindowInterface. Other major breaks include the removal of third-party cookie deprecation APIs, changes to core networking interfaces, and an upgrade of the TypeScript compiler. The first commit of the day was (e1a3d4f) and the last was (5a63e24).

Breaking Changes

  • ⚠️ BREAKING The 'Project Bedrock' refactoring continues, replacing raw Browser* pointers with the narrower BrowserWindowInterface* in many public APIs. This affects core utilities like chrome::FindBrowserWithTab, session service helpers in session_service_lookup.h, and UI coordinators like CastDialogCoordinator. Embedders must update call sites to handle the new interface. (78da6a2) (d9ac938) (05f77bc)
  • ⚠️ BREAKING Public getters on BrowserWindowFeatures for accessing UI controllers like LensOverlayEntryPointController and TabSearchToolbarButtonController have been removed. Access must now be done via the static ::From(browser) method on the respective controller classes. (6d74b29) (a89d948)
  • ⚠️ BREAKING The ui::views::BubbleAnchor type has been refactored from a variant alias into a dedicated class. This changes the public API for views::BubbleDialogDelegateView and other components that create UI bubbles, requiring significant updates to bubble creation code. (1ba6a69)
  • ⚠️ BREAKING Core networking APIs have changed. Methods in net::HttpTransaction that return byte counts now use the net::ByteSize class instead of int64_t. Additionally, the Device Bound Session Credentials (DBSC) API on net::URLRequest was changed from set_allows_device_bound_sessions(bool) to set_device_bound_session_mode(net::DeviceBoundSessionMode). (79bbb97) (b057ca4)
  • ⚠️ BREAKING As part of the ongoing third-party cookie deprecation (3PCD) cleanup, the TpcdMetadataManager and its factory have been deleted entirely. Furthermore, the public method net::CookieSettingsBase::IsExemptFromThirdPartyCookieBlocking and related exemption reasons have been removed. (b7f05fa) (c9934fd)
  • ⚠️ BREAKING For security hardening, the extension downloader will no longer follow HTTP redirects when fetching CRX files. This is a major behavioral change that will break custom or enterprise extension update servers that rely on redirects. (04fad69)
  • ⚠️ BREAKING The TypeScript compiler has been upgraded to v6.0.2, which deprecates older syntax and may require updates to custom WebUI code. The public testing utility PrerenderTestHelper::AddPrerenderUntilScriptAsync also had its signature changed. (83a142e) (e3d4fd8)
  • ⚠️ BREAKING Several other public APIs were changed or removed: sql::Database::ReportMemoryUsage was deleted, the AutofillSuggestionDelegate interface was modified, the kBackgroundMediaCapture feature flag was moved out of the public content::features namespace, and the DataTypeStore::DeleteAllDataAndMetadata signature was updated. (b93f6d1) (5fe250d) (3ecbb63) (bbdfaaf)

Security

  • A Use-After-Free (UAF) vulnerability in ServiceWorkerCacheStorageMatcher was fixed by improving the lifetime management of the script sender. (d17fe11)
  • The Viz service now validates TextureQuad data during deserialization to harden the browser against malicious renderer processes. (acafd83)
  • A path traversal vulnerability on Windows was fixed in ChildProcessSecurityPolicyImpl to better handle parent directory references like ... (8f31946)
  • A one-minute grace period was added for Signed Certificate Timestamp (SCT) validity checking to better tolerate minor clock skew between clients and servers. (f7d2cdc)

Networking

  • The public net::ResolutionDetails struct was expanded to include task_completion_delay and secure_dns_attempted, providing more detailed information about DNS resolutions. (e3c1d6a)
  • A performance issue causing inefficient one-byte reads in HttpStreamParser has been fixed. (4168e93)
  • A bug was fixed in the Connection Allowlist feature to ensure violation reports are sent to the correct endpoint specified by the document's Reporting-Endpoint header. (5564461)

Browser UI & Features

  • The ui::menus::SimpleMenuModel API has been enhanced to support secondary labels, allowing for richer context menus. (a6fb10a)
  • A new context menu item has been added to allow pinning and unpinning the tab search button in the tab strip. (04d2971)
  • An entry point to show and pin the 'Tabs From Other Devices' side panel has been added to the 'Customize Chrome' side panel. (1f82a70)
  • The SyncShowBookmarksLimitExceededError feature, which displays an error when the bookmark storage limit is reached, has been enabled by default. (454badd)
  • The experimental TabStripExperimentService has been removed, with its functionality for updating tab group visuals consolidated into the main TabStripService. (792acce)

Core, Build & Testing

  • A new configuration system was added for Chrome for Testing (CfT), allowing runtime options to be specified via a JSON file with the --chrome-for-testing-config switch. (f908610)
  • A brief but significant breaking change to the allocator_shim::UncheckedAlignedAlloc function signature was introduced and quickly reverted due to build failures. (2b3ee80) (f747855)
  • On Windows, a new feature flag (RemoveGpuJobMemoryLimit) was added to optionally remove the job memory limit for the GPU process, aiding in memory usage measurement for features like WebGPU. (705baec)
  • Several components, including //chrome/browser/feed, //chrome/browser/performance_manager, and //chrome/browser/component_updater, were modularized into their own BUILD.gn files to improve dependency management. (bfec926) (879e523) (c44b7b1)
