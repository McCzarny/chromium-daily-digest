Chromium Digest: 2026-01-15

Overview

This summary covers 665 commits to the Chromium main branch on January 15, 2026, with 483 commits filtered as relevant to developers. The changes span from the first commit ((71ba106)) to the last ((de06b69)). The day was marked by a significant number of breaking changes across APIs for WebApps, Sync, Autofill, Permissions, and Storage, requiring attention from embedders. Notable additions include new browser closing helpers and an observer for extension install policies.

Breaking Changes

  • ⚠️ BREAKING Public API Removal (WebApps): The public method WebAppRegistrar::IsInstallState() has been removed. Embedders must migrate to GetInstallState() to check if a web app is installed. (e849b33)
  • ⚠️ BREAKING Public API Signature Change (FedCM): The IdentityRequestDialogController::ShowAccounts() method signature was changed to include a new completion_callback parameter. Custom FedCM dialog controller implementations will fail to compile without this update. (4229dca)
  • ⚠️ BREAKING Public Interface Change (UI): New pure virtual methods (e.g., IsVisible, Bounds) have been added to the public LocationBar interface. Embedders with custom LocationBar implementations must implement these methods to avoid build failures. (da5f63b)
  • ⚠️ BREAKING Public API Removal (Sync): The virtual method SyncEngine::GetCacheGuidsForAllGaiaIds() has been removed. Embedders should now use the static method SyncTransportDataPrefs::GetCacheGuidsForAllGaiaIds(). (5f70ce6)
  • ⚠️ BREAKING Public API Removal (Autofill): The last_modifier_id() and set_last_modifier_id() methods have been removed from the public AutofillProfile class as part of a data model cleanup. (870f6b1)
  • ⚠️ BREAKING Public API Signature Change (Permissions): The PermissionContextBase::NotifyPermissionSet method signature has been changed to require a PromptOptions parameter. Custom permission context subclasses must be updated. (adc93e0)
  • ⚠️ BREAKING Public API Removal (Autofill): Several deprecated preferences have been removed from the public header components/autofill/core/common/autofill_prefs.h. Code referencing these obsolete preferences will fail to build. (352a82c)
  • ⚠️ BREAKING Public API Signature Change (Storage): Methods for opening and destroying DOM storage databases, such as AsyncDomStorageDatabase::Open, now take a single database_path instead of separate directory and name arguments. Direct usage of these storage component APIs will require code changes. (d3e4a65)
  • ⚠️ BREAKING Behavioral Change (Security): Strict handle checks for sandboxed processes on Windows are now enabled by default. This security hardening may cause crashes or unexpected behavior in embedder code that is not compatible with stricter validation. (3e7aecf)
  • ⚠️ BREAKING Configuration Change (Enterprise): Enterprise policies for SaaS usage reporting have been renamed (e.g., to SaasUsageReportingDomainUrlsForBrowsers). Configurations using the old policy names must be updated to maintain functionality. (4843964)
  • ⚠️ BREAKING File Path Change (On-Device Translation): The translator.h header has been moved from chrome/browser/on_device_translation/ to components/on_device_translation/. Any code including this header from its old location will fail to build. (929002a)
  • ⚠️ BREAKING Mojo Interface Change (Graphics): The gfx::mojom::HDRMetadata mojo interface was changed. The agtm_serialized byte array was replaced with a structured agtm field of type skia.mojom.SkHdrAdaptiveGlobalToneMap, breaking consumers of this interface. (5c20df3)

APIs and Extensions

  • New helper functions, chrome::CloseAllBrowsersWithProfile and chrome::CloseAllBrowsersWithIncognitoProfile, have been added to browser_finder.h, providing a more robust way to close windows for a specific profile. (a193675)
  • The ExtensionInstallPolicyService now includes an Observer class, allowing consumers to be notified when extension-related enterprise policies change. (0bd6c80)
  • The extension update mechanism has been improved to automatically repair corrupted extensions by forcing a re-installation, even if the version number is unchanged. This resolves an earlier rollback of the same feature. (81be1d0) (77291cd)
  • The SessionRestoreObserver interface has a new virtual method, OnSessionRestoreMetadataChanged, to notify observers of changes to session restore metadata. (028f671)
  • On macOS, the public ContentMainDelegate interface has a new virtual method, ShouldRunContentMain(), as part of a refactoring of the application shim initialization process. (b6a3df7)
  • The state management for the extensions toolbar button has been refactored from the view into the ExtensionsToolbarViewModel, changing the public API of these UI classes by moving the State enum and adding GetButtonState(). (1de8411)

Security and Networking

  • A new EcdhKeyExchange class was introduced in remoting/base to establish symmetric keys for encrypting P2P control channels, enhancing security for features like session recording. (7b8d1bd)
  • Trace events have been added to key parts of the network stack, including cookie management and HTTP transactions, to improve performance profiling capabilities. (faceaea)
  • The WebAuthenticationMicrosoftSoftwareUnexportableKeyProvider feature flag for WebAuthn on Windows has been removed. The provider's behavior is now enabled by default and can no longer be disabled via this flag. (aef72c6)
  • A use-after-free bug in the RenderWidgetHostViewAura destruction path on Windows has been fixed. (01c3da8)

Browser UI and Features

  • The global-media-controls-updated-ui feature flag has been removed, making the updated Global Media Controls UI the default on desktop platforms. (6153529)
  • The commerce "product specifications" feature and its associated chrome://product-specifications WebUI have been completely removed from the codebase. (8babf9a)
  • A new "back-to-opener" feature has been implemented, allowing users to navigate back to the opening tab when a newly opened tab is closed. (e9d972c)
  • The chrome://updater internal page on Windows and macOS can now display the updater's current state and applied policies via new methods on BrowserUpdaterClient. (b1d8ebf) (6fadf43)
  • The TabStripControlButton has been updated to support custom colors for hover and ripple effects, enabling more flexible UI theming. (ea742a4)
  • The unlaunched zero-copy-tab-capture feature and its associated flag have been removed from the codebase. (4eb688c)

Infrastructure and Tooling

  • The third-party zip library was upgraded from version 6.0.0 to 7.1.0. (2df3608)
  • In crypto, the Aead::Init() method was deprecated in favor of a new constructor that takes the key directly, simplifying lifetime management. (9989ab0)
  • GPU detection logic on Windows was improved to correctly distinguish between the hardware-accelerated XBox adapter and the software-based WARP adapter, preventing incorrect reporting of software rendering. (bbe3736)
