Chromium Digest: 2026-07-17

Overview

This summary covers 663 total commits on the main branch, with 494 identified as relevant to desktop embedders (Windows and macOS). Development focused on security hardening, modularization of UI components, and API improvements for extensions. The first commit of note is (23ffbe3) and the last is (9eed432).

Breaking Changes

  • ⚠️The public API content::StoragePartition::GetPrivateAggregationDataModel() has been removed. Downstream embedders relying on this interface must remove these calls from their code. (aab9e1e)

Security & Privacy

  • Added .scf and .url files to the Windows Shell Integrated Extension blocklist to prevent automated SMB credential disclosure via Windows Explorer thumbnails. (a94fdc2)
  • Restricted the Notification API so that sandboxed (opaque-origin) frames return a DENIED permission status to prevent cross-origin probing. (1bbf753)

Browser UI & Window Management

  • Improved focus retention for the Omnibox dropdown during native file dialogs and prevented the Omnibox search widget from appearing as an active window on the Windows taskbar. (928e483) (2524f29)
  • Modularized desktop UI components by extracting ExclusiveAccessBubbleViews into a standalone GN target to reduce binary bloat. (325b03a)

Extensions & Developer Tools

  • Streamlined extension manifest handling by adding a templated helper Extension::SetManifestData() to avoid manual string-key parsing. (f79b25a)
  • Enhanced WebSocket debugging by enabling HTTP header support for Web Workers via Network.setExtraHTTPHeaders. (26e0c71)

Infrastructure & Base

  • Refactored base::numerics CheckedNumeric to correctly handle invalid integral representations by returning NaN, improving numerical safety. (4240774)
  • Began moving security-sensitive test harnesses into isolated component targets, such as moving Safe Browsing unit tests to //chrome/browser/safe_browsing. (fc13cfa)
