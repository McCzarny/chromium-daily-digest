Chromium Digest: 2026-05-02

Overview

This digest covers 47 commits from (c45c88b) to (94371de). After filtering for changes relevant to Windows and macOS embedders and excluding areas like Blink, V8, and platform-specific mobile code, 20 commits were analyzed. Key themes include security hardening for extensions and navigation, UI refinements for the new tab page and vertical tabs, and fixes for platform integration on macOS and Windows.

Security

  • Security for service worker event listeners in extensions has been enhanced. New validation checks in the EventRouter ensure that IPC messages to add or remove listeners originate from an authorized process, preventing a compromised renderer from spoofing these requests. (31dce68)
  • A potential vulnerability in subframe error handling has been addressed. Previously, an error page for a subframe could be created with an origin derived from the destination URL. Now, error pages that commit in the current process are forced to use a new, opaque unique origin, mitigating risks associated with a compromised renderer forcing a commit with a victim's precursor. (470a561)
  • The on-device model service now includes additional safety flags for model assets being passed to the sandbox, improving the security posture of this feature. (ebf0ace)

Extensions

  • A crash in the EventRouter has been fixed. The crash occurred when an extension tried to remove a stale, in-memory event listener whose filter was no longer present in persisted preferences. The router now gracefully handles this case by checking for the filter's existence before attempting removal. (ce6c6b2)

Browser UI & Features

  • As part of the ongoing UI refresh, the feature flag for the WebUI toolbar has been renamed from kWebUIRefresh2026 to kWebUIFlexibleToolbar. (c45c88b)
  • The New Tab Page has been updated to remove shortcuts by default, aiming for a simpler default experience. (3a36117)
  • A visual enhancement for vertical tabs with a transparent 'glass' frame has been implemented. The tab strip will now correctly fade to the glass background, improving animations during collapse and expansion. (89fdd0f)
  • A bug causing full-page blanking during screenshot redaction has been fixed. The logic now correctly copies the original bitmap before drawing redaction masks, ensuring the rest of the page remains visible. (c1c7fa4)

Platform Integration & Devices

  • On macOS, a bug that could lead to uninitialized memory access when setting gamepad vibration has been fixed. The code now correctly checks if a force feedback effect was successfully created before attempting to use it, preventing potential crashes. (b90a779)
  • The Chromium updater UI on Windows now supports high contrast modes, improving accessibility by dynamically adopting system colors for its progress window. (3517cbe)
  • An internal refactoring on macOS has removed a manual visibility tracking field in NativeWidgetNSWindowBridge, replacing it with a direct call to the system's -[NSWindow isVisible] method. (91a1300)

Internals & Infrastructure

  • The Widevine test license server infrastructure, which was Linux-only and difficult to maintain, has been removed from the codebase and DEPS. Encrypted media tests that relied on it are now updated to expect failure or be skipped, aligning behavior with other platforms. (1262aed)
  • PWA browser tests have been made more robust by replacing flaky heuristics with a new WebAppPageWaiter utility. This utility uses a new event on WebAppTabHelper to reliably determine when all web app system activity is scheduled for a given page. (4b3f90b)
  • A bug in the GPU command buffer decoder has been fixed. The decoder will no longer prematurely set a texture's 'cleared' flag if the underlying texture allocation fails due to a GL error. (f786ca0)
  • The build system has been refactored to move preference-related files for the accessibility annotator component into their own dedicated prefs build target. (9ca9e21)
