After filtering for changes impacting browser-level development on desktop platforms, this summary highlights 4 major breaking changes and key updates across APIs, UI, Networking, and Code Health.

APIs & Browser Core

  • ⚠️ BREAKING BREAKING: url/ library functions have been renamed to align with the Google C++ style guide (e.g., ConvertFromUTF16 is now ConvertFromUtf16). Embedders directly using these public functions must update their code. (cf08f47)
  • ⚠️ BREAKING BREAKING: A new virtual function, WillCreateURLLoaderFactoryForPrefetch, has been added to the public ContentBrowserClient interface. Embedders must implement this new function to compile. This change is part of an effort to add request integrity headers to prefetch requests. (50aaada)
  • ⚠️ BREAKING BREAKING: The public header chrome/browser/mac/bluetooth_utility.h and its implementation have been removed. While the commit states it was unused internally, any external project on macOS depending on this header will break. (0812c48)
  • ⚠️ BREAKING BREAKING: As part of a PWA sync refactoring, the public method WebApp::SetSyncProto() has been removed. Code that creates WebApp objects from sync data must now use a different approach, as the method for setting the underlying sync proto directly is gone. (bef7e56)
  • The StartWorkerForScope callback for Service Workers is now deferred until the OnStarted event has been processed. This ensures consumers (like extensions) will see the worker as "running" immediately after the callback, fixing a race condition. (a246d71)
  • The topK and temperature parameters in the Language Model API have been deprecated and are now restricted to extension contexts only. Use in regular web pages will trigger console warnings. (05cfd3b)

Browser UI & Features

  • An animation bug in Vertical Tabs has been fixed where the hovercard would animate from incorrect bounds when moving between pinned and unpinned tabs. (b15a283)
  • An experimental "Summarize" button, guarded by a feature flag, has been added to the PDF viewer toolbar. It is not yet functional. (f7fc174)
  • New feature flags and parameters have been added to allow for experimentation with the default browser prompt surface. (379689f)
  • A bug has been fixed in the PWA manifest update job where an 'old' icon could not be found if it only existed in fallback bitmap data, ensuring smoother icon updates. (ad42840)

Networking & Code Health

  • A crash in the SQL-based disk cache (net/disk_cache/sql) has been fixed. The crash occurred when Doom() was called on an entry that was still being created. (2c9fe70)
  • Multiple commits continue the effort to modernize the codebase, including replacing raw pointer-based HexEncode calls with std::span-based versions and simplifying the Windows clipboard ReadAsync template. (26be541) (cc74455) (54dbafc) (c83687b)
  • The remoting protocol implementation continues to be refactored to replace legacy XmlElement usage with well-defined structs, improving type safety and maintainability. (2230288) (8126d3b)

Testing & Infrastructure

  • Several flaky tests have been addressed. A CookieControls IPH test was disabled, and the Readability feature is now disabled in some automated tests to prevent flakiness related to DOM distillation. (d62916c) (15c5f14)
  • Browser tests for StartupBrowserCreator and BrowserNavigatorTest were updated to accommodate ongoing refactors and new features like the WebUI Omnibox popup. (6fca8f0) (2db83a4)
