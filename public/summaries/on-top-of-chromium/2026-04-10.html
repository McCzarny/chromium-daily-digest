Summary for 2026-04-10 on branch 'main'

Chromium Digest: 2026-04-10

Overview

This summary covers 595 commits on the main branch, with 447 identified as relevant to developers building on Chromium. The changes span from the first commit (52f8bb1) to the last (8f5b39c). This digest highlights numerous breaking API changes, security enhancements, and significant updates to core browser components and UI.

Breaking Changes

  • ⚠️ BREAKING The public API for creating GPU contexts, such as viz::ContextProviderCommandBuffer::CreateForRaster, has been changed. The lose_context_when_out_of_memory parameter was removed, and embedders must update their calls. (b7eadee)
  • ⚠️ BREAKING The net::URLRequestJob public interface has been updated. Methods like GetTotalReceivedBytes now return the type-safe base::ByteSize instead of int64_t. Embedders with custom protocol handlers must update their method signatures. (72bf5dc)
  • ⚠️ BREAKING The searchbox.mojom interface was refactored to use a factory pattern. Consumers must now bind a searchbox::mojom::PageHandlerFactory instead of directly binding the PageHandler, breaking custom WebUI implementations. (a43ea83)
  • ⚠️ BREAKING The ModelBrokerClient API in Optimization Guide now uses string-based use_case identifiers instead of a mojom::OnDeviceFeature enum. Embedders using on-device models must update their code. (dfe0d80)
  • ⚠️ BREAKING The base::task::sequence_manager::ThreadControllerImpl class and its factory function CreateSequenceManagerOnCurrentThread have been removed. Embedders must migrate to CreateSequenceManagerOnCurrentThreadWithPump. (36e7905)
  • ⚠️ BREAKING The recently added content::CdmHelper public interface and its header content/public/browser/cdm_helper.h have been reverted and removed completely. (7b96ee6)
  • ⚠️ BREAKING The public constructor for media::EncryptionPattern is now private and has been replaced by a static Create() factory method that returns an std::optional. Direct instantiation will no longer compile. (006ccbd)
  • ⚠️ BREAKING The legacy ManagePasswordsIconView class and its header have been removed as part of a UI migration. Code referencing these components will fail to build. (822db52)
  • ⚠️ BREAKING In the updater's PolicyService, the public function AreUpdatesSuppressedNow has been renamed to AreUpdatesSuppressed and its signature was changed to require an explicit base::Time parameter. (d166a73)
  • ⚠️ BREAKING The AccessibilityAnnotatorBackend::Observer public interface was modified. The OnContentAnnotationsAdded method now requires a const ContentAnnotationsData& parameter, breaking existing observer implementations. (fd9d5b4)
  • ⚠️ BREAKING The FetchEmailOneTimeTokenResponse protobuf message in the One-Time Token Service API has been changed, removing the created_timestamp and received_timestamp fields. (4ce640a)
  • ⚠️ BREAKING The legacy GlicFreUI has been deleted, along with its public headers such as glic_fre_ui.h. Any code referencing this UI will no longer compile. (dc4535b)

Browser & Core APIs

  • The BrowserContext::UniqueId() method is now deprecated. A new BrowserContext::UniqueToken() method returning a more secure base::UnguessableToken should be used instead. (4f81ffc) (f2112b5)
  • Asynchronous methods have been added to PageContentExtractionService, allowing callers to wait for pending page analysis results. (691d55a)
  • A new UnexportableSigningKeyId type was introduced to provide stronger type safety for hardware-backed signing keys. (a2bcfb9)
  • On Windows, a DCHECK failure in VideoCaptureDeviceFactoryWin was fixed by preventing unnecessary restarts of the COM thread, improving stability. (2edfa92)
  • The WebCrypto API now supports ML-KEM-768 and ML-KEM-1024 for post-quantum key encapsulation. (b582009)

Security

  • A Use-After-Free vulnerability in WebContentsImpl::ForSecurityDropFullscreen was fixed by using WeakPtr to safely handle WebContents instances during nested message loops. (899e77f)
  • A re-entrancy bug in TabStripModel that could cause a Use-After-Free during tab move operations has been fixed. (0985a47)
  • For FedCM, a check was added to validate the initiator's origin when processing Set-Login headers in the browser process, enhancing security for subresource loads. (5932bfd)
  • The XKB keyboard layout name parser was hardened to prevent potential path traversal attacks. (73daddb)
  • Integer overflow vulnerabilities in the H.264 parser were fixed by using checked arithmetic. (4986289)
  • A Use-After-Free in the DevTools agent host was patched by preventing double initialization. (5b7fd82)

UI & Browser Features

  • The "Send Tab to Self" feature will now display a toast notification on desktop platforms when a tab is automatically opened. (eecf3ec)
  • The updater UI on Windows now supports high DPI scaling, resulting in a sharper interface. (c154701)
  • The "Default Search Engine Reset" dialog has been refactored to use ui::DialogModel, updating its text and converting the "Learn more" button into an inline link. (6f48ee3)
  • A dangling pointer crash in the bookmark context menu on macOS was resolved by converting a raw_ptr to a WeakPtr. (c31e869)
  • The Task Manager was updated to correctly display prerendered new tabs by adding a new PrerenderNewTabTag. (7c4cbb7)
  • A new enterprise policy has been added to allow overriding the CPU Performance API. (1bff823)

Extensions

  • A crash in the OneTimeMessageHandler has been fixed. The handler now validates that the ScriptContext is still active before processing a message response, preventing a use-after-free. (94a8ea5)
  • A crash in ExtensionSyncService was fixed by adding a null check when retrieving granted permissions, preventing issues with incomplete sync data. (0eeafe7)
  • Content Security Policy (CSP) enforcement was hardened for sandboxed extension service workers. (a0b8451)

Networking

  • The WebSocket implementation over HTTP/3 was enhanced by implementing GetLoadTimingInfo, GetSSLInfo, and GetRemoteEndpoint in the handshake stream. (9a70b09)

Build & Infrastructure

  • The enable_surface_embed build flag has been removed, making the feature's code always available when WebContents is used and simplifying build configurations. (b27035e)
  • On Windows, usage of the Active Template Library (ATL) has been removed from content_main.cc as part of a larger effort to remove ATL from the codebase. (fc1e63c)
  • Numerous components continue to be modularized to improve build structure and dependency hygiene, including push_messaging, nearby_sharing, and headless browser mode. (da19370) (37803b9) (b4167ed)
