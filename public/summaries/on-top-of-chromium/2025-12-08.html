Summary for 2025-12-08 on branch 'main'

Chromium Digest: 2025-12-08

Overview

This summary covers 556 commits on the main branch from commit (a20a7bb) to (d48dfb6). Of these, 396 were identified as relevant to developers building on Chromium, with a focus on APIs, browser UI, security, and extensions. The day includes an unusually high number of breaking API changes across UI, identity, security, and other core services, requiring careful review by embedders.

Breaking Changes

  • ⚠️ BREAKING Multiple public UI interfaces have been changed. TabStripModel::selection_model() now returns a TabStripModelSelectionAdapter instead of a raw ui::ListSelectionModel. Additionally, methods in TabDragContext and TabSlotController now accept TabSlotView* instead of Tab*, requiring updates for custom tab UIs. (58e491f) (14451e1)
  • ⚠️ BREAKING The public EnterpriseIdentityService and its factory have been removed from components/enterprise/signin/. Embedders must migrate away from this service for managed account identification. (5d6bd7b)
  • ⚠️ BREAKING The browser updater interface was refactored into a new public header, updater.h, making previous headers non-public. This is a public API removal and signature change that requires embedders to update their integration with Chromium's updater service. (d4d03c1)
  • ⚠️ BREAKING In WebAuthn, passkey_model.h no longer exposes GetPasskeyByCredentialId. Embedders must migrate to the new, more generic GetPasskey function. (c7b48fb)
  • ⚠️ BREAKING The network service sandbox is no longer enabled by default on Windows, reverting a recent change. This is a major behavioral change to a core security feature that alters the default security posture for embedders on Windows. (e1e1886)
  • ⚠️ BREAKING The Safe Browsing BinaryUploadService::Request was refactored into a new base class, enterprise_connectors::BinaryUploadRequest, affecting embedders using the cloud content scanning service. (84f541f)
  • ⚠️ BREAKING Several component APIs were broken. autofill_client.h lost the virtual method PopupScreenLocation. AsyncDomStorageDatabase::Open no longer accepts a task runner. The AXNodeData mojom interface removed the bool_attributes field. (80af047) (ae4cf43) (75d9332)
  • ⚠️ BREAKING The One-Time-Token (OTP) service API now uses base::expected for its callback, changing the signature for SmsOtpBackend::RetrieveSmsOtp. (2255a7a)
  • ⚠️ BREAKING The chrome::mojom::PluginStatus mojom enum has been updated to remove the obsolete kPlayImportantContent value. Code referencing this value will no longer compile. (243d054)

Security & Privacy

  • The PDF viewer now runs in an out-of-process iframe (OOPIF) by default on Windows and macOS. This improves security and stability by isolating the PDF content from the main browser process. (b7cbb1e)
  • A large number of deprecated preferences, previously scheduled for removal, have been deleted from browser_prefs.cc. While deprecated, this removes public configuration points that may still be in use by automation or embedders. (1cedc7b)
  • The Safe Browsing file analyzer was improved to use the ZIP obfuscated analyzer for files it detects as obfuscated, enhancing protection against malicious archives. (b75a5cf)
  • The enterprise policy PrivacySandboxIpProtectionEnabled has been removed, as the underlying IP Protection feature was canceled. (1bfc15d)

Browser UI

  • A significant performance issue causing UI lag when pasting very large amounts of text into a views::Textfield has been fixed. A new OnBeforePaste hook allows controllers to truncate clipboard content before processing. (81aebdb)
  • A crash in the Omnibox popup that could occur when its remote page is disconnected has been fixed. (064dbc5)
  • A crash in the sign-in interception bubble (DiceWebSigninInterceptionBubbleView) has been fixed for cases where a BrowserView is not present, improving stability in WebUI-based browsers. (8c7c0f3)

Extensions

  • Several fixes were made to improve the reliability of persisted webRequest listeners, a recently added feature. The changes address a race condition during extension updates and improve the handling of security information. (2836073) (4accc7b) (8e98f3d)

APIs & Services

  • The TokenServiceTable in components/signin/public/webdata/ now exposes a new method, GetAllWrappedBindingKeys, to retrieve all binding keys from the database, even if some associated tokens fail to decrypt. (7006399)
  • Network connection handling was improved by preventing duplicate observers in ConnectionChangeNotifier and using base::WeakPtr to avoid potential use-after-free bugs. (c0b4cff) (16a8078)
