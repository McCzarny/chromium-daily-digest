Week 7 of 2026 (2026-02-08 to 2026-02-13)

Chromium Weekly: 2026 Week 7

Overview

The week was dominated by a massive wave of breaking API changes across nearly all major components, requiring significant migration effort for embedders. In total, 2238 relevant commits landed out of an estimated 2983 total, introducing foundational work for an isolated browser process on Windows, significant behavioral changes to browsing history, and numerous updates to the Extensions framework. Key themes include API modernization, security hardening, and continued refactoring of core browser services.

Core API & Interface Modernization

  • ⚠️ BREAKING BREAKING CHANGE: Numerous UI APIs were changed. ui::Clipboard read methods are now asynchronous, ViewAccessibility::OverrideFocus was removed in favor of SetActiveDescendant, and the TabStripModelObserver interface was marked as re-entrant, altering its notification contract. (63af498) (d704671) (30d8cde) (9e81acd)
  • ⚠️ BREAKING BREAKING CHANGE: Identity and Sign-in APIs saw major updates. The DICE sign-in migration is now forced, components::signin::public::identity_manager::AccountInfo members were made private, and PrimaryAccountMutator::SetPrimaryAccount() now requires an explicit access_point. (1581e94) (79c150b) (70eb35f)
  • ⚠️ BREAKING BREAKING CHANGE: Core browser and URL APIs were refactored. BrowserList::CloseAllBrowsersWithProfile() was removed, GURL::ComponentStringPiece() was renamed to GURL::ComponentStringView(), and the public ContentBrowserClient interface requires a new WillCreateURLLoaderFactoryForPrefetch implementation. (d29ce19) (caf65ac) (50aaada)
  • ⚠️ BREAKING BREAKING CHANGE: The media stack was updated, renaming media::VideoFrame::IsMappable() to HasDirectCpuAccess() and removing deprecated media::AudioDecoderConfig constructors that used ChannelLayout in favor of ChannelLayoutConfig. (8465918) (ab38239)

Security & Privacy

  • Foundational work for launching an isolated browser process on Windows has been implemented, including a new command-line switch and a basic COM interface. (0b67c8c) (5655a04)
  • Security was enhanced by adding validation in RenderFrameHostImpl::CreateNewWindow to terminate renderers that send an invalid WindowOpenDisposition, preventing a potential browser process crash. (4d0b870)
  • A new SearchIntegrity component was introduced to monitor search engine configurations against a trusted provider allowlist, and a new WebRtcDiagnosticLogCollectionAllowedForOrigins policy was added for enterprise control over WebRTC diagnostic logging. (bf13717) (22400f0)
  • The VisitedLinkWriter now uses base::CheckedNumeric to calculate shared memory region size, mitigating potential integer overflow vulnerabilities. (da6930a)

Extensions Framework

  • ⚠️ BREAKING BREAKING CHANGE: Several public APIs were removed, including WindowController::CanClose(), DevicePermissionsPrompt::AskForHidDevices(), and the DevicePermissionsPrompt class itself was renamed and its virtual methods removed. (6c171b9) (dbb6168) (d345c49)
  • ⚠️ BREAKING BREAKING CHANGE: The extensions::Message class is now move-only and its internal representation has changed to support Blob types, breaking code that directly accessed its data buffer or did not use std::move. (6f16df7) (d9bea47)
  • A new mechanism ensures an extension's service worker is started at least once per version to reliably register its event listeners, fixing a potential race condition during initialization. (5b375a5)

Browser Behavior & Features

  • ⚠️ BREAKING BREAKING CHANGE: The BrowsingHistoryService now groups similar visits (same hostname and title) on a per-day basis. This is a significant behavioral change that will alter the results of history queries. (f327600)
  • A new public SecurityPrincipal interface was introduced in content/public/browser to represent security contexts, replacing internal classes and leading to the removal of SiteInstance::IsSandboxed(). (5632420) (6d9a124)
  • The Privacy Sandbox notice dialog and its associated WebUI resources at chrome://privacy-sandbox-dialog have been removed from the browser. (39a014e)

Networking

  • ⚠️ BREAKING BREAKING CHANGE: The DnsTransactionFactory::CreateTransaction signature now uses an AttemptMode enum instead of a boolean, and the RestrictedCookieManager::SetCanonicalCookie mojom API now requires a RestrictedCanonicalCookieParamsPtr struct. (17be678) (c91ac9d)
  • ⚠️ BREAKING BREAKING CHANGE: P2P/WebRTC code has been decoupled into a new //services/network/public/cpp:cpp_p2p component. Embedders using this functionality must now add a build dependency on the new target. (e7e9679)
  • The RenderDocument feature and its SkipRendererCancellationThrottle optimization have been enabled by default, which may alter navigation cancellation behavior for improved performance. (b4d0474)

Base & Testing Utilities

  • ⚠️ BREAKING BREAKING CHANGE: Public functions in the url/ library have been renamed to align with the Google C++ style guide, such as ConvertFromUTF16 becoming ConvertFromUtf16. (cf08f47)
  • A new lease-based RAII system for managing thread priorities has been introduced via base::PlatformThread::RaiseThreadTypeLease, which is now the preferred mechanism over SetCurrentThreadType. (612357b)
  • The base::test::TestFuture utility has been enhanced with a kQueue mode, allowing it to queue multiple values for testing scenarios with multiple asynchronous callbacks. (0223ed4)
