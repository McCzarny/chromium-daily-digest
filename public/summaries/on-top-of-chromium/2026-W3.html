Week 3 of 2026 (2026-01-11 to 2026-01-17)

Chromium Weekly: 2026 Week 3

Overview

This week was defined by an exceptionally high volume of breaking changes across core APIs, UI components, and extensions, requiring significant attention from embedders. Out of 1233 total commits, 851 were relevant, introducing major refactorings like the removal of the legacy browser layout system and the base::trace_event handle. Notable new features include the stabilization of Secure Payment Confirmation, a new lightweight chrome.histograms API for WebUI, and the implementation of a 'back-to-opener' navigation feature.

Widespread API and Behavioral Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE: A major behavioral shift in the Extensions API now returns Promises by default for asynchronous calls that omit a callback, even for Manifest V2 extensions. This may alter control flow and message port lifetimes for extensions relying on the previous synchronous-like behavior. (3e0fd5a)
  • ⚠️ BREAKING BREAKING CHANGE: Numerous core APIs were removed or refactored. Key changes include the removal of base::trace_event::TraceEventHandle which breaks tracing integrations, the removal of the base/strings/latin1_string_conversions.h header, and a new signature for base::MemoryPressureListener::OnMemoryPressure. (9801a81) (e4c0c41) (8c3705f)
  • ⚠️ BREAKING BREAKING CHANGE: The legacy browser layout system has been completely removed, including virtual methods on BrowserFrameView like GetBoundsForTabStripRegion(). Embedders with custom frame views must adapt to the new centralized layout logic. (6d6528f) (9c5016d) (8231cda)
  • ⚠️ BREAKING BREAKING CHANGE: The Permissions API was significantly refactored, changing method signatures on permissions::PermissionRequestManager and permissions::PermissionsClient to pass PromptOptions as a direct parameter. (7688ee1) (5d3a4e1) (b47e74e)
  • ⚠️ BREAKING BREAKING CHANGE: Externally installed extensions, such as those deployed via enterprise policy, will now be automatically disabled if an update increases their required permissions, requiring manual administrator intervention to re-enable. (b846067)

APIs and Extensions

  • A new lightweight chrome.histograms API is now available for WebUI pages to log UMA histograms without requiring privileged permissions or complex Mojo setup. (9044a5b)
  • The ApiPrintingMarginsAndScale feature for the Extensions Printing API is now enabled by default, allowing extensions to programmatically control print margins and scaling. (899baf6)
  • New helper functions, chrome::CloseAllBrowsersWithProfile and chrome::CloseAllBrowsersWithIncognitoProfile, have been added to provide a robust way to close all browser windows associated with a specific profile. (a193675)
  • A new public API, content::webid::IdentityCredentialSource, has been added to support FedCM, allowing embedders to directly query for sign-in accounts and build more integrated login UIs. (aa21ef9)

Security and Networking

  • The Secure Payment Confirmation (SPC) feature using browser-bound keys is now stable and enabled by default on macOS and Windows, enabling more secure web payment flows. (56976fd)
  • ⚠️ BREAKING Enterprise Data Controls are now enforced for drag-and-drop operations on desktop platforms, preventing data exfiltration from managed contexts. A new virtual method ContentBrowserClient::GetEndpointForDrag was added to support this. (f29965a)
  • The HTTP cache encryption system has been enhanced. The master encryption key is now stored in profile preferences, and the CacheEncryptionProvider is managed by the ProfileNetworkContextService. (d80d8c1) (6d99411) (2fd5f4d)
  • ⚠️ BREAKING BREAKING CHANGE: The contract for ValidateSignedWebBundleSignatures for Isolated Web Apps has changed; it no longer performs trust validation, which must now be handled separately by the caller. (d675064)

Browser UI and Core Internals

  • A new "back-to-opener" feature has been implemented. When a newly opened tab is closed, the main back button can navigate the user back to the tab that opened it. (e9d972c) (9bdc317)
  • Significant progress was made on the experimental Vertical Tab Strip, including animations for moving tabs between groups and making the top of the toolbar a draggable region. (1378fb4) (89c440f)
  • On Windows, base::StackTrace can now proactively load dbghelp.dll and symsrv.dll, improving symbolization for crash reports originating from some sandboxed processes. (128ded4) (b0778fc)
  • Several UI features have been stabilized and their flags removed, including DesktopPWAsRunOnOsLogin and the updated Global Media Controls UI. Conversely, the commerce "product specifications" feature was removed entirely. (fe87d68) (6153529) (8babf9a)
