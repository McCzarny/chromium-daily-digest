Week 18 of 2026 (2026-04-26 to 2026-05-02)

Chromium Weekly: 2026 Week 18

Overview

This week saw a significant number of breaking changes across core APIs, with 1071 relevant commits out of 1396 total. Major refactors affected fullscreen management, the compositor, password management, and networking delegates, requiring updates from embedders. Security was a key focus, with stricter policies leading to renderer termination for invalid inputs and enhanced sandboxing. Notable user-facing changes include the default enablement of Vertical Tabs and the introduction of a 'Smart Restart' feature on macOS.

API Breaking Changes

  • ⚠️ BREAKING BREAKING CHANGE Fullscreen management has been refactored out of the BrowserWindow public interface. Methods like IsFullscreen() and EnterFullscreen() are now part of the BrowserWindowFullscreenController class, requiring embedders to update calls to, for example, BrowserWindowFullscreenController::From(browser)->IsFullscreen(). (d1db08e)
  • ⚠️ BREAKING BREAKING CHANGE The core compositor API has been refactored. The cc::LayerTreeHostClient class was renamed to cc::LayerTreeHostDelegate, and several public methods were moved from cc::LayerTreeHostImpl to the derived viz::VizLayerTreeHostImpl class. (afd22cd) (a0e7321)
  • ⚠️ BREAKING BREAKING CHANGE The net::URLRequest::Delegate and URLLoaderNetworkServiceObserver mojom interfaces now require a new pure virtual method, OnPlatformLocalNetworkPermissionRequired. Embedders must implement this method to handle new OS-level permission prompts for local network access. (0abb379) (3218389)
  • ⚠️ BREAKING BREAKING CHANGE The webapps::ManifestId type has been changed from a GURL alias to a webapps::ValidManifestId wrapper. This enforces validation at the type level, requiring code to handle potentially invalid IDs and access the GURL via the .value() method. (73319ea)
  • ⚠️ BREAKING BREAKING CHANGE Several core APIs were modified. Navigation headers like browser_navigator.h were moved to a new ui/navigator subdirectory. The views::BubbleDialogDelegate::CreateBubble method now requires an explicit ownership parameter. In the base library, base::RandInt and base::DateIntervalFormat have been removed. (815c5cf) (3301350) (5f591f6) (1c49b4d)
  • ⚠️ BREAKING BREAKING CHANGE The Remoting component was significantly refactored to enforce WebRTC as the sole transport protocol. The SessionConfig and CandidateSessionConfig classes and related APIs like SessionManager::set_protocol_config() have been removed. (32d22ac)

Security

  • ⚠️ BREAKING BREAKING CHANGE The browser will now terminate a renderer process if it sends invalid data, representing a stricter security posture. This includes sending invalid navigation headers or, for extensions, attempting to create a cross-origin SharedWorker. (e0a0f3e) (0e9234d)
  • ⚠️ BREAKING BREAKING CHANGE Filesystem operations on Windows are now more secure. base::DeletePathRecursively will no longer traverse reparse points (e.g., directory junctions), and ui::OpenFileViaShell now uses the explorer's token instead of the browser's to prevent privilege escalation. (aed056d) (a68adec)
  • Extension security has been enhanced by validating event listener IPC messages against the originating renderer process. This prevents a compromised renderer from spoofing requests to add or remove listeners for an extension it does not control. (6271562) (31dce68)
  • The kUseBoringSSLForRandBytes feature is now enabled by default on Windows and POSIX, centralizing the implementation of base::RandBytes on BoringSSL for more consistent and secure random number generation. (be05a9a)

Browser UI & Features

  • The Vertical Tabs feature is now enabled by default on Windows and macOS, with follow-up changes improving drag-and-drop behavior and visual integration with transparent window frames. (20e810a) (bee36ee) (89fdd0f)
  • ⚠️ BREAKING BREAKING CHANGE A new 'Smart Restart' feature has been introduced on macOS to apply updates when no windows are open. To support this, the browser_shutdown::ShutdownPreThreadsStop() function's signature was changed to return a RestartMode enum instead of a boolean. (7002096)
  • ⚠️ BREAKING BREAKING CHANGE Several features have been completely removed from the browser, including the Web Feed (Following) feature, the chrome.virtualKeyboard extension APIs on non-ChromeOS platforms, and the 'card-linked offers' feature from Autofill. (3c759c0) (ac2eccb) (10d5999)
  • The Omnibox received several performance and behavior improvements. History loading for suggestions is now significantly faster due to batch SQL queries, and the popup's display is deferred until visually ready to prevent stalls. (3ed049f) (c708812)

Extensions

  • ⚠️ BREAKING BREAKING CHANGE The terminalPrivate.onProcessOutput() extension API event is now dispatched only to the sender renderer that opened the process, instead of being broadcast to all renderers. Extensions relying on the previous broadcast behavior will break. (4d9c2be)
  • ⚠️ BREAKING BREAKING CHANGE The autofillPrivate extensions API will now return an empty list instead of an error when its data managers are not yet loaded. This changes an error condition to a success condition, which can break logic in extensions that relied on the previous error-handling behavior. (b1e55a8)
  • The architecture for extension MIME type handling is being modernized. A new chrome.mimeHandler.getStreamInfo() API has been added, and internal routing now favors the OOPIF path for better security and performance. (d680baa) (e8aa93f) (db42fd6)

Networking

  • ⚠️ BREAKING BREAKING CHANGE The core networking class net::HashValue is now immutable. The tag-less constructor and non-const span() method have been removed, breaking any code that constructed or modified a HashValue in place. (335a74c) (8823457) (a91b8f4)
  • The Connection-Allowlist HTTP header now supports the webrtc=block parameter, allowing servers to instruct the browser to prevent the creation of RTCPeerConnection objects for a document. (ec55185)
  • Experimental support for ML-DSA (Dilithium) signatures in TLS has been added behind a feature flag, as part of the ongoing effort to integrate post-quantum cryptographic algorithms. (a2638b6)
