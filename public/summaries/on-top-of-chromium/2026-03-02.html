Chromium Digest: 2026-03-02

Overview

This digest covers 607 commits from the main branch on March 2nd, 2026, with 457 identified as relevant to developers working on Chromium-based projects for Windows and macOS. The day was marked by a significant number of breaking changes, particularly the move to a fully asynchronous Clipboard API and the removal of several public classes and methods. The first and last commits in this range are (b8d1bad) and (c3cd25d) respectively.

Breaking Changes

  • ⚠️ BREAKING The ui::Clipboard API is now fully asynchronous. The synchronous methods ReadAvailableTypes, ReadText, ReadAsciiText, ReadBookmark, and ReadData have been removed from the public interface. All consumers must migrate to the asynchronous, callback-based versions. (1d23bcd) (34bc314) (c6bd1aa) (a1527fc) (8a2045e)
  • ⚠️ BREAKING The public helper class BrowserListEnumerator, used for iterating through the BrowserList, has been removed. Alternative iteration patterns must be used. (eaf9b1a)
  • ⚠️ BREAKING The base::MemoryPressureMonitor class and its static Get() accessor have been removed. Embedders must now use the MemoryPressureListener API. Additionally, the deprecated MemoryPressureListenerTags enum has been deleted. (1acb6aa) (dad691e)
  • ⚠️ BREAKING The virtual method GetAnchorView() was removed from the public ToolbarButtonProvider interface. Implementations must be updated to override GetBubbleAnchor(), which returns a views::BubbleAnchor instead of a views::View*. (e8f13b0)
  • ⚠️ BREAKING The signature of net::FileStream::Seek has changed. Its callback now uses base::expected<int64_t, net::Error> instead of Int64CompletionOnceCallback, requiring updates to all callers. (e6d8aba)
  • ⚠️ BREAKING The AppBannerManager class hierarchy was refactored. AppBannerManagerDesktop no longer inherits from AppBannerManager, which will break code relying on this inheritance for casting or polymorphism. (f3143ea)
  • ⚠️ BREAKING The ExtensionsMenuViewModel::Observer::OnActionIconUpdated method signature was changed to include the action's index, breaking existing observer implementations. (2411cda)
  • ⚠️ BREAKING The public API of SadTabView has been altered, and its friend class declaration has been removed from the core ui/views/view.h header. (fc0af00)
  • ⚠️ BREAKING Several GPU and graphics APIs were changed: accelerated image decoding interfaces were removed from gpu_info.mojom, an overload of SharedImageFactory::UpdateSharedImage was removed, gfx::BufferPlane was deleted, and the signature of GLOzone::ImportNativePixmap was changed. (54c67d2) (0a6652b) (2f15cde)
  • ⚠️ BREAKING The entire chrome/browser/follow feature and the WebAuthn CaBLE v1 protocol implementation have been removed from the codebase. (df23f4f) (02514ec)
  • ⚠️ BREAKING The extensions::BookmarksFunction::RunOnReady() method signature was changed to return ResponseAction instead of ResponseValue to support asynchronous clipboard operations. (a1527fc)

API & Core Interfaces

  • As a consequence of the asynchronous clipboard migration, the RenderViewContextMenu constructor signature was changed to accept an is_paste_enabled boolean. (1d23bcd)
  • The DomStorageDatabaseFactory was refactored, renaming its Open() method to Create() and changing its signature to return a base::SequenceBound object directly instead of using a callback. (3000068)
  • The reentrancy policy for several core UI observers, including CompositorObserver and NativeThemeObserver, was explicitly changed to allow reentrancy, which could alter observer behavior in complex UI scenarios. (dd1dfbf)
  • The Direct Sockets implementation was modularized from //chrome/browser into its own component at //chrome/browser/direct_sockets. (f9c102e)

Browser & UI

  • The Send Tab To Self feature was enhanced to propagate the user's scroll position to the receiving device. This is controlled by the kSendTabToSelfPropagateScrollPosition feature flag. (54ee74b)
  • The @memory Autofill feature was improved to correctly position the cursor and replace the specific @@ trigger that invoked the popup. (46944d2)
  • A new RestartabilityMonitor was introduced to gather metrics on what prevents Chrome from automatically restarting for pending updates. (a1db522)
  • The layout of the chrome://browser-switch/internals page was redesigned for better scaling. (4540a72)

Networking & Security

  • The DNS-over-HTTPS (DoH) logic was updated to require a canary domain check to pass before falling back to a secure provider, in addition to a DoH server being available. (ab614c3)
  • Support for Source-Specific Multicast (SSM) was added to UDPSocket via new JoinSourceGroup() and LeaveSourceGroup() methods. (c1ccae1)
  • Tracked preferences will now treat weakly encrypted hashes as invalid if a stronger key is available, enhancing preference security. (92f4fbc)
  • The TcpConnectJob was updated to more accurately report DNS and connection timing metrics, improving performance diagnostics. (16789da)

Performance & Memory

  • The ReportUkm and SynchronizedScrolling features have been disabled by default. (d6a9743) (4ac3496)
  • On Windows, new metrics were added to track how often system memory lists are low, providing better insight into memory pressure. (e724ff2)
  • New metrics were introduced to measure the duration of "zero-disruption" opportunities (e.g., no windows open, screen locked) where Chrome could restart for an update without interrupting the user. (4298304)
