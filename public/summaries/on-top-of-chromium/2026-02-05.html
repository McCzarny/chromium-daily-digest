Overview

This summary covers 723 commits on the main branch, from commit (ffcc9cb) to (483644a). After filtering for changes relevant to developers on Windows and macOS, 556 commits were analyzed. This digest highlights multiple significant breaking changes to core APIs, along with improvements to browser features, networking, security, and testing infrastructure.

Breaking Changes

  • ⚠️ BREAKING The base::File API has been updated. The Read() and Write() methods now require base::span instead of raw pointers and size arguments. All code performing file I/O with base::File must be updated to use the new span-based signatures. (dd02263)
  • ⚠️ BREAKING The widely used testing utility ui_test_utils::WaitForBrowserToClose() has been removed. This function was prone to hangs. Test suites must be migrated to use the more robust ui_test_utils::BrowserDestroyedObserver to await browser closure. (fef9f36) (1eeffe6) (11ba8dd)
  • ⚠️ BREAKING The FeatureVariation struct in components/webui/flags/feature_entry.h now uses base::span. Embedders who define their own feature flags via this public API must update their definitions to match the new struct layout. (2e89d3b)
  • ⚠️ BREAKING In the gin V8 helper library, gin::ArrayBuffer and gin::ArrayBufferView have been modified. The data() and num_bytes() methods were removed in favor of span-based getters (bytes() and mutable_bytes()), and a constructor overload taking v8::Isolate* was also removed. (e490adb) (d570054)
  • ⚠️ BREAKING Public C-style enums in ui/touch_selection have been converted to enum class. Code using these enums must be updated to use scoped names, e.g., ui::TouchSelectionController::ActiveStatus::kInactive. (e1c7925)
  • ⚠️ BREAKING GLES 3.1 function bindings have been removed from ui/gl and gpu/. Embedders relying on these graphics bindings will experience build failures. (9fc655a)
  • ⚠️ BREAKING The public URL parsing functions DoesContainWindowsDriveSpecUntil() and DoesBeginWindowsDriveSpec() have changed their return contract. The "not found" value is now string_view::npos instead of -1. (20e24b0)
  • ⚠️ BREAKING In the graphics stack, ui::gfx::NativePixmap::GetBufferFormatModifier() was renamed to GetFormatModifier(), and the public utility viz::HasEquivalentBufferFormat() was removed entirely. (64a8baa) (c75850d)

Core APIs and Internals

  • The [[nodiscard]] attribute has been added to WebContents::StartPrefetch and BrowserContext::StartPrefetch. This will generate a compiler warning if the returned PrefetchHandle is not used, helping to prevent resource leaks. (63a6759)
  • The TabRestoreService API now includes a RemoveLeastRecentlyUsedEntries method, allowing for efficient bulk removal of session entries with a single UI update. (ba69fd5)
  • The WebAppFilter class now supports bitwise & and | operators, enabling the creation of complex composite filters from simple ones (e.g., IsIsolatedApp() | IsIsolatedSubApp()). (10076c5)
  • The StickyActivationManager is now thread-safe, ensuring that updates to preferences from field trial variations are always posted to the correct sequence to prevent race conditions. (d851b1d)
  • The OpenTabsUIDelegate interface has a new method, GetForeignSessionTimestamps(), to provide more accurate device activity timestamps for the "Send Tab to Self" feature. (e0a4cfa)
  • As part of an ongoing code health effort, many instances of std::unordered_map and std::set have been replaced with the more performant absl::flat_hash_map and absl::flat_hash_set across the codebase. (4d85445) (fa5d26b) (d26ac35) (128090b)

Networking and Security

  • On Windows, the browser can now monitor the OS certificate store for changes in real-time. This enables automatic synchronization of the trust store without requiring a browser restart, a significant improvement for enterprise environments. (515c9bd)
  • Safe Browsing's Client-Side Detection (CSD) will now perform semantic analysis on clipboard content to detect and report commands that may download and execute remote scripts, enhancing protection against clipboard-based attacks. (33374cd)
  • The disk cache backend has been refactored to use a CacheEntryHasher interface. This allows a custom hasher to be used with encryption, preventing URL discovery by checking against URL hash tables. (9e5a48e)
  • The SQL disk cache's write buffering logic has been improved to better handle large data streams and more accurately track memory usage, leading to more efficient I/O and preventing excessive memory consumption. (6048204) (767da05)
  • Foundational plumbing was added to URLLoader to allow clients to provide a pre-configured data pipe for a response body. This is a non-breaking addition that will enable future "zero-copy" response optimizations. (da594d0)
  • The migration to asynchronous OS-level encryption for Sync data is complete, and the kSyncUseOsCryptAsync feature flag has been removed. (f1aed8b) (a741396)

Browser UI and Features

  • A fix has been implemented on macOS to allow drag-and-drop of downloads from PWA windows. A new Mojo interface correctly initiates the drag operation in the separate app shim process. (f36a3d1)
  • On Windows, icon contrast in High Contrast mode has been fixed for hovered or pressed label buttons. Icons are no longer pre-rasterized, allowing their color to be determined dynamically for better visibility. (47c1b1a)
  • The settings page at chrome://settings/autofill has been updated. It no longer redirects and its title is now "Autofill and passwords". (318e8b4) (d18fbe3)
  • A crash in the Autofill bubble manager has been fixed by adding a check to ensure a bubble can be reshown before processing it from the pending queue. (bdb8b48)
  • A potential crash on macOS related to theme preference migration has been mitigated by ensuring the preference is registered before being accessed. (1c09de5)
  • The Windows installer will now probabilistically sample installation and update failures to gather more detailed logs and crash dumps, aiding in debugging generic errors. (e1e954b)

Extensions

  • A use-after-free crash in OneTimeMessageHandler has been fixed. A WeakPtr check now ensures the handler is still alive before closing a message port after a JavaScript reply. (c3abe5e)
  • A critical crash has been fixed that occurred when using a v8::Context after it had been invalidated (e.g., by page navigation). Cleanup tasks are now performed before any JavaScript that could invalidate the context can run. (2740712)
  • As part of an ongoing modernization effort, the internal definitions for the chrome.power and extensions.offscreen APIs were migrated from the legacy JSON schema to WebIDL. (d2b1a83) (6bb9e10)
  • The implementation of the chrome.history API was extracted into its own build target, continuing the effort to modularize browser components. (872e2a6)

Testing

  • The actor-based testing utility TabManagementTool has been enhanced to support activating and closing tabs, in addition to its previous creation capabilities. (9451810)
  • A stable element identifier, kSettingsOverriddenDialogId, has been added to the dialog that appears when an extension overrides settings, making it easier to write reliable interactive UI tests for this surface. (e52af7b)
  • The UIA event waiter for accessibility testing on Windows was updated to improve its ability to watch for events that Chromium does not directly support, enhancing test capabilities. (366011b)
