This digest covers 3 commits. After filtering for changes relevant to Windows and macOS desktop embedders, 2 commits were analyzed. The changes focus on stability improvements in window management and bug fixes within the payments component. No major breaking changes were identified.

UI & Window Management

  • A Use-After-Free vulnerability in TransientWindowManager has been resolved by replacing base::AutoReset with base::WeakAutoReset. This change prevents memory access after the window manager has been destroyed during synchronous callbacks (e.g., window_->Hide()), improving overall browser stability for desktop platforms. (ab2ec9c)

Payments

  • The PaymentLinkManager logic was updated to prevent its state from being modified while a payment link operation is already in progress. This avoids potential race conditions and ensures more predictable behavior for facilitated payments. (2071ecb)
