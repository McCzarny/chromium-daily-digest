Overview

This report summarizes 819 total Chromium commits from July 9, 2026, with 630 relevant to desktop embedder development and platform infrastructure. Browser Architecture & APIs

  • ⚠️Migration of legacy Browser pointer dependencies to BrowserWindowInterface has continued, notably affecting tab management, extension UI controllers, and browser command dispatching. (b35a245) (811bba1) (6db32f1) (63ec66e)
  • ⚠️Update to WebContentsDelegate::WebContentsCreated and CreateParams to use GlobalRenderFrameHostId instead of separate process and frame integer IDs. (313c6e8) (9779066)

Security & Networking

  • Enhanced Mojo security by tagging WebUI processes in tracing and reinforcing gamepad Permission Policy enforcement. Network logs now support NDJSON output via --net-log-file-format. (91c1e45) (c639462) (35e95eb)
  • Strengthened Mojo IPC for remote services and enforced stricter cross-world cache isolation for extension-related resource fetches. (d5b7078) (c4ae60d)

Extensions & Feature Infrastructure

  • Extension messaging using structured cloning is now enabled by default. MimeHandlerStreamManager now synchronously destroys streams when handler extensions unload to prevent state races. (bcc77d2) (4d25a49)
  • Internal refactoring of extension event dispatching to per-context models, gated by kWebRequestPerContextEventDispatch, to reduce IPC overhead. (ed379ef) (db93420)

Platform UI & System Integration

  • Corrected accessibility event signaling for Windows UIA and resolved macOS printing font serialization regressions. Added WASAPI retry mechanisms to improve Windows audio stability. (83148dd) (59a6b32) (ffd1d6d)
