Summary for 2026-01-06 on branch 'main'

Chromium Digest: 2026-01-06

Overview

This report covers 648 commits from the main branch on January 6th, 2026, with 491 commits identified as relevant to developers building on Chromium. The changes span from the first commit (0673bf1) to the last (08418a1). This summary highlights numerous breaking API changes, significant security hardening on macOS, and ongoing refactoring efforts across the UI and core libraries.

Breaking Changes

  • ⚠️ BREAKING [macOS] The loading of third-party in-process plugins is now blocked by default for the CHILD_PLUGIN process type. This security enhancement will break embedders on macOS who rely on this mechanism to load binaries not signed with the same Team ID as the main application. The behavior can be temporarily overridden with the kBlockThirdPartyInProcessPlugins feature flag. (c54ef9c)
  • ⚠️ BREAKING [Windows] Support for detecting MTP/PTP devices via the public StorageMonitor component has been completely removed on Windows. This is a major feature removal that will break embedders relying on this functionality for media device interaction. (aef221a)
  • ⚠️ BREAKING The public utility function base::ByteSwap has been removed from the base library. Embedders should migrate to using std::byteswap from the C++ standard library. (5869e71)
  • ⚠️ BREAKING The signature of the public GPU method FencedAllocatorWrapper::Alloc() has been changed to return a base::span instead of a raw pointer. Code calling this method must be updated to handle the new return type. (de3e9aa)
  • ⚠️ BREAKING The by_display_initiator field has been removed from the public downloads.mojom.Data interface as part of a feature cleanup. Clients of this mojom API will need to be updated. (a2891e1)
  • ⚠️ BREAKING Public serializer functions for session storage and local storage (CreateKey, KeyPrefix, etc.) have been made private by removing them from session_storage_leveldb.h and local_storage_leveldb.h. Any external code relying on these functions will break. (23e8879) (dee78ad)
  • ⚠️ BREAKING The MemoryPressureListener has been removed from the public API of the compositor's cc::ResourcePool class. Embedders interacting with this part of the compositor API will need to remove usage of the listener. (4470afa)
  • ⚠️ BREAKING The public functions RegisterLegacyUsagePrefsForMigration and MigrateLegacyUsagePrefs have been removed from the Optimization Guide's model_execution_prefs.h header as part of a dead code cleanup. (4bd9cc4)
  • ⚠️ BREAKING Strict validation for JSON MIME types is now unconditional, as the StrictJsonMimeTypeTokenValidation feature flag has been removed. This behavioral change may break integrations that relied on the previous, more lenient parsing of JSON responses. (24b17ed)
  • ⚠️ BREAKING The interaction model for tab group headers is now standardized. Left-clicking a header will always toggle its collapsed state, and right-clicking will open the editor bubble. The kLeftClickOpensTabGroupBubble feature flag that allowed customization of this behavior has been removed. (5c3a14f)
  • ⚠️ BREAKING A recent change to the public Cronet networking library has been reverted, removing the was_proxied field from the UrlResponseInfoImpl class. Embedders who adopted this new field will encounter build failures. (cb48ee6)

Security & Privacy

  • The URLAllowlist and URLBlocklist policies were refactored to use the standard sensitive policy mechanism. This improves their visibility on chrome://policy when blocked by an untrusted source, preventing them from being silently ignored. (adfc6d7)
  • Development continues on the experimental Device Bound Session Credentials (DBSC) feature. Changes include adding more detailed event structures to the mojo API for debugging, adding DevTools support behind a flag, and preparing an observer skeleton for the SessionService. (a8949e5) (fd181eb) (3e54720)
  • Internal state tracking for ChildProcessSecurityPolicy was refactored into a new SecurityStateMaps inner class to improve correctness without altering the public API. (589551b)

Browser & UI

  • Public APIs related to an experimental vertical button layout in dialogs have been removed. The allow_vertical_buttons property and its associated methods are no longer available in views::DialogDelegate. (dd407f7)
  • On macOS, a bug was fixed to ensure content areas correctly regain focus when a window is restored from a minimized or hidden state. (7e252e2)
  • On Windows, drag-and-drop functionality was improved to correctly handle dragging multiple files from compressed ZIP folders. (53a893f)
  • A bug causing ampersands (&) to be rendered incorrectly in the "open all" menu items for bookmark folders has been fixed. (249828d)
  • The profile sign-out confirmation dialog was updated to introduce a distinct "Cancel" button, improving the user flow, especially when re-authentication is required. (0508ff4)
  • A large-scale refactoring converted the LayoutConstant enum to an enum class, improving type safety across numerous UI files. (3efff9e) (a303cad) (6c1d3d1)

Extensions

  • The UnpackedInstaller class was moved from chrome/browser/extensions to the more generic extensions/browser directory. This refactoring makes the class more accessible to non-Chrome embedders who need to install unpacked extensions. (e87a1a6)
  • A bug was fixed in the chrome.developerPrivate API that incorrectly fired uninstall events for Theme extensions, which could cause assertion errors. (c12220c)
  • Documentation for several Extension APIs was updated in .idl and .webidl files to use Promise-based language. This change improves documentation and tooling alignment with modern JavaScript but does not alter runtime behavior. (f5b1a2e) (678b098)

Base & Core Internals

  • The base::MemoryPressureListener has been updated to be more stateful. New listeners are now immediately notified of the current memory pressure level upon creation, rather than waiting for the next pressure change. (5c71b78)
  • A new stub module, activity_reporter, has been created and is owned by the BrowserProcess. This aims to centralize activity logging for metrics in the future. (c75e058)
  • The Windows updater was improved to delete orphaned version directories during cleanup and to try uninstalling for a longer duration to increase success rates. (79ec113)
  • Widespread code modernization continues, with many instances of base::Contains() being replaced by std::ranges::contains() and raw pointer/size pairs being replaced with base::span. (e13cb6a) (f301383) (116ad4d)

Feature Lifecycle & Cleanup

  • The feature flag kUserAgentFrozenBuildVersion and its associated parameter have been removed from embedder_support. This solidifies the "0" build number in the User-Agent string and removes a configuration point for embedders. (efdbfb2)
  • The feature flag enable-secure-payment-confirmation-availability-api was removed now that the API is fully launched. (3551439)
  • Obsolete code for cleaning up the PNaCl component was removed from the component updater, as PNaCl itself was removed over six months ago. (99a92f9)
  • The "Read Aloud" feature flag was removed from chrome://flags as the feature moves towards launch. (d52447d)

Build & Testing

  • [Revert] A change that enabled asynchronous hit testing for drag-and-drop operations on macOS was reverted due to a crash regression. This temporarily disables improved drag-and-drop support for out-of-process iframes on Mac. (e7ef7cd)
  • The build system's process for handling sensitive policies has been automated. The build now automatically adds required notices, with a presubmit check to prevent manual additions. (2f59496)
  • The Node.js dependency used for various build tools has been updated to v24.12.0 LTS. (8ced542)
