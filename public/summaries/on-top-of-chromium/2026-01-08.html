Chromium Digest: 2026-01-08

Overview

This summary covers 468 relevant commits out of a total of 620 on the main branch for January 8, 2026. The changes span from the first commit ((f531710)) to the last ((fe6f28d)). This digest highlights several significant breaking API changes in the UI, networking, and storage layers, alongside key enhancements for extensions, policy management, and testing infrastructure.

Browser & UI

  • ⚠️ BREAKING BREAKING CHANGE: The TabStripModel::selection_model() method's return type has been changed from ui::ListSelectionModel to a new tabs::TabStripModelSelectionState wrapper class. Code that directly accesses the selection model will break and must be updated to use the new wrapper, for example by calling selection_model().GetListSelectionModel(). (5242d38)
  • ⚠️ BREAKING BREAKING CHANGE: The public method BrowserView::tabstrip() has been removed and renamed to BrowserView::horizontal_tab_strip_for_testing(). As BrowserView is a core public UI class, this is a breaking API change for any embedder or external tool that relied on it to access the tab strip view. (f55ce20)
  • Drag-and-drop handling for text/uri-list has been improved to correctly parse multiple URLs. A new BookmarkList clipboard format was also added for Windows to support dragging multiple bookmarks. (e3ec71d)
  • Numerous improvements were made to the experimental Vertical Tabs feature, including drag-and-drop support within groups and various UI tweaks to margins, widths, and alert indicators. (1b6124e) (3c34712) (16547a9) (e5031a4) (8a75f68)
  • A bug was fixed where cr-dialog elements in WebUI could be incorrectly closed with the 'Escape' key when the noCancel property was set. (48bf788)
  • A dangling pointer crash in TooltipStateManager was fixed by adding a WindowObserver to clear pointers before window destruction. (cb5183b)
  • A crash in the find-in-page feature was fixed. The crash occurred when searching on a crashed tab (e.g., chrome://crash) by adding validation to ensure the target frame is live. (f12b632)

APIs & Extensions

  • ⚠️ BREAKING BREAKING CHANGE: The sidePanel.close() extension API's behavior has changed. When called with a tabId, it no longer falls back to closing the global side panel. It will now only close a tab-specific panel and will return an error if no such panel is enabled for the specified tab, potentially breaking extensions that relied on the fallback. (cf0517b)
  • The runtime.onMessageExternal listener now supports returning a Promise for asynchronous responses, allowing for more modern async handling in extensions. (b093c11)
  • The DevTools Protocol method Target.createTarget with background=true will no longer focus the window, providing more consistent behavior for creating background targets. (e1038b9)
  • A bug in the chrome.bookmarks API was fixed where it did not correctly format URLs upon creation or update. URLs are now properly sanitized. (4cddd25)
  • A new NavigationThrottle was introduced to intercept navigations for protocol handlers registered by an extension that have not yet been confirmed by the user. (c27a9af)
  • The mechanism for triggering silent PWA manifest updates was refactored to check for updates whenever a manifest is seen or changed, improving reliability. (317f260)

Networking

  • ⚠️ BREAKING BREAKING CHANGE: Deprecated public proxy APIs have been removed from Cronet. This includes constructors for org.chromium.net.Proxy and org.chromium.net.ProxyOptions, requiring embedders to use static factory methods instead. Several methods from Proxy.HttpConnectCallback were also removed. (3670059)
  • ⚠️ BREAKING BREAKING CHANGE: The disable_idle_sockets_close_on_memory_pressure parameter has been removed from the public network.mojom.NetworkContextParams interface. Embedders programmatically creating a NetworkContext with this parameter will experience a build failure. (6cbcf51)
  • Behavioral Breaking Change: Support for the chromotocol protocol has been disabled on the Chrome Remote Desktop host. This protocol was used by the now-delisted native iOS client, and any tools still attempting to use it will no longer be able to connect. (656334f)

Security & Policy

  • A new group policy, WebAppInstallForceList, has been implemented to allow enterprise administrators to prevent users from installing web apps from the browser on Windows and macOS. (06504a5)
  • The kSyncUseOsCryptAsync feature is now enabled by default, moving asynchronous OS-level encryption for sync data closer to a full launch. (837aa57)
  • On macOS, the UnexportableKeyProviderMac was refactored to support prefix matching when deleting keys from the Keychain, aligning its behavior with key retrieval methods. (45163bd)
  • The kAdditionalOpaqueOriginEnforcements feature flag was removed, making its security behavior permanent and simplifying related code. (c85efb1)

Base, Testing & Infrastructure

  • ⚠️ BREAKING BREAKING CHANGE: The signature of the public method OzonePlatform::IsNativePixmapConfigSupported has changed. It now takes a viz::SharedImageFormat instead of a gfx::BufferFormat, which is a breaking change for embedders who implement custom Ozone platforms. (01619b6)
  • ⚠️ BREAKING BREAKING CHANGE: The GetLevelDB() method has been removed from the public DomStorageDatabase interface in components/services/storage. Embedders using this to access the underlying database for local/session storage will need to remove these calls. (2f26e1c)
  • The PrerenderTestHelper (content/public/test/prerender_test_util.h) has a breaking change for test suites: the AddPrerender and GetHostForUrl methods now return a PrerenderHostId instead of a FrameTreeNodeId. (477aec0)
  • For chromedriver users, element screenshots are no longer scaled. This aligns behavior with the WebDriver specification but may require updates to visual regression tests. (cbb1ce2)
  • A new helper function, base::RandomChoice(), was added to base/rand_util.h to provide a standardized way of selecting a random element from a container. (9610cfd) (fe6f28d)
  • As part of a long-term refactoring, several components were migrated from the global MemoryPressureMonitor to use the more granular MemoryPressureListener interface. (b6f01dc) (029efeb) (0c3608f)
  • Direct usage of chai.js in WebUI tests is now banned in favor of the chai_assert.ts wrapper to improve test infrastructure. (3dc6f5e) (afdf971)
