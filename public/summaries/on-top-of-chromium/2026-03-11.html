Chromium Digest: 2026-03-11

Overview

This release includes 668 commits, with 511 identified as relevant to developers building on Chromium for Windows and macOS. The day was marked by a significant number of breaking changes affecting core APIs in networking, file I/O, UI, and identity management, requiring immediate attention from embedders. Other notable work includes security hardening with the network service sandbox now enabled by default on Windows, continued modularization of the chrome/browser directory, and the final removal of several features and their associated flags. Breaking Changes

  • ⚠️ BREAKING The base::File class's public method int WriteAtCurrentPos(const char* data, int size) has been made private. Code must be updated to use the base::span-based overload. (42d167b)
  • ⚠️ BREAKING The callback signature for the core networking functions ClientSocketPool::Preconnect and net::PreconnectSocketsForHttpRequest has changed from base::OnceCallback<void(int)> to base::OnceCallback<void(bool)>. Callbacks must be updated to accept a boolean success status instead of a net::Error code. (ad97a90)
  • ⚠️ BREAKING The signature of the virtual method InputHandler::ScrollEnd has been modified to include a new compensated_scroll_delta parameter. Custom subclasses of InputHandler will need to be updated. (9ec5ed7)
  • ⚠️ BREAKING The public API content::ChildProcessHost::LaunchAsChildProcess and the associated CHILD_PLUGIN helper process type have been removed. Embedders using this method to launch child processes must find an alternative. (12d359c)
  • ⚠️ BREAKING The PrivacySandboxActivityTypesService and its factory have been removed. Any embedder code using this ProfileKeyedService will no longer compile. (44dc583)
  • ⚠️ BREAKING The constructor for signin::internal::PrimaryAccountManager and the signin::IdentityTestEnvironment::BuildIdentityManagerForTests utility now require a metrics::ProfileMetricsService instance, breaking custom initialization of the identity stack. (de412c3)
  • ⚠️ BREAKING The factory method AutofillSuggestionController::GetOrCreate now requires an additional AutofillSuggestionTriggerSource argument. All calls to this method must be updated. (a29ad42)
  • ⚠️ BREAKING The public API of AccessibilityAnnotationService has been changed. The AddObserver(), RemoveObserver(), and GetEntities() methods have been removed in favor of a new GetEntityDataProvider() method. (77cb6c6)
  • ⚠️ BREAKING The direct getter upgrade_notification_controller() has been removed from the public BrowserWindowFeatures class. Access is now through the static method UpgradeNotificationController::From(browser()). (1542adb)
  • ⚠️ BREAKING The value of the public constant ui::display::kDefaultDisplayId has been changed from 0xff to -2, which can break custom display or window management logic. (1983a07)
  • ⚠️ BREAKING The underlying type of the ui::KeyboardCode enum is now explicitly sized, which may break code that performs unsafe casting or makes assumptions about the enum's size. (e9e9cf3)
  • ⚠️ BREAKING The legacy FindBarIcon UI class has been completely removed from the location bar implementation. Embedders that customized or referenced this class must adapt to the new page action-based UI. (fd062ba)
  • ⚠️ BREAKING The TabSearchContainer UI view class and its identifier kTabSearchContainerElementId have been removed from the tab strip, breaking any code that directly references them. (d43a49d)
  • ⚠️ BREAKING The deprecated "Click to Call" feature has been removed, including the public enum value PageActionIconType::kClickToCall. Code referencing this type will fail to compile. (47aad4d)
  • ⚠️ BREAKING The navigator.storage.estimate() API will now always report a fixed quota in Incognito mode to prevent privacy leaks. This is a significant behavioral change to a stable API. (c48f71d)
  • ⚠️ BREAKING On macOS, the "safe mode" for video capture and its corresponding feature flag kRetryGetVideoCaptureDeviceInfos have been removed. This removes a stable feature that allowed retrying device enumeration after a crash. (a252ae8)

APIs & Core Internals

  • The SidePanelRegistry has been refactored to use tab-scoped user data. A new From(TabInterface*) method is introduced to reduce dependencies on the Browser object. (2970137)
  • A new helper function, Uuid::AsInteger(), was added to base::Uuid to convert its string representation to a 128-bit integer. (54458bb)
  • The geolocation permission prompt now supports an "approximate-only" state, which may require updates for embedders with custom permission UIs. (03df9c4)

Browser UI & Features

  • A new HitTestCallback was added to ui::views::FrameView, providing a uniform hook for embedders to customize hit-testing and implement draggable regions in non-client areas. (7c1ec2a)
  • Several features were finalized and their flags removed, including EnableNtpEnterpriseShortcuts (now default), password-form-clientside-classifier, and multiple Autofill/Sync flags for loyalty cards. (b2574b0) (e484c32) (b937d39) (ba1bc87) (76c9ae2)
  • Foundational work was done to migrate the chrome://policy page to Mojo, managed by the new kPolicyPageMojoMigration feature flag. (b6222f0)
  • On macOS, a privacy issue was fixed where a user's full profile path was written to the clipboard when copying a bookmark. (f7d0665)
  • A new feature flag, kHistoryEnableWalMode, was added to experiment with enabling SQLite's Write-Ahead Logging (WAL) mode for the History database to improve performance. (79a5694)

Networking & Security

  • The network service sandbox is now enabled by default on Windows, significantly hardening the browser's security posture. (a0ad012)
  • A security vulnerability was fixed where the Authorization header was not stripped on cross-origin fetch() redirects when keepalive was used. (5fb6646)
  • For the new Built-In AI APIs, browser-side permission policy checks have been added. If a compromised frame attempts to use these APIs without permission, the renderer process will be terminated. (a3d6e7c)
  • The policy engine now supports policies that can have both machine and user scope values active simultaneously, with ChromeDataRegionSetting being the first to use this. (e60b2de)
  • Rollout is beginning for the client certificate provisioning policies: AutoSelectCertificateForUrl and ProvisionManagedClientCertificateFor*. (9bd8cfd)

Extensions

  • The migration of extension API tests from Manifest V2 to V3 continues for APIs like chrome.extension, management, and history, reinforcing the deprecation of MV2. (0b8b89d) (35e7b66) (c4f4143) (36f2f4c) (bdbefc8)
  • Initial support for the "devtools_page" manifest key was added to the core extensions platform. (5e32adf)
  • A crash in the chrome.cookies API was fixed by wrapping Mojo callbacks to ensure they are always invoked, preventing crashes if the network service disconnects during an API call. (70dd2cc)
  • A crash was fixed that could occur when an extension with a service worker was reloaded by ensuring stale context initialization messages are safely ignored. (d12bcc8)

Testing & Infrastructure

  • The ongoing modularization of chrome/browser continues, with page_info, data_sharing, engagement, security_events, manta, and performance_monitor being moved into their own build targets. (f7272c2) (b49f647) (1703de6) (d58da74) (d8243dd) (5e70d3b)
  • The Bookmarks component's unit tests were migrated from a custom helper to the standard MockBookmarkModelObserver with gMock expectations, improving test clarity. (e096b12)
  • The autotest tool was improved to automatically find and run the relevant test file when a corresponding implementation file (.cc) is changed. (bd89b93)
