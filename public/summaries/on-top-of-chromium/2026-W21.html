Week 21 of 2026 (2026-05-17 to 2026-05-23)

Chromium Weekly: 2026 Week 21

Overview

This week was characterized by a significant number of breaking API changes across the codebase, impacting core browser functions, UI, media, and extensions. Key developments include major progress on the Manifest V2 deprecation, substantial build system modularization, and critical security hardening in the extensions and networking stacks. Across the 7 days, there were 523 relevant commits out of 692 total.

Core APIs & Browser Internals

  • ⚠️BREAKING CHANGE Numerous public helper functions in chrome/browser/ui/browser_finder.h, such as chrome::FindTabbedBrowser and chrome::GetIncognitoBrowserCount, have been removed across several commits. Embedders must migrate to the newer ProfileBrowserCollection and GlobalBrowserCollection APIs for querying browser state. (1d867aa) (e9e4c74) (686390b)
    BrowserUI
  • ⚠️BREAKING CHANGE The core content::ContentBrowserClient interface has a new pure virtual method, GetContainerPolicyOverrideForCommit. All embedders must implement this method, which allows overriding a frame's container policy at commit time. (6da6b23)
    BrowserSecurityExtensions
  • ⚠️BREAKING CHANGE Several core media and UI API signatures were updated, requiring embedder changes. AudioManager::GetAudio...DeviceNames now returns a bool, media::VideoCaptureDevice::Client::OnIncomingCapturedImage requires a natural_size parameter, and WebContentsDelegate::OnDidBlockNavigation requires initiator_url and initiator_origin. (d1b707c) (636ff60) (64a041d)
    MediaBrowserNavigation
  • ⚠️BREAKING CHANGE The deprecated SyncService methods IsSyncFeatureEnabled() and IsSyncFeatureActive() have been removed. Callers must now use GetUserSettings()->GetSelectedOsTypes() or GetActiveDataTypes(). (9efc428)
    Sync
  • ⚠️BREAKING CHANGE The experimental Compute Pressure API has been completely removed due to an information leak vulnerability. All related files, feature flags, and logic have been deleted from the codebase. (4d118b5) (d235c83) (a8f43ad) (448f7cd)
    SecurityPrivacyBlink
  • A new enterprise policy, HttpHeaderInjection, has been introduced. While the implementation is still in progress, this will allow administrators to inject custom HTTP headers into requests for specified URL patterns. (a6a8295)
    PolicyNetworkEnterprise

Security & Privacy

  • A critical cross-origin secret leak in the Extensions Web Request API was fixed. The fix detects request ID collisions that could route callbacks to the wrong extension, aborts the request, and terminates the offending renderer process. (048923c)
    SecurityExtensionsNetwork
  • ⚠️BREAKING CHANGE The WebNN API is being hardened by moving graph validation into a separate utility process. This introduces new WebNNCompilerService and WebNNCompilerContext mojom interfaces, creating a new security boundary. (a1a5ab0)
    AISecurityGPU
  • ⚠️BREAKING CHANGE When a Progressive Web App (PWA) is uninstalled, its windows are now forcibly closed, bypassing beforeunload handlers. This fixes a potential origin spoofing vulnerability where a window could remain open without its custom PWA frame. (d47269c)
    SecurityBrowser
  • On Windows, support has been added to force Address Space Layout Randomization (ASLR) for sandboxed child processes, making them harder to exploit. Embedders can override this behavior via the new ContentBrowserClient::ShouldForceASLR() method. (09eaeae)
    SecurityWindows

Extensions

  • ⚠️BREAKING CHANGE The kExtensionManifestV2Disabled feature flag has been removed, permanently forcing all Manifest V2 extensions into the "disable" phase. This is a major milestone in the MV2 deprecation, and embedders can no longer use the flag to delay the transition. (e768ae9)
    ExtensionsBrowser
  • ⚠️BREAKING CHANGE The ExtensionKeybindingRegistry::Delegate class has been removed. The ExtensionKeybindingRegistry constructor now requires a TabListInterface*, abstracting away platform differences for getting the active tab. (f08221c)
    ExtensionsUI
  • A Time-of-Check Time-of-Use (TOCTOU) race condition in the speech recognition API that allowed extensions to bypass audio capture permissions has been fixed. (0086720)
    ExtensionsSecurityMedia

UI & UX

  • ⚠️BREAKING CHANGE The FullscreenObserver class has been removed from FullscreenController. Consumers must migrate to the new RegisterOnFullscreenStateChanged method, which uses a base::CallbackListSubscription pattern for safer lifetime management. (00d519b)
    UIBrowser
  • ⚠️BREAKING CHANGE The ConfirmInfoBar class is now an abstract base class. Direct instantiation will fail; developers must use the new static factory method ConfirmInfoBar::Create() instead. (0cda611)
    UIBanner
  • A new feature for global Most-Recently-Used (MRU) tab switching with Ctrl-Tab is being implemented under the kCtrlTabMru flag. This allows users to toggle between the two most recent tabs across all windows. (2e673d0)
    UITabsBrowser
  • The session restore service now correctly restores split tab layouts and the tab groups within them, improving persistence of complex window arrangements after a browser restart or crash. (a39bde7) (2897e0b)
    TabsBrowserUI

Build & Testing

  • ⚠️BREAKING CHANGE The build system has undergone significant modularization. The monolithic //chrome/browser:browser and //chrome/browser/ui:ui targets have been broken into smaller, header-only source sets. This will break builds for embedders with direct dependencies on old file locations. (a3e448d) (9694359)
    BuildBrowserUI
  • ⚠️BREAKING CHANGE The base::FileTracing::Provider interface has been removed. File tracing is now handled internally within base::File using Perfetto, removing the public API for supplying a custom file tracing provider. (6855e06)
    BaseInfraPerformance
  • ⚠️BREAKING CHANGE Support for applying Chrome policies in the headless shell has been completely removed. The headless_use_policy GN flag and all associated logic are gone, which will break embedders who relied on policies for configuration. (fcdf7e8)
    BrowserPolicyBuild
  • The build system now supports TypeScript v7 via the use_typescript_go GN argument, modernizing the toolchain for developers working on WebUI components. (dd56ca1) (1487cbd)
    BuildInfraUI
