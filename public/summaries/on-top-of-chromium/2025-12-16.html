Chromium Digest: 2025-12-16

Overview

This summary covers 563 commits on the main branch for December 16th, 2025, with 421 identified as relevant to developers. The changes span from commit (9356cc6)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((9356cc6)) to (4296b27)" target="_blank" rel="noopener noreferrer" class="text-sky-500 hover:text-sky-300 font-mono">((4296b27)) and include a significant number of breaking API changes across UI, networking, security, and the build system.

UI & Views

  • ⚠️ BREAKING The ui/views framework now enforces child list immutability during iteration in production builds via a CHECK. Previously a debug-only check, this will now cause a hard crash if a View's children are mutated while being iterated over, exposing latent bugs in UI code. (4b55b0b)
  • ⚠️ BREAKING Several public static methods for querying incognito browser states, such as IsOffTheRecordBrowserActive(), have been removed from BrowserList. Developers must migrate to the newer functions available in browser_finder.h. (bde2c0b) (d4b2a8e)
  • ⚠️ BREAKING The cc::LayerTreeSettings API has been updated for more granular overscroll control. The enable_elastic_overscroll boolean has been replaced by enable_elastic_overscroll_on_root and enable_elastic_overscroll_for_subscroll, requiring updates for embedders who configure compositor settings. (f1f1620)
  • A bug on macOS that prevented the bookmark menu from opening on hover when in full-screen mode has been fixed. (81072e1)
  • The experimental 'Tab Organization' feature has been removed. This cleans up related APIs in TabOrganizationService and removes associated metrics. (e1c53e2)

Security & Enterprise

  • ⚠️ BREAKING The public safe_browsing::BinaryUploadService::Request class has been removed. Embedders using this service for content scanning must migrate to the enterprise_connectors::BinaryUploadRequest class. (afc8753)
  • ⚠️ BREAKING The static factory method safe_browsing::BinaryUploadService::GetForProfile has been removed. Its logic is now available via the new free function enterprise_connectors::GetBinaryUploadServiceForConnector. (5ee7771)
  • ⚠️ BREAKING The REVOKE_SYNC_DISALLOWED value has been removed from the public SigninClient::SignoutDecision enum. Custom SigninClient implementations that check for this state will need to be updated. (10226a6)

APIs & Core Internals

  • ⚠️ BREAKING The public LaunchGoogleChrome function in the Windows GCAPI no longer requires the caller to uninitialize COM. This simplifies the API but is a breaking change to its documented contract. (b387b30)
  • The Permissions API has been extended with a new GEOLOCATION_APPROXIMATE permission type, preparing for future support of approximate location requests. (92e4b27)
  • The base::PassKey utility has been improved to support multiple types, allowing a function's visibility to be restricted to a set of approved calling classes instead of just one. (72446fb)
  • The experimental prefetch_with_subresources action for Speculation Rules has been removed. This also removes the now-unused public content::SpeculationHostDelegate interface. (277fc6f)

Networking

  • ⚠️ BREAKING The net::GetHttpReasonPhrase() API has been reworked. Its signature now accepts an int instead of net::HttpStatusCode, and it will no longer return nullptr for unknown codes, preventing potential crashes. (f32bb69)

Build System & Platform

  • ⚠️ BREAKING Build support for ARMv6 and ARM configurations with NEON disabled (arm_use_neon=false) has been removed. Projects compiling for these older ARM targets will no longer build. (e6b1ab2)
  • ⚠️ BREAKING On Windows, the IProcessLauncher::LaunchCmdLine method in the Updater's public COM interface has been reverted to return E_NOTIMPL, making it non-functional. Clients relying on this method to launch processes will be broken. (6684d0a)
  • On Windows, the installer can now install the Windows App Runtime package to support WebNN, and a deadlock during this installation process has been fixed by moving the operation to a background thread. (d9435e7) (dd0ebe0)

Extensions

  • Minor internal refactoring moved the kMaxMessageBytes constant to message_port.mojom for better visibility and updated the SandboxedUnpacker to reduce string conversions. (d01145a) (fbaeecf)

Feature & Flag Removals

  • Several stale feature flags for shipped or abandoned features have been removed, including kDestroySystemProfiles, kFedCmAlternativeIdentifiers, and TabScrollingButtonPosition. (8c267f0) (06811e3) (c07c1f1)
