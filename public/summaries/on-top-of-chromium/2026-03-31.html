Chromium Digest: 2026-03-31

Overview

This summary covers 710 commits to the Chromium main branch on March 31, 2026, with 549 identified as relevant to developers building on the platform. This period includes a large number of breaking changes to core APIs in UI, services, and extensions. The first relevant commit was (39c2ad8) and the last was (bb2ee6a).

APIs & Services

  • ⚠️ BREAKING BREAKING: The public method ContentBrowserClient::UpdateEnabledBlinkRuntimeFeaturesInIsolatedWorker has been removed. Embedders overriding this method to control Blink features in Service Workers must remove their overrides. (b01d62a)
  • ⚠️ BREAKING BREAKING: The public API for AccountSettingService has been refactored. Feature-specific methods like IsWalletPrivacyContextualSurfacingEnabled() are removed in favor of a generic GetBoolean(const AccountSetting& setting) method. Callers must update to the new generic interface. (19e350b)
  • ⚠️ BREAKING BREAKING: The public method signature for QueryContextualizer::Contextualize has changed. The callback signature is modified and the session_handle parameter is removed, requiring updates for any embedder using this contextual search component. (a95f201)
  • ⚠️ BREAKING BREAKING: The user_gesture boolean parameter has been removed from the RequestPermission and RequestPermissions methods in the public permission.mojom interface. The browser now determines transient activation internally. (5168b89)
  • ⚠️ BREAKING BREAKING: The public printing API has changed with the removal of PrintedDocument::GetMetafile(). Embedders must now use the new PrintedDocument::HasDocument() and PrintedDocument::GetDocumentData() methods to access the printed document's data. (3ac886f)
  • ⚠️ BREAKING BREAKING: The public method IsAllowedBy3pcdHeuristicsGrantsSettings has been removed from CookieSettingsBase as part of deleting the obsolete kTpcdHeuristicsGrants feature. Embedders relying on this method will have their builds broken. (8214edf)
  • ⚠️ BREAKING BREAKING: The behavior of the public RunIsolatedChrome COM interface on Windows has changed. It now passes non-switch arguments after a -- separator, which can break callers who relied on the previous argument handling. (068662f)
  • The TabStrip API has been enhanced to support closing entire tab groups, in addition to individual tabs. (7cd1de4)
  • A bug in the Web Speech API has been fixed where audio from speechSynthesis could persist after page navigation. Active utterances are now correctly canceled when the primary page changes. (0793f4f)

UI & Views

  • ⚠️ BREAKING BREAKING: The public virtual method BrowserWindow::UserChangedTheme() has been removed. Embedders with custom BrowserWindow implementations must now use the new BrowserWindowThemeObserver callback mechanism to listen for theme changes. (a863957)
  • ⚠️ BREAKING BREAKING: Core public drag-and-drop methods in ui/views have been renamed. For example, RunShellDrag is now RunDragDropLoop and OnWidgetDragComplete is now OnWidgetDragDropCompleted. Embedders using views::WidgetObserver or views::MenuDelegate must update to the new names. (e246bab)
  • ⚠️ BREAKING BREAKING: The browser_controls_api.mojom interface for WebUI has been updated. Methods like Back(), Forward(), and ReloadFromClick() are no longer fire-and-forget and now return a result via a callback, breaking existing clients. (41af281)
  • ⚠️ BREAKING BREAKING: The public static method SearchboxHandler::SetupWebUIDataSource has been removed and replaced with SearchboxHandler::GetWebUIDataSourceDict. WebUI implementations using the old helper method will break. (e51273e)
  • ⚠️ BREAKING BREAKING: The signature of the ShowContentSettingsBubble method in the public Mojo interface toolbar_ui_api.mojom has been changed. Embedders using this API to build custom toolbars will need to update their code. (023a4ca)
  • The "Send Tab to Self" device picker now displays more granular timestamps, such as "Active 5 minutes ago", for recently used devices. (38f613e)
  • Hover cards are now supported for tab group headers in the horizontal tab strip, providing more information on hover or focus. (e8ce621)
  • PWA uninstallation logic has been improved to properly clean up OS integration resource directories, resolving an issue on Windows where .ico files were left behind. (2a27c7f)

Extensions

  • ⚠️ BREAKING BREAKING: The CWSInfoService has been moved from //chrome/browser/extensions to //extensions/browser. Projects must update the include path from chrome/browser/extensions/cws_info_service.h to extensions/browser/cws_info_service.h to avoid build failures. (d57d493)
  • A critical vulnerability in the chrome.debugger API was fixed. It previously allowed an extension to bypass web_accessible_resources checks and navigate subframes to private extension pages. (5baca06)
  • Extension keybindings are now correctly registered on browser launch, ensuring they are active without requiring manual intervention by the user. (9923357)

Security

  • The networking stack was hardened against HTTP Request Smuggling by fixing an integer overflow in ElementsUploadDataStream that could lead to an incorrect Content-Length header. (4c99d24)
  • The RestrictedCookieManager now reports a bad message and terminates the renderer connection if site_for_cookies or top_frame_origin are incorrect, hardening the API against misuse. (225000a)
  • The client_side_redirect_url in the BeginNavigation IPC message is now properly validated and filtered by the browser process, closing a potential security gap. (59cf813)
  • DevTools security was improved by correctly enforcing file access restrictions for view-source:file:// URLs, which could previously be bypassed. (1f5d394)
  • The core base::PickleIterator has been refactored to use the safer base::SpanReader, enforcing stricter C++ alignment and data handling rules to prevent unsafe buffer usage. (a630cf8) (e98ba17)

Performance

  • A new "Inline Script Cache" feature has been introduced behind the kInlineScriptCache flag. It creates a persistent code cache for inline scripts to improve subsequent page load times. (514b324) (6e5c441)
  • The Performance Manager's logic for Memory Saver has been improved. It now waits for a page to be fully loaded and idle before considering title or favicon changes as user activity, preventing initial load churn from being misclassified. (8da9611)

Build, Testing & Infrastructure

  • ⚠️ BREAKING BREAKING: The legacy bool FromMojom(Input, Output*) signature for Mojo's EnumTraits has been removed. All custom Mojo traits must now use a modern signature that returns the value directly or via std::optional. (a1c3e63)
  • ⚠️ BREAKING BREAKING: The kFlagNotEnabled enum value has been removed from the AppUpdateDataReadResult enum in chrome/browser/web_applications/commands/app_update_data_read_command.h. This will cause compilation failures for code that switches on this enum. (3efe506)
  • Integration with Rust has been improved. In developer builds, a Rust panic! will now trigger a LOG(FATAL) in C++, providing more detailed crash reports that include C++, Java, and task runner stack traces. (89012c0)
  • The base/traits_bag.h header has been refactored to use C++20 concepts, providing much clearer compiler errors for invalid trait combinations. (5a5b956)
  • The Mojom wire format is now officially documented, providing a reference for developers working with Mojo IPC. (c289c51)
  • A bug in the Windows accessibility implementation has been fixed to prevent returning uninitialized memory from get_columnHeaderCells and get_rowHeaderCells, which could lead to crashes in COM marshalling. (c81f01b)
