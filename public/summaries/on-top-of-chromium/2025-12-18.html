Chromium Digest: 2025-12-18

Overview

On 2025-12-18, the Chromium main branch saw 637 commits, with 466 identified as relevant to developers on Windows and macOS. This day was marked by a significant number of breaking changes across core utilities, APIs, and UI components, requiring embedder attention. Key themes include the modernization of extension APIs, refactoring of security and privacy settings, and numerous UI refinements. The day's changes began with commit (08603f6) and concluded with (5a487f7).

Breaking Changes

  • ⚠️ BREAKING The core tracing API has been refactored, removing the TraceLog::EnabledStateObserver and TraceLog::AsyncEnabledStateObserver interfaces. Embedders must now use the new per-session TraceSessionObserver to monitor tracing status. (8e0db64)
  • ⚠️ BREAKING The base::LRUCache constructor signature has changed to accept an std::optional<size_type> for its max size, with std::nullopt replacing 0 as the NO_AUTO_EVICT sentinel. The Put() method can now fail if the cache's max size is zero. (119215c)
  • ⚠️ BREAKING The public method AreAllThirdPartyCookiesBlocked() and its observer OnBlockAllThirdPartyCookiesChanged() have been removed from the TrackingProtectionSettings API. Embedders must find alternative ways to monitor this state. (3100d0f)
  • ⚠️ BREAKING The viz::ReturnedResource struct, part of the public Viz compositing API, has been modified. The gpu::SyncToken sync_token field was replaced with gpu::SharedImageExportResult export_result, requiring updates for clients interacting with the graphics stack. (f04f79f)
  • ⚠️ BREAKING The method signature for midi::MidiManagerClient::ReceiveMidiData has changed. It now accepts a base::span<const uint8_t> instead of a raw pointer and length, requiring implementation updates. (70146e2)
  • ⚠️ BREAKING The public method BrowserView::vertical_tab_strip_region_view() has been removed. Embedders must now use tab_strip_view() and perform a cast. (62bc150)
  • ⚠️ BREAKING The command-line switch kReduceUserAgentPlatformOsCpu has been removed. The reduced User-Agent string for platform and OS CPU is now the default and only behavior. (28d0394)
  • ⚠️ BREAKING The public Mojo interface storage::mojom::SessionStorageControl has been modified. Several methods no longer include a callback parameter, which will break clients using the old signature. (fe3f59a)
  • ⚠️ BREAKING On Windows, the sandbox::BrokerServices::SpawnTargetAsync method signature has changed from using const wchar_t* to std::wstring_view for path and command line arguments. (2a990dd)
  • ⚠️ BREAKING The AutofillEnableNewFopDisplayDesktop feature flag was removed, making the new UI for displaying Forms of Payment (FOPs) permanent and non-optional. (1d52844)
  • ⚠️ BREAKING The header file ui/compositor/scoped_animation_duration_scale_mode.h has been removed. Code must be updated to include ui/gfx/scoped_animation_duration_scale_mode.h instead. (417c9ad)
  • ⚠️ BREAKING The command identifier IDC_CONTENT_CONTEXT_CLOSE_GLIC has been removed from the public header chrome/app/chrome_command_ids.h, breaking integrations that reference it. (c24eca8)

APIs and Extensions

  • As part of an ongoing modernization effort, the web_accessible_resources, sharedModule, and protocolHandlers extension manifest keys and APIs have been converted from the legacy JSON schema to the modern WebIDL format. (36bc943) (bc0d14a) (390c9b5)
  • The trust checker for Isolated Web Apps was updated to correctly check against blocklists and user-install allowlists. (46ab2ce)
  • Media access permission requests (e.g., for voice search) are now correctly handled for the Omnibox WebUI popup. (ae16873)

Security and Privacy

  • Enterprise policies IncognitoModeBlocklist and IncognitoModeAllowlist have been renamed to IncognitoModeUrlBlocklist and IncognitoModeUrlAllowlist respectively, which also changes the underlying preference names. (c78a55b) (43e427d) (3539075)
  • A garbage collection mechanism was introduced to automatically delete unexportable cryptographic keys associated with obsolete or off-the-record user profiles, improving security hygiene. (4528d34) (a6e868f)
  • The Geolocation API on macOS now uses ellipsoidalAltitude instead of altitude to better align with the WGS84 ellipsoid standard. This will result in different altitude data being reported. (b5ab370)
  • A privacy enhancement was introduced to redact password fields from screenshots by leveraging geometry data from the page content extraction service. (08603f6)
  • A bug in the WebAuthn "Change PIN" flow that could lead to a crash has been resolved. (a93fbc7)
  • The File System Access API will now be more performant for .json and .md files, as they have been added to the list of file types considered safe by Safe Browsing. (783acab)

Browser UI and Features

  • The browser theme color will now update to match the tab group color when entering Tab Group Focus Mode. (dd65203)
  • The password save dialog has been updated to a wider, horizontal layout, with experimental vertical layout support being removed. (726a03a)
  • In narrow browser windows, the Tab Hover State Preview will now be rendered below the toolbar to avoid UI crowding. (3066834)
  • The layout manager for vertical tabs now supports animations for newly added tabs, improving visual feedback. (8878325)
  • The "Save and Fill" credit card dialog was refactored to provide a smoother UI transition without a jarring "pop" effect. (9ba434d)
  • A bug was fixed where the bookmark page action icon would incorrectly appear on the New Tab Page. (dfd5c85)
  • The Omnibox WebUI popup will now correctly hide when the active tab is changed. (929b694)
  • A new feature flag, InfobarPrioritization, was added to about:flags to allow testing of new infobar ranking logic. (fb9290d)

Core Utilities and Infrastructure

  • Multiple components, including sync and safe browsing, have been updated to use SimpleURLLoader::DownloadHeadersOnly for network requests where the response body is not needed, reducing memory usage. (24885fb) (dda8db0) (fdc6928)
  • As a test of C++23 feature adoption, base::to_underlying has been aliased to the standard library's std::to_underlying. (4b06d6f)
  • The ContactInfoSyncBridge now commits metadata and model changes atomically to the database, improving data consistency. (8750b89)
  • A new DeviceStatisticsRequest class has been introduced to the sync service to retrieve data about an account's syncing devices. (d5618b4) (55d130c)
  • On Windows, the RunDeElevated utility will now launch processes in the foreground, making its behavior consistent with elevated launches. (1d0efe4)
  • The CompositorAnimationRunner has been optimized to remove an unnecessary frame rate limit, which could cause dropped frames in UI animations. (457ef2d)

Testing

  • New interactive UI tests were added for the vertical tab dragging feature. (82bda9b)
  • The WebAppIntegrationTestDriver was fixed to resolve test flakiness related to intent picker state. (e7ba886)
  • Several flaky tests were fixed or re-enabled across PWAs, Safe Browsing, and general browser test suites. (8d3f41e) (b4f00d8) (bcfbe5a)
