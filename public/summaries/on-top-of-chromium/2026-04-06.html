Chromium Digest: 2026-04-06

Overview

On April 6th, 2026, 215 of 282 total commits were identified as relevant for developers building on Chromium for desktop platforms. This summary covers changes from commit (95b6d18) to (0c355e9), highlighting two significant breaking changes in the compositor and accessibility APIs, the introduction of a new TabStrip API, and notable updates to networking and security.

Breaking Changes

  • ⚠️ BREAKING BREAKING: The public cc::LayerTreeHostClient interface has been changed. The virtual method WillBeginImplCommit was removed, and its logic merged into WillCommit. Embedders with custom LayerTreeHostClient implementations must remove the WillBeginImplCommit override to avoid build failures. The cc::TextureLayer::Create factory method signature was also updated. (7010b87)
  • ⚠️ BREAKING BREAKING: A stability check has been promoted from a debug-only DCHECK to a production CHECK on Windows to prevent re-entrant calls to the public BrowserAccessibilityManager::OnAccessibilityEvents method. Embedders whose code triggers this condition will now experience a production crash, hardening the API but requiring potential code updates. (c52ae89)

Browser & UI

  • A new TabStrip API is being introduced to allow programmatic updates to the tab strip. Initial commits add the service definition and an Update method, starting with support for updating a TabGroup's visual data. (ea91004) (4a588dd)
  • A race condition causing a dangling pointer in BackgroundModeManager was fixed. The fix ensures old status icon menus are kept alive until the context menu update is fully complete, preventing crashes. (620a047)

Networking & Security

  • Detailed DNS resolution information (e.g., source of result like cache, DoH, or system resolver) is now propagated through the network stack. This data is available in LoadTimingInternalInfo and can be used for more granular performance analysis. (2e629dc) (3c5e3ba) (dd8d63c) (7288496)
  • A new feature flag, kIgnoreCorsPreflightPolicy, was added to the Network Service. When enabled, this flag will ignore the kPreventPreflight policy, moving towards its deprecation. (55b0c31)
  • A security fix prevents a potential user activation bypass. System-triggered media suspensions, such as audio focus loss, no longer grant a transient user activation token to the frame. (55823cd)
  • The experimental ServiceWorkerAutoPreload feature for subresources has been removed. This includes the removal of the kAutoPreload enum value from the public blink.mojom.ServiceWorkerFetchHandlerBypassOption interface. (2bb2c50)
  • Bug fixes for Service Workers were landed, addressing a potential use-after-free in ServiceWorkerVersion and a bug that caused requests to stall when a fetch event was restarted. (405dcf0) (bc87e02)
  • The list of bulk HSTS (HTTP Strict Transport Security) preloaded entries has been updated. (baeca77)

Extensions

  • Several internal enterprise extension APIs, including enterprise.platformKeys and enterprise.networkingAttributes, were migrated from JSON schema to WebIDL as part of an ongoing refactoring. This is not expected to impact extension behavior. (19c7e7f) (f610734) (3440252)
  • The EnterpriseExtensionDOMActivityTelemetry feature, which collects DOM activity for enterprise-installed extensions, has been enabled by default. Its runtime state is now tracked via a synthetic field trial for performance analysis. (1775f9a) (00f7844)

Testing & Infrastructure

  • A new InteractionTestUtilSimulatorWebUI was implemented to allow automated tests to interact with tracked WebUI elements using the same APIs as for Views and WebContents, improving testability of WebUI components. (3609c24)
  • As part of the ongoing modularization of //chrome/browser, source files related to downloads were moved into a new //chrome/browser/download component, breaking a dependency cycle. (38b45dc) (5adc86a)
  • A large-scale code health effort fixed "Include What You Use" (IWYU) violations related to base/logging.h across many files, improving build hygiene. (3be1f60)
