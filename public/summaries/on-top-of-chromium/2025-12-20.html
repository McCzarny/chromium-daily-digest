Chromium Digest: 2025-12-20

Overview

This digest covers 39 commits, with a focus on 13 changes relevant to browser APIs, extensions, and core utilities on desktop platforms. Key updates include two breaking changes to the Extensions and Permissions APIs, requiring embedder attention. Other changes involve internal refactoring of navigation and translation components, UI icon updates, and improvements to build tooling and testing infrastructure. Extensions

  • ⚠️ BREAKING BREAKING: The const qualifier has been removed from several virtual methods in ExtensionFunctionDispatcher::Delegate, including GetExtensionWindowController() and GetAssociatedWebContents(). This API signature change will cause build failures for any embedder subclassing the delegate and implementing the methods with const. (d6c7658)
  • Internal logic for setting a tab's opener has been consolidated to share code between the tabs.create() and tabs.update() API implementations. This is a non-functional refactoring to reduce duplication. (64505fc)
  • Documentation within extension_function_dispatcher.h has been updated to replace the deprecated RenderViewHost (RVH) acronym with the modern RenderFrameHost (RFH). (4c32fca)

Permissions

  • ⚠️ BREAKING BREAKING: The pure virtual function GetTrackingProtectionSettings has been removed from the public permissions::PermissionsClient interface. While unused within Chromium, this will break builds for any embedder that provides a custom implementation of this client interface. (25d5686)

Browser Internals & Refactoring

  • The on-device translation feature was refactored to centralize dependencies and constants into components/on_device_translation, improving modularity and reducing direct coupling to components like host_content_settings_map_factory. (fa60b0c) (4e1dd4b)
  • The logic for calculating back/forward navigations that skip history entries (e.g., ad navigations) has been refactored within NavigationControllerImpl. This internal change makes the calculation more granular in preparation for future features. (e6624c2)

UI

  • A new v8_off.icon is now used in the omnibox in branded builds to indicate when V8 optimization is disabled. (dfd876c) (0d27f04)
  • A minor UI text change was made to remove a period from the 'unsupported currency' error dialog. (6f5f39c)

Build & Tooling

  • The proto_extras build system now allows developers to specify custom serialization logic for individual protobuf messages, offering greater flexibility than the default behavior. (3028c0b)
  • Legacy tracing macros (TRACE_EVENT_FLOW) have been migrated to the modern TRACE_EVENT syntax in ui/events as part of an ongoing code health effort. (a0c1ce5)

Testing

  • Unit tests for the media::AudioBus class have been templatized using TYPED_TEST_SUITE. This code health improvement reduces code duplication and simplifies the process of adding tests for new audio sample formats. (3b2d3af)
