Chromium Digest: 2026-05-13

Overview

On May 13, 2026, the Chromium main branch saw 682 commits, with 499 identified as relevant to developers. This summary highlights major API changes, security enhancements, and performance improvements. The day's activity started with commit 5b6de9caa4cd00624e25a4dea1a7adcf9eea2e70 and concluded with 010fba64a502640c492a2d53d47613cbe5881ae6.

Breaking Changes

  • ⚠️The <webview> tag and chrome.webviewTag API have been removed from Chrome Apps on Windows, Mac, and Linux. This feature is now restricted to ChromeOS only, breaking a stable and widely-used API on desktop platforms. (dfa28c4)
    ExtensionsBrowser
  • ⚠️Several PartitionAlloc headers have been moved into an internal/ subdirectory. This is a public API change that will break any build that directly includes headers like partition_page.h or partition_root.h from their old location. (cd836a8)
    BaseBuild
  • ⚠️The core compositor class cc::LayerTreeMutatorClient has been renamed to cc::LayerTreeMutatorDelegate, and its SetClient method is now SetDelegate. This will break embedders who implement or use this animation interface. (34492d7)
    CompositorBlink
  • ⚠️The public SendTabToSelfModelObserver C++ interface has been significantly changed. All method names now require an On prefix (e.g., EntriesAddedRemotely is now OnEntriesAddedRemotely), and the OnSendTabToSelfModelLoaded method has been removed entirely, breaking all existing observer implementations. (35d38a6) (2ca3334)
    SyncBrowser
  • ⚠️The remoting::AudioInjector C++ interface was refactored. It no longer inherits from protocol::AudioStub, and the InjectAudioPacket() method was replaced by a new pure virtual method SetSampleInfo(). Custom audio injection implementations will need to be updated. (5259bac)
    MediaWebRTC
  • ⚠️On Android, the public Java interface ChromeAndroidTaskFeature has changed. The onAddedToTask(long) method was removed and onAddedToTask(InitInfo) is now a mandatory method, which will cause compilation errors for any class implementing this interface. (5af4b45)
    Android
  • ⚠️The PasswordStoreBackend::InitBackend method signature has been changed, removing the AffiliatedMatchHelper* parameter. This will break any custom PasswordStoreBackend implementations. (4fdbebe)
    PasswordManagerStorage
  • ⚠️The public class AccessibilityAnnotationSyncBridge and its usages within SyncServiceFactory have been completely removed. Embedders using this sync data type will be broken. (bc57ea3)
    SyncAccessibility
  • ⚠️On ChromeOS, the GetPolicies method has been removed from the crosapi::mojom::LocalPrinter interface, breaking clients that use it to fetch printing policies. (28102da)
    ChromeOSPrinting

Security & Privacy

  • The PartitionAllocWithAdvancedChecks (MiracleObject) memory safety feature has been enabled by default for the Browser UI process, helping to detect and prevent Use-After-Free vulnerabilities. (34c2ad4)
    SecurityPerformanceBase
  • A site isolation bypass in the Web Serial API for non-default StoragePartitions (e.g., in guest views) has been fixed by blocking access for HTTP/HTTPS origins in those partitions. (16e3c60)
    SecurityBlinkStorage
  • The Windows network service sandbox is now more robust. It brokers all connections to non-publicly-routable addresses to prevent the OS firewall from blocking legitimate local network traffic from sources like VPNs or virtual machine NICs. (3129fd2)
    SecurityWindowsNetwork
  • A privacy leak where cross-origin images inside SVG <mask> elements could be rendered to a <canvas> without tainting it has been fixed. (4855238)
    BlinkSecurityGPU
  • A Use-After-Free crash in DeepScanningRequest has been fixed. The fix prevents synchronous cancellation of a scan request from prematurely deleting the request object while it is still in use. (64a9675)
    SecurityDownloads

Web Platform & Performance

  • The async clipboard read() and readText() APIs were updated to use non-blocking Mojo IPCs in the renderer. This prevents the browser UI from freezing when pasting content from applications that are slow to render it. (a274f36)
    BlinkPerformanceUI
  • The border shorthand property was fixed to correctly reset border-image to its initial value, aligning with CSS specifications and improving web compatibility. (3318b59)
    BlinkCSS
  • Layout performance has been improved by caching SkStrike objects per font, reducing expensive digest computations. This provides a notable benefit for pages with complex text, especially CJK characters. (51e12ce)
    BlinkPerformanceFonts
  • The CSS shape-outside property now supports interpolation when both the start and end shapes share the same <shape-box>, enabling smoother animations and transitions. (11370e4)
    BlinkCSS
  • A renderer crash in the File System Access API has been fixed. The crash occurred when writing a WebAssembly-backed buffer larger than INT_MAX bytes to a SyncAccessHandle. (8965c32)
    BlinkStorageWebAssembly

Android & iOS

  • ⚠️On Android, the TabGroupModelFilterObserver::didCreateNewGroup method signature was changed from (Tab, TabGroupModelFilter) to (Tab, TabModel). Embedders using this observer will need to update their implementations. (2c5f60a)
    AndroidTabs
  • ⚠️On Android, the org.chromium.base.ObserverList Java class now restricts direct access to its mObservers field, marking it @VisibleForTesting. This breaks any production code that was directly accessing this list. (dc6bac0)
    AndroidBase
  • On Android, Chrome will now terminate its own process upon a system locale change. This is a proactive measure to prevent crashes from locale mismatches on the next startup. (1e3c007)
    Android
  • ⚠️On iOS, the public OverlayUIDidFinishDismissal method in the OverlayRequestCoordinatorDelegate interface now passes a safe OverlayRequestId instead of a raw OverlayRequest*, breaking implementations but preventing a class of Use-After-Free bugs. (c3eb345)
    iOSUI

Desktop & Core Internals

  • AppArmor profiles are now included for all Chrome/Chromium Linux installer packages (.deb, .rpm). This fixes sandbox and crash reporting issues on newer Linux distributions like Ubuntu 24.04. (d4c2b79)
    InfraLinuxSecurity
  • The public C++ API on FormFieldData for the experimental Email Verification feature has been changed. The nonce()/set_nonce() methods have been renamed to challenge()/set_challenge(). (0846112)
    AutofillBlink
  • The app_shell binary and its related code, such as media_capture_util, have been fully deprecated and removed from the extensions system. (e2a0758) (937f006)
    ExtensionsBuild
