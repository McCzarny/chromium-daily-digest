Chromium Digest: 2026-04-22

Overview

This daily summary for the main branch on 2026-04-22 covers 649 commits, with 479 identified as relevant to developers. The changes span from the first commit d80eb100687de7caeb714de06922e98b1949b68d to the last d8b766aca6cf560a969aa30424d6f1745b9acd65. This period includes a large number of breaking changes due to significant API and build system refactoring.

Breaking Changes

  • ⚠️ BREAKING The long-standing BrowserList API (browser_list.h) has been completely removed. The public API surface of browser_finder.h was also reduced with the removal of FindBrowserWithID, FindBrowserWithWindow, FindBrowserWithActiveWindow, and FindBrowserWithUiElementContext. Embedders must migrate to newer mechanisms like BrowserWindowInterfaceIterator. (499cfae) (ba141b8) (af10f4e)
  • ⚠️ BREAKING A major build system refactoring moved browser and application lifetime headers (e.g., application_lifetime.h, browser_shutdown.h) from the monolithic //chrome/browser target to a new //chrome/browser/lifetime target. Embedders must explicitly add a dependency on the new target. (94c8534)
  • ⚠️ BREAKING The Web Feed feature has been completely removed. This deletes numerous C++ classes, Java bridges (WebFeedBridge), feature flags (kWebFeedAwareness), and public API methods like FeedActionDelegate.openWebFeed(). (c292b11) (1a47ae7)
  • ⚠️ BREAKING The pure virtual method GetURLLoaderFactory() has been removed from the public Profile interface and moved to its parent content::BrowserContext class. Code calling this method on a Profile object will no longer compile. (5436f47)
  • ⚠️ BREAKING Several public Java methods have been removed from the Sync and Signin services on Android, including SyncService.isSyncFeatureEnabled(), SyncService.hasSyncConsent(), and SigninManager.revokeSyncConsent(). (f42b9fc) (0639353) (960d5db)
  • ⚠️ BREAKING The Autofill "Plus Address" feature has been removed. This includes the removal of GetPlusAddressDelegate() from the AutofillClient interface and related SuggestionType enum values. (ff574ae) (d65a964)
  • ⚠️ BREAKING The default behavior of ContentBrowserClient::IsFullCookieAccessAllowed now returns false for storage keys created with a nonce, restricting cookie access in sandboxed or isolated contexts. This may affect embedder cookie handling. (7042a41)
  • ⚠️ BREAKING The signature for WebAppInstallationAcceptanceCallback has changed to support decoupling PWA installation from its launch. Callbacks now receive an additional closure to trigger the post-install launch action. (2a13b24)
  • ⚠️ BREAKING The public constructor for TabMenuModel now requires a TabStripModel* parameter, breaking existing instantiations. (37ca7f3)
  • ⚠️ BREAKING In Blink, the PathBuilder::CurrentPath() method now returns a Path by value instead of a const Path& to prevent use-after-free bugs. This is a public API signature change. (f3b97ba)
  • ⚠️ BREAKING On Android, the default HTTP status code for navigations without response headers has changed from 200 to 0. This affects the NavigationHandleObserver interface and may impact error handling logic. (ca8ad98)
  • ⚠️ BREAKING The JNI generation tooling no longer allows parameters for the @CalledByNative Java annotation. All instances of @CalledByNative("...") must be updated to @CalledByNative. (30d45a3)
  • ⚠️ BREAKING The google_toolbox_for_mac third-party dependency has been removed. Embedders that transitively depended on this library will experience build failures. (bcd5847)
  • ⚠️ BREAKING The public API of AimEligibilityService was altered by removing the virtual method GetCountryCode(). Subclasses that override this method will no longer compile. (6ce8693)
  • ⚠️ BREAKING Public methods for showing "send tab to self" bubbles have been removed from the BrowserWindow interface. This functionality is now exposed through a dedicated BrowserWindowFeature. (ab08725)
  • ⚠️ BREAKING The build_tflite_with_nnapi build flag has been removed. Embedders relying on this flag to enable the TFLite NNAPI delegate will have their builds broken. (6caa8b6)

Security

  • DevTools has been hardened by disallowing devtools:// access via Target.createTarget for untrusted clients and restricting file:// resource loading from remote frontends. A new --allow-unsafe-devtools-remote-file-loading flag is available as an escape hatch. (28c26ea) (8e5202f)
  • A Time-of-Check to Time-of-Use (TOCTOU) vulnerability in DataObjectItem::GetAsFile was fixed by adding sequence number validation for clipboard items. (e59c8af)
  • A vulnerability where third-party SharedWorkers could bypass SameSite cookie restrictions when creating WebSocket connections has been fixed. (2b49183)
  • Security for Mojo IPC was hardened on Windows by improving remote process ID detection and adding an option to verify the server process's integrity level to prevent named pipe squatting attacks. (0e083d3)
  • A Use-After-Free in WebViewImpl::Minimize was fixed by using a weak pointer for a delayed task, and another in Page::OrdinaryPages() was fixed by copying the collection before iteration. (b1a2341) (7428099)
  • A Server-Side Request Forgery (SSRF) vulnerability in Cast was fixed by adding URL validation for media image URLs received from Cast devices. (16b9265)

Blink & Web Platform

  • The MergeFixedLayers and MergeStickyLayers rendering optimizations are now enabled by default to improve performance. (d80eb10)
  • The initiatorUrl for ResourceTiming is now enabled for dedicated and shared workers, providing more granular attribution for fetched resources. (dc78e7e)
  • A bug in AudioBufferSourceNode was fixed to correctly handle cases where start() is called without a source buffer attached, aligning with the Web Audio API specification. (ab5a5fd)
  • SVG filters using userSpaceOnUse will now correctly resolve percentage values against the viewport of the element where the filter is used, not where it was defined. (de5c226)
  • The <selectedcontent> element was updated to support <select multiple>, allowing it to clone and display content from all selected options. (3606758)
  • Video captions will now dynamically update when OS caption settings change on Windows, improving accessibility. (f593405)

Progressive Web Apps (PWA)

  • A graceful recovery flow was implemented for when the Web App Database is corrupted or fails to open. Instead of crashing, the browser will now attempt to salvage data, show a profile error dialog, and reset the PWA system. (6ffaab9)

Desktop & UI

  • The UI for the "Tabs from Other Devices" side panel has been implemented, showing a device picker and a list of tabs. (4f374b7)
  • Support for context menus on pinned action toolbar buttons was implemented as part of the WebUI toolbar migration. (1b3a82e)
  • A significant internal refactoring moved the WebContentsModalDialogManager delegate logic out of the Browser class and into a new BrowserWindowModalDialogDelegate. (083c2a8)
  • The chrome://policy page has been modernized by porting its promotion banner and export logic from legacy WebUI messages to Mojo. (e3cc2f5)

Android

  • The TranslatePrefs class was updated to store a list of recent target languages instead of a single one, in preparation for a richer language selection UI. (c691383)
  • Reload support was implemented for the native PDF page viewer, guarded by the InlinePdfV2 feature flag. (5dd34c7)
  • The media notification icon was changed from a sound icon to the Chrome icon to avoid user confusion. (9ab824f)
  • The list of supported media codecs for the MediaLauncherActivity was explicitly enumerated to prevent attempts to open unsupported file types. (17d2fdf)

iOS

  • Enterprise policy checks were implemented for passkey creation, ensuring compliance with PasswordManagerEnabled and other related policies. (dbab372)
  • Logic has been implemented to protect the Tab Grid from screenshots based on enterprise policies. (1ebf897)
  • The AuthenticationService API was updated to add new methods that do not require a ConsentLevel argument, as part of a larger cleanup effort. (e6b6aed)
  • A race condition was fixed in the Payment Methods UI that could cause unresponsiveness during rapid tapping. (24a6fcf)

Internals & Code Health

  • A new helper, base::WeakValueTable, was introduced for managing thread-safe shared caches of weakly-owned values. (dd21b7b)
  • A new public key encryption API (crypto::encrypt::Encrypt/Decrypt) was added to the crypto library. (34f8ea3)
  • A new helper function, ScaleByMemoryLimit, was introduced in base/memory_coordinator to provide a safe way to scale memory cache sizes based on system memory limits. (64428d6)
  • The media app accessibility code was extracted from the monolithic chrome/browser target into its own source_set to improve build times and dependency management. (cf9b018)
  • A large number of files were updated to migrate comments referencing old crbug.com issue IDs to the new issue tracker format. (abac920)
