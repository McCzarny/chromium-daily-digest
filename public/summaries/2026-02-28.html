Chromium Digest: 2026-02-28

Overview

This digest covers 60 commits, from (da64ff6) to (c1640e1). This week features several significant API refactorings affecting UI components and services, including breaking changes to the Android BottomSheet controller, the WebUI toolbar's mojom API, and Chrome Remote Desktop's configuration on Linux. Other notable changes include security hardening for sandboxed popups and continued development of experimental web platform features.

UI & UX

  • ⚠️ BREAKING BREAKING: The public Java interface ManagedBottomSheetController on Android has been updated. The method setBottomControlsHeight(int) was renamed to setBottomControlsOffset(int). Embedders using this component will need to update their method calls to avoid build failures. (2f25ade)
  • ⚠️ BREAKING BREAKING: The browser_controls_api.mojom interface has been refactored. Several UI-specific methods and observers, including showContextMenu, onPageInitialized, and onNavigationControlsStateChanged, have been moved to a new, dedicated toolbar_ui_api.mojom interface. Clients of the old mojom API will need to bind to the new ToolbarUIService for this functionality. (06faf85)
  • The user interaction for site search mode on Android has been improved, allowing users to exit the mode by pressing backspace in an empty omnibox. (80d11ba)
  • A bug was fixed to prevent collapsed tab groups from expanding during a drag-and-drop operation. (533d98e)
  • Various internal refactorings and bug fixes were made to Android toolbars, Omnibox, and the History page UI. (0269280) (83618f7) (812bfaa)

Services & Components

  • ⚠️ BREAKING BREAKING: In the Chrome Remote Desktop host component, the public function GetConfigDirectoryPath() has been removed from remoting/base/file_path_util_linux.h. It is replaced by a new GetConfigDir() function and a different configuration directory structure on Linux. Downstream projects building against CRD host libraries will need to adapt to the new functions and file layout. (6d6ef8e)
  • The update_client component now cancels pending tasks at shutdown to improve reliability. (9d2e5ec)
  • Internal improvements were made to SessionStorageImpl to better manage its database connection state, preventing operations before the database is fully open. (21f02c2)

Security

  • Security has been hardened by adding browser-process validation for the allow-popups sandbox attribute. Previously, this was only checked in the renderer, and this change adds a second layer of defense against sandboxed frames incorrectly creating popups. (a12e651)

Web Platform

  • A bug in form constraint validation has been fixed. readonly <button> and <select> elements were incorrectly barred from validation, which was against the HTML specification. This change corrects the behavior to align with the spec. (64e3f4b)
  • Work continues on experimental features, including the implementation of click interaction for the <login> element to dispatch a FedCM request, and support for single-axis scroll containers for sticky positioning (behind a flag). (c297fdf) (4286ec2)
  • A potential stack overflow in the compositor has been fixed by adding cycle detection during the calculation of AnchorPositionOffset for position: anchor elements. (d3bdf5f)

Extensions

  • The content_settings API has been ported to desktop Android, enabling extensions on that platform to manage content settings. (4290a34)
  • The contentScripts API schema was converted from the legacy JSON format to the modern WebIDL format as part of ongoing modernization efforts. (c1640e1)
  • A new DOMActivityLogger has been integrated to support enterprise telemetry for extension activity, gated by the kEnterpriseExtensionDOMActivityTelemetry feature flag. (78d9afe)
