Summary for 2026-05-11 on branch 'main'

Chromium Digest: 2026-05-11

Overview

This daily summary for the Chromium main branch on May 11, 2026, covers 596 commits, with 456 identified as relevant to developers. The changes span from the first commit b458747572a9 to the last 6ab45a706ac3 and include a significant number of breaking changes across the build system, networking, and various public C++ and Mojom APIs.

Breaking Changes

  • ⚠️The monolithic //chrome/browser build target has been broken into many smaller source_set targets. Embedders must update their BUILD.gn files to depend on the new, more specific targets instead of the old umbrella target. (a235e83)
    BuildBrowser
  • ⚠️Multiple core networking functions in net/socket and net/http have been changed to accept a target_network parameter. This is a widespread public API change affecting ConnectJobFactory::CreateConnectJob, HttpProxySocketParams, and others. (6f90916)
    Network
  • ⚠️Two new pure virtual methods, SurfaceEmbedChildWebContentsAttached and SurfaceEmbedChildWebContentsDetached, have been added to the public observer interface content::WebContentsObserver. Subclasses must be updated. (4a7f0a4)
    UIPerformance
  • ⚠️A new pure virtual function GetWebContentsFromWindowIfCaptureHandleAllowed has been added to the public interface content::ContentBrowserClient. Embedders subclassing this client must implement the new method. (3221e4a)
    MediaUISecurity
  • ⚠️The public media::AudioHandler interface now includes a new pure virtual function, CopyPartialFramesTo. All subclasses must implement this function to compile. (67d7e23)
    Media
  • ⚠️The signature of the virtual method MaybeReportPasswordReuseDetected in the public class PasswordProtectionServiceBase has changed to include a const ReferrerChain& parameter. Overriding subclasses will fail to compile. (3b5907c)
    PasswordManagerSecurityEnterprise
  • ⚠️The string representation of the public API content::DesktopMediaID has changed to support native picker sessions. Code that manually parses or generates these ID strings will break. (045d84e)
    MediaWebRTC
  • ⚠️The signature of the public mojom interface device::mojom::XRFrameDataProvider::SubmitFrameDrawnIntoTexture has changed to include a new camera_sync_tokens parameter. (02a16b8)
    XRMedia
  • ⚠️The return type of the public Web Crypto API SubtleCrypto.verify() has been changed from Promise<any> to Promise<boolean> to align with the specification. Web applications relying on the previous non-standard behavior may break. (14ff7bb)
    BlinkCryptoSecurity
  • ⚠️The long-deprecated app_shell binary and its associated sources have been removed from extensions/shell. Any remaining build dependencies on this target will break. (75cd2c6)
    ExtensionsBuild
  • ⚠️The signature of the public mojom interface network::mojom::RestrictedCookieManager::SetCookieFromString has changed, removing the get_version_shared_memory parameter and CookiesResponse return value. (05514ce)
    NetworkPrivacy
  • ⚠️Media parsing has been made stricter to reject MP4 files with duplicate track IDs, per specification. Previously playable but non-compliant files will now fail to load. (63afd1b)
    MediaBlink
  • ⚠️The GetOemData method has been removed from the crosapi.mojom.TelemetryProbeService interface. Clients on ChromeOS must update to use DebugDaemonClient::GetLog instead. (c89c1be)
    ChromeOS
  • ⚠️The public constructor for TextBubble.Builder on Android has been changed, removing the isAccessibilityEnabled parameter. Callers must update to use the new signature. (121ecaa)
    AndroidUI
  • ⚠️The signature of the virtual method ContentSettingsProvider::SetWebsiteSetting has been changed from taking base::Value&& to const base::Value&. Custom provider implementations must be updated. (41de479)
    PrivacyBlink
  • ⚠️The type of the public member encoded_body_size in net::HttpResponseInfo has been changed from std::optional<int64_t> to std::optional<base::ByteSize>, which is a source and binary breaking change. (ec45984)
    Network

Security

  • A TOCTOU vulnerability in the WebUSB API was fixed by locking interface requests during pending device state changes. (7e97805)
    SecurityUSBBlink
  • Input event routing was hardened by adding validation for the FrameSinkId in async hit-test responses, preventing a compromised renderer from redirecting input to arbitrary cross-origin frames. (1bdeadd)
    SecurityUICompositor
  • A use-after-unmap vulnerability in the Web Serial API on Windows was resolved by ensuring data pipe handles are kept alive until all pending I/O completes. (c1ee3a3)
    SecurityBlinkWindows
  • On-demand spell check requests are now restricted to run only after explicit user interactions with a focused element, mitigating potential user dictionary leak attacks. (46881f7)
    SecurityUIPrivacy
  • A vulnerability was fixed where GuestViews could bypass WebHID enterprise policies, allowing a malicious host to hijack HID access from an embedded origin. (63d25d3)
    SecurityExtensions
  • A potential information leak in the Gamepad API, where browser stack data could be copied into shared memory, has been fixed. (f7b517d)
    SecurityBlinkMedia

Build & Infrastructure

  • The enable_swift_cxx_interop flag was removed from iOS build configurations as the feature is no longer being actively pursued. (1ad2be3)
    BuildiOS
  • A change to roll out the faster Apple linker for non-official macOS builds was landed and then reverted due to build breakages. (a086c9b) (df36205)
    BuildmacOS
  • Dependencies for the libafl fuzzer were added for testing purposes. (03b7245) (f219890) (e8fac8e)
    BuildTestingSecurity

Networking

  • A 'maximum priority' mode was added for WebSockets, intended for internal browser use cases, which uses MAXIMUM_PRIORITY and LOAD_IGNORE_LIMITS flags. (d09d6a6)
    Network
  • A potential crash in SpdyStream was resolved by deferring stream destruction during a session drain, changing an internal synchronous operation to asynchronous. (eb721a8)
    Network
  • A bug fix in SSPI authentication on Windows restricts delegation flags to the Negotiate scheme, preventing a crash in NTLM authentication. (0996e94)
    NetworkWindowsSecurity

Blink & Rendering

  • Support was added for ruby-overhang: spaces, allowing ruby annotations to overhang adjacent whitespace. (d935f57)
    BlinkCSS
  • The paint implementation for text-decoration-skip-spaces was added. (7c50cb6)
    BlinkCSS
  • A crash was fixed in typed arithmetic calc() expressions involving multiplication of non-number types. (57c87d4)
    BlinkCSS
  • ⚠️The cc::LayerTreeHost::SetNeedsAnimate signature was reverted. This is a breaking change for any code that had adapted to the new signature introduced in a very recent commit. (c6623ae)
    CompositorPerformance

Performance & Memory

  • The compositor's ResourcePool will now discard all resources when the viewport is resized, significantly reducing memory spikes during window resizing. (d6a5cb9)
    PerformanceCompositor
  • A new base::MultiMemoryConsumer class was introduced to allow a single class to manage multiple granular memory interventions, as part of a migration away from MemoryPressureListener. (a5ea88b) (61dfb28)
    PerformanceBase
  • ⚠️The non_main_world_entries() method was removed from the public class V8DetailedMemoryProcessData via a revert, breaking any code that had adopted the new API. (4c257ac)
    V8Performance

Password Manager & Autofill

  • Multiple components, including SavedPasswordsPresenter and PasswordReuseDetector, were refactored to use the new StoredCredential struct instead of the legacy PasswordForm. (25838de) (e32cf6b)
    PasswordManagerAutofill
  • Several expired WebAuthn feature flags, including kWebAuthnSignalApiHidePasskeys and kPasskeyUnlockManager, were cleaned up from the codebase. (8c48777) (89f764e) (1f4bde8)
    PasswordManagerBlink

Accessibility (A11y)

  • A major internal refactoring on Windows removed custom ATL dependencies from ui/accessibility, replacing them with modern WRL and hand-written COM implementations. Public COM interface compatibility should be unaffected. (54afab9)
    AccessibilityWindows
  • Strict validation was added to AXTree updates to reject invalid node IDs, preventing potential crashes and tree corruption. (50d789f)
    AccessibilityBlink

Feature Removals & Reverts

  • A behavioral change that made the bookmark bar visible on custom New Tab Pages was reverted due to user feedback. (01e71df)
    UIBookmarks
  • A change removing Mac power observers for CADisplayLink was reverted as it caused unresponsiveness after sleep/wake cycles. (3dc3598)
    macOSPerformanceCompositor
  • The kClientSideDetectionDeprecateDOMModel feature flag and related code for an old client-side phishing detection model have been removed. (38d0d2f)
    SecurityPrivacy
