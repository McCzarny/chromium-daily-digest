Chromium Digest: 2026-04-02

Overview

This daily summary for the main branch on April 2, 2026 covers 699 commits, with 522 identified as relevant to developers. This was a particularly active day for API changes, introducing numerous breaking changes across core components, Blink, services, and the DevTools protocol. Key feature work includes enabling lazy loading for media by default and continued refactoring for the Bedrock UI abstraction project. The first and last commits of this period are (6a50027) and (8c20471) respectively.

Core, UI & Networking

  • ⚠️ BREAKING BREAKING: The constructor for base::Thread has been changed to inject its delegate directly, removing the simple Thread(const std::string& name) overload. Code must be updated to use new overloads, for example by providing a Thread::Delegate or using Thread::Restartable{} for restartable threads. (7d91034)
  • ⚠️ BREAKING BREAKING: As part of the Bedrock UI abstraction project, the public function chrome::OpenEmptyWindow now returns a BrowserWindowInterface* instead of a Browser*. Callers must be updated to use the new interface type. (4357394)
  • ⚠️ BREAKING BREAKING: The public method WebView::SetCrashedOverlayView has been removed from ui/views/controls/webview/webview.h. Embedders must find an alternative approach to provide a custom view for crashed web contents. (b5d130c)
  • The NetworkAsyncRetryOnTooManyConnectionErrors feature is now enabled by default, which should improve network reliability by retrying certain connection errors asynchronously. (44348b2)
  • A new GN argument, partition_alloc_enable_system_free_fallback, has been introduced. This allows embedders to enable a fallback to the system free() for pointers not allocated by PartitionAlloc, which can address crashes in third-party vendor libraries on Android. (ed8ed77)
  • The global ServiceProcessHost::Observer is being migrated to a per-instance callback model via a new WithObserver option. This allows clients to register for notifications from a specific service instance, avoiding the need to filter global notifications. (ca7b044)

Blink & Web Platform

  • ⚠️ BREAKING BREAKING: To fix a user activation bypass vulnerability, a new pure virtual method UnlockBackgroundPlayback() has been added to the public WebMediaPlayer interface. Embedders with custom media player implementations must add this method to their classes. (75237fb)
  • ⚠️ BREAKING BREAKING: The core public Blink API method blink::WebString::FromUTF16() has been renamed to FromUtf16() to conform to style guides. Embedder code calling this utility must be updated. (cf3132f)
  • The loading="lazy" attribute is now enabled by default for <video> and <audio> elements, deferring the loading of offscreen media to improve initial page load performance. (8744f9e)
  • SharedWorker is now enabled by default on Android, bringing platform parity for this feature. (8bcc568)
  • To align with a W3C SVG Working Group resolution, CSS width and height properties are no longer honored as presentation attributes on <use> elements. Sizing will now fall back to attribute values, which may change the rendering of some SVGs. (642867f)
  • The HTML parser has been fixed to no longer treat custom elements as 'formatting elements', which could change parsing behavior for malformed HTML containing custom elements, bringing it closer to spec compliance. (8d8417c)
  • The MediaCapabilities.decodingInfo() API is being updated to consider the encryptionScheme from the keySystemConfiguration. This change, behind the KeySystemTrackConfigurationEncryptionScheme flag, may cause queries that previously returned 'Supported' to now correctly return 'Not Supported' for certain encrypted media. (9c2ad85)

Services & Components

  • ⚠️ BREAKING BREAKING: The signature for SyncUserSettings::SetInitialSyncFeatureSetupComplete() in components/sync/service/sync_user_settings.h has changed, removing the SyncFirstSetupCompleteSource parameter. (448b9a0)
  • ⚠️ BREAKING BREAKING: The observer method OnAccountSettingDataUpdated in components/account_settings/account_setting_service.h has changed its signature to report setting updates individually rather than as a collection. (2af3fe1)
  • ⚠️ BREAKING BREAKING: Numerous public OAuth2 scope constants (e.g., kOAuth1LoginScope) have been removed from google_apis/gaia/gaia_constants.h and made internal. Embedders using these constants for authentication flows will be broken. (4fd8668)
  • ⚠️ BREAKING BREAKING: The signature of the utility function omnibox::AdjustTextForCopy has changed. It no longer accepts an OmniboxClient* and instead requires several new, explicit parameters. (c31f877)
  • ⚠️ BREAKING BREAKING: The public mojom interface storage::mojom::IndexedDBControl has been changed, removing the ForceCloseReason parameter from the ForceClose method. (d954490)
  • ⚠️ BREAKING BREAKING: The public API struct blink::RecordContentToVisibleTimeRequest, used for performance monitoring, has changed its signature. It now uses a list of VisibilityEvent structs instead of simple boolean flags. (a71337a)

ChromeOS & Ash

  • ⚠️ BREAKING BREAKING: The entire video_conference.mojom Crosapi interface has been deleted. The video conferencing pipeline now uses native types internally. (645e372)
  • ⚠️ BREAKING BREAKING: The GetOptionGroups method has been removed from the document_scan.mojom Crosapi interface as part of a service refactor. (57ede81)
  • ⚠️ BREAKING BREAKING: Several user session-related preference key strings (e.g., kSessionLengthLimit) have been moved from the general chrome/common/pref_names.h to the Ash-specific ash/constants/ash_pref_names.h. Embedders building for Ash must update their references. (d9e2c11)

Android & WebView

  • ⚠️ BREAKING BREAKING: The public API for AwPrefetchManager in Android WebView has changed. The methods setMaxPrefetches and setPrefetchTtlSeconds now require a primitive int instead of a nullable Integer. New clear...() methods must be used to reset values. (5f953a6)
  • The kMojoUseEventFd feature is now enabled by default on Android (excluding WebView) to improve InputDelay and INP metrics. (04033a0)
  • Scaffolding has been added for a new 'Tabs from other devices' side panel on Android, though the feature is not yet user-visible. (b0e4272)

iOS

  • Support for extracting and filling <input type="date"> fields has been added to the iOS Autofill pipeline, controlled by the kAutofillSupportDateInput feature flag. (c969422)
  • The Assistant UI on iPad has been updated to support a side-panel layout. (f83b1a9)

Developer Tools

  • ⚠️ BREAKING BREAKING: The getAffectedUrlsForThirdPartyCookieMetadata command has been removed from the public DevTools Storage protocol (Storage.pdl). (c8d0ad5)
  • ⚠️ BREAKING BREAKING: The DevTools protocol commands Storage.setInterestGroupTracking and Storage.getInterestGroupDetails have been functionally disabled as part of the deprecation of the Protected Audience API. They will now return empty or no-op results. (bf74505)
  • A security vulnerability has been fixed where a compromised renderer could hijack DevTools sessions via Fenced Frames. Validation for devtools_frame_token is now global, and collisions will terminate the renderer process. (a77ddcd)

Performance

  • A significant O(N^2) performance issue in TextCodec::Encode was fixed by removing redundant reserve() calls inside the encoding loop, improving performance to O(N). (ac243d6)
  • A performance regression in IndexedDB was fixed by skipping redundant validation checks in KeyPrefix::Decode. (f6f6307)
  • To reduce memory allocations on the audio processing hot path, an optional input pool was added to ConvertingAudioFifo. (25e2800)
