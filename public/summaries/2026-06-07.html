Chromium Digest: 2026-06-07

Overview

This digest covers 10 commits integrated into Chromium between 2026-06-07. The changes span media engine stability, network protocol enhancements, accessibility improvements, and build system infrastructure. Commits range from (9dc8ccb) to (7c53e46).

Blink & Network

  • Reverted the refactor of HTMLMediaElement and WebMediaPlayerImpl to use FrameVisibilityObserver due to test regressions, restoring the previous RendererWebMediaPlayerDelegate visibility signaling flow. (7c53e46)
  • Enhanced the WebTransport Mojo interface to support optional stream priorities. New WebTransportStreamPriority structure allows developers to specify send groups and ordering, improving scheduling granularity. (d214fc3)

UI, Accessibility & Fonts

  • Added an accessibility delegate to Android site search settings to explicitly support expand and collapse actions for screen readers. (cf832ff)
  • Improved font rendering by forcing specific codepoints with Emoji_Presentation=Yes to use the native platform color emoji font on Windows, ensuring consistency with macOS. (7680799)
  • Refined UI interactions: the reload button's animation is now restricted to manual clicks, and experimental Glic task notifications are now throttled to appear only on the first turn. (f920558) (89b1282)

Infrastructure & Maintenance

  • Updated Commit Queue configurations to make chromeos-betty trybots critical, ensuring system-level stability for ChromeOS targets. (af34f1a)
  • Cleaned up exit-time destructors in extension unit tests to comply with upcoming global compiler warning requirements, and imported latest translation resources. (328eef3) (9dc8ccb)
