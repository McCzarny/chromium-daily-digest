Chromium Digest: 2026-06-19

Overview

Chromium development on 2026-06-19 saw 444 total commits, with 286 identified as having significant impact on core infrastructure and features. The development cycle ranged from commit (c86d8dd) to (d1f6ecf), focusing heavily on memory safety, infrastructure hardening, and platform-specific UI improvements.

Memory Safety and Code Hardening

  • Continued 'spanification' initiative by replacing pointer/length arithmetic with base::span, including updates to base::MemoryMappedFile and cc::paint::PaintOp APIs. (f709879) (3df86b1) (76a1a75) (79c5b55)
  • Promoted critical DCHECK statements to CHECK in content/browser/renderer_host as part of the ongoing code-hardening initiative. (7945763) (7941933) (7931149)

Graphics and Platform Infrastructure

  • Removed support for the D3D9 ANGLE backend, including Mojo enums and initialization paths. Embedders relying on legacy Windows graphics drivers should verify configuration. (0bd6138)
  • Enabled LTO for blink_perf_tests, yielding 10-16% improvement in style calculation and parsing performance. (7967780) (7962959)

UI and Feature Development

  • Substantial UI development for Android vertical tabs, including group color synchronization and 'vertical spine' decoration. (03d14d6) (7d87f63)
  • Centralized omnibox position tracking for iOS by migrating legacy agents into a unified LayoutState. (7965378) (7928007)

Build and Dependency Management

  • Shifted Android builds to is_component_build=false by default to improve performance and leverage modern linkers. (7896376) (7931652)
  • Removed legacy ENABLE_DESKTOP_ANDROID_EXTENSIONS build flag and dropped support for legacy JNI registration in jni_zero. (32d3194) (91168a9)
