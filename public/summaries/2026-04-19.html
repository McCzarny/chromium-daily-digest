Chromium Digest: 2026-04-19

Overview

This digest covers 29 commits, highlighting two major breaking changes in UI components and Chrome Remote Desktop. Other key developments include significant internal refactoring of the networking stack's disk cache, numerous test enhancements and bug fixes for the HTML canvas element, and ongoing build system modularization.

UI & Views

  • ⚠️ BREAKING A refactor of tab hover cards introduces a breaking change by replacing the HoverCardAnchorTarget::GetAnchorView() virtual method with a new pure virtual method, GetAnchor(), which returns a views::BubbleAnchor instead of a views::View*. Embedders or custom UI components subclassing HoverCardAnchorTarget must update their implementation to provide the new anchor type. (174be1f)
  • Various UI enhancements were made, including persisting the window size for auto Picture-in-Picture on Android, reordering buttons on the profile creation screen for better readability, and temporarily disabling animations on the first-run sign-in page. (b5af6f8) (e73f207) (88eebcc)

Chrome Remote Desktop

  • ⚠️ BREAKING The host setup logic on Linux has been updated, introducing a breaking change to the remoting::StartHostAsRoot function. Its signature has been changed from int StartHostAsRoot(int argc, char** argv) to int StartHostAsRoot(), removing its parameters. Scripts or tools that call this function directly will need to be updated. (24cd9c6)

Networking

  • The SQL-based disk cache backend received significant internal updates to improve asynchronous task management. New classes like SqlAsyncTaskManager and SqlTrackedSequenceBound were introduced to provide more reliable synchronization for testing purposes, ensuring that complex, multi-step operations can be tracked to completion without relying on RunUntilIdle(). (0e4d1c9) (d809804) (38c4f09)

Blink & Web Platform

  • Extensive work was done to improve the robustness of the 2D canvas implementation. This includes adding comprehensive web tests for context loss and restoration scenarios, especially when combined with GPU process loss or invalid canvas sizes. A bug was also fixed where a resource dispatcher was not correctly recreated after a context was restored, affecting desynchronized canvases. (9b65e50) (676e2fd) (70940fb) (cc55330) (3c95ea3) (0ccd8f8) (ff41cf0)
  • Regular Web Platform Test (WPT) imports bring in new tests and expectations. (17970ac) (91342bb) (a942ac1)

Autofill & Payments

  • Autofill sees several updates, including new metrics to differentiate between suggestions for cards saved externally (e.g., via Google Play) versus newly-synced cards. On iOS, the logic for displaying passport details was improved. The in-product help bubble for saved cards was also updated to mention 'Google Wallet' for brand consistency. (6fb6cb0) (d481fcf) (7ba1c54)

Build & Infrastructure

  • Ongoing modularization efforts continue, with //chrome/browser/support_tool and //chrome/browser/ntp_customization being extracted into their own source sets. The build dependency script install-build-deps.py was updated for compatibility with Debian 'sid'. (0f60250) (ba57552) (5698a61)
  • Several expired histograms were removed from the codebase, and CI was updated to run gtests on betty for a percentage of Googler CLs. (3db3a99) (2cbe2ff) (2a3b985)

Security

  • A security vulnerability was addressed in the VA-API JPEG encoder (VaapiJpegEncodeAccelerator) by adding an explicit bounds check to prevent a potential out-of-bounds write during the encoding process. (c24fe50)
