Chromium Digest: 2026-05-06

Overview

This summary covers 558 relevant commits out of 741 total from the main branch on May 6th, 2026. This digest highlights numerous breaking changes to core C++ interfaces and web platform APIs, the shipment of new CSS features, and significant security fixes. First commit: (dcf44d8), Last commit: (056c439).

Breaking Changes

  • ⚠️The public Android WebView API getAwContents() now blocks and waits for Chromium to initialize. This can cause deadlocks or ANRs in applications that previously called it on the UI thread without expecting it to block. (97e5ca6)
    Android
  • ⚠️The scrollIntoView() JavaScript API has changed its default alignment. The block and inline options now default to "auto" instead of "start" and "nearest", which may alter scrolling behavior on websites relying on the previous defaults. (5972c54)
    BlinkCSS
  • ⚠️The media::AudioManager interface underwent rapid, breaking changes. Methods GetAudioInputDeviceNames and GetAudioOutputDeviceNames were changed from returning void to bool, and then reverted back to void. Embedders must ensure their implementation matches the final void return type. (8ffee52) (470bf4c)
    MediaAudio
  • ⚠️A new pure virtual function, RemoveGoogleGroupsFromPrefsForDeletedProfiles, has been added to the variations::VariationsServiceClient public interface. All embedders must now implement this method to handle profile data cleanup. (8c8d5ff)
    EnterpriseProfiles
  • ⚠️For privacy, the Long Tasks API now redacts detailed attribution (containerId, containerName, containerSrc) for long tasks originating in cross-origin iframes. Performance monitoring tools relying on this data will be affected. (f34f0ba)
    BlinkPerformancePrivacy
  • ⚠️The web-serial-wired-devices-android feature has been removed. This constitutes a removal of a stable, public API for websites that were using the Web Serial API to interact with wired serial devices on Android. (1113e33)
    BlinkAndroidUSB
  • ⚠️The PaymentsAutofillClient interface has been updated, removing several virtual methods including LoadRiskData and GetVirtualCardBenefitDetails. Embedders implementing this interface must update their code. (99e0d54)
    PaymentsAutofill
  • ⚠️The public method set_owned_by_client() has been removed from the foundational ui::views::View class. Code should now manage view ownership directly. (04fbab8)
    UI
  • ⚠️Global functions for managing metrics reporting consent (e.g., ChangeMetricsReportingState) have been moved into the metrics:: namespace, breaking existing callers. (f723259)
    BasePrivacy
  • ⚠️The extensions::MimeTypesHandler::GetHandler() method was renamed to Get() and its signature changed to accept a reference. Additionally, extensions::Extension::GetManifestData() now returns a const pointer. (d1eb114) (9d8833a)
    Extensions
  • ⚠️On iOS, the public method FetchCanSigninToChrome was removed from SystemIdentityManager, and the web::ScriptMessage constructor signature was changed to require a url::Origin. (e11c0f8) (f25be9b)
    iOS

Web Platform & Blink

  • New CSS features have been shipped: URL request modifiers (cross-origin(), referrer-policy(), integrity()) for resources fetched via CSS are now enabled. (e1c6eac)
    BlinkCSS
  • New CSS media pseudo-classes (:playing, :paused, :seeking, :buffering, :stalled, :muted) are now enabled by default, allowing easier styling based on media state. (cefbd1e)
    BlinkCSSMedia
  • The popover=hint attribute, which creates non-modal, auto-dismissing popovers, has been shipped and is now enabled by default. (a4e8e34)
    BlinkHTML
  • To prevent stack overflow crashes from malicious CSS, depth limits are now enforced on flat math expression chains (e.g., 1px * 1px * ...) during parsing. (1a31de5)
    BlinkCSSSecurity
  • For WebAudio, the promise resolution order for setSinkId() has been corrected to fire the onstatechange event after the promise resolves, not before, improving spec compliance. (262fe80)
    BlinkMedia
  • In WebCodecs, H.264 content with a 4:4:4 chroma subsampling profile will now be routed to the software decoder to improve compatibility, as hardware support is rare. (08a73f0)
    Media

Security & Privacy

  • A Use-After-Free vulnerability in LegacyRenderWidgetHostHWND on Windows was fixed by adding WeakPtr guards to handle object destruction during synchronous Win32 calls. (73cfb55)
    SecurityWindowsUI
  • A security vulnerability was fixed where a compromised renderer could bypass microphone permission checks on Android. The browser now explicitly rejects speech recognition requests that attempt this bypass. (a10d218)
    SecurityAndroidMedia
  • A fix was implemented on Android to prevent a malicious webpage from using the soft keyboard to obscure the fullscreen entry notification, which could enable UI spoofing. (fb7aa25)
    SecurityAndroidUI
  • On iOS, regular expressions in WKContentRuleList are now anchored to prevent malicious URLs from bypassing blocking rules via query strings or fragments. (c8fe8b3)
    SecurityiOS
  • Several vulnerabilities were addressed in Chromecast, including potential navigation provenance laundering, a heap buffer overflow, an out-of-bounds read, and a potential use-after-free in the BLE implementation. (cd5d219) (6183ca6) (978af60) (ecf7dbf)
    SecurityMediaBluetooth

Embedder APIs, UI & Services

  • A new, safer CreateBubble method that returns a std::unique_ptr<Widget> was introduced to modernize bubble creation and help prevent memory leaks. (1a3139e)
    UIBase
  • On iOS, the web::WebFrame API was enhanced with new CallAsyncJavaScriptFunction methods to allow native code to call JavaScript functions that return Promises. (dd55163)
    iOS
  • For Android WebView, support was added for the new AndroidX navigate API, providing a more structured way for embedders to trigger navigations. (4a35add)
    Android
  • New context menu items "Copy video frame" and "Download video frame" have been implemented for videos on Android. (7062176) (839877c)
    AndroidUIMedia
  • A new framework for automatically recording animation performance metrics has been introduced, and several UI components like the side panel and vertical tab strip have been migrated to use it. (c83c394) (e8eb851) (7596b60)
    UIPerformance

Build & Infrastructure
