Chromium Digest: 2026-01-02

Overview

This digest covers 156 commits from the Chromium repository, from commit (3b7bf59) to (c998bad). It highlights significant updates across UI, Blink, Security, Media, and core platform components, including several major breaking changes affecting embedders.

UI & UX

  • ⚠️ BREAKING ui/aura API Change: The WindowEventDispatcher::ObserverNotifier class and its corresponding observer interface have been removed. This was a legacy component from the Mustash project. Embedders directly observing window events through this mechanism will need to migrate to using PlatformEventObserver in NativeEventObserver or other appropriate event listeners. (47e1cd3)
  • Multiple enhancements were made to the experimental 'Contextual Tasks' feature, including UI updates for error pages, header visibility, and compose box behavior. Eligibility checks were also made stricter, requiring users to be signed in. (c998bad) (ac75fc6) (7cf78fb) (1581f04) (772c913) (3bac276) (461fbe8) (5af2f24)
  • The New Tab Page 'Realbox' now supports multiline input, allowing text to wrap instead of overflowing. (693bb90) (475056c) (58cfcbb)
  • The Vertical Tabs feature now implements the GLIC (General Language Interface for Chrome) underline to indicate tab sharing state and updates tab group visuals on theme changes. (656e822) (ee7129f)

Blink & Rendering

  • ⚠️ BREAKING LayoutBox API Removal: The public method LayoutBox::ParentBox() has been removed from Blink's core layout engine. Downstream projects that rely on this method for traversing the layout tree must now use safer alternatives like LayoutObject::ContainingScrollContainer() or other appropriate traversal methods. (cfc08f9)
  • ⚠️ BREAKING GlobalCacheStorage Refactoring: GlobalCacheStorage has been refactored to attach to ExecutionContext via the Supplementable interface, instead of being attached to WindowOrWorkerGlobalScope. This changes the access pattern for this global object, and embedders interacting with it directly will need to update their code. (e268bfb)
  • The CSS random() function has received expanded support, enabling its use within image-set(), color values (color-mix, rgb, etc.), and ident() functions. This allows for property-dependent random values based on caching keys derived from the property name and value index. (c8d4dfe) (858f19e) (b0749af) (a935cd9) (9854bd8)
  • Performance of the scroll-target-group CSS property has been optimized by integrating it with the OrderedScopeTree infrastructure, replacing full DOM traversals with more efficient, localized updates. (22efc8c)
  • A fix has been implemented to avoid subpixel inaccuracies in multi-column layout progression, leading to more consistent and predictable rendering that aligns better with other browsers. (f7eff36)

Security & Privacy

  • ⚠️ BREAKING Safe Browsing PingManager API Removal: Key public methods for reporting security events, ReportThreatDetails and ReportSafeBrowsingHit, have been removed from PingManager. Embedders using this API to send hit reports or threat details will need to find alternative mechanisms, as this functionality is no longer exposed. (bb63549)
  • ⚠️ BREAKING WebAuthn EnclaveManagerInterface Renaming: Virtual methods within the public EnclaveManagerInterface have been renamed from snake_case to PascalCase (e.g., get_wrapped_secret is now GetWrappedSecret). This is a breaking signature change for any embedder that implements or directly calls this interface for passkey enclave management. (36fb252)
  • ⚠️ BREAKING iOS Safe Browsing Preference Change: The iOS-specific feature flag for SBER (Safe Browsing Enhanced Reporting) deprecation has been removed, and the behavior is now controlled by the general kSafeBrowsingEnhancedProtection feature. This unifies preference management and represents a significant behavioral change for embedders relying on the previous iOS-specific logic. (8869b97)
  • AffiliationService Signature Change: The GetAffiliationsAndBranding method in AffiliationService has changed its signature from accepting a const GURL& to a const std::string& for the facet URI. This will break embedders who implement or call this service for managing website affiliations. (2b5dedb)
  • The Privacy Sandbox 'Mode B' rollback is now being enforced unconditionally to ensure all users are migrated off the experimental configuration. (1bcd25c)
  • A fix was added for Local Network Access (LNA) to allow fetch calls in shared workers within Chrome Apps to correctly access private network resources without an unnecessary permission prompt. (f061a97)

Media

  • ⚠️ BREAKING HLS Stats Reporting API Removal: The HlsStatsReporter class and several related histograms for HLS (HTTP Live Streaming) have been removed. This includes methods like NotifyAdaptation and NotifyLiveContent. Embedders who relied on this public API for monitoring HLS playback statistics will need to remove their dependencies. (33305e2)
  • A new histogram, Media.TrackChangePipelineStatus, has been added to monitor pipeline failures following a track change, which will help diagnose issues with the feature. (313cb9b)
  • The V4L2 stateless decoder initialization logic was updated to better follow the specification, ensuring the EXT_CTRLS ioctl is sent before CAPTURE format negotiation for stateless decoders. (ecf0dfb)

ChromeOS (Ash)

  • ⚠️ BREAKING AuthRequest Interface Refactoring: The public AuthRequest interface, used for in-session authentication, has been refactored. The NotifyAuthSuccess and NotifyAuthFailure methods have been replaced with a single NotifyAuthResult(AuthResult) method. This is a breaking change for any component or embedder with custom AuthRequest implementations. (126909d)
  • The authentication controller has been extended to support Payments Autofill, introducing a new PaymentsAutofillAuthRequest and the ability to treat 'no auth factor available' as a success case for this flow. (126909d) (33a0a1b) (f58e691) (e40a7c8)
  • A fix was implemented in the enrollment state fetcher to continue with the enrollment process even when device ownership status is unknown, preventing a potential denial-of-service vector. (4decd46) (6214dd1)

Android

  • ⚠️ BREAKING Test Framework Breaking Change: The Android test transit framework has undergone a major refactoring. Core concepts like CarryOn have been renamed to State, and associated methods like pickUpCarryOn are now enterState. This will break any downstream test suites that utilize this framework for UI testing. (a223bda)
  • A null check was added in CustomTabLocator to prevent a crash when the IntentDataProvider is null, improving stability. (3ae4f41)
  • Remaining androidx.annotation.Nullable annotations in the keyboard accessory feature have been migrated to the Chromium-specific org.chromium.build.annotations.Nullable. (46db5c7)

iOS

  • The Assistant Sheet implementation was further developed by integrating the Mediator layer for Gemini and AIM, and a new AssistantCommands protocol was introduced to standardize communication between parent and child coordinators. (427a0bc) (4bb9128)
  • Several obsolete feature flags have been removed, including EnhancedSafeBrowsingPromo, IPHPriceNotificationsWhileBrowsing, and NotificationSettingsMenuItem, as their corresponding features have fully launched. (bb466b7) (339d4a4) (993910a)
  • Significant code refactoring occurred within the toolbar and browser view components, moving them out of legacy directories to be reused in modern UI implementations. (a06f57f) (8f4f9ab)

Code Health & Infrastructure

  • A race condition in mojo::core::ChannelLinux was fixed by adding more comprehensive locking around the write state, preventing a potential crash during channel shutdown. (24d2da0)
  • A large number of commits were dedicated to code health, including fixing Clippy warnings in Rust code, removing dangling pointers, migrating away from unsafe buffer usage, and refactoring internal APIs to use std::u16string for error messages. (2c53a6a) (4a5c112) (9eb5643) (d9657d9) (5980162) (7c4ffb7)
  • Numerous histograms across various features like Permissions, Settings, WebApps, and Media had their expiration dates extended to continue data collection and analysis. (e2aa5f1) (0ee3d20) (e8a9b4d) (6bae201) (a0cfd68) (cd3b788) (740965a) (d0b4d16) (a6136b3) (1458dc3) (1efa7d1) (69e40ca)
  • Several reverts were processed to address test failures and regressions, including changes related to AIM screenshots and bookmark sync error UI. (e24b6bb) (a3d6029) (375d02e)
