Chromium Digest: 2026-06-20

Overview

This report summarizes 17 recent Chromium commits ranging from performance-oriented architecture changes to routine bug fixes and maintenance. The analyzed commits span from hash (0f1207d) to (2513b16), with a focus on Android platform stability and internal storage service refactoring.

Performance & Resource Management

  • Introduced an immutable Java-side cache for the tab hierarchy in TabCollectionTabModelImpl, reducing the overhead of tree traversal operations by amortizing them per mutation. (7097855)
  • Updated GPU process logic to enable fallback to software composition without requiring a full process restart, explicitly managing GrContextType fallbacks. (2a30e27)
  • Refactored LocalStorage metric collection to move logic from DomStorageContextWrapper to the Storage Service; removed internal DomStorageDatabase::GetPath() method. (d98576c)

UI, Accessibility & Features

  • Added BottomBarAppMenuUpdateBadgeController for Android UI lifecycle management and adjusted New Tab Page (NTP) feature behaviors by disabling module auto-removal for Android. (ae1f975) (a2d74b1)
  • Improved accessibility in Tab Search by dynamically adjusting the aria-activedescendant attribute based on user input, preventing premature focus shifts for screen reader users. (0f1207d)

Stability & Bug Fixes

  • Fixed Android drilldown app menu crashes by correcting item type identification and registering view binder height providers correctly. (68bb2f8) (604fe22)
  • Reverted memory leak fixes in SwipeRefreshHandler and EGL image internal usage fixes that caused regressions in swipe-to-refresh gestures and hardware rendering on legacy devices. (8d9dd8d) (cc54c35) (2513b16)
