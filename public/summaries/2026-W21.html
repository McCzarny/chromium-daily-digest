Chromium Weekly: 2026 Week 21

Overview

A significant week for Chromium development, with 2487 relevant commits out of 3385 total across 6 days. The week was dominated by an unprecedented number of breaking changes aimed at improving componentization, security, and API consistency. The most impactful change is the mandatory upgrade to Java 21 for building Android targets. Other major highlights include the introduction of post-quantum key agreement in WebCrypto, the removal of the 'borderless' web app display mode, and a massive, multi-day refactoring of core browser and Android APIs.

Breaking Changes: Build System & Core APIs

  • ⚠️BREAKING CHANGE The required Java version for building Android targets has been upgraded from Java 17 to Java 21. All developers and embedders building for Android must update their JDK. (2e2c668)
    BuildAndroid
  • ⚠️BREAKING CHANGE A major effort to improve modularity involved splitting monolithic build targets. The //chrome/browser:browser and //chrome/browser/ui:ui targets were broken into many smaller source_set targets, which will break downstream build configurations. (a3e448d) (9694359)
    BuildInfraBrowser
  • ⚠️BREAKING CHANGE Numerous core C++ interfaces received new pure virtual methods, requiring embedders to update their implementations. Notable examples include ContentBrowserClient::GetContainerPolicyOverrideForCommit, blink::WebAutofillClient::IsAutofillableElement, and FrameWidget::SendEarlyFinalBeginMainFrame. (6da6b23) (4f8eaf4) (87c8e7c)
    BrowserBlinkAutofill
  • ⚠️BREAKING CHANGE As part of a large-scale refactoring, many widely-used helper functions and classes were removed. This includes browser lookup functions in browser_finder.h, FullscreenObserver, and BrowserFocusController::Delegate, requiring migration to newer, more componentized APIs like ProfileBrowserCollection. (1d867aa) (00d519b) (64c1527)
    BrowserUITabs
  • ⚠️BREAKING CHANGE Buffer mapping commands (MapBufferRange, UnmapBuffer) have been removed from the public GLES2Interface as a security hardening measure. Embedders using this low-level GPU API directly must adapt. (7cc1fc6)
    GPUSecurity
  • ⚠️BREAKING CHANGE The public Tab Strip API has been moved from //chrome/browser/ui/tabs/tab_strip_api to //components/browser_apis/tab_strip, requiring embedders to update header paths. (d10436b)
    UIBrowserBuild

Breaking Changes: Android & Web Platform

  • ⚠️BREAKING CHANGE A massive, multi-day refactoring of Android's tab management APIs has removed numerous public Java classes, including TabGroupModelFilter, TabModelHolder, and IncognitoTabModelHolder. Embedders must migrate to the consolidated TabModel API. (e51e8f4) (561cded) (c0a50ed) (5bc1da8)
    AndroidTabsUI
  • ⚠️BREAKING CHANGE Support for the "borderless" value in the Web App Manifest display member and the display-mode CSS media query has been completely removed. Web apps relying on this feature will fall back to a different display mode. (8df25b6)
    BlinkUICSS
  • ⚠️BREAKING CHANGE The BackgroundFetchManager.fetch() API, when called from a Service Worker, will now throw a NotAllowedError by default. This behavioral change will break existing applications unless their origin is added to a server-configurable allowlist. (2976e22)
    ServiceWorkerBlinkNetwork
  • ⚠️BREAKING CHANGE Over 20 enterprise policies have been removed, including AppLaunchAutomation and LegacySameSiteCookieBehaviorEnabled. Administrative scripts relying on these policies will no longer function. (bf54c2e) (94ea7ec)
    PolicyEnterprise
  • ⚠️BREAKING CHANGE The SelectFileDialog public API on Android now strictly rejects scheme-less URIs and any scheme other than content://, breaking embedders who were passing other URI types like file://. (a652043)
    AndroidSecurityUI

Security & Privacy

  • Support for the X-wing hybrid key encapsulation mechanism (ML-KEM-768 and X25519) has been implemented in WebCrypto, enabling post-quantum key agreement for TLS and other protocols. (31f3485)
    CryptoBlinkSecurity
  • Security was hardened across multiple components. Key changes include enforcing icon hash validation for policy-installed PWAs, preventing symlink traversal vulnerabilities in downloads, and forcing ASLR for sandboxed child processes on Windows. (75d3397) (3cf6610) (09eaeae)
    SecurityPolicyDownloads
  • A framework for tracking password element coordinates has been implemented. This lays the groundwork for future capabilities to redact sensitive information from screenshots and other screen captures. (cf85b4f)
    PasswordManagerPrivacyUI
  • The password manager's check for cross-origin iframes was improved to validate the entire ancestor chain, preventing potential password leakage in complex 'sandwich' (A-B-A) iframe scenarios. (b80505d)
    PasswordManagerSecurity

Web Platform

  • New CSS features have been implemented, including a new max-lines syntax for line-clamp, support for closest-corner and farthest-corner keywords in shape functions, and parsing support for the scroll-axis-lock property. (7eec7df) (925f513) (91a0fbc)
    BlinkCSS
  • The SelectiveClipboardFormatRead feature, which allows sites to request specific clipboard formats, has been enabled by default, giving web applications more control over paste operations. (137220d)
    BlinkUISecurity
  • ⚠️The setPreferredSinkId() web API is now restricted to top-level documents. Calls from within an iframe will now throw a DOMException, a behavioral breaking change to enhance security. (1e43d45)
    MediaBlinkSecurity

UI & Platforms

  • A new global Ctrl-Tab MRU (Most Recently Used) switching feature was implemented, allowing users to quickly toggle between the two most recent tabs across all browser windows. (2e673d0)
    UITabs
  • On Android, windowed Incognito mode is now enabled by default on large-screen devices, and DevTools will now be shown in a separate activity to improve the debugging experience. (e57ef26) (5be7d36)
    AndroidUIDevTools
  • The Translate bubble UI has been improved with a new searchable target language selection view, replacing the legacy combobox to better handle a large number of languages. (c03576a)
    UII18n
  • The Borealis (Steam on ChromeOS) installer WebUI and related code have been removed, deprecating the standard installation flow for the feature. (2102169)
    ChromeOSUI
