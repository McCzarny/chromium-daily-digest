Chromium Digest: 2026-06-06

Overview

A total of 56 commits were analyzed. The changesets span core components like security isolation, graphics pipelining, ChromeOS modernization, and cryptography, bounded between commit (63014b8) and (329bbab).

Security & Process Isolation

  • Cross-World Script Isolation (Scoped Custom Element Registries): Closes a security vulnerability where isolated script environments (e.g., extension content scripts) could access the main world's custom element registries. WebIDL accessors document.customElementRegistry and element.customElementRegistry now validate caller world IDs to prevent leakage of V8 constructors and promises. (40b2818)
  • BrowserContext Sandbox State Teardown: Added cleanup of origin-isolation opt-in maps (origin_isolation_opt_ins_and_outs_) in ChildProcessSecurityPolicyImpl when a BrowserContext is removed. Teardown execution is shifted to NotifyWillBeDestroyed to resolve destruction order null-dereference crashes. (af7cf3e)
  • Secure Scheme Validations (PDFs and Calendar): Implemented strict scheme restrictions to block potential URL-based exploits. Ash Quick Settings calendar now rejects non-HTTP/HTTPS schemes for video-conferencing, and Android's PdfCoordinator enforces path whitelisting on shared content URIs to protect private file providers. (a24a935) (288c628)

Graphics, Composition & Media Internals

  • HDR Metadata Pipeline Refactor: Refactored cc::PaintImageGenerator to natively store gfx::HDRMetadata assigned inside its constructor instead of querying cc::ImageHeaderMetadata. This permits generator subclasses (like VideoImageGenerator) to cleanly propagate video frame HDR properties. (8f0906a)
  • GPU Backend IPC Optimization: Avoids unnecessary transmission of shader cache handles from the browser to the GPU host by verifying active Skia backend rendering context status inside GpuHostImpl::DidInitialize(). (1172a59)

ChromeOS & Ash System Architecture

  • Decoupling from ProfileHelper: Migrated several ChromeOS controllers and tab helpers (including Google One Offer, network enrollment, media clients, and casting) from high-level ash::ProfileHelper dependencies to lower-level BrowserContext and ash::BrowserContextHelper abstractions. (3382059) (4a8688a) (33788ef) (04b4602) (a091b39)
  • Window Aura Deletion Safeguards: Corrected state-tracking bugs in Aura window event targeting blockers to bypass restoring targeting policies and updating active states of windows that are actively undergoing destruction. (63014b8) (329bbab)

WebUI & Experimental Features

  • WebUI Context Menu Alignment: Updated contextual menus to support adaptive layout alignments based on viewport clearance thresholds, limiting menu max heights via CSS properties and setting scroll parameters to prevent clipping flyouts. (72374e1) (a538de6)
  • Glic Widget & Settings: Refactored the experimental Glic widget's hit-testing on Windows to allow proper border resizing interactions, and coupled system settings so that disabling web-actuation disables experimental trigger parameters. (169f06e) (50b7486)
  • Autofill AI Caching & Prefetching: Integrated predictive context constraints into PersonalContextAccessManagerImpl to prefetch ambient autofill context when enabled and store results under a 30-minute time-to-live cache TTL. (304b4e9) (5340cf5) (c42e142)

Cryptography, Infrastructure & Testing

  • BoringSSL Cryptographic Randomness Integration: Re-enabled the kUseBoringSSLForRandBytes feature flag by default across supported platforms, using BoringSSL for standard cryptographic random byte sequences rather than operating system-specific calls. (76e1478)
  • TextInputClient Mac Cleanup: Cleaned up stale feature flags kTextInputClient and kTextInputClientIPCTimeout on macOS, hardcoding the fallback synchronous input timeout sequence directly in TextInputClientMac as a member initialized to 1500ms. (ee1af4f)
  • Feedback JSON Parser Sandboxing Removal: Optimized the private feedback extensions API by replacing out-of-process DataDecoder instances with synchronous base::JSONReader calls directly inside simpler, faster URLLoader callback context execution paths. (0a7c77e)
