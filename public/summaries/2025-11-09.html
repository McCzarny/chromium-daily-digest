Chromium Digest: 2025-11-09

Overview

A total of 107 commits were made to the 'main' branch. After filtering, 17 relevant commits were analyzed. The day's changes span from commit (96c4358) to (cfa4cb1).

Performance

  • Introduces a new mechanism to defer navigations until a "site familiarity" verdict is computed. This verdict, based on browsing history and Safe Browsing data, determines whether V8 optimizations are enabled for a given site. The goal is to enhance security and performance by selectively applying optimizations. The change adds SiteFamiliarityProcessSelectionDeferringCondition to control process selection during navigation. (9beb380)

Blink Engine

  • Major update to the HTML-in-Canvas feature: the setHitTestRegions API has been removed in favor of always-on hit testing based on CSS transforms. This simplifies the API for developers and is a necessary step to better support features like anchor positioning for custom form controls. (3a9e6b2)
  • Refactors Local Storage by implementing LocalStorageLevelDB::WriteMetadata(). This new method centralizes the process of writing metadata, such as last-access and last-modified times, to the underlying LevelDB database. This improves the structure of LocalStorageImpl when handling map unloading and metadata updates. (64b770d)
  • Aligns with the HTML specification by converting the partial interface for NavigatorPlugins into an interface mixin. This is a structural code health improvement with no change in behavior, made possible by recent updates to the IDL compiler. (5a21b9a)

Bug Fixes

  • Fixes a crash in --single-process mode by adding a null-pointer check for RendererMemoryCoordinatorPolicy, which does not exist in this mode. (7b5a317)
  • Prevents a potential crash in the omnibox (realbox) by handling a null case for the expandedSearchboxShowVoiceSearch flag in ui/webui/resources/cr_components/searchbox/searchbox.ts. (f519a52)
  • The logic for IsShareImageEnabledForProfile is updated to correctly check for managed accounts, ensuring enterprise policies are respected for this feature. (903dd22)

Developer Tools

  • Adds a new command-line switch, --webnn-ort-ep-device, allowing developers to force the WebNN service to use a specific ONNX Runtime execution provider device. This provides fine-grained control for testing and debugging on specific hardware like a particular GPU. (98e6bd5)
  • Improves debugging of the compositor by printing the background-color of a layer when dumping the layer tree. (905c682)

Infrastructure

  • A large-scale code cleanup removes superfluous #include statements for file_util.h across more than 20 directories, improving build hygiene. (502b6c6)
  • Continues the migration of network-stack Java tests from JUnit's Assert.assertThat to the more readable Truth framework. (594a5ac)
  • A new helper method, ButtonTestApi::NotifyMouseClick, is introduced to simplify UI testing by allowing the triggering of button clicks without constructing a full MouseEvent. (0db0398)
  • A new Web Platform Test (WPT) import (f62764048) and updates to test expectations were landed. (cfa4cb1) (f3b75cf)
